Provisions all shared regional infrastructure that sandboxes in a region use.

Run this once per region before your first 'km create' targeting that region.
Safe to re-run — idempotent via Terraform. Missing modules are skipped with a
warning. Modules requiring unconfigured env vars are skipped gracefully.

Usage:
  km init [flags]

Examples:
  # Initialize the default region (us-east-1)
  km init

  # Initialize a specific region
  km init --region ca-central-1

  # Use a non-default AWS profile
  km init --region us-east-1 --aws-profile klanker-application

Modules Deployed (in dependency order):
  1. network             — VPC, public subnets, internet gateway, security groups
  2. dynamodb-budget     — DynamoDB table for per-sandbox budget tracking
  3. dynamodb-identities — DynamoDB table for sandbox identity/key management
  4. ses                 — SES email infrastructure (requires KM_ROUTE53_ZONE_ID)
  5. s3-replication      — S3 artifact replication bucket (requires KM_ARTIFACTS_BUCKET)
  6. ttl-handler         — Lambda for TTL-based auto-destroy (requires KM_ARTIFACTS_BUCKET)

Environment Variables (optional):
  KM_ROUTE53_ZONE_ID   — Route53 hosted zone ID for SES DNS validation
  KM_ARTIFACTS_BUCKET  — S3 bucket name for Lambda artifacts

  Modules whose required env var is unset are skipped with a warning.
  Other modules still deploy normally.

Prerequisites:
  - AWS credentials configured (profile: klanker-application)
  - km-config.yaml present (run 'km configure' first)
  - Terraform and Terragrunt installed

Network outputs (VPC ID, subnet IDs, security group ID) are stored in
Terraform state and automatically referenced by 'km create'.
