Show OTEL telemetry and AI spend summary for a sandbox.

Displays a unified view of AI observability data from two sources:
the HTTP proxy's budget-enforcer (DynamoDB) and the OTel Collector's
telemetry export (S3).

Usage:
  km otel <sandbox-id>
  km otel <sandbox-id> --prompts
  km otel <sandbox-id> --events
  km otel <sandbox-id> --tools
  km otel <sandbox-id> --timeline

Examples:
  # Summary: budget by provider, S3 file counts, latest metrics
  km otel sb-a1b2c3d4

  # Use numeric shorthand from km list
  km otel 1

  # List all user prompts with timestamps
  km otel 1 --prompts

  # Full event stream (prompts, API calls, tool calls)
  km otel 1 --events

  # Tool call history with parameters, duration, success/failure
  km otel 1 --tools

  # Conversation timeline with per-turn cost rollup
  km otel 1 --timeline

Views:

  Default (no flags):
    Budget Enforcer — per-model AI spend grouped by provider
    (Bedrock vs Anthropic Direct/Max), S3 telemetry file counts
    and sizes, latest Claude Code self-reported metrics snapshot.

  --prompts:
    All user prompts with timestamps and session IDs.
    Internal messages (task-notifications, system-reminders) are
    filtered out automatically.

  --events:
    Chronological event stream showing prompts, API requests
    (model, tokens, cost, duration, cache stats), tool decisions,
    and tool results.

  --tools:
    Tool call history grouped by session. Shows tool name, duration,
    result size, and key parameters (file paths for Read/Write/Edit,
    commands for Bash, patterns for Grep/Glob, descriptions for Agent).
    Failed calls marked with a cross symbol.

  --timeline:
    Conversation turns grouped by user prompt. Each turn shows all
    API calls with model, tokens, cost, cache read/creation, and
    duration. Tool usage count and per-turn cost subtotal. Ends with
    grand total across all turns.

Data Sources:

  Budget data comes from DynamoDB (real-time, updated per-request).
  OTEL data comes from S3 (batched by the OTel Collector, ~10s delay).
  The budget-enforcer tracks spend via HTTP proxy MITM interception.
  The OTEL data is Claude Code's own telemetry export.
