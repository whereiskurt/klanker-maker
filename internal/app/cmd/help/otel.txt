Show OTEL telemetry and AI spend summary for a sandbox.

Displays a unified view of AI observability data from two sources:
the HTTP proxy's budget-enforcer (DynamoDB) and the OTel Collector's
telemetry export (S3).

Usage:
  km otel <sandbox-id>

Examples:
  # Show telemetry for a sandbox
  km otel sb-a1b2c3d4

  # Use numeric shorthand from km list
  km otel 1

Output Sections:

  Budget Enforcer (HTTP proxy → DynamoDB):
    Per-model AI spend tracked by the MITM proxy during streaming
    responses. Models are grouped by provider:
    - Bedrock: model IDs starting with "anthropic." (e.g. anthropic.claude-sonnet-4-6)
    - Anthropic Direct (Max/API): bare model IDs (e.g. claude-opus-4-6)

  OTEL Telemetry (OTel Collector → S3):
    File counts and sizes for each signal type (logs, metrics, traces)
    exported by the OTel Collector sidecar to S3.

  OTEL Metrics Snapshot:
    Latest Claude Code self-reported metrics from S3, including
    token usage, cost, session count, and active time.

The budget-enforcer data is real-time (updated per-request).
The OTEL data has a ~10s batch delay from the collector.
