Note on OTEL_S3_BUCKET: The systemd unit sets `OTEL_S3_BUCKET={{ .KMArtifactsBucket }}` because KMArtifactsBucket is the correct S3 bucket for OTEL data export. The artifacts bucket (`s3://<account>-km-artifacts`) stores all sandbox operational data including sidecar binaries, tracing configs, and telemetry output. The collector config (`sidecars/tracing/config.yaml`) references `${OTEL_S3_BUCKET}` as an env var for its S3 exporter destination. This is intentionally the same bucket -- OTEL telemetry data (traces/, logs/, metrics/ prefixes) belongs alongside other sandbox artifacts, not in a separate bucket.
