- `handleDestroy(w, r)` -- POST /api/sandboxes/{id}/destroy. Get sandbox ID from path. Call the same destroy logic used by `km destroy` -- import from `internal/app/cmd` or reuse `pkg/terragrunt` + `pkg/aws` directly. For v1: shell out to `km destroy {id}` as a subprocess (simpler than importing internal packages with Cobra dependencies). Return 200 on success with HTMX `HX-Trigger: sandbox-destroyed` header to refresh table. Return 500 with error message on failure.

- `handleExtendTTL(w, r)` -- PUT /api/sandboxes/{id}/ttl. Read `{duration: "2h"}` from JSON body. Parse duration. Delete existing EventBridge schedule via `pkg/aws.DeleteTTLSchedule()`, create new schedule at `now + duration` via `pkg/aws.CreateTTLSchedule()`. Return 200 with updated TTL remaining. Need to add SchedulerAPI to Handler struct (narrow interface already exists in pkg/aws/scheduler.go).

- `handleQuickCreate(w, r)` -- POST /api/sandboxes/create. Read `{profile: "filename.yaml"}` from JSON body. Shell out to `km create profiles/{profile}` as subprocess. Return 202 with message "Sandbox creation started". Long-running -- the HTMX response can trigger a polling refresh of the dashboard.

2. Wire new routes in main.go:
- `POST /api/sandboxes/{id}/destroy` -> handleDestroy
- `PUT /api/sandboxes/{id}/ttl` -> handleExtendTTL
- `POST /api/sandboxes/create` -> handleQuickCreate
- Add SchedulerAPI client to Handler initialization
