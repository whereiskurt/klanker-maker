  <behavior>
    - Test: POST /api/sandboxes/{id}/destroy returns 200 and calls Destroyer with correct sandbox ID
    - Test: POST /api/sandboxes/{id}/destroy for unknown sandbox returns 404
    - Test: PUT /api/sandboxes/{id}/ttl with valid JSON body {"duration":"2h"} returns 200 with updated TTL
    - Test: PUT /api/sandboxes/{id}/ttl with invalid duration returns 400
    - Test: POST /api/sandboxes/create with {"profile":"base.yaml"} returns 202
    - Test: POST /api/sandboxes/create with nonexistent profile returns 400
  </behavior>
  <action>
1. Add action handlers to `cmd/configui/handlers.go`:

Define narrow interfaces for testability:
- `Destroyer` interface with `Destroy(ctx context.Context, sandboxID string) error` -- wraps the destroy logic. In production, this shells out to `km destroy {id}` as a subprocess (simpler than importing internal packages with Cobra dependencies). In tests, use a mock that records calls.
- `TTLExtender` interface with `ExtendTTL(ctx context.Context, sandboxID string, duration time.Duration) error` -- wraps delete + recreate EventBridge schedule via `pkg/aws.DeleteTTLSchedule()` and `pkg/aws.CreateTTLSchedule()`.
- `SandboxCreator` interface with `Create(ctx context.Context, profilePath string) error` -- wraps `km create profiles/{profile}` subprocess call.

Add these fields to the Handler struct.

Implement handlers:
- `handleDestroy(w, r)` -- POST /api/sandboxes/{id}/destroy. Get sandbox ID from path. Call Destroyer.Destroy(). Return 200 on success with HTMX `HX-Trigger: sandbox-destroyed` header to refresh table. Return 500 with error message on failure.
- `handleExtendTTL(w, r)` -- PUT /api/sandboxes/{id}/ttl. Read `{duration: "2h"}` from JSON body. Parse duration via time.ParseDuration. Call TTLExtender.ExtendTTL(). Return 200 with updated TTL remaining. Return 400 for invalid duration format.
- `handleQuickCreate(w, r)` -- POST /api/sandboxes/create. Read `{profile: "filename.yaml"}` from JSON body. Validate the profile file exists in profilesDir. Call SandboxCreator.Create(). Return 202 with message "Sandbox creation started". Return 400 if profile file not found.

2. Create `cmd/configui/handlers_actions_test.go`:
- Define mock structs for Destroyer, TTLExtender, SandboxCreator that record calls and return configurable errors
- Use `os.MkdirTemp` for a test profilesDir, pre-populate with a `base.yaml` file for the create test
- Test all behaviors listed above using httptest.NewRequest/NewRecorder
- No real AWS calls -- all interactions go through the narrow mock interfaces

3. Wire new routes in main.go:
- `POST /api/sandboxes/{id}/destroy` -> handleDestroy
- `PUT /api/sandboxes/{id}/ttl` -> handleExtendTTL
- `POST /api/sandboxes/create` -> handleQuickCreate
- Create production implementations of Destroyer/TTLExtender/SandboxCreator that shell out to km CLI or call pkg/aws directly
- Add SchedulerAPI client to Handler initialization for TTLExtender
