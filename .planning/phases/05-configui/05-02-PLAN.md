    - Test: POST /api/validate with invalid YAML returns 200 + non-empty error array with path and message
    - Test: POST /api/validate with empty body returns 400
    - Test: GET /api/profiles returns JSON array of profile filenames from profiles/ directory
    - Test: GET /api/profiles/{name} returns raw YAML content of the named profile
    - Test: GET /api/profiles/{name} for nonexistent profile returns 404
    - Test: PUT /api/profiles/{name} writes YAML body to profiles/{name} file
    - Test: PUT /api/profiles/{name} with path traversal (../) returns 400
    - Test: GET /editor returns 200 with full editor page
  </behavior>
  <action>
1. Create `cmd/configui/handlers_editor.go` -- profile editor handlers as methods on Handler:

- `handleEditorPage(w, r)` -- renders editor.html full page. If `?profile=name` query param present, loads that profile's content into template data.
- `handleValidate(w, r)` -- POST only. Reads body (max 64KB via io.LimitReader). Calls `profile.Validate(raw)`. Returns JSON array of ValidationError. Empty array on valid input.
- `handleProfileList(w, r)` -- reads `h.profilesDir`, returns JSON array of `{name, hasExtends}` for each .yaml file. Check each file for `extends:` line to populate hasExtends flag.
- `handleProfileGet(w, r)` -- `r.PathValue("name")` to get profile name. Sanitize: reject if contains `..` or `/`. Read file from `h.profilesDir/{name}`. Return raw YAML bytes with `text/yaml` content type.
- `handleProfileSave(w, r)` -- PUT. `r.PathValue("name")` to get name. Sanitize same as above. Read body (max 64KB). Run `profile.Validate()` first -- if errors, return 422 with error JSON (save still succeeds but warns). Write to `h.profilesDir/{name}` using `os.WriteFile` with 0644 permissions. Return 200.

2. Create `cmd/configui/handlers_editor_test.go` -- use `os.MkdirTemp` for a test profiles directory. Populate with sample YAML files. Use httptest.NewRequest + httptest.NewRecorder for each test case. Verify status codes, content types, and response body content.
