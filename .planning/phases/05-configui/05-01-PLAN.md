2. Create `cmd/configui/handlers.go` -- define the Handler struct and all HTTP handler methods:

```go
type Handler struct {
    tmpl        *template.Template
    lister      SandboxLister    // narrow interface for ListAllSandboxesByS3
    finder      SandboxFinder    // narrow interface for FindSandboxByID
    profilesDir string
}
```

Define narrow interfaces following project convention:
- `SandboxLister` interface with `ListSandboxes(ctx, bucket) ([]SandboxRecord, error)` -- wraps ListAllSandboxesByS3
- `SandboxFinder` interface with `FindSandbox(ctx, sandboxID) (*SandboxLocation, error)` -- wraps FindSandboxByID

Implement handlers:
- `handleDashboard(w, r)` -- full page or HTMX partial (check `HX-Request` header). Calls lister, renders dashboard.html or sandbox_rows partial.
- `handleSandboxesPartial(w, r)` -- same as handleDashboard but always returns rows partial (used by HTMX polling endpoint).
- `handleSandboxDetail(w, r)` -- `r.PathValue("id")` to get sandbox ID, calls finder, renders sandbox_detail partial. Returns 404 if ErrSandboxNotFound.
- `handleSchema(w, r)` -- serves `profile.SchemaJSON()` as application/json with Cache-Control header.
- Helper: `isHTMXRequest(r) bool` checks `HX-Request` header.
- Helper: `renderTemplate(w, name, data)` wraps tmpl.ExecuteTemplate with error handling.

3. Create `cmd/configui/handlers_test.go` -- use `net/http/httptest` with mock lister/finder that return canned SandboxRecord/SandboxLocation data. Test all behaviors listed above. Parse templates from testdata or use minimal inline template strings for test isolation.

4. Create `cmd/configui/main.go` -- the server entrypoint:
- `//go:embed templates` and `//go:embed static` for asset embedding
- Parse templates with `template.Must(template.ParseFS(...))`
- Create AWS clients (S3, ResourceGroupsTaggingAPI) using `config.LoadDefaultConfig`

  - `GET /` -> handleDashboard

- Listen on CONFIGUI_ADDR env var (default `:8080`)
- Use zerolog for structured request logging (project standard)
- Accept `--bucket` flag or `KM_BUCKET` env var for the S3 bucket name (default `tf-km`)
