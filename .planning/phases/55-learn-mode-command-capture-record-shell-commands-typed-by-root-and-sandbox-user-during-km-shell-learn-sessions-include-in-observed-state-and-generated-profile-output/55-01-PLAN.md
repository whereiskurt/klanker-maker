  <name>Task 1: Add RecordCommand/Commands to Recorder with tests</name>
  <files>pkg/allowlistgen/recorder.go, pkg/allowlistgen/recorder_test.go</files>
  <behavior>
    - RecordCommand("apt install curl") records the command
    - RecordCommand("") is a no-op (empty/whitespace ignored)
    - RecordCommand("apt install curl") called twice stores only once (dedup)
    - Commands() returns commands in first-seen order, not alphabetical
    - RecordCommand("cmd A"), RecordCommand("cmd B"), RecordCommand("cmd A") -> Commands() returns ["cmd A", "cmd B"]
    - Commands() on empty Recorder returns empty slice (not nil)
    - RecordCommand is safe for concurrent use (mutex-protected)
  </behavior>
  <action>
Add two new fields to the Recorder struct:
- `commandSeen map[string]struct{}` for O(1) dedup lookup
- `commandOrdered []string` for preserving first-seen order

Initialize both in NewRecorder().

Add `RecordCommand(cmd string)` method:
- Trim whitespace, return if empty
- Lock mutex
- If not in commandSeen, add to both commandSeen and commandOrdered
- Unlock

Add `Commands() []string` method:
- Lock mutex, copy commandOrdered to new slice, unlock, return
- Return `[]string{}` (not nil) when empty — matches pattern of other accessors

Follow the exact same mutex pattern as RecordDNSQuery/DNSDomains. Commands() should NOT sort (unlike DNSDomains/Hosts/Repos) because order is semantically meaningful for commands.

Add tests in recorder_test.go (can extend existing file or add new test functions):
- TestRecordCommand_Basic: record 3 commands, verify Commands() returns them in order
- TestRecordCommand_Dedup: record same command twice, verify only one in output
- TestRecordCommand_Empty: record empty/whitespace strings, verify ignored
- TestRecordCommand_OrderPreserved: record A, B, A again, verify [A, B] not [A, A, B]
  </action>

  </verify>
  <done>RecordCommand deduplicates while preserving order; Commands() returns ordered first-seen slice; all tests pass</done>
