<objective>
Wire command capture into the EC2 and Docker learn mode integration paths.

Purpose: Plan 01 built the Recorder/Generator library layer. This plan connects it to the actual capture mechanisms: the userdata PROMPT_COMMAND hook writes commands to a log file, the eBPF enforcer reads it at flush time, and the operator-side code (shell.go) round-trips the Commands field through JSON and feeds it to the Recorder for profile generation. Docker path reads from the audit-log container.

Output: Complete end-to-end command capture for both EC2 and Docker learn sessions.
</objective>
