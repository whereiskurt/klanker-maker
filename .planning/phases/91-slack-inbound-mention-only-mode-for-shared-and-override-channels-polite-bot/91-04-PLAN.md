<objective>
Teach `km slack init` and `km slack rotate-token` to capture and cache the bot's `user_id` from
the `auth.test` response into SSM at `{prefix}slack/bot-user-id`. POL-07 + POL-08.

Purpose: The bridge Lambda's mention-scan needs the bot user ID. The existing
`CachedBotUserIDFetcher` already fetches it lazily, but the SSM cache enables:
1. `km doctor` to sanity-check the cache without a live Lambda cold-start (Plan 05)
2. Plan 03's `KM_SLACK_BOT_USER_ID` env-var injection to prime the cache before any Slack API call
3. Operators to inspect the cached value via `aws ssm get-parameter`

This plan adds the SSM write half of the pipeline — Plan 03 reads it via `get_env` in terragrunt.

Output: New `AuthTestWithUserID` method on the Slack client + interface extension + SSM write in
both init flows + two live tests.
</objective>

    // ok=false; caller decides whether to fail the whole flow or warn and continue.
    func (c *Client) AuthTestWithUserID(ctx context.Context) (string, error) {
        var resp struct {
            OK     bool   `json:"ok"`
            Error  string `json:"error,omitempty"`
            UserID string `json:"user_id"`
        }

            return "", err
        }
        if !resp.OK {
            return "", &SlackAPIError{Method: "auth.test", Code: resp.Error}

        return resp.UserID, nil
    }
