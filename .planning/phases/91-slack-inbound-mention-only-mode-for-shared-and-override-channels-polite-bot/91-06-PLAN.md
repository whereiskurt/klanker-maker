       a) **Overview** (1 paragraph): the km-slack bridge previously processed every inbound
          message; in a corporate Slack workspace with the bot installed into a shared channel

          for shared (Mode 1) and operator-override (Mode 3) channels, the bot only reacts when
          the message text contains `<@{bot_user_id}>`. Per-sandbox `#sb-{id}` channels (Mode 2)
          keep current every-message behaviour because the bot is the primary participant.

       b) **Per-mode defaults table:**
          | Mode | Channel | Default `mention_only` | Why |
          | --- | --- | --- | --- |
          | 1 | shared (e.g. `#km-notifications`) | `true` | reduce noise in corporate channels |
          | 2 | per-sandbox `#sb-{id}` | `false` | bot is the primary participant |
          | 3 | operator override (`notifySlackChannelOverride`) | `true` | operator controls the channel |

       c) **Profile field reference:**
          ```yaml
          spec:
            cli:
              notifySlackEnabled: true
              # Tri-state: nil = mode default, true = force polite, false = force chatty
              notifySlackInboundMentionOnly: true
          ```

       d) **Override examples** (3-4 yaml fixtures showing common operator intents):
          - Force polite in a per-sandbox channel (rare; e.g. multiple operators sharing a sandbox)
          - Force chatty in a shared channel (testing only)
          - Default Mode 1 polite (just notifySlackEnabled, nothing else)

       e) **Bridge env vars (locked):**
          ```
          KM_SLACK_MENTION_ONLY  — "true"|"false"; default "false" (back-compat)
          KM_SLACK_BOT_USER_ID   — bot user_id for mention scan; primes CachedBotUserIDFetcher
          ```
          Both injected via the lambda-slack-bridge Terraform module's environment block. Operators
          export `KM_SLACK_MENTION_ONLY=true` (and optionally `KM_SLACK_BOT_USER_ID=$(aws ssm get-parameter --name /km/slack/bot-user-id --query Parameter.Value --output text)`) before `km init --sidecars`.

       f) **New `km doctor` check:** `Slack bot-user-id cache` — WARNs when at least one local
          profile has mention-only effective AND `{prefix}slack/bot-user-id` is unset/empty.
          Remediation: `km slack init --force` (re-runs auth.test, caches user_id).

       g) **Rollout sequence** (after upgrading km to a Phase 91 build):
          ```sh
          make build                                 # rebuild km binary with new schema
          export KM_SLACK_MENTION_ONLY=true          # opt-in to polite-bot (install-level default)
          km slack init --force                      # re-runs auth.test, caches bot_user_id at SSM
          export KM_SLACK_BOT_USER_ID=$(aws ssm get-parameter --name /km/slack/bot-user-id --query Parameter.Value --output text)
          km init --sidecars                         # redeploys bridge Lambda with new env vars
          km doctor                                  # verifies Slack bot-user-id cache OK
          # For sandboxes created before Phase 91:
          km destroy <sandbox-id> && km create <profile>   # picks up the new field
          ```

          - "`km doctor` reports `Slack bot-user-id cache` WARN" → run `km slack init --force` to
            re-cache; verify with `aws ssm get-parameter --name /km/slack/bot-user-id`.

          - "Mention-scan never fires; cold-start is slow" → confirm `KM_SLACK_BOT_USER_ID` is
