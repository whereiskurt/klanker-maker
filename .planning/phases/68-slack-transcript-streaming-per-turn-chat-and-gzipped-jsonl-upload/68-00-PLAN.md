**File 3 — `pkg/compiler/testdata/notify-hook-stub-km-slack.sh`:**
Mirror the existing `notify-hook-stub-km-send.sh` pattern. Records the subcommand (`post` / `upload` / `record-mapping`) + flags + body to a log file the test reads back. Returns deterministic success JSON for `post` (so the auto-thread-parent path captures a ts) and exit-0 for `upload` / `record-mapping`.

```sh
#!/usr/bin/env bash
# Stub for tests: records subcommand + flags + stdin body to KM_SLACK_STUB_LOG.
# Mirrors pkg/compiler/testdata/notify-hook-stub-km-send.sh.
set -euo pipefail
log="${KM_SLACK_STUB_LOG:-/tmp/km-slack-stub.calls}"
{
  echo "---"
  echo "subcommand: ${1:-}"
  shift || true
  echo "args: $*"
} >> "$log"
case "${1:-}" in
  post)
    # post returns a deterministic ts (used for auto-thread-parent caching)
    echo '{"ok":true,"ts":"1700000000.000100"}'
    ;;
  upload|record-mapping|*)
    # both succeed silently
    :
    ;;
esac
exit 0
```
