<task type="auto">
  <name>Task 2: Update Terraform ec2spot module with AMI resolution, root_block_device, and hibernation</name>
  <files>infra/modules/ec2spot/v1.0.0/variables.tf, infra/modules/ec2spot/v1.0.0/main.tf</files>
  <action>
    1. In `variables.tf`, add three new module-level variables:
       ```hcl
       variable "root_volume_size_gb" {
         type        = number
         description = "Root EBS volume size in GB. 0 = use AMI default."
         default     = 0
       }

       variable "hibernation_enabled" {
         type        = bool
         description = "Enable EC2 hibernation. Requires on-demand instances and encrypted root volume."
         default     = false
       }

       variable "ami_slug" {
         type        = string
         description = "AMI slug for base image resolution. Supported: amazon-linux-2023, ubuntu-24.04, ubuntu-22.04"
         default     = "amazon-linux-2023"
       }
       ```

    2. In `main.tf`, replace the hardcoded `data "aws_ami" "base_ami"` block with a parameterized version using a locals map:
       ```hcl
       locals {
         ami_filters = {
           "amazon-linux-2023" = {
             name_pattern = "al2023-ami-2023.*-x86_64"
             owner        = "amazon"
           }
           "ubuntu-24.04" = {
             name_pattern = "ubuntu/images/hvm-ssd-gp3/ubuntu-noble-24.04-amd64-server-*"
             owner        = "099720109477"
           }
           "ubuntu-22.04" = {
             name_pattern = "ubuntu/images/hvm-ssd/ubuntu-jammy-22.04-amd64-server-*"
             owner        = "099720109477"
           }
         }
         resolved_ami_slug = var.ami_slug != "" ? var.ami_slug : "amazon-linux-2023"
       }
       ```
       Update `data "aws_ami" "base_ami"` to use `local.ami_filters[local.resolved_ami_slug].name_pattern` for the name filter and `local.ami_filters[local.resolved_ami_slug].owner` for owners. Keep `most_recent = true`, architecture `x86_64`, and virtualization-type `hvm`.

    3. Add `root_block_device` dynamic block to `aws_spot_instance_request.ec2spot`:
       ```hcl
       dynamic "root_block_device" {
         for_each = var.root_volume_size_gb > 0 ? [1] : []
         content {
           volume_size           = var.root_volume_size_gb
           volume_type           = "gp3"
           delete_on_termination = true
         }
       }
       ```
       Note: No `encrypted` or `hibernation` on spot instances (AWS does not support hibernation for spot).

    4. Add `root_block_device` dynamic block AND `hibernation` argument to `aws_instance.ec2_ondemand`:
       ```hcl
       dynamic "root_block_device" {
         for_each = var.root_volume_size_gb > 0 || var.hibernation_enabled ? [1] : []
         content {
           volume_size           = var.root_volume_size_gb > 0 ? var.root_volume_size_gb : null
           volume_type           = "gp3"
           encrypted             = var.hibernation_enabled
           delete_on_termination = true
         }
       }

       hibernation = var.hibernation_enabled
       ```
       The `root_block_device` is also rendered when `hibernation_enabled` is true (even without explicit size) because hibernation requires `encrypted = true` on the root volume.
  </action>
