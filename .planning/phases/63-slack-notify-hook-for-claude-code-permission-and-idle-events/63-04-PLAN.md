<objective>
Extend the Phase 62 inline `km-notify-hook` heredoc in `pkg/compiler/userdata.go` for multi-channel dispatch, and add four new env-var keys to the `/etc/profile.d/km-notify-env.sh` emission. The hook script gains a `sent_any` variable, parallel email + Slack branches gated on `KM_NOTIFY_EMAIL_ENABLED` and `KM_NOTIFY_SLACK_ENABLED`, and the cooldown updates iff at least one channel succeeded. RESEARCH.md correction #4 (`*bool` semantics for `notifyEmailEnabled`/`notifySlackEnabled`) is operationalized here: nil pointer means "don't emit the env var, let the hook's `:-1` default keep email on" — this is what preserves Phase 62 backward compat.

Purpose: Wire the schema additions from Plan 01 into the compiler so a profile with `notifySlackEnabled: true` produces a sandbox whose `/etc/profile.d/km-notify-env.sh` carries `KM_NOTIFY_SLACK_ENABLED=1`, `KM_SLACK_CHANNEL_ID=…`, `KM_SLACK_BRIDGE_URL=…`, and whose `/opt/km/bin/km-notify-hook` knows to dispatch to both email and Slack.

This plan does NOT include the `km-slack` binary (that's Plan 05) or the runtime injection of channel ID and bridge URL (those land in Plan 08 `km create`). What it DOES do is make the compiler ready: a profile with `notifySlackEnabled: true` and `notifySlackChannelOverride: "C0123ABC"` already produces a working env file with the override channel pinned at compile time.

Output:
- Phase 62 hook heredoc replaced with the multi-channel `sent_any` body from CONTEXT.md.
- `/etc/profile.d/km-notify-env.sh` emission extended with the four new keys, conditional on the `*bool` pointer states.
- Tests cover: Slack-only profile, email-only profile, both-on profile, both-off profile, *bool unset (Phase 62 backward compat path), `notifySlackChannelOverride` baked in at compile time.
</objective>
