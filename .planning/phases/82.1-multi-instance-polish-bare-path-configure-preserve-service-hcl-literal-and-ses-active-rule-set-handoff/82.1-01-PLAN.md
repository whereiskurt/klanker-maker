<objective>
Extend the `km configure` preserve-on-re-run guard at `configure.go:145` to also fire when `--output-dir` is absent (the default operator workflow). Currently the guard condition is `outputDir != ""`, which silently skips the existing-config read on every bare invocation, meaning operators who run `km configure` without `--output-dir` to rotate an unrelated field can reset their `resource_prefix` back to "km" without warning.

Purpose: Close Gap 1 from Phase 82 VERIFICATION.md. The CLAUDE.md claim "Re-running `km configure` preserves the existing `resource_prefix`" must be true for ALL invocation modes, not just the --output-dir path.

Output: Modified `configure.go` with an extended preserve guard; two new TDD tests in `configure_test.go` covering the bare-path scenario.
</objective>
