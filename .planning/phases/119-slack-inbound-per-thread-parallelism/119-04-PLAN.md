<objective>
Layer 2 — the novel piece: rewrite the sandbox-side Slack inbound poller bash in
pkg/compiler/userdata.go from a serial blocking executor into a BOUNDED-CONCURRENT
dispatcher. This is the only behaviorally-novel work in the phase; everything else mirrors
existing patterns. It is bash embedded in a Go heredoc → invisible to Go goldens → the
live E2E in Plan 05 is the real gate (Research Pitfall 6).

Five composed changes, all inside the poller loop (~userdata.go:1670–2122):

1. **Counting semaphore (P119-B):** batch-receive up to N = min(cap,10); dispatch each message
   in a `( ... ) &` subshell; block with `wait -n || true` + an in-flight counter so ≤ cap run
   at once. `MAX="${KM_SLACK_MAX_CONCURRENCY:-1}"` (default 1 ⇒ serial). bash ≥ 4.3 guaranteed on
   all AMIs → `wait -n` is safe (no fallback idiom).
2. **Ack-after-completion (P119-D):** move the `delete-message` from its current spot (~:2072,
   before the post) to the very END of the subshell, AFTER `km-slack post` succeeds. The message
   stays in-flight for the whole turn so FIFO won't release the thread's next message early. This
   reverses the L2069 "Ack first" comment — update it.
3. **Visibility heartbeat (P119-E):** replace the one-shot 300s extension (~:1701) with a per-message
   background ticker that calls `change-message-visibility --visibility-timeout 360` every ~120s
   until the turn finishes; `kill` it after the turn. `|| true` swallows the harmless
   ReceiptHandleIsInvalid after delete.
4. **RUN_ID uniqueness (Pitfall 1):** `RUN_ID="$(date -u +%Y%m%dT%H%M%SZ)-$$-$RANDOM"` computed
   INSIDE the subshell so concurrent jobs never clobber `/workspace/.km-agent/runs/$RUN_ID`.
5. **Idempotency guard / option C (P119-F):** read `last_processed_event_ts` off the km-slack-threads
   row at turn start; if it == this message's inbound event ts/id, SKIP the turn + post (delete only).
   After a successful post, write `last_processed_event_ts` = this event's ts in the same put-item
   that already writes the row (~:2055). Fail-OPEN (a read error → process normally). Source the
   event id from the SQS message body / dedup id — see the implementation note.

cap=1 MUST degenerate to today's serial loop (MAX=1 ⇒ at most one in-flight ⇒ effectively serial,
ack-after still applies which is correct/harmless serially).

Purpose: P119-B/D/E/F — the runtime concurrency contract.
Output: rewritten poller bash + hand-patched frozen golden.
</objective>
