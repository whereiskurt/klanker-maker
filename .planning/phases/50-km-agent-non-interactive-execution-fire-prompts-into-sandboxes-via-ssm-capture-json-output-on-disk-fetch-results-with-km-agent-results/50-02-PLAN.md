    - AGENT-05: `km agent list sb-abc123` lists all runs as a table: RUN_ID | STATUS | OUTPUT_SIZE

    - Results fetches output via SSM SendCommand `cat` of the output file (not via SSM stdout capture, which truncates at 24KB)
  </behavior>
  <action>
    1. Add `results` subcommand to the agent parent command in `NewAgentCmd`:
       - `km agent results <sandbox-id> [--run <run-id>] [--json]`
       - `--run` (string): specific run ID (timestamp). If omitted, fetches latest.
       - `--json` (bool): output raw JSON without formatting (default: true since output IS json)
       - Implementation (`runAgentResults`):
         a. Fetch sandbox, check status, extract instance ID (same pattern as run subcommand)
         b. If `--run` is specified, use that run ID directly
         c. If `--run` is omitted, first send SSM command to find latest run:
            `sudo -u sandbox bash -c 'ls -1t /workspace/.km-agent/runs/ 2>/dev/null | head -1'`
         d. Send SSM command to cat the output file:
            `sudo -u sandbox cat /workspace/.km-agent/runs/<RUN_ID>/output.json`
         e. Poll GetCommandInvocation, print StandardOutputContent to stdout
         f. NOTE: SSM truncates at 24KB. For this phase, accept the limitation. If output is truncated (check if JSON is valid), print a warning. Future enhancement: S3 upload for large outputs.

    2. Add `list` subcommand to the agent parent command:
       - `km agent list <sandbox-id>`
       - Implementation (`runAgentList`):
         a. Fetch sandbox, check status, extract instance ID
         b. Send SSM command:
            ```
            sudo -u sandbox bash -c '
              for d in /workspace/.km-agent/runs/*/; do
                [ -d "$d" ] || continue
                id=$(basename "$d")
                status=$(cat "$d/status" 2>/dev/null || echo "unknown")
                size=$(stat -f%z "$d/output.json" 2>/dev/null || stat -c%s "$d/output.json" 2>/dev/null || echo "0")
                echo "$id|$status|$size"
              done
            '
            ```
            Note: Use `stat -c%s` (Linux) since sandboxes are EC2 Linux instances. The macOS `stat -f%z` is not needed on sandbox side.
         c. Poll GetCommandInvocation, parse pipe-delimited output
         d. Print formatted table: RUN_ID | STATUS | SIZE

    3. Add tests to `agent_test.go`:
       - `TestAgentResults`: mock SSM returns output.json content, verify it's printed to stdout
       - `TestAgentResults_SpecificRun`: verify --run flag uses the specified run ID in the cat command
       - `TestAgentResults_NoRuns`: mock SSM returns empty output for ls command, verify "no agent runs found" message
       - `TestAgentList`: mock SSM returns pipe-delimited run entries, verify table output

    Both subcommands use the same `SSMSendAPI` interface defined in Plan 01 for dependency injection.
