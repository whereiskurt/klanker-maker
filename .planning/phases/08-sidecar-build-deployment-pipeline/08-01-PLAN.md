<task type="auto">
  <name>Task 1: Create Dockerfiles for all 4 sidecars</name>
  <files>sidecars/dns-proxy/Dockerfile, sidecars/http-proxy/Dockerfile, sidecars/audit-log/Dockerfile, sidecars/tracing/Dockerfile</files>
  <action>
Create multi-stage Dockerfiles for the 3 Go sidecars and a simple Dockerfile for the tracing sidecar.

For dns-proxy and http-proxy Dockerfiles:
- Stage 1 (builder): FROM golang:1.25-alpine, WORKDIR /build, COPY go.mod go.sum and run go mod download, COPY . . (full module root), RUN CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -o /sidecar ./sidecars/{name}/
- Stage 2 (final): FROM scratch, COPY --from=builder /sidecar /sidecar, ENTRYPOINT ["/sidecar"]

For audit-log Dockerfile:
- Same pattern but build target is ./sidecars/audit-log/cmd/ (NOT ./sidecars/audit-log/ which is package auditlog, not package main)

For tracing Dockerfile:
- FROM otel/opentelemetry-collector-contrib:latest
- COPY config.yaml /etc/otelcol-contrib/config.yaml

CRITICAL: All Go sidecar Dockerfiles expect the repo root as Docker build context (invoked with -f sidecars/{name}/Dockerfile .). This is because sidecars import pkg/aws, pkg/lifecycle, etc.
  </action>
  <verify>
    <automated>for f in sidecars/dns-proxy/Dockerfile sidecars/http-proxy/Dockerfile sidecars/audit-log/Dockerfile sidecars/tracing/Dockerfile; do test -f "$f" && echo "OK: $f" || echo "MISSING: $f"; done</automated>
  </verify>
  <done>All 4 Dockerfiles exist. Go sidecar Dockerfiles use multi-stage build with golang:1.25-alpine builder and scratch final. Tracing Dockerfile uses otelcol-contrib base. audit-log Dockerfile builds from ./sidecars/audit-log/cmd/ path.</done>

<task type="auto">
  <name>Task 2: Create root Makefile with sidecars and ecr-push targets</name>
  <files>Makefile, .gitignore</files>
  <action>
Create a Makefile at the repo root with the following targets:

Variables at top:
- GOOS := linux, GOARCH := amd64, CGO_ENABLED := 0
- VERSION ?= latest
- REGION ?= $(shell aws configure get region)
- ACCOUNT_ID = $(shell aws sts get-caller-identity --query Account --output text)
- ECR_REGISTRY = $(ACCOUNT_ID).dkr.ecr.$(REGION).amazonaws.com
- KM_ARTIFACTS_BUCKET ?= (from env, no default — guarded)
- SIDECARS := dns-proxy http-proxy audit-log

Target: `sidecars` (phony)
1. Guard: fail with clear error if KM_ARTIFACTS_BUCKET is empty
2. Cross-compile each sidecar binary into build/ directory:
   - go build -o build/dns-proxy ./sidecars/dns-proxy/
   - go build -o build/http-proxy ./sidecars/http-proxy/
   - go build -o build/audit-log ./sidecars/audit-log/cmd/  (note cmd/ subdirectory)
3. Upload all 3 binaries to s3://$(KM_ARTIFACTS_BUCKET)/sidecars/{name}
4. Upload sidecars/tracing/config.yaml to s3://$(KM_ARTIFACTS_BUCKET)/sidecars/tracing/config.yaml
5. Print success message with S3 paths

Target: `ecr-push` (phony, depends on ecr-login and ecr-repos)
1. docker buildx build --platform linux/amd64 for each of the 4 sidecars
2. Each build uses -f sidecars/{name}/Dockerfile . (repo root context)
3. Tags: $(ECR_REGISTRY)/km-{name}:$(VERSION)
4. Uses --push flag

Target: `ecr-login` (phony)
- aws ecr get-login-password | docker login

Target: `ecr-repos` (phony)
- For each of km-dns-proxy, km-http-proxy, km-audit-log, km-tracing: describe-repositories || create-repository

Target: `build-sidecars` (phony) — cross-compile only, no S3 upload (for local testing)
- Same go build commands as sidecars target but no S3 upload

Also update .gitignore: add `build/` line if not already present (it is not — current .gitignore only ignores root-level binary names like /dns-proxy).
  </action>
  <verify>
    <automated>test -f Makefile && grep -q "^sidecars:" Makefile && grep -q "^ecr-push:" Makefile && grep -q "build/" .gitignore && echo "OK" || echo "FAIL"</automated>
  </verify>
  <done>Makefile exists at repo root with sidecars, ecr-push, ecr-login, ecr-repos, and build-sidecars targets. KM_ARTIFACTS_BUCKET is guarded. audit-log builds from cmd/ subdirectory. .gitignore includes build/ directory.</done>
</task>
