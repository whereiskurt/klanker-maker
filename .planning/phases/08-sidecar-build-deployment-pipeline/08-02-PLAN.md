<objective>
Fix the ECS service.hcl compiler to emit resolvable ECR image URIs instead of literal `${var.dns_proxy_image}` strings that Terraform cannot resolve.

Purpose: Currently, `km create` for ECS substrate produces service.hcl with `image = "${var.dns_proxy_image}"` — but the ECS Terraform module declares no such variable. This causes `terragrunt plan` to fail. The compiler must produce real URIs like `123456789012.dkr.ecr.us-east-1.amazonaws.com/km-dns-proxy:latest`.

Output: Fixed service_hcl.go + new test
</objective>
