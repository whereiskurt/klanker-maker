# Plan 45-01: Multipart MIME support in pkg/aws

**Wave 1** — Foundation. Both bash scripts and Go CLI depend on the Go library understanding multipart MIME with attachments.

## Goal

Extend `buildRawMIME` to produce multipart/mixed MIME when attachments are present, and extend `ParseSignedMessage` to extract attachments from multipart MIME messages. Signature still covers only the text body part.

## Tasks

### Task 1: Extend buildRawMIME for multipart/mixed

**File:** `pkg/aws/identity.go`

- Add `attachments []Attachment` parameter to `buildRawMIME` (new `Attachment` struct: `Filename string`, `Data []byte`)
- When `len(attachments) == 0`: current behavior (single-part text/plain)
- When `len(attachments) > 0`: produce multipart/mixed with:
  - Part 1: text/plain body (the signed content)
  - Part N+1: application/octet-stream, Content-Disposition: attachment; filename="...", Content-Transfer-Encoding: base64
- Generate MIME boundary using crypto/rand

### Task 2: Extend SendSignedEmail for attachments

**File:** `pkg/aws/identity.go`

- Add `attachments []Attachment` parameter to `SendSignedEmail`
- Pass attachments through to `buildRawMIME`
- Signature still covers `bodyToSign` only (text body, not attachments)
- Update all existing callers (if any) to pass `nil` for attachments

### Task 3: Extend ParseSignedMessage for multipart MIME

**File:** `pkg/aws/mailbox.go`

- Add `Attachments []Attachment` field to `MailboxMessage` struct
- In `ParseSignedMessage`: detect multipart/mixed Content-Type
- If multipart: iterate parts, first text/plain part becomes Body, remaining parts with Content-Disposition: attachment become Attachments
- If not multipart: current behavior (body = full message body)
- Signature verification uses the text body part only

### Task 4: Tests for multipart buildRawMIME

**File:** `pkg/aws/identity_test.go`

- Test: buildRawMIME with no attachments produces single-part (backward compat)
- Test: buildRawMIME with 1 attachment produces valid multipart/mixed
- Test: buildRawMIME with 2 attachments, verify boundary, Content-Disposition headers
- Test: signature covers body only, not attachment data

### Task 5: Tests for multipart ParseSignedMessage

**File:** `pkg/aws/mailbox_test.go`

- Test: ParseSignedMessage with multipart/mixed extracts body + attachments
- Test: ParseSignedMessage with single-part (backward compat)
- Test: signature verification on multipart message uses body part only
- Test: encrypted + multipart message (body is ciphertext, attachments present)

### Task 6: Round-trip test

**File:** `pkg/aws/identity_test.go`

- Build multipart MIME with buildRawMIME (body + 2 attachments)
- Parse with ParseSignedMessage
- Verify body matches, attachments match, signature verifies
