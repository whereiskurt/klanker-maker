# Plan 45-04: km email send / km email read Go CLI commands

**Wave 3** — Depends on Plan 01 (multipart MIME in pkg/aws). Independent of Plans 02/03 (bash scripts).

## Goal

Add `km email send` and `km email read` Cobra subcommands for operator-side orchestration of inter-sandbox signed email with attachments, authoritative Ed25519 verification, and auto-decryption.

## Tasks

### Task 1: Create email parent command

**File:** `internal/app/cmd/email.go` (new file)

- `NewEmailCmd(cfg)` returns a `*cobra.Command` with `Use: "email"`, `Short: "Send and read signed sandbox email"`
- Parent command only — no RunE, just holds subcommands
- Register `send` and `read` subcommands

### Task 2: km email send command

**File:** `internal/app/cmd/email.go`

`km email send --from <sandbox-id> --to <sandbox-id> --subject <subject> --body <file> [--attach file1,file2,...]`

Flags:
- `--from` (required): sender sandbox ID — used to fetch signing key from SSM
- `--to` (required): recipient sandbox ID — resolved to email address via DynamoDB identity lookup
- `--subject` (required): email subject line
- `--body` (required): path to file containing email body (use `-` for stdin)
- `--attach`: comma-separated list of file paths for attachments

Logic:
1. Validate sandbox ID format for both `--from` and `--to`
2. Read body from file (or stdin if `-`)
3. Read attachment files into `[]kmaws.Attachment`
4. Resolve sender email: `{from}@{emailDomain}` (emailDomain from config)
5. Resolve recipient email: `{to}@{emailDomain}`
6. Look up sender's identity record to get encryption policy
7. Call `kmaws.SendSignedEmail(...)` with attachments
8. Print confirmation: `Sent signed email from {from} to {to} (subject: {subject}, attachments: N)`

### Task 3: km email read command

**File:** `internal/app/cmd/email.go`

`km email read <sandbox-id> [--json] [--raw]`

Args:
- `sandbox-id` (required, positional): the sandbox whose mailbox to read

Flags:
- `--json`: output as JSON array
- `--raw`: dump raw MIME bytes (one message per invocation — requires `--message-id` or reads latest)

Logic:
1. Validate sandbox ID format
2. Get artifacts bucket from config
3. Get email domain from config
4. Call `kmaws.ListMailboxMessages(...)` to get S3 keys
5. For each message:
   a. `kmaws.ReadMessage(...)` to get raw MIME
   b. Fetch sender's public key from DynamoDB (if X-KM-Sender-ID present)
   c. `kmaws.ParseSignedMessage(...)` with the sender's public key
   d. **Auto-decrypt**: if `msg.Encrypted && !msg.Plaintext`:
      - Fetch recipient's X25519 private key from SSM
      - Fetch recipient's X25519 public key from DynamoDB
      - Call `kmaws.DecryptFromSender(privKey, pubKey, ciphertext)`
      - Replace body with decrypted plaintext
   e. Save attachments info for display
6. Output format:
   - Default: human-readable table (index, from, subject, sig status, encrypted, body preview)
   - `--json`: JSON array of message objects
   - `--raw`: raw MIME bytes to stdout
