# Plan 45-03: In-sandbox km-recv bash script

**Wave 2** — Depends on Plan 01 (MIME contract). Can run in parallel with Plan 02.

## Goal

Create the `/opt/km/bin/km-recv` bash script that reads and displays signed emails from `/var/mail/km/new/*` with best-effort signature verification. Pure bash + AWS CLI + openssl.

## Tasks

### Task 1: Write km-recv script

**File:** `pkg/compiler/userdata.go` (new heredoc block after km-send)

Script: `/opt/km/bin/km-recv`

```
Usage: km-recv [--json] [--watch] [--no-move]
```

Logic:
1. Parse args: `--json` (machine output), `--watch` (poll loop), `--no-move` (don't move to processed)
2. Read env: `KM_SANDBOX_ID`
3. Scan `/var/mail/km/new/*` for .eml files
4. For each message:
   a. Parse MIME headers with basic bash (grep/sed): From, To, Subject, X-KM-Sender-ID, X-KM-Signature, X-KM-Encrypted
   b. Extract body (everything after first blank line, or first MIME part)
   c. **Best-effort signature verification:**
      - If X-KM-Sender-ID present and X-KM-Signature present:
        - Lookup sender's public key: `aws dynamodb get-item --table-name km-identities --key '{"sandbox_id":{"S":"$SENDER_ID"}}' --query 'Item.public_key.S' --output text`
        - If key found: convert base64 pubkey to PEM, verify with `openssl pkeyutl -verify`
        - Set status: `verified` / `failed` / `no-key`
      - If no signature header: status = `unsigned`
   d. If `--json`: output JSON object per message
   e. If default: print human-readable summary line
   f. If not `--no-move`: move to `/var/mail/km/processed/`
5. If `--watch`: sleep 5s, repeat scan

### Task 2: Ed25519 public key PEM conversion helper

Within km-recv script, a function `ed25519_pubkey_to_pem()` that:
- Takes base64-encoded 32-byte Ed25519 public key
- Builds SubjectPublicKeyInfo DER: fixed 12-byte prefix `302a300506032b6570032100` + 32-byte pubkey
- Base64-encodes DER, wraps in PEM headers
- Writes to temp file, returns path

### Task 3: MIME parser helper functions

Within km-recv script:
- `parse_headers()`: extract header values from raw MIME (handles folded headers)
- `extract_body()`: extract body from single-part or first text/plain part of multipart
- `list_attachments()`: list attachment filenames from multipart parts
- `save_attachment()`: decode base64 attachment to file (for future use)

```

    Body: First 80 chars of message body here...

[2] From: sb-def67890  Subject: handoff      Sig: ✗ failed    Enc: yes
    Body: [encrypted - cannot decrypt in bash]

[3] From: external@example.com  Subject: hello  Sig: — unsigned  Enc: no
    Body: First 80 chars of message body here...
```
