# Plan 45-02: In-sandbox km-send bash script

**Wave 2** — Depends on Plan 01 (MIME contract established). Can run in parallel with Plan 03.

## Goal

Create the `/opt/km/bin/km-send` bash script that sends Ed25519-signed emails with optional attachments from inside a sandbox. Pure bash + AWS CLI + openssl. Deployed via `userdata.go`.

## Tasks

### Task 1: Write km-send script

**File:** `pkg/compiler/userdata.go` (new heredoc block after km-mail-poller)

Script: `/opt/km/bin/km-send`

```
Usage: km-send --to <addr> --subject <subject> [--body <file>] [--attach file1,file2,...] [message on stdin]
```

Logic:
1. Parse args: `--to` (required), `--subject` (required), `--body` (file path, optional — reads stdin if omitted), `--attach` (comma-separated file paths, optional)
2. Read env: `KM_SANDBOX_ID`, `KM_EMAIL_ADDRESS` (set by km-identity.sh)
3. Validate: error if KM_SANDBOX_ID or KM_EMAIL_ADDRESS not set
4. Read body from file or stdin into temp file
5. Fetch Ed25519 private key from SSM:
   ```bash
   PRIVKEY_B64=$(aws ssm get-parameter --name "/sandbox/$KM_SANDBOX_ID/signing-key" --with-decryption --query 'Parameter.Value' --output text)
   ```
6. Convert base64 privkey to PEM format for openssl:
   - Decode base64 → 64-byte raw Ed25519 private key
   - Extract 32-byte seed (first 32 bytes)
   - Wrap in PKCS8 DER envelope → base64 → PEM
7. Sign body with openssl:
   ```bash
   SIGNATURE=$(openssl pkeyutl -sign -inkey "$PEM_FILE" -rawin < "$BODY_FILE" | base64 -w0)
   ```
8. Build raw MIME:
   - Headers: From, To, Subject, X-KM-Sender-ID, X-KM-Signature, MIME-Version
   - If no attachments: single-part text/plain
   - If attachments: multipart/mixed with boundary, text body as first part, each attachment base64-encoded as subsequent parts
9. Send via SES:
   ```bash
   aws sesv2 send-email --from-email-address "$KM_EMAIL_ADDRESS" \
     --destination "ToAddresses=$TO" \
     --content "Raw={Data=$(base64 -w0 < "$MIME_FILE")}"
   ```
10. Print confirmation: `[km-send] Sent signed email to $TO (sig: ${SIGNATURE:0:12}...)`

### Task 2: Ed25519 PEM conversion helper function

Within km-send script, a function `ed25519_privkey_to_pem()` that:
- Takes base64-encoded 64-byte Ed25519 private key
- Extracts 32-byte seed (first 32 bytes of decoded key)
- Builds PKCS8 DER: fixed 16-byte prefix `302e020100300506032b657004220420` + 32-byte seed
- Base64-encodes DER, wraps in PEM headers
- Writes to temp file, returns path

This is the trickiest part — Ed25519 PKCS8 DER has a fixed structure so the prefix is constant.

### Task 3: Multipart MIME builder function

Within km-send script, a function `build_mime()` that:
- Takes: from, to, subject, sender_id, signature, body_file, attachment_files (array)
- Generates random boundary string
- Writes MIME headers
- If no attachments: plain text body
- If attachments: multipart/mixed with each file base64-encoded
- Returns path to temp MIME file
