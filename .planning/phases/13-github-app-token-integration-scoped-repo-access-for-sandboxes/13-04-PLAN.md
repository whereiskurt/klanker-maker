    **create.go:** Add two new steps after Step 12c (budget-enforcer deploy), guarded by `p.Spec.SourceAccess.GitHub != nil`:

    Step 13a: Generate GitHub App installation token.
    1. Read app client ID from SSM `/km/config/github/app-client-id`
    2. Read private key PEM from SSM `/km/config/github/private-key` (with decryption)
    3. Read installation ID from SSM `/km/config/github/installation-id`
    4. Extract repos and permissions from profile's `sourceAccess.github.allowedRepos`
    5. Call `github.GenerateGitHubAppJWT(appClientID, privateKeyPEM)`
    6. Call `github.ExchangeForInstallationToken(ctx, jwt, installationID, repos, perms)`
    7. Call `github.WriteTokenToSSM(ctx, ssmClient, sandboxID, token, kmsKeyARN, false)` -- overwrite=false for initial write
    8. Non-fatal: log warning and continue if any step fails (matches budget-enforcer pattern from Phase 06-06: "sandbox provisioned even if GitHub token fails")
    9. Print `Step 13a: GitHub App installation token stored in SSM`

    Step 13b: Deploy github-token/ Terragrunt directory.
    1. Create `github-token/` subdirectory in sandbox dir
    2. Write `artifacts.GitHubTokenHCL` to `github-token/terragrunt.hcl`
    3. Run `terragrunt apply` in the github-token/ directory (non-fatal)
    4. Print `Step 13b: GitHub token refresher Lambda deployed for {sandboxID}`

    Step 7c: Destroy github-token resources.
