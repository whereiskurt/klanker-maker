<feature>
  <name>GitHub App Token Library</name>
  <files>pkg/github/token.go, pkg/github/token_test.go, cmd/github-token-refresher/main.go</files>
  <behavior>
    - GenerateGitHubAppJWT(appClientID, privateKeyPEM) returns RS256 JWT:
      - iss = appClientID
      - iat = now - 60s (clock drift buffer)
      - exp = now + 10 minutes (GitHub max)
      - Signed with RS256
    - GenerateGitHubAppJWT parses PKCS#1 PEM (-----BEGIN RSA PRIVATE KEY-----) correctly
    - GenerateGitHubAppJWT parses PKCS#8 PEM (-----BEGIN PRIVATE KEY-----) correctly
    - GenerateGitHubAppJWT returns error for non-RSA keys
    - GenerateGitHubAppJWT returns error for invalid PEM
    - ExchangeForInstallationToken(ctx, jwt, installationID, repos, perms) -> (token, error):
      - Sends POST to https://api.github.com/app/installations/{id}/access_tokens
      - Sets Authorization: Bearer {jwt}, Accept: application/vnd.github+json, X-GitHub-Api-Version: 2022-11-28
      - Request body has repositories (short names) and permissions
      - Returns token string from 201 response
      - Returns error with status code and body for non-201 responses
    - repoShortName("org/repo") -> "repo", repoShortName("repo") -> "repo"
    - CompilePermissions(["clone"]) -> {"contents": "read"}
    - CompilePermissions(["fetch"]) -> {"contents": "read"}
    - CompilePermissions(["push"]) -> {"contents": "write"}
    - CompilePermissions(["clone", "push"]) -> {"contents": "write"} (write supersedes read)
    - CompilePermissions(["clone", "fetch"]) -> {"contents": "read"}
    - Lambda handler logs structured JSON to CloudWatch on each token generation: {"event": "token_generated", "sandbox_id": "...", "allowed_repos": [...], "permissions": {...}, "success": true/false}
    - Lambda handler logs structured JSON on failure: {"event": "token_generation_failed", "sandbox_id": "...", "error": "...", "allowed_repos": [...]}
  </behavior>
  <implementation>
    Use golang-jwt/jwt/v5 for JWT generation. First `go get github.com/golang-jwt/jwt/v5`. Use crypto/rsa + crypto/x509 + encoding/pem from stdlib for key parsing. Try ParsePKCS1PrivateKey first, fall back to ParsePKCS8PrivateKey.

    For ExchangeForInstallationToken, use net/http stdlib. In tests, use httptest.NewServer to stub the GitHub API. Make the GitHub API base URL injectable via a package-level variable (defaulting to "https://api.github.com") so tests can point at httptest server.

    WriteTokenToSSM(ctx, ssmClient, sandboxID, token, kmsKeyARN, overwrite) writes to `/sandbox/{sandbox-id}/github-token` as SecureString. Use existing aws-sdk-go-v2/service/ssm patterns from pkg/aws/. Define a narrow SSMAPI interface (PutParameter) for testability -- follows the S3RunAPI, CWLogsAPI narrow interface pattern established in Phase 3.

    cmd/github-token-refresher/main.go: Lambda handler that:
    1. Reads event payload (sandbox_id, installation_id, ssm_parameter_name, kms_key_arn, allowed_repos, permissions)
    2. Reads private key PEM from SSM at /km/config/github/private-key
    3. Reads app client ID from SSM at /km/config/github/app-client-id
    4. Calls GenerateGitHubAppJWT
    5. Calls ExchangeForInstallationToken
    6. Calls WriteTokenToSSM with overwrite=true
    7. Logs structured token generation audit event to CloudWatch (JSON with sandbox_id, allowed_repos, permissions, success/failure) -- this satisfies the ROADMAP "Token audit" requirement
    Uses aws-lambda-go (already in go.mod). Same structure as cmd/budget-enforcer/main.go.

    Token audit logging: Use `log/slog` with JSON handler (consistent with Go structured logging). On success, log at INFO level: `{"event": "token_generated", "sandbox_id": "...", "allowed_repos": [...], "permissions": {...}}`. On failure, log at ERROR level: `{"event": "token_generation_failed", "sandbox_id": "...", "error": "...", "allowed_repos": [...]}`. CloudWatch automatically captures Lambda stdout as log events. Test by verifying log output in Lambda handler test (capture slog output with a bytes.Buffer handler).
  </implementation>
</feature>
