<feature>
  <name>pkg/quota multi-window atomic counter + resolution</name>
  <files>pkg/quota/quota.go, pkg/quota/resolve.go, pkg/quota/quota_test.go</files>
  <behavior>
    Record:
    - For each window WITH a limit set on `limit` (lifetime/perHour/perDay), build the SK
      (`lifetime`, `hour#{epoch/3600}`, `day#{epoch/86400}`), issue UpdateItem with
      `UpdateExpression: "ADD #c :one"` (`#c=count`, `:one=1`), `ReturnValues: ALL_NEW`, and on
      first-create also `SET ttl = :exp` for hour/day rows (lifetime: no ttl). Read the returned
      count → WindowResult{exceeded: count > limit}.
    - PK = `{sandboxID}#{action}`. TTL: hour rows ~ now+2h, day rows ~ now+2d (epoch seconds).
    - Decision: Tripped = any WindowResult.Exceeded; WorstWindow = the exceeded window with the
      smallest reset horizon (lifetime worst, then day, then hour — pick the most severe present);
      OnBreach = limit.OnBreach (default warn when empty).
    - Action with NO windows configured ⇒ return Decision{} with no DDB calls (dormant).

    ResolveLimits(profile map, defaults map) → resolved map:
    - For each action, for each window: profile value wins if set, else default if set, else nil
      (unlimited). OnBreach: profile wins, else default, else "warn".
    - A profile that sets only github_pr.perHour still inherits defaults' github_pr.lifetime.

    Test cases (turn plan 01 stubs green):
    - TestRecord: hourBucket(2026-06-27T12:34:56Z) == unix/3600; dayBucket == unix/86400.
    - TestResolveLimits: per-window precedence + OnBreach precedence.
    - TestDecision: counts [16 vs 15 hour, 40 vs 50 day] ⇒ Tripped, WorstWindow=hour.
    - TestAtomicADD: mock QuotaAPI captures UpdateItemInput; assert UpdateExpression contains "ADD".
    - TestTTL: lifetime UpdateItemInput has no ttl in ExpressionAttributeValues; hour/day do, ~2h/~2d.
  </behavior>
