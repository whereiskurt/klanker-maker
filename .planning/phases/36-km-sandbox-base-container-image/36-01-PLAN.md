<task type="auto">
  <name>Task 1: Create containers/sandbox/Dockerfile</name>
  <files>containers/sandbox/Dockerfile</files>
  <action>
Create `containers/sandbox/Dockerfile` based on `amazonlinux:2023-minimal`:

1. Use `FROM amazonlinux:2023-minimal` as the base image
2. Install system packages via `dnf install -y`: shadow-utils, tar, gzip, unzip, jq, git, ca-certificates, python3, procps-ng, findutils, which, less, vim-minimal, hostname, iputils, bind-utils, curl
3. Install AWS CLI v2 via official zip method:
   ```
   curl -fsSL "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o /tmp/awscliv2.zip
   unzip -q /tmp/awscliv2.zip -d /tmp
   /tmp/aws/install
   rm -rf /tmp/aws /tmp/awscliv2.zip
   ```
4. Install gosu 1.17 for linux/amd64:
   ```
   curl -fsSL "https://github.com/tianon/gosu/releases/download/1.17/gosu-amd64" -o /usr/local/bin/gosu
   chmod +x /usr/local/bin/gosu
   gosu nobody true  # verify
   ```
5. Create sandbox user: `useradd -m -d /home/sandbox -s /bin/bash -u 1000 sandbox`
6. Create workspace: `mkdir -p /workspace && chown sandbox:sandbox /workspace`
7. Create helper dirs: `mkdir -p /opt/km/bin`
8. COPY entrypoint.sh to `/opt/km/entrypoint.sh` and `chmod +x`
9. Set `WORKDIR /workspace`
10. Set `ENTRYPOINT ["/opt/km/entrypoint.sh"]`
11. Set `CMD ["/bin/bash"]`

Layer optimization: combine dnf install into one RUN, combine AWS CLI install into one RUN, combine gosu install into one RUN, combine user/dir creation into one RUN. Clean dnf cache (`dnf clean all`) in the same layer as install.

Do NOT include SSM agent (per locked decision — km shell uses docker exec / kubectl exec).
Do NOT include km-mail-poller or km-upload-artifacts as separate binaries — these are bash functions in entrypoint.sh (per locked decision: shell script helpers, not Go binaries).
  </action>

  </verify>

<task type="auto">
  <name>Task 2: Create containers/sandbox/entrypoint.sh</name>
  <files>containers/sandbox/entrypoint.sh</files>
  <action>
Create `containers/sandbox/entrypoint.sh` as a direct port of `pkg/compiler/userdata.go` logic. Each section is a bash function. Use `[km-entrypoint]` log prefix (matching `[km-bootstrap]` on EC2).

Structure:
```bash
#!/bin/bash
set -euo pipefail

log() { echo "[km-entrypoint] $*"; }
log_warn() { echo "[km-entrypoint] WARNING: $*" >&2; }
log_fail() { echo "[km-entrypoint] FATAL: $*" >&2; exit 1; }
```

Implement these functions in order, porting from userdata.go:

**1. setup_ca_trust() — CRITICAL**
Port from userdata.go lines ~560-597. If `KM_PROXY_CA_CERT_S3` is set:
- `aws s3 cp "${KM_PROXY_CA_CERT_S3}" /etc/pki/ca-trust/source/anchors/km-proxy-ca.crt` or `log_fail`
- `update-ca-trust`
- Export `SSL_CERT_FILE=/etc/pki/tls/certs/ca-bundle.crt`
- Export `REQUESTS_CA_BUNDLE`, `CURL_CA_BUNDLE`, `NODE_EXTRA_CA_CERTS` to same path
If not set, log skip and return 0.

**2. inject_secrets() — CRITICAL**
Port from userdata.go lines ~130-155. If `KM_SECRET_PATHS` is set:
- `IFS=',' read -ra PATHS <<< "${KM_SECRET_PATHS}"`
- For each path: `aws ssm get-parameter --name "${path}" --with-decryption --query "Parameter.Value" --output text` or `log_fail`
- Derive env name from basename: uppercase, replace hyphens with underscores
- `export "${env_name}=${val}"`

**3. inject_otp_secrets() — OPTIONAL**
Port from userdata.go lines ~158-168. If `KM_OTP_PATHS` is set:
- Same as inject_secrets but after reading, delete the parameter: `aws ssm delete-parameter --name "${path}" 2>/dev/null || true`
- On fetch failure: `log_warn` and continue (OTP secrets may have been consumed already)

**4. setup_profile_env() — OPTIONAL**
If `KM_PROFILE_ENV` is set:
- Decode: `echo "${KM_PROFILE_ENV}" | base64 -d | jq -r 'to_entries[] | "\(.key)=\(.value)"'`
- Export each key=value pair
- On failure: `log_warn` and continue

**5. setup_github_credentials() — OPTIONAL**
Port from userdata.go GitHub credential helper (~lines 200-250). If `KM_GITHUB_TOKEN_SSM` is set:
- Fetch token from SSM
- Write GIT_ASKPASS script to `/opt/km/bin/git-askpass.sh` that echoes the token
- `chmod +x`, export `GIT_ASKPASS=/opt/km/bin/git-askpass.sh`
- On failure: `log_warn` and continue

**6. setup_git_ref_enforcement() — OPTIONAL**
Port from userdata.go git ref enforcement. If `KM_GITHUB_ALLOWED_REFS` is set:
- Write a git pre-push hook template to `/opt/km/bin/km-pre-push-hook`
- Install in sandbox user's git template dir: `/home/sandbox/.config/git/hooks/pre-push`
- On failure: `log_warn` and continue

**7. restore_rsync_snapshot() — OPTIONAL**
Port from userdata.go rsync restore. If `KM_RSYNC_SNAPSHOT` is set:
- Download from S3: `aws s3 cp "s3://${KM_ARTIFACTS_BUCKET}/snapshots/${KM_SANDBOX_ID}/${KM_RSYNC_SNAPSHOT}.tar.gz" /tmp/snapshot.tar.gz`
- Extract to /workspace: `tar xzf /tmp/snapshot.tar.gz -C /workspace`
- Fix ownership: `chown -R sandbox:sandbox /workspace`
- On failure: `log_warn` and continue

**8. run_init_commands() — OPTIONAL**
If `KM_INIT_COMMANDS` is set:
- Decode base64 JSON array: `echo "${KM_INIT_COMMANDS}" | base64 -d | jq -r '.[]'`
- Execute each command with `bash -c "$cmd"` as root
- Log each command before executing
- On individual command failure: `log_warn "initCommand failed: $cmd"` and continue to next

**9. start_mail_poller() — OPTIONAL**
Port mail poller from userdata.go lines ~388-429. If `KM_EMAIL_ADDRESS` is set:
- Define the mail poller as an inline bash function that polls SES inbox via S3
- Start in background: `_mail_poller &`
- On failure: `log_warn` and continue

**10. upload_artifacts() — helper function**
Port artifact upload from userdata.go. Upload `/workspace` contents to S3:
- `aws s3 sync /workspace "s3://${KM_ARTIFACTS_BUCKET}/artifacts/${KM_SANDBOX_ID}/" --exclude ".git/*"`
- Called by SIGTERM handler and can be called manually

**11. SIGTERM handler**
Install trap before main execution:
```bash
_shutdown() {
    log "Shutdown signal received - uploading artifacts..."
    upload_artifacts || log_warn "Artifact upload failed during shutdown"
    exit 0
}
trap '_shutdown' TERM INT
```

**12. Also install shutdown handler in sandbox user profile**
Write `/etc/profile.d/km-shutdown.sh` that traps SIGTERM in the sandbox user's bash session (since `exec gosu` replaces PID 1 and the entrypoint trap is gone):
```bash
_km_shutdown() { /opt/km/bin/km-upload-artifacts 2>/dev/null; }
trap '_km_shutdown' TERM INT EXIT
```
Write `/opt/km/bin/km-upload-artifacts` as a small shell script that calls the upload_artifacts logic.

**13. Main execution and user drop**
At the bottom of the script:
```bash
log "=== km-sandbox entrypoint starting ==="
setup_ca_trust
inject_secrets
inject_otp_secrets
setup_profile_env
setup_github_credentials
setup_git_ref_enforcement
restore_rsync_snapshot
run_init_commands
start_mail_poller

# Write env vars to file so sandbox user inherits them via profile.d
env | grep -E '^(KM_|SSL_|REQUESTS_|CURL_|NODE_EXTRA_|GIT_|AWS_|SANDBOX_|HTTP_PROXY|HTTPS_PROXY|NO_PROXY|OTEL_|CLAUDE_)' > /etc/profile.d/km-env.sh 2>/dev/null || true
sed -i 's/^/export /' /etc/profile.d/km-env.sh 2>/dev/null || true

log "Dropping to sandbox user..."
exec gosu sandbox "${@:-/bin/bash}"
```

IMPORTANT: All `export` statements must be at the top level of the script (not inside subshells) so they're inherited by `exec gosu`. Write env vars to `/etc/profile.d/km-env.sh` as a backup for interactive shells.

Mirror the section numbering and comments from userdata.go for easy cross-reference.
