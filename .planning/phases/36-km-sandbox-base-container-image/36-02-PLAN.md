Make two changes to `pkg/compiler/service_hcl.go`:

**Change 1: Replace MAIN_IMAGE_PLACEHOLDER (line ~596)**

Replace:
```go
mainImage := "MAIN_IMAGE_PLACEHOLDER"
if p.Spec.Runtime.InstanceType != "" {
    // For ECS, instanceType holds the Fargate task size (e.g. "512/1024")
    // We keep the placeholder here; real image set at provision time.
}
```

With:
```go
// mainImage uses the same ECR registry as sidecars — km-sandbox:{version}
// This replaces the MAIN_IMAGE_PLACEHOLDER that existed before the sandbox
// container image was created in Phase 36.
mainImage := sidecarImage("sandbox")
```

Note: `sidecarImage` is a closure defined at line ~626. Move the `mainImage` assignment to AFTER the `sidecarImage` definition (or move `sidecarImage` definition earlier). The `sidecarImage` helper already produces the correct format: `{ecrRegistry}/km-sandbox:{imageTag}`.

**Change 2: Add KM_* env vars to ecsHCLParams struct**

Add these fields to `ecsHCLParams`:
```go
// Entrypoint env vars (Phase 36 — km-sandbox container entrypoint configuration)
KMSandboxID       string // always set
KMArtifactsBucket  string // S3 bucket for artifacts/CA/snapshots
KMStateBucket      string // S3 bucket for metadata
KMProxyCACertS3    string // s3://bucket/sidecars/km-proxy-ca.crt
KMSecretPaths      string // comma-separated SSM paths
KMOTPPaths         string // comma-separated OTP SSM paths
KMInitCommands     string // base64-encoded JSON array of shell commands
KMRsyncSnapshot    string // snapshot name to restore
KMGitHubTokenSSM   string // SSM path for GitHub token (entrypoint)
KMGitHubAllowedRefs string // comma-separated allowed refs (entrypoint)
KMProfileEnv       string // base64-encoded JSON env vars
KMOperatorEmail    string // operator notification address
```

**Change 3: Populate the new fields in generateECSServiceHCL**

In the `params := ecsHCLParams{...}` block, add the new field assignments:
- `KMSandboxID: sandboxID`
- `KMArtifactsBucket: artifactBucket`
- `KMStateBucket: stateBucket` (derive from config or env, similar to artifactBucket)
- `KMProxyCACertS3: fmt.Sprintf("s3://%s/sidecars/km-proxy-ca.crt", artifactBucket)` (if artifactBucket is set)
- `KMSecretPaths:` join the profile's secret refs as comma-separated SSM paths
- `KMOTPPaths:` join the profile's OTP secret refs as comma-separated paths
- `KMInitCommands:` base64-encode the profile's `Spec.Execution.InitCommands` as a JSON array (use `encoding/json` + `encoding/base64`)
- `KMRsyncSnapshot:` from profile if set
- `KMGitHubTokenSSM:` use the existing `GitHubSSMPath` field value
- `KMGitHubAllowedRefs:` derive from profile's allowed refs
- `KMProfileEnv:` base64-encode the profile's env vars as a JSON object
- `KMOperatorEmail:` from config/env

**Change 4: Add KM_* env vars to the ECS HCL template**

In the `ecsServiceHCLTemplate` constant, in the main container's `environment` block (after existing env vars like SANDBOX_ID, KM_LABEL, HTTP_PROXY, etc.), add:
