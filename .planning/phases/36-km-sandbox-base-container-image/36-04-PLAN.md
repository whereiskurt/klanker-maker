<objective>
Build the km-sandbox image locally and run a smoke test that verifies the entrypoint boots, handles missing env vars gracefully, drops to the sandbox user, and responds to SIGTERM — all without requiring AWS credentials or infrastructure.
</objective>

<context>
After Plans 01 and 03, the Dockerfile, entrypoint.sh, and Makefile target exist. This plan verifies they actually work end-to-end by building and running the image locally.

The smoke test must work without AWS credentials — it validates the container mechanics (user drop, signal handling, step skipping) not the AWS integrations (SSM, S3, SES). AWS integration testing happens in Phase 37 when the Docker substrate connects to real AWS services.
</context>

<tasks>

<task id="1" type="execute">
<title>Create smoke test script</title>
<files>scripts/smoke-test-sandbox.sh</files>
<action>
Create `scripts/smoke-test-sandbox.sh` — a self-contained bash script that:

1. **Builds the image:**
   ```bash
   docker build --platform linux/amd64 -t km-sandbox:smoke-test containers/sandbox/
   ```

2. **Test 1 — Minimal boot (no env vars):**
   Run the container with no KM_* env vars. The entrypoint should:
   - Skip all optional steps (CA trust, secrets, rsync, mail, GitHub) with WARNING messages
   - Drop to sandbox user
   - Exit cleanly
   ```bash
   docker run --rm --name km-smoke-minimal \
     -e KM_SANDBOX_ID=smoke-test-001 \
     km-sandbox:smoke-test \
     bash -c 'whoami && echo "BOOT_OK"'
   ```
   Assert: output contains `sandbox` (whoami) and `BOOT_OK`.
   Assert: output contains `WARNING` (skipped steps).
   Assert: exit code 0.

3. **Test 2 — Environment passthrough:**
   Run with profile env vars and verify they're exported:
   ```bash
   docker run --rm --name km-smoke-env \
     -e KM_SANDBOX_ID=smoke-test-002 \
     -e KM_PROFILE_ENV=$(echo '{"MY_VAR":"hello","OTHER_VAR":"world"}' | base64) \
     km-sandbox:smoke-test \
     bash -c 'echo "MY_VAR=$MY_VAR OTHER_VAR=$OTHER_VAR"'
   ```
   Assert: output contains `MY_VAR=hello OTHER_VAR=world`.

4. **Test 3 — initCommands execution:**
   Run with a simple initCommand and verify it ran:
   ```bash
   docker run --rm --name km-smoke-init \
     -e KM_SANDBOX_ID=smoke-test-003 \
     -e KM_INIT_COMMANDS=$(echo '["touch /tmp/init-marker"]' | base64) \
     km-sandbox:smoke-test \
     bash -c 'test -f /tmp/init-marker && echo "INIT_OK" || echo "INIT_FAIL"'
   ```
   Assert: output contains `INIT_OK`.

5. **Test 4 — SIGTERM graceful shutdown:**
   Start container in background, send SIGTERM, verify clean exit:
   ```bash
   docker run -d --rm --name km-smoke-sigterm \
     -e KM_SANDBOX_ID=smoke-test-004 \
     km-sandbox:smoke-test \
     bash -c 'sleep 300'
   sleep 2
   docker kill --signal SIGTERM km-smoke-sigterm
   sleep 3
   # Container should have exited (--rm auto-removes)
   docker inspect km-smoke-sigterm 2>&1 | grep -q "No such object" && echo "SIGTERM_OK"
   ```
   Assert: output contains `SIGTERM_OK` (container exited and was removed).

6. **Test 5 — User and workspace verification:**
   ```bash
   docker run --rm --name km-smoke-user \
     -e KM_SANDBOX_ID=smoke-test-005 \
     km-sandbox:smoke-test \
     bash -c 'id -u && id -un && pwd && test -w /workspace && echo "WORKSPACE_WRITABLE"'
   ```
   Assert: output contains `1000` (UID), `sandbox` (username), `/workspace` (pwd), `WORKSPACE_WRITABLE`.

7. **Test 6 — Installed tools check:**
   ```bash
   docker run --rm --name km-smoke-tools \
     -e KM_SANDBOX_ID=smoke-test-006 \
     km-sandbox:smoke-test \
     bash -c 'git --version && jq --version && aws --version && gosu --version && echo "TOOLS_OK"'
   ```
   Assert: output contains `TOOLS_OK`.

Script should print a summary at the end:
```
km-sandbox smoke test results:
  ✓ Minimal boot (no env vars)
  ✓ Environment passthrough
  ✓ initCommands execution
  ✓ SIGTERM graceful shutdown
  ✓ User and workspace
  ✓ Installed tools

6/6 passed
```

Exit code 0 if all pass, 1 if any fail.

Make the script executable: `chmod +x scripts/smoke-test-sandbox.sh`
</action>
<verify>
