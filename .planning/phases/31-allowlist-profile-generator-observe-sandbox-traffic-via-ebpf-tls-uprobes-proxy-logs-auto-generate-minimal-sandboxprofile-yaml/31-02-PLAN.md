**pkg/ebpf/resolver/resolver.go:**
Add a `DomainObserver` field to `ResolverConfig`:
```go
// DomainObserver, if non-nil, is called for every DNS query with the domain
// (trailing dot stripped) and whether it was allowed. Used by the allowlist
// generator's learning mode.
DomainObserver func(domain string, allowed bool)
```
In `handleQuery`, after the existing `zerolog` log statement that logs the query (around line 210), add:
```go
if r.cfg.DomainObserver != nil {
    r.cfg.DomainObserver(domain, allowed)
}
```
The `domain` variable is already available in `handleQuery` scope. The `allowed` boolean is already computed by `IsAllowed`. This is a nil-safe callback -- no behavior change when DomainObserver is not set.

**internal/app/cmd/ebpf_attach.go:**
Add a `--observe` bool flag to the ebpf-attach command.
Add a `--observe-output` string flag defaulting to `/tmp/km-observed.json`.

In `runEbpfAttach`, when `--observe` is true:
1. Create `recorder := allowlistgen.NewRecorder()`

5. Log at info level: "observe mode: wrote N DNS domains, N hosts, N repos to {path}"

**internal/app/cmd/observe.go:**
Create a `km observe` command (Cobra) with:
- `Use: "observe <sandbox-id>"`
- `Short: "Generate a minimal SandboxProfile from observed sandbox traffic"`
- `Long:` explains the learning mode workflow (create permissive sandbox, run workload, run km observe)
- Args: `cobra.ExactArgs(1)` for sandbox-id
- Flags:
  - `--base` (string, default ""): base profile name for `Extends` field
  - `--output` (string, default ""): write to file instead of stdout
  - `--source` (string, default "auto"): where to read observed data ("auto" tries S3 artifacts bucket first, falls back to SSM; "local" reads a local JSON file path via `--observed-json`)
  - `--observed-json` (string, default ""): path to local observed JSON file (for testing or when ebpf-attach wrote locally)

**RunE logic:**
1. Resolve sandbox-id (use existing `ResolveSandboxRef` pattern from other commands)
2. If `--observed-json` is set, read that local file directly
3. Otherwise: fetch the observed JSON from S3 artifacts bucket using `s3://{artifacts-bucket}/{sandbox-id}/km-observed.json` (use existing S3 helpers from `pkg/aws/`)
4. Parse JSON into `ObservedState` struct: `{"dns": []string, "hosts": []string, "repos": []string}`
5. Create `allowlistgen.NewRecorder()`, populate it by calling `RecordDNSQuery`, `RecordHost`, `RecordRepo` for each entry
6. Call `recorder.GenerateYAML(base)` to get the YAML bytes
7. If `--output` is set, write to file; otherwise write to stdout
8. Print a stderr summary: "Generated profile with N DNS suffixes, N hosts, N repos"
