<objective>
Wire the allowlistgen Recorder into the sandbox runtime (ebpf-attach) and add a `--learn` flag to `km shell` that runs the sandbox in learning mode and generates a SandboxProfile YAML on the host when the shell exits.

Purpose: Operators can run `km shell --learn`, exercise their workload, exit the shell, and find a ready-to-use minimal profile YAML on their host machine. Observed data is also appended to S3 so future `km observe` (deferred) can aggregate multiple sessions.
Output: Working `--learn` flag on `km shell`, `--observe` flag on `ebpf-attach`, DNS resolver observation hook, S3 timestamped learn session uploads.
</objective>

```go
// DomainObserver, if non-nil, is called for every DNS query with the domain
// (trailing dot stripped) and whether it was allowed. Used by the allowlist
// generator's learning mode.
DomainObserver func(domain string, allowed bool)
```
In `handleQuery`, after the existing `zerolog` log statement that logs the query (around line 210), add:
```go
if r.cfg.DomainObserver != nil {
    r.cfg.DomainObserver(domain, allowed)
}
```
The `domain` variable is already available in `handleQuery` scope. The `allowed` boolean is already computed by `IsAllowed`. This is a nil-safe callback -- no behavior change when DomainObserver is not set.

**internal/app/cmd/ebpf_attach.go:**
Add two new flag variables in NewEBPFAttachCmd:
- `observe` (bool, default false): `--observe` enables learning mode
- `observeOutput` (string, default `/tmp/km-observed.json`): `--observe-output` local JSON output path

Pass `observe` and `observeOutput` through to `runEbpfAttach` by extending its parameter list (add `observe bool, observeOutput string` at the end).

In `runEbpfAttach`, when `observe` is true:
1. Import and create `recorder := allowlistgen.NewRecorder()`. Import path: `"github.com/whereiskurt/klankrmkr/pkg/allowlistgen"`.
2. If TLS consumer exists (`tlsConsumer != nil`), register `recorder.HandleTLSEvent` as an additional handler via `tlsConsumer.AddHandler(recorder.HandleTLSEvent)` right after the existing BedrockAuditHandler registration.
3. Set `resolverCfg.DomainObserver` to a closure: `func(domain string, allowed bool) { recorder.RecordDNSQuery(domain) }` -- record ALL queries (both allowed and denied) since in learning mode we want to capture what the workload actually needs.
4. After the shutdown signal handling block (after `cancel()` is called and TLS/audit consumers are closed), serialize the recorder state. Define a local struct:
```go
type observedState struct {
    DNS   []string `json:"dns"`
    Hosts []string `json:"hosts"`
    Repos []string `json:"repos"`
}
```
Populate with `recorder.DNSDomains()`, `recorder.Hosts()`, `recorder.Repos()`. Marshal with `json.MarshalIndent(state, "", "  ")`.
5. Write atomically to `observeOutput`: write to `observeOutput + ".tmp"`, then `os.Rename` to `observeOutput`.
6. Upload to S3 at timestamped key. Use the `KM_ARTIFACTS_BUCKET` env var (already set in sandbox environment). The S3 key should be `learn/{sandboxID}/{timestamp}.json` where timestamp is `time.Now().UTC().Format("20060102T150405Z")`. Use a direct `s3.PutObject` call (create a minimal S3 client from the default credential chain inside the sandbox -- the sandbox IAM role grants S3 access). If `KM_ARTIFACTS_BUCKET` is empty or S3 upload fails, log a warning but do not fail -- the local file is the primary output.
7. Log at info level: `"observe mode: wrote N DNS domains, N hosts, N repos to {path} (S3: {key or 'skipped'})"`.

Note: `ebpf_attach.go` has a `//go:build linux && amd64` constraint. The S3 upload code belongs here since it runs inside the sandbox (not on the operator's host).

```go
func generateProfileFromJSON(data []byte, base string) ([]byte, error) {
    var state struct {
        DNS   []string `json:"dns"`
        Hosts []string `json:"hosts"`
        Repos []string `json:"repos"`
    }
    if err := json.Unmarshal(data, &state); err != nil {
        return nil, fmt.Errorf("parse observed state: %w", err)
    }
    rec := allowlistgen.NewRecorder()
    for _, d := range state.DNS {
        rec.RecordDNSQuery(d)
    }
    for _, h := range state.Hosts {
        rec.RecordHost(h)
    }
    for _, r := range state.Repos {
        rec.RecordRepo(r)
    }
    return rec.GenerateYAML(base)
}
```
