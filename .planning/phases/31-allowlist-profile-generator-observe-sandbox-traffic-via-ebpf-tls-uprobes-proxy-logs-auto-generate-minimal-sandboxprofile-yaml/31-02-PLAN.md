<objective>
Wire the allowlistgen Recorder into the sandbox runtime and add a `--learn` flag to `km shell` that works on BOTH EC2 and Docker substrates. On EC2, observation happens via ebpf-attach --observe (TLS uprobes + DNS resolver hook) with S3 upload. On Docker, observation happens by parsing dns-proxy and http-proxy container logs post-exit. Both paths feed into the same Recorder to generate a SandboxProfile YAML on the host.

Purpose: Operators can run `km shell --learn` on any substrate, exercise their workload, exit the shell, and find a ready-to-use minimal profile YAML on their host machine. Observed data is also uploaded to S3 for future aggregation.
Output: Working `--learn` flag on `km shell` (EC2 + Docker), `--observe` flag on `ebpf-attach`, DNS resolver observation hook, Docker proxy log parser, S3 learn session uploads.
</objective>

**pkg/allowlistgen/proxylog.go:**
Create `ParseProxyLogs` -- the Docker-equivalent data path that parses zerolog JSON from dns-proxy and http-proxy container logs and feeds into a Recorder.

```go
package allowlistgen

import (
    "bufio"
    "encoding/json"
    "fmt"
    "io"
    "net"
)

// proxyLogEntry represents a single zerolog JSON line from a proxy container.
type proxyLogEntry struct {
    EventType string `json:"event_type"`
    Domain    string `json:"domain"`    // dns_query events
    Host      string `json:"host"`      // CONNECT / blocked events
    Owner     string `json:"owner"`     // github_repo_allowed events
    Repo      string `json:"repo"`      // github_repo_allowed events
    Allowed   bool   `json:"allowed"`   // dns_query events
}

// ParseProxyLogs reads zerolog JSON lines from DNS proxy and HTTP proxy log
// readers and feeds observed traffic into the provided Recorder. This is the
// Docker substrate equivalent of the eBPF TLS handler + DNS observer path
// used on EC2.
//
// Either reader may be nil (skipped). Returns an error only on malformed JSON
// that prevents further parsing; individual malformed lines are skipped.
func ParseProxyLogs(dnsLogs, httpLogs io.Reader, recorder *Recorder) error {
    if dnsLogs != nil {
        if err := parseDNSProxyLogs(dnsLogs, recorder); err != nil {
            return fmt.Errorf("parse dns proxy logs: %w", err)
        }
    }
    if httpLogs != nil {
        if err := parseHTTPProxyLogs(httpLogs, recorder); err != nil {
            return fmt.Errorf("parse http proxy logs: %w", err)
        }
    }
    return nil
}
```

Implement the two internal parsers:

`parseDNSProxyLogs(r io.Reader, rec *Recorder) error` -- scan line by line with `bufio.Scanner`. For each line, `json.Unmarshal` into `proxyLogEntry`. If `EventType == "dns_query"` and `Domain != ""`, call `rec.RecordDNSQuery(entry.Domain)`. Record ALL queries (allowed and denied) since in learning mode we want to capture what the workload needs. Skip lines that fail to unmarshal (non-JSON debug lines).

`parseHTTPProxyLogs(r io.Reader, rec *Recorder) error` -- scan line by line. For each line, unmarshal into `proxyLogEntry`.
- If `EventType == "github_repo_allowed"` and `Owner != ""` and `Repo != ""`, call `rec.RecordRepo(entry.Owner + "/" + entry.Repo)`.
- If `EventType` is any of `"github_mitm_connect"`, `"http_blocked"`, or any event with a non-empty `Host` field, strip port using `net.SplitHostPort` (fall back to raw Host if no port) and call `rec.RecordHost(host)`.
- Skip lines that fail to unmarshal.
