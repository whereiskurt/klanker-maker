    Create `docs/e2e-verification-checklist.md` covering Items 3-6 from Phase 21 scope. Structure as a step-by-step operator procedure:

    **Section 1: Sidecar E2E Verification (Item 3)**
    Prerequisites: A running sandbox (EC2 or ECS) provisioned via `km create`

    1. DNS Proxy:
       - Inside sandbox: `nslookup blocked.example.com` -- expect NXDOMAIN or timeout
       - Inside sandbox: `nslookup {allowed-domain}` -- expect resolution
       - Check CloudWatch logs at `/km/sandboxes/{id}/dns-proxy` for query entries

    2. HTTP Proxy:
       - Inside sandbox: `curl -v https://blocked-host.com` -- expect proxy 403 or connection refused
       - Inside sandbox: `curl -v https://{allowed-host}` -- expect 200
       - Check CloudWatch logs at `/km/sandboxes/{id}/http-proxy` for CONNECT entries

    3. Audit Log:
       - Run a few commands inside sandbox
       - Check CloudWatch log group `/km/sandboxes/{id}/` for audit entries
       - Verify command timestamps and content are captured

    4. OTel Tracing:
       - Inside sandbox, run a workload that makes HTTP requests
       - Check configured OTel collector endpoint for traces with sandbox-id tag
       - Verify trace context propagation through proxy (X-Trace-Id headers preserved)

    **Section 2: GitHub Repo Cloning (Item 4)**
    Prerequisites: GitHub App configured via `km configure github`, `km doctor` GitHub check passes

    1. Provision sandbox with `sourceAccess.github` configured for a specific repo
    2. Inside sandbox: `git clone https://github.com/{org}/{allowed-repo}` -- expect success
    3. Inside sandbox: `git clone https://github.com/{org}/{non-allowed-repo}` -- expect auth failure
    4. Inside sandbox: `git push origin main` to non-allowed ref -- expect rejection (if push restrictions configured)

    **Section 3: Inter-Sandbox Email (Item 5)**
    Prerequisites: Two running sandboxes (A and B) with email configured

    1. Sandbox A: call `SendSignedEmail` to B's address (`{B-id}@sandboxes.{domain}`)
    2. Sandbox B: call `ListMailboxMessages` -- expect message from A
    3. Sandbox B: call `ReadMessage` + `ParseSignedMessage` -- expect `SignatureOK=true`
    4. Verify message content matches what A sent

    **Section 4: Email Allow-List Enforcement (Item 6)**
    Prerequisites: Sandbox with `allowedSenders: ["self"]` (restrictive)

    1. Send email from a third sandbox (not self) to the restricted sandbox
    2. Call `ParseSignedMessage` with the restricted sandbox's allow list
    3. Expect `ErrSenderNotAllowed` or `SignatureOK=false` (sender not on list)
    4. Send self-mail (same sandbox to itself) -- expect success

    Each section includes: prerequisites, exact commands, expected outcomes, and what failure looks like.
