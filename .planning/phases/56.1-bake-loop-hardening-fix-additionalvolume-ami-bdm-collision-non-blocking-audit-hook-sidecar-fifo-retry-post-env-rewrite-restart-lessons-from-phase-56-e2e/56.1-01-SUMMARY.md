# Phase 56.1 Plan 01: BDM Collision Fix for additionalVolume + Raw AMI Launch Summary

**One-liner:** AMI BDM collision detection at compile time picks the first non-colliding device from /dev/sd[f-p] and threads it through Compile into a new Terraform variable `additional_volume_device_name` replacing the hardcoded `/dev/sdf` in the ec2spot module.

## What Was Built

### New Helpers

**`pkg/aws.AMIBDMDeviceNames(ctx, client, amiID) ([]string, error)`**
- Calls `DescribeImages` (already in `EC2AMIAPI` interface) and extracts `BlockDeviceMappings[*].DeviceName`
- Returns `(nil, nil)` for empty amiID or zero images ‚Äî safe as a no-op for non-raw-AMI paths
- Wraps errors with `"describe images %s for BDM: %w"` for caller-side `errors.Is`

**`pkg/compiler.pickAdditionalVolumeDevice(amiDevices []string) string`**
- Selects the first device from `/dev/sdf` through `/dev/sdp` not occupied by the AMI's BDMs
- Normalizes `/dev/xvdX` as equivalent to `/dev/sdX` (NVMe alias handling per RESEARCH Open Question 2)
- Returns `/dev/sdf` as default when input is nil/empty ‚Äî backwards-compatible

**`pkg/compiler.IsRawAMIID(s string) bool`**
- Exported wrapper for the existing unexported `isRawAMIID` ‚Äî allows `internal/app/cmd/create.go` to guard BDM lookups without importing the regex directly

### Signature Changes

**`compiler.Compile`** ‚Äî added `amiBDMDeviceNames []string` as 5th parameter
- ECS and Docker paths ignore it (pass nil)
- EC2 path threads it into `generateEC2ServiceHCL` ‚Üí `ec2HCLParams.AdditionalVolumeDeviceName`

**`generateEC2ServiceHCL`** ‚Äî added `amiBDMDeviceNames []string` as 8th parameter
- `ec2HCLParams.AdditionalVolumeDeviceName` = `pickAdditionalVolumeDevice(amiBDMDeviceNames)`
- Template emits `additional_volume_device_name = "{{ .AdditionalVolumeDeviceName }}"` in the `module_inputs` block

### Terraform Changes

**`infra/modules/ec2spot/v1.0.0/variables.tf`**
- Added `additional_volume_device_name` variable, `type = string`, `default = "/dev/sdf"`

**`infra/modules/ec2spot/v1.0.0/main.tf`**
- Replaced hardcoded `device_name = "/dev/sdf"` with `device_name = var.additional_volume_device_name` in `aws_volume_attachment.additional`

### Call Site Updates

Three Compile call sites in `internal/app/cmd/create.go` updated:
- Line ~430 (docker path): passes `nil` ‚Äî no BDM lookup needed for docker
- Line ~480 (EC2 AZ retry loop): BDM lookup runs ONCE before the loop; `amiBDMDevices` threaded through
- Line ~1687 (runCreateRemote): same guard + lookup pattern

`internal/app/cmd/budget.go` and all test files pass `nil` (pre-existing behavior preserved).
