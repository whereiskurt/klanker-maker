<objective>
Eliminate the EC2 launch failure that occurs when an operator launches a sandbox from a baked AMI whose own block device mappings already declare `/dev/sdf` AND the profile requests an `additionalVolume`. The Terraform module currently hardcodes `/dev/sdf` as the additional-volume attach point at `infra/modules/ec2spot/v1.0.0/main.tf:582`; EC2 rejects the launch because the device name is taken by the AMI itself.

The fix detects the collision at compile time using a new `pkg/aws.AMIBDMDeviceNames` helper, picks the first non-colliding device from the canonical AWS additional-volume range (`/dev/sdf` through `/dev/sdp`), threads that device name through the compiler into a new Terraform variable `additional_volume_device_name`, and replaces the hardcoded `/dev/sdf` in `main.tf` with that variable.

Purpose: Bake-loop end-to-end. Phase 56 introduced the bake (`km ami bake`) and Phase 33.1 introduced raw-AMI-ID support (`spec.runtime.ami: ami-xxxxxxxx`); Phase 56.1 fixes the launch path so the bake then relaunch loop is actually usable when the source profile has `additionalVolume` set (as `profiles/dc34.ami.yaml` does ‚Äî the profile that triggered the bug).

- `pkg/aws/ec2_ami.go` ‚Äî `AMIBDMDeviceNames` helper using existing `EC2AMIAPI.DescribeImages`

       ```go
       // AMIBDMDeviceNames returns the device names from BlockDeviceMappings for an AMI.
       // Returns (nil, nil) when amiID is empty or no images are returned.
       // Used by the compiler to detect /dev/sdf collision before emitting additionalVolume HCL.
       func AMIBDMDeviceNames(ctx context.Context, client EC2AMIAPI, amiID string) ([]string, error) {
           if amiID == "" { return nil, nil }
           out, err := client.DescribeImages(ctx, &ec2.DescribeImagesInput{ImageIds: []string{amiID}})
           if err != nil { return nil, fmt.Errorf("describe images %s for BDM: %w", amiID, err) }
           if len(out.Images) == 0 { return nil, nil }
           var devices []string
           for _, bdm := range out.Images[0].BlockDeviceMappings {
               if bdm.DeviceName != nil { devices = append(devices, *bdm.DeviceName) }
           }
           return devices, nil
       }
       ```

    2. Add `pickAdditionalVolumeDevice` + `additionalVolumeDeviceCandidates` to `pkg/compiler/service_hcl.go`. Place near `isRawAMIID` (line 27). Per RESEARCH.md ¬ß"Selecting next non-colliding device name", with NVMe alias normalization (Open Question 2):

       ```go
       var additionalVolumeDeviceCandidates = []string{
           "/dev/sdf","/dev/sdg","/dev/sdh","/dev/sdi","/dev/sdj",
           "/dev/sdk","/dev/sdl","/dev/sdm","/dev/sdn","/dev/sdo","/dev/sdp",
       }
       func pickAdditionalVolumeDevice(amiDevices []string) string {
           occupied := make(map[string]bool, len(amiDevices)*2)
           for _, d := range amiDevices {
               occupied[d] = true
               // Treat /dev/xvdX as occupying /dev/sdX and vice versa (NVMe alias)
               if strings.HasPrefix(d, "/dev/xvd") {
                   occupied["/dev/sd"+d[len("/dev/xvd"):]] = true
               }
               if strings.HasPrefix(d, "/dev/sd") {
                   occupied["/dev/xvd"+d[len("/dev/sd"):]] = true
               }
           }
           for _, c := range additionalVolumeDeviceCandidates {
               if !occupied[c] { return c }
           }
           return "/dev/sdf"
       }
       ```
