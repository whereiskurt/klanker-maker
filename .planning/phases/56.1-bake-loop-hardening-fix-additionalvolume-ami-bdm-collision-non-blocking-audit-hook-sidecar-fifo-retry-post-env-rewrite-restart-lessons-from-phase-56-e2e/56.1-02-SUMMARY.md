# Phase 56.1 Plan 02: Bake-Loop Hardening — Audit Hook, FIFO Retry, Sidecar Restart Summary

**One-liner:** Non-blocking timeout-tee FIFO writes in bash + 10-attempt retry loop in km-audit-log + post-cloud-init sidecar restart to fix three runtime hazards on AMI-backed sandbox launch.

---

## Objective

Eliminate three runtime hazards discovered during Phase 56 bake-loop end-to-end testing:

- **Bug 2 (login-shell deadlock):** `_km_audit` PROMPT_COMMAND hook blocked indefinitely on FIFO write when km-audit-log had not yet opened the read end.
- **Bug 3 (sidecar silent audit failure):** km-audit-log opened the audit FIFO exactly once; on boot-race or baked-AMI re-launch the pipe didn't exist, causing silent fallback to `/dev/null` stdin.
- **Bug 4 (stale sidecar identity):** Baked AMI carried previous sandbox's `SANDBOX_ID`/`CW_LOG_GROUP` in systemd `Environment=` directives; cloud-init rewrote the unit files but never restarted the sidecars.

---

## Bugs Fixed

### Bug 2: Login-Shell Deadlock (`pkg/compiler/userdata.go` line 487 / 514)

**Root cause:** The `_km_audit()` PROMPT_COMMAND hook used a bare redirect:
```bash
printf '...' > /run/km/audit-pipe 2>/dev/null
```
Opening a FIFO `O_WRONLY` blocks until a reader is present. `2>/dev/null` suppresses stderr but does nothing about the blocking `open()` syscall.

**Fix:** Replace with the timeout-tee pattern:
```bash
printf '...' | timeout 0.1 tee /run/km/audit-pipe > /dev/null 2>/dev/null || true
```
`timeout 0.1` bounds the FIFO write at 100ms. `tee` opens the pipe; its stdout is discarded. `|| true` ensures PROMPT_COMMAND never receives a non-zero exit.

Applied to both `_km_audit()` and `_km_heartbeat()` (same pattern, same vulnerability).

### Bug 3: Sidecar FIFO Single-Attempt (`sidecars/audit-log/cmd/main.go` line 178)

**Root cause:** Single `os.OpenFile(pipePath, os.O_RDWR, 0)` attempt. If `/run/km/` didn't exist or the FIFO hadn't been created yet (boot race on AMI re-launch, systemd ordering), the sidecar silently fell back to stdin — which is `/dev/null` on EC2.

**Fix:** Extract `openAuditPipeWithRetry(pipePath string, maxAttempts int, baseBackoff time.Duration) (*os.File, error)` helper that:
1. Calls `os.MkdirAll(filepath.Dir(pipePath), 0755)` to create `/run/km/` if absent.
2. Calls `syscall.Mkfifo(pipePath, 0666)` if the path doesn't exist (ignores EEXIST).
3. Calls `os.OpenFile(pipePath, os.O_RDWR, 0)` — O_RDWR avoids blocking when no reader/writer is on the other end.
4. On failure, sleeps `attempt * baseBackoff` and retries.

Default parameters: `maxAttempts=10, baseBackoff=500ms` → cumulative wait ~27.5s before giving up and falling back to stdin.

### Bug 4: Stale Sidecar Identity (`pkg/compiler/userdata.go` end of script)

**Root cause:** Baked AMIs carry `Environment=SANDBOX_ID=<old-sandbox>` baked into `/etc/systemd/system/km-*.service`. Cloud-init rewrites the unit files (via `cat > /etc/systemd/system/km-*.service` heredocs) and runs `systemctl daemon-reload`, but never issues a restart. Sidecars continue logging under the bake-source's CloudWatch group.

**Fix:** Add as the last cloud-init step before `SANDBOX_READY`:
```bash
systemctl restart km-audit-log km-dns-proxy km-http-proxy km-tracing 2>/dev/null || true
```
Placed after: sidecar binary download from S3, FIFO creation, all `cat >` unit file writes, and `systemctl daemon-reload`. The `2>/dev/null || true` guards against conditionally absent units (e.g., km-dns-proxy is omitted in eBPF-only mode).

## Operator Deploy Steps Required

These changes require two operator actions before they take effect on remote sandboxes:

1. **`make build`** — rebuilds the `km` binary with the new userdata template. Already done in this plan (km v0.1.404, commit 1822f84). Next `km create` will use the new template.

2. **`km init --sidecars`** — uploads the new `km-audit-log` binary (with the FIFO retry loop) to S3. Remote sandboxes download sidecars from S3 during cloud-init (Section 5); without this step, existing and new sandboxes will use the old single-attempt binary. See project memory: "Schema change → km init --sidecars".
