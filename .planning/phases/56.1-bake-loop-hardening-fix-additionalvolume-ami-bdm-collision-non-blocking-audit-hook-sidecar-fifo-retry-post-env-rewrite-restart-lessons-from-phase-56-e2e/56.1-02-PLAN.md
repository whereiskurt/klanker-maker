<objective>
Eliminate three runtime hazards observed when running an AMI-backed sandbox end-to-end after Phase 56 completed:

1. **Login-shell deadlock (Bug 2):** The `_km_audit` PROMPT_COMMAND hook in `pkg/compiler/userdata.go:487` writes to `/run/km/audit-pipe` with a bare `>` redirect. When the FIFO has no reader (the common race when `km-audit-log` hasn't opened the pipe yet), `open(O_WRONLY)` blocks indefinitely. Every login command hangs. Fix: replace the redirect with a `timeout 0.1 tee /run/km/audit-pipe > /dev/null 2>/dev/null || true` pattern that bounds the wait at 100ms. Apply the same pattern to the `_km_heartbeat` writer.

2. **Sidecar single-attempt FIFO open (Bug 3):** `sidecars/audit-log/cmd/main.go:178` calls `os.OpenFile(pipePath, O_RDWR, 0)` exactly once. If `/run/km/` doesn't exist or the FIFO hasn't been created, the sidecar logs an error and silently falls back to stdin (which is `/dev/null` on EC2 — zero events, silent audit). On baked-AMI re-launches and on `systemctl restart km-audit-log` after `rm -rf /run/km`, this races. Fix: retry up to 10 times with linear 0.5s backoff, creating the directory and FIFO defensively on each iteration; only after exhausting retries does the sidecar fall back to stdin.

3. **Stale sidecar identity on baked AMI (Bug 4):** When sandbox B is launched from an AMI baked off sandbox A, the AMI carries A's systemd unit files (`/etc/systemd/system/km-*.service`) with `Environment=SANDBOX_ID=A` baked in. Cloud-init rewrites `/etc/profile.d/km-identity.sh` with B's identity but never restarts the sidecars; they continue logging under sandbox A's CloudWatch group. Fix: after the existing `systemctl daemon-reload` (line 523), AND after the sidecar binaries are downloaded from S3 (section 5), AND after the FIFO is created (lines 420-422), issue `systemctl restart km-dns-proxy km-http-proxy km-audit-log km-tracing` to re-read the unit files which the compiler template re-rendered with B's identity.

Purpose: Make the bake-loop end-to-end usable. Phase 56's `km ami bake` produces an AMI; Phase 33.1 + Phase 56.1 Plan 01's BDM fix lets that AMI launch; *this* plan ensures that when it launches, login shells respond, audit logs flow, and sidecars carry the new sandbox's identity rather than the bake-source's.

       ```go
       // openAuditPipeWithRetry opens (and creates if necessary) a FIFO at pipePath,
       // retrying up to maxAttempts times with linear backoff (attempt N waits N*baseBackoff
       // before the next try). Cumulative wait for default (10, 500ms) is ~27.5s.
       //
       // On each iteration the function:
       //   1) Ensures filepath.Dir(pipePath) exists (os.MkdirAll, mode 0755).
       //   2) Re-creates the FIFO if missing (syscall.Mkfifo, mode 0666; ignores EEXIST).
       //   3) Attempts os.OpenFile(pipePath, os.O_RDWR, 0). O_RDWR avoids the
       //      classic FIFO open() block when no reader/writer is present.
       //
       // Returns the open *os.File on success, or the LAST error after exhausting attempts.
       func openAuditPipeWithRetry(pipePath string, maxAttempts int, baseBackoff time.Duration) (*os.File, error) {
           if pipePath == "" {
               return nil, fmt.Errorf("openAuditPipeWithRetry: empty pipePath")
           }
           var lastErr error
           for attempt := 1; attempt <= maxAttempts; attempt++ {
               if mkErr := os.MkdirAll(filepath.Dir(pipePath), 0755); mkErr != nil {
                   lastErr = mkErr
                   log.Warn().Err(mkErr).Int("attempt", attempt).Msg("audit-log: cannot create pipe dir")
               } else {
                   // Create FIFO if missing (ignore EEXIST).
                   if _, statErr := os.Stat(pipePath); os.IsNotExist(statErr) {
                       if mkfErr := syscall.Mkfifo(pipePath, 0666); mkfErr != nil && !os.IsExist(mkfErr) {
                           lastErr = mkfErr
                           log.Warn().Err(mkfErr).Int("attempt", attempt).Msg("audit-log: mkfifo failed")
                       }
                   }
                   f, openErr := os.OpenFile(pipePath, os.O_RDWR, 0)
                   if openErr == nil {
                       return f, nil
                   }
                   lastErr = openErr
                   log.Warn().Err(openErr).Int("attempt", attempt).Str("pipe", pipePath).Msg("audit-log: retrying pipe open")
               }
               if attempt < maxAttempts {
                   time.Sleep(time.Duration(attempt) * baseBackoff)
               }
           }
           return nil, lastErr
       }
       ```
