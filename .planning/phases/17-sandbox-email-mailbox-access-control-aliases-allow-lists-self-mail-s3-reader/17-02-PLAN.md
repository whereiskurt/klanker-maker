1. Create `pkg/aws/mailbox.go` with package `kmaws`.

2. Define `MailboxS3API` narrow interface:
   ```go
   type MailboxS3API interface {
       ListObjectsV2(ctx context.Context, input *s3.ListObjectsV2Input, opts ...func(*s3.Options)) (*s3.ListObjectsV2Output, error)
       GetObject(ctx context.Context, input *s3.GetObjectInput, opts ...func(*s3.Options)) (*s3.GetObjectOutput, error)
   }
   ```

3. Define `MailboxMessage` struct:
   ```go
   type MailboxMessage struct {
       MessageID   string // extracted from S3 key (last path segment)
       S3Key       string // full S3 object key
       From        string
       To          string
       Subject     string
       Body        string // decrypted plaintext body
       SenderID    string // X-KM-Sender-ID header value
       SignatureOK bool   // true if Ed25519 signature verified
       Encrypted   bool   // true if X-KM-Encrypted: true was present
       Plaintext   bool   // true if no X-KM-Signature header
   }
   ```

4. Implement `ListMailboxMessages(ctx, client MailboxS3API, bucket, sandboxID, emailDomain string) ([]string, error)`:
   - List all objects under `mail/` prefix (flat — per research decision Option A)
   - Return all S3 keys without filtering by recipient. Caller is responsible for filtering via ParseSignedMessage if needed.
   - Handle pagination if there are more than 1000 objects (use ContinuationToken loop)

5. Implement `ReadMessage(ctx, client MailboxS3API, bucket, s3Key string) ([]byte, error)`:
   - GetObject, read body into []byte, close body reader
   - Return raw MIME bytes

6. Implement `ParseSignedMessage(rawMIME []byte, receiverSandboxID, pubKeyB64 string, allowedSenders []string) (*MailboxMessage, error)`:
   - Use `net/mail.ReadMessage(bytes.NewReader(rawMIME))` to parse headers and body
   - Extract: From, To, Subject from standard headers
   - Extract: X-KM-Sender-ID, X-KM-Signature, X-KM-Encrypted from custom headers
   - Read body into string via `io.ReadAll(msg.Body)`
   - If X-KM-Encrypted == "true" and body appears base64-encoded, note Encrypted=true (actual decryption requires receiver's NaCl private key — accept the ciphertext body for now, document that caller must call DecryptFromSender separately)
   - If X-KM-Signature is present:
     - Call `VerifyEmailSignature(pubKeyB64, body, sigB64)` — set SignatureOK=true if nil error, false otherwise
     - Do NOT return error on signature failure — set SignatureOK=false so caller can decide
   - If X-KM-Signature is absent, set Plaintext=true
   - Allow-list enforcement:
     - Self-mail (senderID == receiverSandboxID): always permitted, regardless of allowedSenders
     - Otherwise: call `MatchesAllowList(allowedSenders, senderID, "", receiverSandboxID)` — if false, return `ErrSenderNotAllowed` (define as package-level sentinel)
     - Note: senderAlias parameter is empty string here because we don't have the sender's alias in the MIME headers. Future enhancement could add X-KM-Sender-Alias header.
   - Return populated MailboxMessage

7. Define `ErrSenderNotAllowed = errors.New("sender not on allow-list")` as package-level sentinel.

8. Write comprehensive tests in `mailbox_test.go`:
   - Mock MailboxS3API struct capturing inputs, returning canned responses
   - Build test MIME messages using the same `buildRawMIME` format from identity.go (X-KM-Sender-ID, X-KM-Signature headers + body)
   - Test all behaviors listed above
