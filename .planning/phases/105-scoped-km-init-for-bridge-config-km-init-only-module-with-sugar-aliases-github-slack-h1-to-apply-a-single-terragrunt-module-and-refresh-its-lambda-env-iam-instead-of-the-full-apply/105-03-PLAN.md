<objective>
Implement the tier-1 scoped apply happy path:

- `RunInitScopedWithRunner(runner InitRunner, repoRoot, region, module string, dryRun, acceptDestroys bool) error`
  — the testable core. Filters `regionalModules()` to the one named module, runs the
  envReqs check, prints a tier+module banner, and (when `dryRun=false`) calls
  `runner.Apply(target.dir)` with the per-module timeout. When `dryRun=true` it prints a
  "would apply" summary and returns without applying. Includes email-handler ARN capture.
  (Tier-2 ses gate logic is added in Plan 04 — this plan handles tier-1 modules; for `ses`
  it may delegate to a `gatedApply` hook that Plan 04 fills in, or return a clear "tier-2
  handled in Plan 04" path. Recommended: structure RunInitScopedWithRunner so the tier-2
  branch calls an injectable `scopedGateFunc` that defaults to a passthrough no-op in this
  plan and is replaced in Plan 04.)

- `runInitScoped(cfg, awsProfile, region, verbose, module string, dryRun, acceptDestroys bool) error`
  — the production wrapper. Loads AWS config + validates credentials, then runs the
  NON-OBVIOUS side effects that the bridges depend on (the bridges read commands/bot-id from
  SSM at cold start, NOT only from the Lambda env block):
    1. ExportTerragruntEnvVars(cfg)   — recompute KM_* env (required; else get_env() returns "")
    2. EnsureSlackBotUserIDFromSSM(ctx, cfg, awsCfg)  — unconditional (no-op when not slack)
    3. if module == "lambda-github-bridge" && len(cfg.Github.Commands) > 0: PublishGitHubCommandsToSSM(...)
    4. if module == "lambda-h1-bridge"     && len(cfg.H1.Programs)  > 0: PublishH1CommandsToSSM(...)
  Then construct the terragrunt runner and call RunInitScopedWithRunner.

- Rebind the `runInitScopedFunc` package var (declared as a stub in Plan 02) to `runInitScoped`.

Purpose: Deliver the actual fast-iteration deploy path with correct SSM side effects.
Output: Working `km init --github/--slack/--h1/--email` for tier-1 modules + tests.
</objective>
