<objective>
Implement the tier-2 gated path for `--only ses`. The gate is a PRE-APPLY destroy-class
check on a SINGLE module — it MUST NOT call `RunInitPlanFunc` (which plans all ~27 modules
and never applies). Instead it calls `planModule()` for just the ses module, evaluates with
`planreport.Evaluate([...]Report, acceptDestroys)`, and:
  - if result.Blocked → print trip block, return error (apply NOT called)
  - if result.Trips but overridden (acceptDestroys) → print trip block + override note, continue
  - else → continue
Then it replicates the ses special-casing from `RunInitWithRunner` (init.go:1845-1870):
`InitSESPreflight(ctx)` + `runner.Reconfigure(ctx, target.dir)` (ses moved v1→v2 source — omitting
Reconfigure silently wedges the backend).

This is wired by rebinding the `scopedGateFunc` package var (stub from Plan 03) to the real
implementation. RunInitScopedWithRunner already invokes scopedGateFunc for gated modules before
the dry-run/apply decision — so on dry-run the gate plan still runs (showing what would apply)
and on apply the gate blocks protected destroys.

Purpose: Make `--only ses` the deliberate, gated "did the dns and all that" path.
Output: Real tier-2 gate + ses preflight + the 3 tier-2 tests passing.
</objective>
