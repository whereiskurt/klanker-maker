<objective>
Ship the operator escape hatch and the doctor visibility for the durable store. `km slack adopt <alias> <channelID>` lets an operator seed the alias→channel mapping for a genuinely orphaned channel (validate `^C[A-Z0-9]+$` + confirm bot membership via `conversations.info` is_member + write-through to BOTH the DDB store and the SSM by-name cache) so the next `km create` on that alias resolves O(1). Depends on 104-03 (the `pkg/aws.SlackChannelStore` helper + config getter + IAM that adopt and the doctor check both use).

Purpose: An actionable manual path for orphaned channels (no YAML edit needed) + operator visibility into the new table.

</objective>
