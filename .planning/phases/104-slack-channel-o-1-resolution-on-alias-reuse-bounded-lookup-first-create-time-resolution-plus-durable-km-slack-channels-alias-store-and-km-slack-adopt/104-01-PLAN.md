<objective>
Land the P0+P1 core of the O(1) Slack channel resolution fix — pure Go, no infra. This is the heart of the incident fix: bound `FindChannelByName` (page cap + ctx-per-page + typed `ErrScanCapExceeded`), add the `IsChannelNotFound` classifier so only a definitive `channel_not_found` invalidates a stored mapping, and restructure `resolveSlackChannel` (Mode-2) into a lookup-first, budgeted state machine that NEVER falls through to an unbounded `conversations.list` scan on a transient `conversations.info` blip (the exact defeater that wedged the 900s create-handler).

The durable DDB store arrives in plan 104-03; this plan introduces the `SlackChannelStore` interface (in the cmd package) with a nil-tolerant resolver so the resolver works today against the SSM by-name cache and is ready to be wired to DDB later.

Purpose: Eliminate the unbounded-scan wedge at create time. A reused alias resolves O(1); a transient hiccup is bounded-retried then optimistically trusted; the workspace is never enumerated unless explicitly opted-in via `KM_SLACK_MAX_SCAN_PAGES>0`.
Output: Bounded `FindChannelByName`, `ErrScanCapExceeded`, `IsChannelNotFound`, and a lookup-first/budgeted `resolveSlackChannel` with `slack_resolve` observability.
</objective>
