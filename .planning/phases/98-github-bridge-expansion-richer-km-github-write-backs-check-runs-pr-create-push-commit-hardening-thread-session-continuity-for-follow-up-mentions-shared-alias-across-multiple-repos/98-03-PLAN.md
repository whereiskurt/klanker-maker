Shared alias = two entries with DIFFERENT match but SAME explicit alias → both Resolve to that alias. This already works.

doctor.go (internal/app/cmd/) — existing doctor checks pattern. Read the GitHub-related checks already present (Phase 97 added github_app_configured etc.) and add the new collision check alongside them. The check reads cfg's github.repos and reports:
- WARN "alias collision" when two entries with the SAME match-owner namespace resolve to the same alias UNINTENTIONALLY (heuristic: an entry omits `alias` (defaults to gh-{owner}-{repo}) yet that default equals another entry's explicit alias).
- WARN "overlapping match" when an exact match and a glob both match a plausible repo (e.g. `owner/repo` and `owner/*`) — exact wins, but warn so the operator knows the glob is partially shadowed.
- Intentional shared alias (multiple entries with the SAME explicit alias) is NOT a warning — it is the supported feature. Only warn on the unintentional/ambiguous cases.
