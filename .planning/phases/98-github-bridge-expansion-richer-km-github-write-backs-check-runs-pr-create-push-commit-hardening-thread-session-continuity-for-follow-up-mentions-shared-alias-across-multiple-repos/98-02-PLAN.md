<objective>
Add thread/session continuity for GitHub: a `(repo, number) → {sandbox_id, agent_session_id}` mapping backed by the `km-github-threads` DDB table (created in 98-00) so follow-up @-mentions in the same PR/issue continue the same agent session, plus a thread-bypass so replies in a known thread skip the re-@-mention requirement (mirrors Phase 91.3 Slack thread-bypass at pkg/slack/bridge/events_handler.go:311-339).

This plan owns the shared bridge files (interfaces.go, aws_adapters.go, webhook_handler.go). 98-04 (resume + cold-create) builds on top of this plan's changes to the same files — hence 98-04 is in a later wave.

Continuity data lives ONLY in km-github-threads, never in km-sandboxes — this deliberately sidesteps the SandboxMetadata lossy round-trip footgun (RESEARCH Pitfall 5): no new km-sandboxes attribute, no marshal/unmarshal changes.

Purpose: Natural back-and-forth with the bot in a PR thread without re-mentioning, and session continuity so the agent remembers prior turns.
Output: DynamoGitHubThreadStore adapter, thread-bypass + Upsert wiring in Handle(), bridge IAM v1.1.0, poller session write.
</objective>
