<objective>
Fix the broken cold-create path (Phase 97: built but never exercised — see 97-VERIFICATION.md § Cold path) AND add stopped-sandbox auto-resume, unified into ONE dispatch decision per the design spec: resume a paused/stopped aliased box if one exists, cold-create only when the alias is truly absent. These are coupled because today the warm path is status-agnostic — it finds a stopped row, enqueues to a dead queue, and the cold path never fires (the stopped row holds the alias).

The four cold-create defects (RESEARCH GH-COLD-CREATE):
1. EventBridge detail omits sandbox_id → bridge generates `gh-{8hex}` (compiler.GenerateSandboxID semantics).
2. artifact_prefix is doubled (`{prefix}/profiles/{profile}.yaml`) → set to `github-profiles/{profileSlug}` (a directory; create-handler appends `.km-profile.yaml`).
3. No profile staged in S3 → `km init` pre-stages each github.repos profile (deduped by slug) to `github-profiles/{slug}/.km-profile.yaml`, plus its SOPS bundle.
4. No Claude auth on a fresh box → github-review.yaml gets `spec.secrets.sopsFile` (Phase 89 SOPS, NOT Bedrock) so the box self-authenticates.

This plan builds on 98-02's bridge changes (same files: interfaces.go, aws_adapters.go, webhook_handler.go) → it is in a later wave. It also ADDS the EC2 resume IAM into the v1.1.0 module 98-02 created, and adds `dynamodb-github-threads` to `regionalModules()` (GREENing the 98-00 guard test).

Purpose: "configure-once, stop, GitHub wakes it" + first-ever working cold-create with self-auth.
Output: ResolveByAliasWithStatus, EC2Resumer, fixed EventBridgeAdapter, preStageGitHubProfiles, regionalModules entry, EC2 IAM, SOPS-enabled profile.
</objective>
