<feature>
  <name>Natural language time parser and schedule name sanitizer</name>
  <files>pkg/at/parser.go, pkg/at/parser_test.go</files>
  <behavior>
    --- ScheduleSpec type ---
    - ScheduleSpec has fields: Expression (string, e.g. "at(2026-04-10T15:00:00)" or "cron(0 15 ? * 5 *)"), IsRecurring (bool), HumanExpr (string, the original input)

    --- Parse function ---
    - Parse("10pm tomorrow", now) returns ScheduleSpec with at() expression, IsRecurring=false
    - Parse("next tuesday at 2pm", now) returns at() expression for next Tuesday 14:00 UTC
    - Parse("in 30 minutes", now) returns at() expression 30 minutes from now
    - Parse("tonight at 11pm", now) returns at() expression for today 23:00 UTC
    - Parse("every thursday at 3pm", now) returns cron(0 15 ? * 5 *), IsRecurring=true
    - Parse("every day at 9am", now) returns cron(0 9 * * ? *), IsRecurring=true
    - Parse("daily at 9am", now) returns cron(0 9 * * ? *), IsRecurring=true
    - Parse("every monday at 8:30am", now) returns cron(30 8 ? * 2 *), IsRecurring=true
    - Parse("every friday at 5pm", now) returns cron(0 17 ? * 6 *), IsRecurring=true
    - Parse("every 2 hours", now) returns rate(2 hours), IsRecurring=true
    - Parse("every 30 minutes", now) returns rate(30 minutes), IsRecurring=true
    - Parse("gibberish nonsense", now) returns error
    - Parse("", now) returns error

    --- ValidateCron function ---
    - ValidateCron("cron(0 15 ? * 5 *)") returns nil (valid)
    - ValidateCron("0 15 * * 5") returns error (unix cron, wrong format)
    - ValidateCron("cron(0 15 15 * THU *)") returns error (both day-of-month and day-of-week set, neither is ?)
    - ValidateCron("garbage") returns error

    --- SanitizeScheduleName function ---
    - SanitizeScheduleName("kill-sb-a1b2c3d4-10pm-tomorrow") returns "kill-sb-a1b2c3d4-10pm-tomorrow" (unchanged, valid)
    - SanitizeScheduleName("kill sb-a1b2c3d4 at 10pm!") returns "kill-sb-a1b2c3d4-at-10pm" (spaces to dashes, strip invalid chars)
    - SanitizeScheduleName(string > 64 chars) returns truncated to 64 characters
    - SanitizeScheduleName("") returns error

    --- GenerateScheduleName function ---
    - GenerateScheduleName("kill", "sb-a1b2c3d4", "10pm tomorrow") returns "km-at-kill-sb-a1b2c3d4-10pm-tomorrow"
    - GenerateScheduleName("create", "", "every thursday at 3pm") returns "km-at-create-every-thursday-at-3pm"
    - Result is always valid per EventBridge constraints (sanitized, <= 64 chars)

    --- EventBridge day-of-week mapping ---
    - Thursday maps to 5 (NOT 4 as in unix cron)
    - Sunday maps to 1, Monday maps to 2, Saturday maps to 7
  </behavior>
  <implementation>
    Install `olebedev/when` dependency: `go get github.com/olebedev/when@v1.1.0`

    Create `pkg/at/parser.go`:
    - Define `ScheduleSpec` struct: `Expression string`, `IsRecurring bool`, `HumanExpr string`
    - `Parse(expr string, now time.Time) (ScheduleSpec, error)` — split strategy:
      1. Check if expr contains recurring keywords ("every", "each", "weekly", "daily", "monthly") -> custom recurring parser
      2. Otherwise -> `olebedev/when` for one-time date parsing -> format as `at(YYYY-MM-DDThh:mm:ss)`
    - Custom recurring parser (~50 lines regex):
      - `every <day> at <time>` -> `cron(M H ? * DOW *)` using EventBridge day-of-week mapping (1=SUN, 2=MON, ..., 7=SAT)
      - `every day at <time>` / `daily at <time>` -> `cron(M H * * ? *)`
      - `every <N> hours` -> `rate(<N> hours)`
      - `every <N> minutes` -> `rate(<N> minutes)`
    - `ValidateCron(expr string) error` — basic structural validation of EventBridge cron expressions (6 fields, day-of-month/day-of-week mutual exclusion with ?)
    - `SanitizeScheduleName(name string) (string, error)` — replace spaces with `-`, strip chars not in `[0-9a-zA-Z-_.]`, truncate to 64 chars, error on empty result
    - `GenerateScheduleName(command, sandboxID, timeExpr string) string` — builds `km-at-{command}-{sandboxID}-{timeExpr}`, sanitized

    Time parsing for recurring expressions: parse the time portion (e.g., "3pm", "8:30am", "14:00") using time.Parse with multiple formats ("3pm", "3:04pm", "15:04") to extract hour and minute.

    PITFALL: EventBridge cron day-of-week is 1-indexed with 1=SUN. Map: sun=1, mon=2, tue=3, wed=4, thu=5, fri=6, sat=7. Do NOT use unix cron mapping.
    PITFALL: When day-of-week is specified, day-of-month MUST be `?` and vice versa.
    PITFALL: EventBridge cron has 6 fields (includes year). Always append `*` for year field.
  </implementation>
</feature>
