    **sandbox_id.go:** `GenerateSandboxID() string` using `github.com/google/uuid`. Format: `sb-` + first 8 hex chars of UUID (no dashes). Add `go get github.com/google/uuid@latest`.

    **compiler.go:** Define types and main entry point:
    ```go
    type CompiledArtifacts struct {
        SandboxID   string
        ServiceHCL  string // content for service.hcl
        UserData    string // content for user-data.sh (EC2 only; empty for ECS)
        SGEgressRules []SGRule
        IAMPolicy   *IAMSessionPolicy
        SecretPaths []string // SSM parameter paths to inject
    }
    type SGRule struct {
        FromPort    int
        ToPort      int
        Protocol    string
        CIDRBlocks  []string
        Description string
    }
    type IAMSessionPolicy struct {
        MaxSessionDuration int      // seconds
        AllowedRegions     []string
    }
    func Compile(p *profile.SandboxProfile, sandboxID string, onDemand bool) (*CompiledArtifacts, error)
    ```
    Compile() validates substrate field is "ec2" or "ecs", then dispatches to substrate-specific generators. For this task, implement the EC2 path only; ECS path returns error "ecs substrate not yet implemented" (Task 2 adds it).

    **service_hcl.go:** Generate service.hcl using `text/template`. For EC2, the template must match the structure in `infra/live/sandboxes/_template/terragrunt.hcl` — specifically, the `locals` block with `sandbox_id`, `substrate_module = "ec2spot"`, and `module_inputs` containing the ec2spots variable structure matching `infra/modules/ec2spot/v1.0.0/variables.tf`. Use text/template with strict quoting — never fmt.Sprintf for HCL values per research anti-pattern.

    **userdata.go:** Generate user-data.sh using `text/template`. Include: (1) SSM agent install and start, (2) IMDSv2 token verification (curl PUT, fail if empty), (3) Loop over allowed secret paths calling `aws ssm get-parameter --with-decryption` and exporting as env vars, (4) GitHub token injection if sourceAccess.github is set (export GITHUB_TOKEN from SSM path `/km/github/app-token`), (5) SANDBOX_READY logger signal. Use the exact script pattern from RESEARCH.md.

    **security.go:** Three compilation helpers:
    - `compileSGRules(p)` returns baseline egress: TCP 443 to 0.0.0.0/0 (SSM agent + HTTPS), UDP 53 to 0.0.0.0/0 (DNS). Phase 3 tightens these when proxy sidecars handle filtering.
    - `compileIAMPolicy(p)` reads identity.roleSessionDuration (parse Go duration string to seconds, default 3600), identity.allowedRegions.
    - `compileSecrets(p)` reads identity.allowedSecretPaths → list of SSM ARN paths. If sourceAccess.github is set, appends `/km/github/app-token`.

    **Tests in compiler_test.go:** Write tests FIRST (RED), then implement (GREEN). Use testdata profiles loaded via `profile.Parse()` from the existing package. Create `pkg/compiler/testdata/ec2-basic.yaml` and `pkg/compiler/testdata/ec2-with-secrets.yaml` as minimal valid profiles for testing.
