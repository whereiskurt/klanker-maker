<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: End-to-end AWS verification of create and destroy</name>
  <what-built>
    Full km create/destroy pipeline for EC2 and ECS substrates. The operator needs to verify that real AWS resources are provisioned and torn down correctly.
  </what-built>
  <how-to-verify>
    **Test 1: EC2 Spot Sandbox (PROV-01, PROV-09, PROV-11, NETW-01, NETW-05, PROV-08)**
    1. Run: `./km create profiles/open-dev.yaml` (substrate: ec2, spot: true by default)
    2. Note the sandbox ID printed (e.g. sb-a1b2c3d4)
    3. Verify in AWS Console (EC2 > Instances):
       - Spot instance is running with tag `km:sandbox-id = sb-a1b2c3d4`
       - Instance metadata: IMDSv2 enforced (http-tokens = required)
       - Security group has egress rules for 443/TCP and 53/UDP only
    4. Verify in AWS Console (VPC > Security Groups):
       - sandbox_mgmt SG has the expected egress rules
    5. Verify SSM access: `aws ssm start-session --target <instance-id> --profile klanker-application`

    **Test 2: EC2 On-Demand Override (PROV-11)**
    1. Run: `./km create --on-demand profiles/open-dev.yaml`
    2. Verify the instance is NOT a spot instance (check Instance Lifecycle in console)

    **Test 3: ECS Fargate Spot Sandbox (PROV-09, PROV-10, PROV-12)**
    1. Create an ECS profile (edit substrate to "ecs" in a test profile)
    2. Run: `./km create test-ecs-profile.yaml`
    3. Verify in AWS Console (ECS > Clusters):
       - Cluster exists with sandbox name
       - Task is running with FARGATE_SPOT capacity provider
       - Task definition has 5 containers (main + 4 sidecar placeholders)

    **Test 4: Destroy EC2 (PROV-02)**
    1. Run: `./km destroy sb-<ec2-sandbox-id>`
    2. Verify: Instance terminated, VPC deleted, no orphaned resources
    3. Check: `aws resourcegroupstaggingapi get-resources --tag-filters Key=km:sandbox-id,Values=sb-<id>` returns empty

    **Test 5: Destroy ECS (PROV-02)**
    1. Run: `./km destroy sb-<ecs-sandbox-id>`
    2. Verify: Task stopped, service deleted, cluster removed, no orphans

    **Test 6: Secrets Injection (NETW-06, NETW-07, NETW-08)**
    1. Create a test SSM parameter: `aws ssm put-parameter --name /km/test/secret --value "test123" --type SecureString --profile klanker-application`
    2. Create a profile that references this secret path
    3. Run: `./km create test-secrets-profile.yaml`
    4. SSM into the instance and verify: `echo $test_secret` shows the value
    5. Clean up: destroy sandbox and delete SSM parameter
  </how-to-verify>
