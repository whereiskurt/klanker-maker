  <action>
    Create `internal/app/cmd/create.go` implementing `km create <profile.yaml> [--on-demand]`.

    Follow the established Cobra pattern from validate.go: `NewCreateCmd(cfg *config.Config) *cobra.Command`.

    **Command flow (runCreate function):**
    1. Parse profile path from args[0]
    2. Load and parse profile via `profile.Parse(data)` (same pattern as validate.go)
    3. If profile has `extends`, resolve via `profile.Resolve()` (same as validate.go)
    4. Validate profile via `profile.Validate()` and `profile.ValidateSemantic()` -- fail early if invalid
    5. Generate sandbox ID via `compiler.GenerateSandboxID()`
    6. Load AWS config via `awspkg.LoadAWSConfig(ctx, cfg.AWSProfile)` -- default profile "klanker-terraform"
    7. Validate AWS credentials via `awspkg.ValidateCredentials(ctx, awsCfg)` -- fail early if no valid creds
    8. Compile profile via `compiler.Compile(profile, sandboxID, onDemand)`
    9. Create sandbox dir via `terragrunt.CreateSandboxDir(repoRoot, sandboxID)`
    10. Populate sandbox dir via `terragrunt.PopulateSandboxDir(sandboxDir, artifacts.ServiceHCL, artifacts.UserData)`
    11. Create Runner and call `runner.Apply(ctx, sandboxDir)` -- streams output in real time
    12. On success: print sandbox ID and summary. On failure: attempt cleanup of sandbox dir.

    **Flags:**
    - `--on-demand` (bool, default false): overrides `spot: true` in the profile
    - `--aws-profile` (string, default "klanker-terraform"): AWS CLI profile to use

    **Output format:**
    ```
    Creating sandbox sb-a1b2c3d4 (substrate: ec2, spot: true)...
    [terragrunt output streams here]
    Sandbox sb-a1b2c3d4 created successfully.
    ```

    **Error handling:** If Apply() fails, print error and attempt `CleanupSandboxDir()`. Do NOT attempt destroy -- the resources may be partially created and require manual cleanup.

    **Register in root.go:** Add `rootCmd.AddCommand(NewCreateCmd(cfg))` in NewRootCmd.

    **Gitignore:** Add `infra/live/sandboxes/sb-*/` to `.gitignore` so per-sandbox directories are not committed. The `_template/` directory IS committed (already exists).

    **NETW-05 (IMDSv2):** The ec2spot module already has `http_tokens = "required"`. No additional code needed in the create command -- this is enforced at the Terraform module level. Document this in a code comment.

    **NETW-07 (SOPS):** SOPS decryption happens at provision time via site.hcl's `run_cmd("sops", "--decrypt", ...)` pattern. The compiler writes SSM parameter ARNs into tfvars; user-data decrypts at boot using the instance IAM role. No additional SOPS handling needed in the create command -- document this in a code comment.

    - TestDestroyCmd_NotFound: destroy with unknown sandbox ID returns "sandbox not found" error

  <action>
    Create `internal/app/cmd/destroy.go` implementing `km destroy <sandbox-id>`.

    Follow the established Cobra pattern: `NewDestroyCmd(cfg *config.Config) *cobra.Command`.

    **Command flow (runDestroy function):**
    1. Parse sandbox ID from args[0]. Validate format: must match `sb-[a-f0-9]{8}`.
    2. Load AWS config via `awspkg.LoadAWSConfig(ctx, cfg.AWSProfile)`
    3. Validate AWS credentials
    4. Discover sandbox via `awspkg.FindSandboxByID(ctx, tagClient, sandboxID)` -- fail if not found
    5. Print: `Destroying sandbox <id> (<resource_count> resources)...`
    6. Reconstruct sandbox directory path: `infra/live/sandboxes/<sandboxID>/`
    7. If sandbox directory exists locally, use it. If not, re-create it from template and populate with minimal service.hcl (just sandbox_id and substrate_module needed for Terragrunt state key resolution).
    8. **For EC2 substrate:** Get spot instance ID from `runner.Output()` -> `awspkg.GetSpotInstanceID()`. Call `awspkg.TerminateSpotInstance()` to explicitly terminate the instance BEFORE running destroy. This is critical per research Pitfall 1: `aws_spot_instance_request` destroy cancels the request but leaves the instance running.
    9. Run `runner.Destroy(ctx, sandboxDir)` -- streams output in real time
    10. On success: `CleanupSandboxDir()` to remove local directory
    11. Print: `Sandbox <id> destroyed successfully.`

    **Flags:**
    - `--aws-profile` (string, default "klanker-terraform"): AWS CLI profile to use
    - `--force` (bool, default false): skip confirmation prompt (future use; for now always proceeds)

    **Substrate detection:** Determine substrate from the sandbox's Terraform state. The `runner.Output()` call before destroy returns module outputs that indicate whether it's ec2spot or ecs. If ec2spot, the output contains `spot_instance_id`. If no `spot_instance_id` in output, skip the termination step (it's ECS or on-demand EC2).

    **Register in root.go:** Add `rootCmd.AddCommand(NewDestroyCmd(cfg))` in NewRootCmd.
