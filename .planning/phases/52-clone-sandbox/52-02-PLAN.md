<objective>
Implement `km clone <source> [new-alias]` command that duplicates a running sandbox by fetching its stored profile from S3, optionally staging workspace files through SSM+S3, and running the existing create pipeline to provision a fully independent clone.

Purpose: Enables quick sandbox duplication for fan-out agent workflows and workspace reuse without manually locating profile YAML files.

Output: Working `km clone` command with `--alias`, `--no-copy`, `--count N` flags; registered in root command; unit tested.
</objective>

    Actually, re-reading research more carefully: the staging key should be `artifacts/{clone-id}/staging/workspace.tar.gz` (per clone ID), and each clone's userdata unconditionally checks for its own staging key.
