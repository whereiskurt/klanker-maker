<objective>
Port the EC2 resume + DDB status-flip primitives from the GitHub bridge into the Slack
bridge package, with interfaces and unit tests. This is the foundational layer the
EventsHandler resume branch (Plan 02) and the wiring (Plan 03) build on.

This is a near-verbatim port of `pkg/github/bridge/aws_adapters.go`, with two deliberate
Slack-specific differences:
1. The Slack `SandboxStatusWriter` interface has ONLY `SetStatusRunning` — NO
   `DeleteSandboxRow` (the Slack bridge has no cold-create publisher, so there is no
   orphaned-row delete path).
2. The Slack `DynamoSandboxStatusWriter.Client` is typed `DDBUpdateItemAPI` (the existing
   slack-bridge interface at aws_adapters.go:1233), NOT the GitHub bridge's
   `DynamoUpdateItemClient` (which carries DeleteItem). This avoids RESEARCH.md Pitfall 6.

Purpose: Adds StartInstances capability to the one bridge that knows a sandbox is paused.
Output: New interfaces, adapters, sentinel error, and unit tests — all in pkg/slack/bridge.
</objective>

// SandboxResumer starts a stopped or paused EC2 sandbox instance. Phase 114:
// used by EventsHandler at the step-9 paused branch — start the box so the on-box
// poller boots and drains the already-enqueued message. Errors are non-fatal in the

}

func (w *DynamoSandboxStatusWriter) SetStatusRunning(ctx context.Context, sandboxID string) error {
    _, err := w.Client.UpdateItem(ctx, &dynamodb.UpdateItemInput{
        TableName: awssdk.String(w.TableName),
