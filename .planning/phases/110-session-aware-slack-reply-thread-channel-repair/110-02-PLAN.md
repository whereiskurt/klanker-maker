<objective>
Add the Ed25519-signed `lookup-thread` bridge action. A sandbox sends a signed
envelope carrying a `session_id`; the bridge Queries the `session-index` GSI
(Plan 01), filters results to rows owned by the requesting sandbox, and returns
`{found, channel_id, thread_ts, agent_type}`. This preserves the
sandbox-never-reads-DDB boundary: the sandbox never touches DDB; the bridge does
the Query and enforces ownership.

This plan also adds the shared `SessionID` envelope field consumed by the
sandbox (Plan 03) and operator (Plan 04) reply paths. The field MUST be inserted
at the correct alphabetical position to keep canonical-JSON signing deterministic.

Purpose: Give sandboxes a signed path to resolve session id → bound thread.
Output: ActionLookupThread, SessionID field, LookupBySession adapter, handler dispatch.
</objective>
