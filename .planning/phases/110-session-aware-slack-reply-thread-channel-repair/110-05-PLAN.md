<objective>
Add four operator-side cleanup/repair commands for stale or wrong Slack thread and
channel mappings, mirroring the `km slack adopt` pattern (operator creds, direct DDB):

- `km slack threads <sandbox-id|--alias>` — read-only list of km-slack-threads rows
  (Scan + FilterExpression sandbox_id = :sid; O(n), documented in help).
- `km slack forget-thread (--session | --thread+--channel)` — DeleteItem on km-slack-threads.
- `km slack prune-threads [sandbox] [--dry-run]` — validate each row's channel via
  Slack conversations.info; drop dead-channel/dead-thread rows. --dry-run lists only.
- `km slack forget-channel <alias>` — DeleteItem on km-slack-channels (same table as adopt).

Reply/repair also detects a dead `slack_channel_id` (manually-deleted channel) and
clears the assumption — surface that in prune-threads' dead-channel detection.

Purpose: Operator hygiene for stale/wrong session/thread/channel mappings.
Output: 4 repair subcommands registered under km slack.
</objective>
