<objective>
Build the data foundation for inbound Slack: a new `km-slack-threads` DynamoDB table module, a v1.1.0 bump of the `dynamodb-sandboxes` module that adds a `slack_channel_id-index` GSI without breaking the existing PK/SK, and the live Terragrunt configs that instantiate them. Purpose: Two storage surfaces drive the whole phase — the bridge needs `(channel_id) → (sandbox_id, queue_url)` resolution (v1.1.0 GSI), and the poller needs `(channel_id, thread_ts) → claude_session_id` persistence (new table). Both use Phase 63's existing `dynamodb-slack-nonces` module as the architectural reference. The Phase 66 shim avoids a hard dependency on Phase 66 shipping first — Plans 03/04/05/06 use `cfg.GetResourcePrefix()` from day one and Phase 66 simply migrates the body when it lands.

   // to "km" when unset. Phase 66 populates this from km-config.yaml; Phase
   // 67 callers use this helper directly so they remain forward-compatible.
   func (c *Config) GetResourcePrefix() string {
       if c == nil || c.ResourcePrefix == "" {
           return "km"
       }
       return c.ResourcePrefix
   }

   // GetSlackThreadsTableName returns the slack-threads table name with the
   // resource prefix applied, falling back to "km-slack-threads" when both
   // SlackThreadsTableName and ResourcePrefix are unset.
   func (c *Config) GetSlackThreadsTableName() string {
       if c == nil {
           return "km-slack-threads"

       if c.SlackThreadsTableName != "" {
           return c.SlackThreadsTableName
