```yaml

kind: SandboxProfile

  name: inbound-uat
```

## Operator-confirmed extended validations (beyond the original UAT script)

The operator's UAT round 3 on `l11` validated three behaviours that go beyond
the literal UAT script and provide additional confidence:

1. **Resume an older thread mid-stream** — opened a thread that had been
   silent for several turns, posted a new message, and Claude continued
   where the older session left off. Confirms `(channel, thread_ts) →
   session_id` lookup in DDB works for not-just-recent threads.
2. **Multiple parallel threads stay isolated** — interleaved messages
   across two open threads in the same channel, each thread maintained its
   own session and produced answers consistent with that thread's history.
3. **No fallback string + no double-posts + no systemd warnings** — the
   three failure-mode signatures from the post-Plan-67-12 commits
   (`e0e76e5`, `bc058af`) are gone. Confirms:
   - Stop hook gate on `KM_SLACK_THREAD_TS` (set BEFORE Claude launches)
     correctly suppresses the Stop hook's Slack branch on inbound turns
   - Poller exports `AWS_REGION` so `km-slack post` does not fall back to
     `us-east-1` blindly
   - Parallel `/etc/km/notify.env` (systemd-format, no `export` prefix) is
     loaded by `EnvironmentFile=` and `journalctl -u km-slack-inbound-poller`
     has zero `Ignoring invalid environment assignment` warnings

These three validations are the strongest signal that Gap A and the two
follow-up commits closed the failure modes diagnosed in `67-UAT.md` round 1.
