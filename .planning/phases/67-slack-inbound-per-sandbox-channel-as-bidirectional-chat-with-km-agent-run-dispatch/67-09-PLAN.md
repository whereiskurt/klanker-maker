<objective>
Extend the existing `km slack init` command with the operator-side configuration for inbound: capture and store the Slack signing secret (separate from the bot token), verify the Slack App has the additional Events API scopes (channels:history, groups:history), and print the bridge `/events` URL the operator must paste into the Slack App Events Subscriptions panel.

Purpose: Without these knobs, `km slack init` produces a fully outbound-capable bridge but a dead inbound surface. The operator hits "Save" in Slack App config and gets a `url_verification` failure because the bridge can't validate signatures (no signing secret). This task closes that loop.

Output: Extensions to `km slack init` (no new command), tests, no infra changes (the SSM parameter is created at first run with KMS encryption from the existing bot-token KMS key).
</objective>

// getSSMParameterKMSKey reads an existing SecureString parameter's KeyId
// (the KMS key alias or ARN used to encrypt it). Used to ensure the new
// signing-secret param is encrypted with the same key as the bot token.
func getSSMParameterKMSKey(ctx context.Context, ssmClient ssmAPI, name string) (string, error) {
    out, err := ssmClient.DescribeParameters(ctx, &ssm.DescribeParametersInput{
        Filters: []ssmtypes.ParametersFilter{
            {Key: ssmtypes.ParametersFilterKeyName, Values: []string{name}},
        },
    })
    if err != nil { return "", err }
    if len(out.Parameters) == 0 { return "", fmt.Errorf("param %s not found", name) }
    return awssdk.ToString(out.Parameters[0].KeyId), nil
}
