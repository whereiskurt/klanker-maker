### Summary

Phase 56.2 goal is achieved at the code level. Both bugs are addressed:

**Bug A (cgroup scope):** `km-bootstrap.service` (Type=oneshot, RemainAfterExit=yes, Before=amazon-ssm-agent.service) runs `/usr/local/bin/km-bootstrap` on every boot via `systemctl enable`. The script idempotently recreates `/sys/fs/cgroup/km.slice/km-${KM_SANDBOX_ID}.scope` with correct chown/chmod. Both cgroup.procs write sites in km-sandbox-shell and km-cgroup.sh are hardened with compound-command form `{ echo $$ > ...; } 2>/dev/null || true` that actually suppresses the redirect-open ENOENT (the bare form leaked errors because bash opens redirect targets before applying `2>/dev/null`).

**Bug B (/run/km tmpfs):** The bootstrap script unconditionally recreates `/run/km/` with correct ownership, and conditionally re-creates `/run/km/learn-commands.log` when `LearnMode=true`. The audit-pipe FIFO is intentionally not created here (owned by km-audit-log's FIFO-retry loop from Phase 56.1).

All 7 requirements (P56.2-01 through P56.2-07) are covered by 4 automated tests that pass. The km binary builds cleanly. The `internal/app/cmd` failures in `go test ./...` are pre-existing, documented in `phases/56/deferred-items.md`, and not introduced by this phase.
