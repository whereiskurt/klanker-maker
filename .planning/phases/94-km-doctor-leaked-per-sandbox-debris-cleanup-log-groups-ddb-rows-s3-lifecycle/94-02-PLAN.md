CRITICAL FINDING (94-RESEARCH.md Item 1 + 94-CONTEXT.md revised Detection rules): the per-sandbox
Lambda log groups were HISTORICALLY created with the literal `km-` prefix in Terraform
(budget-enforcer/main.tf:260, github-token/main.tf:159), NOT `${var.resource_prefix}` — which is
exactly why destroy/ttl deletes missed them and ~271 leaked on the kph install. Plan 94-05 migrates
the CREATE side to the dynamic prefix, so going forward new sandboxes get `{prefix}-`/ `/{prefix}/`
groups. Existing sandboxes keep their legacy `km-`/ `/km/` groups (not retroactively renamed).
Therefore the doctor check MUST detect under BOTH name shapes, deduped by log-group name. On the
default `km` install the two shapes are identical (`km`==`km`) and the dedup collapses them.

<interfaces>
The four families × two name shapes (legacy literal + dynamic prefix). For DescribeLogGroups
LogGroupNamePrefix filters, run each prefix; `p := cfg.GetResourcePrefix()`:
| Family | Legacy filter | Prefixed filter | Extract sandbox-id |
| budget-enforcer Lambda | /aws/lambda/km-budget-enforcer- | /aws/lambda/{p}-budget-enforcer- | strip the matched filter prefix |
| github-token-refresher Lambda | /aws/lambda/km-github-token-refresher- | /aws/lambda/{p}-github-token-refresher- | strip the matched filter prefix |

| ECS sidecars | /km/sidecars/ | /{p}/sidecars/ | 3rd path component |
When p=="km" a filter pair is identical — issue the DescribeLogGroups once per DISTINCT filter
string (de-dup the filter list) and accumulate groups into a map keyed by full log-group name so a
group seen twice is counted once. Skip the shared eBPF groups (/klanker-maker/...) — not per-sandbox.

Management groups to set retention on but NEVER delete (single, not per-sandbox):
/aws/lambda/{prefix}-create-handler, -ttl-handler, -email-handler[-{region}], -slack-bridge.

sandbox-id → ListSandboxes(ctx,false) active map → orphan diff → OK if none → WARN+hint if
dryRun||!deleteLogs → else delete loop with deleted/failed counts → return WARN after delete
(preserves audit trail). Hint pattern: "use --dry-run=false --delete-logs to delete" /
"use --delete-logs to delete".

Pagination (DescribeLogGroups uses NextToken; LogGroup.RetentionInDays is *int32, nil = unset):
