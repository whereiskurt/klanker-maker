<objective>
Add sandbox aliasing — human-friendly names that resolve to sandbox IDs.

Purpose: Operators can refer to sandboxes by short aliases (e.g., `orc`, `wrkr`) instead of hex IDs. Aliases come from `--alias` flag on `km create` or auto-generated from profile-level `metadata.alias` template with incrementing suffix. Aliases are stored in S3 metadata.json and resolved by scanning active sandboxes.

Output: Alias field in metadata, --alias flag on create, profile-level alias template, ResolveSandboxRef alias path, km list alias column.

Design note: Current implementation scans S3 metadata for alias resolution (O(n) with sandbox count). Future optimization: add alias to DynamoDB km-identities table with a GSI for O(1) lookup. This is a TODO for when sandbox count makes S3 scan a bottleneck.
</objective>
