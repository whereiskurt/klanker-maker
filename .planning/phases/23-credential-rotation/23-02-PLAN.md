  <action>
Create `internal/app/cmd/roll.go` with:

1. **RollDeps struct** following DoctorDeps pattern:
   ```go
   type RollDeps struct {
       SSMClient    RollSSMAPI      // PutParameter, GetParameter, GetParametersByPath, SendCommand
       KMSClient    RollKMSAPI      // DescribeKey, RotateKeyOnDemand
       S3Client     RollS3API       // PutObject, GetObject (proxy CA)
       DynamoClient RollDynamoAPI   // PutItem, GetItem (km-identities)
       CWClient     RollCWAPI       // CreateLogGroup, CreateLogStream, PutLogEvents
       ECSClient    RollECSAPI      // ListTasks, StopTask
       EC2Client    RollEC2API      // DescribeInstances
       Lister       SandboxLister   // ListSandboxes for bulk enumeration
   }
   ```

2. **Narrow interfaces** for each client (following doctor.go pattern). RollSSMAPI extends with SendCommand. RollKMSAPI has DescribeKey + RotateKeyOnDemand. RollECSAPI has ListTasks + StopTask.

3. **NewRollCmd(cfg) / NewRollCmdWithDeps(cfg, deps)** — Cobra command at `roll creds` (subcommand pattern: `roll` parent with `creds` child). Flags:
   - `--sandbox <id>` — single sandbox mode (CRED-02)
   - `--platform` — platform-only mode (CRED-03)
   - `--github-private-key-file <path>` — new GitHub App PEM file
   - `--force-restart` — force ECS task restart after proxy CA rotation
   - `--json` — JSON output mode (follow doctor.go pattern)

4. **runRollCreds(cmd, cfg, deps, sandboxID, platformOnly, githubKeyFile, forceRestart, jsonOutput)** — main orchestration:

   **All mode (no flags, CRED-01):**
   a. Rotate platform: proxy CA (RotateProxyCACert), KMS (RotateKeyOnDemand), GitHub key (if --github-private-key-file provided)
   b. Enumerate running sandboxes via deps.Lister.ListSandboxes(ctx, false)
   c. For each running sandbox: RotateSandboxIdentity + ReEncryptSSMParameters
   d. After proxy CA rotation: restart proxies on running sandboxes (SSM SendCommand for EC2, StopTask for ECS if --force-restart)
   e. Write audit events for each step

   **Sandbox mode (--sandbox <id>, CRED-02):**
   a. RotateSandboxIdentity for the specified sandbox
   b. Force GitHub token refresh via HandleTokenRefresh
   c. ReEncryptSSMParameters for the sandbox
   d. Write audit events

   **Platform mode (--platform, CRED-03):**
   a. RotateProxyCACert
   b. KMS RotateKeyOnDemand (call DescribeKey first to verify symmetric key)
   c. If --github-private-key-file: read PEM, validate JWT generation, write to SSM at /km/config/github/private-key via PutParameter Overwrite=true
   d. Restart proxies on running sandboxes (CRED-05)
   e. Write audit events

5. **restartProxies(ctx, deps, sandboxes, forceRestart)** for CRED-05:
   - For EC2 sandboxes: SSM SendCommand with AWS-RunShellScript to download new CA from S3 and restart km-http-proxy
   - For ECS sandboxes: if forceRestart, call StopTask to force task replacement; otherwise log info that new CA picked up on next task start
   - Fire-and-forget pattern (don't wait for command completion per research Pitfall 3)

6. **GitHub PEM validation**: Read file, parse PEM block, parse PKCS1/PKCS8 private key, attempt to generate a JWT (using existing GenerateJWT from pkg/github), verify success before writing to SSM.

7. **Output formatting**: Progress lines during rotation (similar to doctor.go check output). Summary at end: "N credentials rotated, M failures". --json flag outputs structured JSON summary.

8. **Error handling**: Per-sandbox errors are collected, not fatal. Platform errors are fatal (abort with error). Summary printed at end showing successes and failures.
