<objective>
Implement the `km doctor` command that validates the entire platform setup — config, AWS credentials, bootstrap resources, per-region infrastructure, GitHub App config, SCP policy, and active sandboxes — and outputs a structured health report with actionable remediation.

Purpose: Operators need a single command to verify their platform is correctly configured before running `km create`. This enables CI gating (`km doctor && km create`) and fast debugging of misconfigured environments.

Output: `internal/app/cmd/doctor.go`, `doctor_test.go`, `help/doctor.txt`, updated `root.go`
</objective>
