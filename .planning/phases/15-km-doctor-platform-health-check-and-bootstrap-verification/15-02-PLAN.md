  <action>
Extend `internal/app/cmd/configure_github.go` with the `--setup` manifest flow:

1. **Add `--setup` flag** to the existing `newConfigureGitHubCmdCore` function. When `--setup` is true, RunE calls `runConfigureGitHubSetup()` instead of `runConfigureGitHub()`.

2. **buildManifestJSON()** — returns JSON string for the GitHub App manifest:
   ```json
   {
     "name": "klanker-maker-sandbox",
     "url": "https://github.com/whereiskurt/klankrmkr",
     "public": false,
     "default_permissions": {"contents": "write"},
     "hook_attributes": {"url": "https://example.com", "active": false}
   }
   ```

3. **openBrowser(url string) error** — cross-platform browser open using `exec.Command`. darwin: `open`, linux: `xdg-open`, windows: `cmd /c start`. Use `.Start()` not `.Run()` to avoid blocking.

4. **receiveManifestCode(ctx, timeout) (string, int, error)** — starts local HTTP server on `127.0.0.1:0`, serves `/github-app-setup` endpoint. Extracts `code` query parameter. Returns code and port. Shuts down server after receiving code or timeout (5 minutes). Uses `net.Listen("tcp", "127.0.0.1:0")` for random port.

5. **manifestConversionResponse struct** — fields: `ID int64`, `ClientID string`, `PEM string`, `WebhookSecret string`, `HTMLURL string`, `InstallationsCount int` (json tags matching GitHub API: `id`, `client_id`, `pem`, `webhook_secret`, `html_url`, `installations_count`).

6. **exchangeManifestCode(ctx, baseURL, code string) (*manifestConversionResponse, error)** — POST to `{baseURL}/app-manifests/{code}/conversions` with `Accept: application/vnd.github+json`. Parse JSON response into manifestConversionResponse. Use a package-level `var githubManifestBaseURL = "https://api.github.com"` for test injection (follows `GitHubAPIBaseURL` pattern from `pkg/github/token.go`).

7. **fetchInstallations(ctx, baseURL string, appID int64, pemKey string) ([]installationInfo, error)** — create JWT from App ID + PEM (reuse `pkg/github` JWT creation if exported, otherwise inline the JWT creation), call `GET {baseURL}/app/installations`. Return slice of `installationInfo{ID int64, Account string}`. If the JWT creation is complex, consider calling the existing `pkg/github` helpers.

8. **runConfigureGitHubSetup(ctx, ssmClient SSMWriteAPI, out io.Writer, cfg *config.Config, force bool)** — orchestrates the full flow:
   a. Build manifest JSON
   b. Start local callback server (receiveManifestCode)
   c. Construct GitHub URL: `https://github.com/settings/apps/new?manifest=<url-encoded-manifest>`
   d. Add `redirect_url=http://127.0.0.1:{port}/github-app-setup` to manifest JSON before encoding
   e. Print URL and attempt to open browser (non-fatal if browser open fails — operator can copy URL)
   f. Wait for code from callback server
   g. Exchange code via POST to GitHub API
   h. Write PEM to temp file
   i. Try to fetch installations to get installation ID
   j. If installations found: call `putSSMParam` for app-client-id, private-key, installation-id
   k. If no installations: call `putSSMParam` for app-client-id, private-key only; print instructions to install the App and run `km configure github --installation-id <ID>` afterward
   l. Print success message with App URL

Tests use `httptest.NewServer` to mock both the GitHub manifest exchange API and the installations API. The callback server test starts the real local server and hits it with an HTTP client. Use `NewConfigureGitHubCmdWithDeps` with mock SSMWriteAPI to verify SSM writes.

Important: The `redirect_url` must be included IN the manifest JSON body (not as a query parameter). GitHub's manifest flow redirects to this URL with the `code` parameter after the operator clicks "Create GitHub App".
