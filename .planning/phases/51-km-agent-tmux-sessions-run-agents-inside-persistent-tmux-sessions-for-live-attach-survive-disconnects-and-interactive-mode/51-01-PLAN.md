<objective>
Wrap non-interactive agent execution in persistent tmux sessions so agents survive SSM disconnects and operators can later attach to view output.

Purpose: Currently `km agent run` executes Claude directly via SSM SendCommand -- if the SSM connection drops, the operator loses visibility. Wrapping in tmux provides session persistence, scrollback, and the foundation for live attach.

Output: Modified `BuildAgentShellCommands` that generates tmux-wrapped commands, updated signature returning RUN_ID, and updated tests.
</objective>
