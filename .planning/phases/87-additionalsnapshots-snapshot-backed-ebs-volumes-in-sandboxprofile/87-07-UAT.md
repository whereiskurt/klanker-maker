## UAT-1 — Single snapshot, auto-device

**Profile:** `profiles/uat/87/uat-1.yaml`

### Setup

```bash
# 1. Create a small EBS volume to snapshot from
VOL=$(aws ec2 create-volume \
  --region us-east-1 \
  --availability-zone us-east-1a \
  --size 1 \
  --volume-type gp3 \
  --query 'VolumeId' \
  --output text)
aws ec2 wait volume-available --region us-east-1 --volume-ids "$VOL"
echo "Volume: $VOL"

# 2. Create a snapshot
SNAP=$(aws ec2 create-snapshot \
  --region us-east-1 \
  --volume-id "$VOL" \
  --description "phase-87-uat-1" \
  --query 'SnapshotId' \
  --output text)
aws ec2 wait snapshot-completed --region us-east-1 --snapshot-ids "$SNAP"
echo "Snapshot: $SNAP"
```

### Execute

1. Edit `profiles/uat/87/uat-1.yaml` — replace `snap-REPLACE_UAT1` with `$SNAP`
2. `km validate profiles/uat/87/uat-1.yaml` — must pass
3. `km create profiles/uat/87/uat-1.yaml --wait`
4. `km shell <sandbox-id>` — inside:
   ```bash
   mount | grep /opt/uat1
   ls -la /opt/uat1
   cat /proc/mounts | grep /opt/uat1
   lsblk
   ```

### Expected

Mount present at `/opt/uat1`. Device is `/dev/xvdf` (or `/dev/nvme1n1` on Nitro instances).
`km destroy` cleans up the materialised volume; source snapshot survives.

### Teardown

```bash

  --region us-east-1 \

  --query 'Volumes[*].VolumeId' \
  --output text

# Cleanup source resources
aws ec2 delete-snapshot --region us-east-1 --snapshot-id "$SNAP"
aws ec2 delete-volume --region us-east-1 --volume-id "$VOL"
```

### Pass criteria

- [ ] Mount present at `/opt/uat1`
- [ ] Device auto-allocated (no explicit device in profile)
- [ ] `km destroy` cleans materialised volume (describe-volumes returns empty for sandbox-id tag)
