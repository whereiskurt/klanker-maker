**2. Implement execDockerShell:**

```go
func execDockerShell(ctx context.Context, sandboxID string, root bool, execFn ShellExecFunc) error {
    containerName := fmt.Sprintf("km-%s-main", sandboxID)
    args := []string{"exec", "-it"}
    if root {
        args = append(args, "-u", "root")
    }
    args = append(args, containerName, "/bin/bash")
    c := exec.CommandContext(ctx, "docker", args...)
    c.Stdin, c.Stdout, c.Stderr = os.Stdin, os.Stdout, os.Stderr
    return execFn(c)
}
```

The container name `km-{sandboxID}-main` matches the compose project name `km-{sandboxID}` with service name `main`, which Docker Compose names as `km-{sandboxID}-main-1`. However, we use `container_name: km-{sandboxID}-main` in the generated compose (set by Plan 01), so the fixed name works.

Note: Unlike EC2 (SSM) and ECS (ECS Exec), docker exec does not need region, instance ID, or cluster ARN. The sandbox ID is sufficient to derive the container name.

**stop.go changes:**

`runStop()` currently assumes EC2 (DescribeInstances + StopInstances). Add substrate detection at the top of runStop():

1. Load S3 metadata to get substrate (use the same pattern as destroy.go — read metadata from S3 via `awspkg.ReadMetadata()`).
2. If `meta.Substrate == "docker"`:
   - Find compose file at `~/.km/sandboxes/{sandboxID}/docker-compose.yml`
   - Run `docker compose -f {path} -p km-{sandboxID} stop`
   - Print "Sandbox {sandboxID} stopped. Use 'km resume {sandboxID}' to restart."
   - Return early (skip EC2 code)
3. Else: existing EC2 path unchanged.

**pause.go changes:**

Same pattern as stop.go:

1. Load S3 metadata to get substrate.
2. If `meta.Substrate == "docker"`:
   - Find compose file at `~/.km/sandboxes/{sandboxID}/docker-compose.yml`
   - Run `docker compose -f {path} -p km-{sandboxID} pause`
   - Print "Sandbox {sandboxID} paused. Use 'km resume {sandboxID}' to unpause."
   - Return early
3. Else: existing EC2 path unchanged.

Note: `docker compose pause` freezes containers (SIGSTOP); `docker compose stop` sends SIGTERM. Both are valid for the docker substrate. The pause.go command maps naturally to `docker compose pause` and stop.go maps to `docker compose stop`.

Add a helper in a shared location (or duplicate in each file since it's small):
```go
func dockerComposePath(sandboxID string) string {
    homeDir, _ := os.UserHomeDir()
    return filepath.Join(homeDir, ".km", "sandboxes", sandboxID, "docker-compose.yml")
}

func runDockerCompose(ctx context.Context, sandboxID string, args ...string) error {
    composePath := dockerComposePath(sandboxID)
    if _, err := os.Stat(composePath); os.IsNotExist(err) {
        return fmt.Errorf("docker-compose.yml not found at %s — sandbox may have been cleaned up", composePath)
    }
    allArgs := append([]string{"compose", "-f", composePath, "-p", "km-" + sandboxID}, args...)
    cmd := exec.CommandContext(ctx, "docker", allArgs...)
    cmd.Stdout, cmd.Stderr = os.Stdout, os.Stderr
    return cmd.Run()
}
```
