<objective>
Add `docker` as a valid substrate to the JSON schema and Go semantic validator, and implement the `compileDocker()` compiler path that generates a `docker-compose.yml` from a SandboxProfile.

Purpose: This is the foundation for the Docker substrate — all CLI commands (create, destroy, shell, stop, pause) depend on the compiler producing a valid docker-compose.yml. Schema validation must pass before any profile with `substrate: docker` can be used.

Output: `compose.go` with `compileDocker()` + `generateDockerCompose()`, updated schema and validator accepting `docker`, test fixtures and unit tests.
</objective>
