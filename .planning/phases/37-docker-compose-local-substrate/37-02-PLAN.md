```go
    return runCreateDocker(ctx, cfg, awsCfg, resolvedProfile, sandboxID, artifacts, verbose)
}
```
This skips ALL Terragrunt code (no runner.Apply, no CreateSandboxDir, no budget-enforcer apply).

**3. Implement `runCreateDocker()`:**
This function handles the full docker create workflow:

a. **Create sandbox directory:** `~/.km/sandboxes/{sandboxID}/` using `os.MkdirAll`

b. **Create IAM roles via SDK** (not Terraform):
   - Sandbox role: `km-docker-{sandboxID}-{region}` with trust policy allowing both `ec2.amazonaws.com` AND the operator's account root (for STS AssumeRole). Attach inline policy from `artifacts.IAMPolicy` (compile the same IAM policy as EC2 uses). Tag with `km:sandbox-id`.
   - Sidecar role: `km-sidecar-{sandboxID}-{region}` with narrower permissions (DynamoDB budget table, S3 audit/OTEL prefix). Tag with `km:sandbox-id`.
   - Per Pitfall 4: after creating roles, poll `iam.GetRole()` until available, then wait 5 seconds before AssumeRole.

c. **AssumeRole for scoped credentials:**
   - Call `sts.AssumeRole` on sandbox role -> get sandbox credentials
   - Call `sts.AssumeRole` on sidecar role -> get sidecar credentials
   - Call `sts.AssumeRole` with a minimal inline policy (only `sts:AssumeRole` on the two sandbox roles) -> get operator-scoped credentials for cred-refresh container

d. **Inject credentials into compose YAML:**
   Replace placeholder values in `artifacts.DockerComposeYAML` with actual credentials and role ARNs. Use `strings.ReplaceAll()` for each placeholder.

e. **Write docker-compose.yml** to `~/.km/sandboxes/{sandboxID}/docker-compose.yml`

f. **Write .km-ttl file** with TTL expiry timestamp (ISO8601) to `~/.km/sandboxes/{sandboxID}/.km-ttl`

g. **Run `docker compose up -d`:**
   ```go
   cmd := exec.CommandContext(ctx, "docker", "compose", "-f", composeFilePath, "-p", "km-"+sandboxID, "up", "-d")
   ```
   Use `-p` flag to set project name as `km-{sandboxID}` for consistent container naming. Stream output if verbose. Use the existing `os/exec` pattern from shell.go.

h. **Write S3 metadata** using existing `awspkg.WriteMetadata()` with `Substrate: "docker"`. Same call as EC2/ECS path.

i. **Write MLflow run** using existing pattern from create.go.

j. **Print success banner** with sandbox ID, TTL expiry, and `km shell {sandboxID}` hint.

**Important considerations:**
- Do NOT run any Terragrunt commands for docker substrate
- Do NOT create budget-enforcer or github-token Terragrunt modules (those are cloud-side; the budget Lambda still fires on its normal schedule and reads from DynamoDB which the http-proxy sidecar writes to)
- GitHub token SSM parameter creation: if profile has GitHub config, create the SSM parameter with the token (same as the existing github-token Terragrunt module does, but via SDK)
- The `onDemand` parameter is irrelevant for docker — ignore it
