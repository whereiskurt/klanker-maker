## Scenarios

### Scenario 1 — Happy path metering

**Goal:** A Codex CLI call through the proxy produces a `BUDGET#ai#gpt-*` DynamoDB row with `spentUSD > 0`.

1. Create a Codex sandbox:

   ```bash
   km create profiles/codex.yaml
   # If profiles/codex.yaml does not exist, start from the bak2 file:
   # cp profiles/learn.v2.codex.yaml.bak2 profiles/codex-uat.yaml
   # km create profiles/codex-uat.yaml
   ```

2. Wait for the sandbox to reach `running` state:

   ```bash
   km list | grep <sandbox-id>
   # Look for status=running
   ```

3. Trigger a Codex call through the proxy:

   ```bash
   km agent run <sandbox-id> --prompt "Echo back the word hello." --wait
   ```

4. List all budget rows for the sandbox to find the model name used:

   ```bash
   eval $(km env)
   aws dynamodb query \
     --table-name "${KM_RESOURCE_PREFIX:-km}-budgets" \
     --key-condition-expression 'PK = :pk' \
     --expression-attribute-values '{":pk":{"S":"SANDBOX#<sandbox-id>"}}' \
     --output json | jq '.Items[]'
   ```

5. Fetch the specific OpenAI model row (adjust SK to the model seen in step 4):

   ```bash
   aws dynamodb get-item \
     --table-name "${KM_RESOURCE_PREFIX:-km}-budgets" \
     --key '{"PK":{"S":"SANDBOX#<sandbox-id>"},"SK":{"S":"BUDGET#ai#gpt-5.3-codex"}}' \
     --query 'Item' --output json
   ```

6. **Pass criterion:**
   - Item exists
   - `spentUSD` field present and non-zero (e.g., `{"N": "0.0012"}`)
   - `inputTokens > 0`, `outputTokens > 0`
   - `last_updated` is recent (within last 5 minutes)
   - SK matches `BUDGET#ai#gpt-*` pattern

### Scenario 2 — km status surfaces the row

**Goal:** `km status` renders the OpenAI model spend in the AI cost breakdown.

1. Run km status for the sandbox from Scenario 1:

   ```bash
   km status <sandbox-id>
   ```

   Or filter to the AI section:

   ```bash
   km status <sandbox-id> | grep -A 10 "AI\|ai\|openai\|gpt"
   ```

2. **Pass criterion:** Output shows a per-model breakdown that includes the gpt-5.x (or equivalent) entry from Scenario 1. Claude spend should be absent for a Codex-only sandbox.

### Scenario 3 — Budget exhaustion 403 (optional)

**Goal:** When the OpenAI AI budget is exhausted, the proxy returns a 403 with the OpenAIBlockedResponse JSON and logs `event_type=ai_budget_exhausted_preflight`.

1. Set a tiny budget limit. Either:
   - Destroy and recreate with `spec.budget.ai.maxSpendUSD: 0.001` in the profile
   - Or directly update DynamoDB (advanced — adjust `maxSpendUSD` on the sandbox row)

2. Make another Codex call:

   ```bash
   km agent run <sandbox-id> --prompt "Echo hello again." --wait
   ```

3. **Pass criterion:**
   - Codex CLI surfaces a budget-exhausted or 403 error
   - CloudWatch logs for the http-proxy sidecar show:
     ```
     event_type=ai_budget_exhausted_preflight host=api.openai.com
     ```
   - Log group: typically `/km/sandboxes/<sandbox-id>/http-proxy` or check `km otel <sandbox-id> --events`

## Troubleshooting

| Symptom | Check |
|---|---|
| No DDB row after Codex call | `km otel <sandbox-id> --events` — look for `ai_spend` events; verify proxy intercept is active |
| `api.openai.com` connection refused | Check `spec.cli.agent: codex` is set — compiler gate only adds host to L7 list when agent=codex |
| `spentUSD=0` despite row existing | Model name in response not matched by rate table in `openai.go` — check `event_type=openai_unknown_model` in logs |
| Budget exhaustion not triggering | Verify `spec.budget.ai.maxSpendUSD` is set below the actual cost of a turn |
| km status doesn't show OpenAI row | Confirm the DDB row PK/SK format matches `BUDGET#ai#gpt-*` pattern expected by km status renderer |
