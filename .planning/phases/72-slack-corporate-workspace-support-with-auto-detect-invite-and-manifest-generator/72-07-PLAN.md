```go
// After existing operator-invite block, add:

// Phase 72: profile-driven auto-invite for additional operators.
// Each email is run through the EnsureMemberByEmail orchestrator with
// Interactive=false (km create may run from km at / scheduled contexts —
// no stdin prompt is appropriate). Non-fatal: SkippedExternal and Failed
// emit stderr warnings; the create proceeds.
inviteList := profile.Spec.CLI.NotifySlackInviteEmails
if len(inviteList) > 0 {
    inviteAPI := slackInviteAPIAdapter(api) // helper from Plan 72-05; reuse
    for _, email := range inviteList {
        email = strings.TrimSpace(email)
        if email == "" {
            continue
        }
        res, err := slack.EnsureMemberByEmail(ctx, inviteAPI, chID, email, slack.EnsureMemberOpts{
            Interactive: false,
            // Prompter intentionally nil — non-interactive path never calls it
        })
        switch res {
        case slack.InvitedDirect, slack.InvitedConnect:
            log.Info().Str("email", email).Str("channel", chID).Str("result", res.String()).Msg("Auto-invited operator to Slack channel")
        case slack.AlreadyMember:
            log.Debug().Str("email", email).Str("channel", chID).Msg("Operator already in Slack channel — no-op")
        case slack.SkippedExternal:
            // Visible stderr warning with actionable follow-up command.
            fmt.Fprintf(os.Stderr,
                "[warn] %s is not a member of the Slack workspace; not sending Connect invite.\n  To send one: km slack invite --external %s --channel %s\n",
                email, email, chID,
            )
        case slack.Failed:
            // Fail-soft per CONTEXT.md: warn and continue.
            fmt.Fprintf(os.Stderr,
                "[warn] Slack invite failed for %s on channel %s: %v (non-fatal — sandbox provisioning continues)\n",
                email, chID, err,
            )
        }
    }
}
```

NOTE on placement: The loop MUST run AFTER the bot has joined the channel (so
`InviteUserToChannelStrict` doesn't fail with `not_in_channel`) and AFTER the channel-creation
fully succeeds (chID is valid). The existing operator-invite block already gates on these
conditions; place this loop directly after.

NOTE on `slackInviteAPIAdapter`: this helper was introduced in Plan 72-05's
`internal/app/cmd/slack_invite.go`. Same package — directly callable. If 72-05 chose to
extend the cmd-level `SlackAPI` interface (option a) instead of adapt at call sites, the
adapter call may be a direct cast `api.(slack.InviteAPI)` instead. Match Plan 72-05's
chosen approach.

NOTE on the cmd-level `api` variable: in `create_slack.go` the existing variable is likely
named `api` and is the same SlackAPI interface used in slack.go. After Plan 72-05 extended
