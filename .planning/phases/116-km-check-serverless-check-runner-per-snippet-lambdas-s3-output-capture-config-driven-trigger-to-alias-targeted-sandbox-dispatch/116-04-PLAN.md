  - `handler(event, context)` is the Lambda entrypoint

  - Read KM_CHECK_NAME, KM_ARTIFACTS_BUCKET (required), KM_CHECK_TRIGGER (json, default {}),

  - Env build precedence (LATER WINS): copy os.environ -> for each SSM secret path
    fetch WithDecryption and set env[BASENAME_UPPER] -> if event.get('env') update over.
  - subprocess.run([sys.executable, 'snippet.py'], env=..., capture_output=True,
    text=True, timeout=remaining-5s). Capture stdout. On non-zero exit, still write
    stdout (may be empty) to S3 and log stderr; do not crash the handler.
  - Always write stdout verbatim to s3://{bucket}/check-runs/{name}/{ts}/output.json
    BEFORE evaluating the trigger.
  - If KM_CHECK_TRIGGER has no when_py -> return {'triggered': False, 'reason':'no trigger', 'output_key': key}.
  - Try json.loads(stdout). On JSONDecodeError: print 'check_output_not_json: {name}'
    and return triggered=False (output already captured). NEVER trigger on non-JSON.
  - Wrap when_py: pred_src = 'def _pred(out):\n' + indent each line by 4 spaces; exec
    into a globals dict; call _pred(out). Accept bool OR (bool, reason). On exception
    in predicate -> log + treat as not-triggered (fail-closed for dispatch).
  - On truthy: expand prompt ({{reason}} + {{out.<field>}} for each top-level key),
    then events.put_events ONE CheckDispatch entry with the Detail shape above. Return
    triggered=True with reason + output_key.
Keep it dependency-light (boto3 + stdlib only — boto3 is in the Lambda runtime).
