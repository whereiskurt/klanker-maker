## Bug K — corrected root cause + low-hanging follow-ups (final)

A live check-triage box (narrow allowlist) settled the Bug K root cause definitively:
the box booted clean (`SANDBOX_READY`) but **claude was never installed** — no herdr
install ran, no proxy denials. The agent CLI install is **profile-driven via
`initCommands`, NOT a platform default.** learn.v2 works only because its big
initCommands block installs claude (+goose/codex); check-triage's `yum install -y git
jq` installs no agent. So the earlier detection (K1) + allowlist (K2) theories were
secondary — the real fix is **install the agent in the profile**.

- **Real fix (12bc9a9c):** check-triage initCommands now `yum install nodejs/npm` +
  trust the MITM proxy CA (`NODE_EXTRA_CA_CERTS`, persisted to profile.d) + `npm
  install -g @anthropic-ai/claude-code@2.1.132`. **Validated live** on a narrow-allowlist
  box: `npm install` exit 0 → `/usr/bin/claude` 2.1.132. Allowlist trimmed to what the
  npm path needs (`.npmjs.org`/`.amazonaws`/`api.anthropic`; dropped unused herdr/node).
- K1 detection fix (d9fa2e33) is kept as a valid robustness improvement (detect the
  agent as the sandbox user, matching how it's invoked).
