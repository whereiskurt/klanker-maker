<objective>
Add Tier 2 Block Kit rendering on top of the Tier 1 tokenizer from Plan 01,
extend the bridge envelope/dispatch to carry a Blocks field, and flip the
Phase 68 streaming hook in `pkg/compiler/userdata.go` from `--render=plain`
(no-op default) to `--render "${KM_SLACK_RENDER:-blocks}"` so Claude's
transcript turns appear as structured Slack messages with header / section /
context / divider blocks instead of literal markdown.

Purpose: Eliminate the visual noise of headings, dividers, and tool one-liners
all blending into one wall of mrkdwn text. The streaming hook is the only
caller that flips to `blocks`; existing Phase 62/63/67 callers stay on `plain`
default. This is PR2 of the two-PR phasing — depends on Plan 01 (tokenizer +
Tier 1 mrkdwn) being merged and soaked in production first.

Output: New `pkg/slack/blocks.go` with `RenderBlocks` builder; bridge changes
to carry/dispatch Blocks; `cmd/km-slack/main.go` `--render=blocks` execution
path; the one-line bash flip in `pkg/compiler/userdata.go _km_stream_drain`;
and a manual verification checkpoint after `make build && km init --sidecars
&& km init --lambdas` to confirm a real sandbox transcript renders cleanly in
Slack.
</objective>

  <behavior>
    RenderBlocks fully implements BLK-01..BLK-10 per CONTEXT.md § Tier 2 Block Kit transforms.

    Mapping rules:
    - **BLK-01 header block:** Line `^# X` → `{"type":"header","text":{"type":"plain_text","text":"X"}}`. Strip backticks/asterisks/underscores from inner text (BLK-09). If stripped > 150 chars, hard-truncate to 147 + `…` (BLK-10).
    - **BLK-02 section for h2/h3:** Lines `^## ` or `^### ` become bold (`*X*`) inside a `section` block: `{"type":"section","text":{"type":"mrkdwn","text":"*X*"}}`. Adjacent paragraph text coalesces into the same section under 3000 chars.
    - **BLK-03 context block for tool lines:** Lines matching `^🔧 \w+: ` become `{"type":"context","elements":[{"type":"mrkdwn","text":"🔧 …"}]}`. HTML-escape `<`/`>`/`&` only — no other transforms (hook already pre-formatted).
    - **BLK-04 divider:** `^\s*(---|\*\*\*|___)\s*$` → `{"type":"divider"}`. NO auto-dividers between other block types.
