<objective>
Add a tokenizer-based markdown→Slack mrkdwn transformer (`pkg/slack/mrkdwn.go`)
plus the `--render` flag wiring on `km-slack post`. This is PR1 of the two-PR
phasing in CONTEXT.md and RESEARCH.md § Two-PR Sequencing. After this plan
ships, operators can opt into `--render=mrkdwn` per call (or via
`KM_SLACK_RENDER=mrkdwn`) and the streaming hook keeps emitting `--render=plain`
(default) so existing Phase 62/63/68 callers are unchanged. PR2 (plan 02) adds
Tier 2 Block Kit and flips the streaming hook.

Purpose: Eliminate today's production failures (`***heading***` literal
asterisks, `# heading` rendering with visible `#`, pipe-tables in proportional
font with column drift) by transforming Claude's CommonMark-ish output into
valid Slack mrkdwn before it reaches the bridge. Robustness is the priority —
the test investment (unit + property + fuzz + corpus) is non-negotiable.

Output: New `pkg/slack/mrkdwn.go` with `Mrkdwnify(string) string`; complete
test moat in `pkg/slack/mrkdwn_test.go` covering REND-01..REND-13;
`MaxRenderedBytes = 35000` constant in `pkg/slack/payload.go`; `--render` flag
+ overflow truncation + env override wired into `cmd/km-slack/main.go` with
tests in `cmd/km-slack/main_test.go` covering REND-14..REND-16.
</objective>

    The renderer functionally satisfies REND-01..REND-13. Specifically:
    - Tokenizer produces three segment kinds: text, code-span, code-fence.
      Code-fence detection takes priority over code-span (check ``` before `).
      A code-fence is opened by a line whose first non-space chars are ``` and
      closed by a matching ``` line (any info string after ``` is preserved).
      A code-span is delimited by single backticks within a line.
    - Transforms run ONLY on text segments. Code segments are emitted
      byte-for-byte.
    - Tier 1 transforms applied to text segments, in this order:
      1. HTML-escape `&` → `&amp;`, `<` → `&lt;`, `>` → `&gt;` (must be
         FIRST so subsequent transforms don't double-escape inserted
         angle brackets).
      2. Link conversion: `[label](url)` → `<url|label>`. The brackets
         we insert are literal Slack link syntax — they have already been
         protected from escaping because escaping ran first on the original
         text only.
      3. Bold collapse: `**x**` → `*x*`. Use a non-greedy matcher; do NOT
         match across newlines.
      4. Strikethrough: `~~x~~` → `~x~` (similarly non-greedy).
      5. Heading map: at line start, `^# `, `^## `, `^### ` → `*…*` for
         the rest of the line (trim trailing whitespace).
      6. Horizontal-rule drop: a line matching `^\s*(---|\*\*\*|___)\s*$`
         is removed entirely (don't leave a blank line; keep one
         surrounding newline at most so adjacent paragraphs don't fuse).
      7. Pipe-table fence: identify runs of ≥2 consecutive lines matching
         `^\s*\|.*\|\s*$`. Wrap each such run in triple-backtick fences.
         Solo single-line matches are left unchanged. Lines starting with
         🔧 are tool lines and excluded from the heuristic (Tier 1 leaves
         them alone; Tier 2 routes them to context blocks).

      easiest way: each transform's output is itself in the target form

      `**x**` pattern; HTML-escaped output `&lt;` no longer contains `<`,
