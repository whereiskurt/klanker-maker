Create `pkg/ebpf/tls/` package directory and `pkg/ebpf/tls/bpf/` subdirectory.

**ssl_common.h** — shared definitions:
- `#include <linux/bpf.h>` and `#include <bpf/bpf_helpers.h>` (system headers, same as pkg/ebpf/bpf.c)
- `#define MAX_PAYLOAD_LEN 16384` (16KB, TLS max fragment)
- `#define MAX_CONN_ENTRIES 65536`
- Library type constants: `LIB_OPENSSL 1`, `LIB_GNUTLS 2`, `LIB_NSS 3`, `LIB_GO 4`, `LIB_RUSTLS 5`
- Direction constants: `DIR_WRITE 0`, `DIR_READ 1`
- `struct ssl_event` ring buffer event: `{ u64 timestamp_ns; u32 pid; u32 tid; u32 fd; u32 remote_ip; u16 remote_port; u8 direction; u8 library_type; u32 payload_len; u8 payload[MAX_PAYLOAD_LEN]; }`
- `struct conn_info` for connection correlation: `{ u32 remote_ip; u16 remote_port; u16 local_port; }`
- `struct pid_fd_key` for hash map key: `{ u32 pid; u32 fd; }`
- `struct ssl_read_args` for stashing SSL_read buf pointer between entry and return: `{ u64 buf_ptr; u32 fd; }`
- Ring buffer map: `struct { __uint(type, BPF_MAP_TYPE_RINGBUF); __uint(max_entries, 16 * 1024 * 1024); } tls_events SEC(".maps");` (16MB)
- Connection map: `struct { __uint(type, BPF_MAP_TYPE_HASH); __uint(max_entries, MAX_CONN_ENTRIES); __type(key, struct pid_fd_key); __type(value, struct conn_info); } conn_map SEC(".maps");`
- Per-library enable map: `struct { __uint(type, BPF_MAP_TYPE_HASH); __uint(max_entries, 16); __type(key, u8); __type(value, u8); } lib_enabled SEC(".maps");`
- SSL_read args stash map: `struct { __uint(type, BPF_MAP_TYPE_HASH); __uint(max_entries, 4096); __type(key, u64); __type(value, struct ssl_read_args); } ssl_read_args_map SEC(".maps");`

**openssl.bpf.c** — OpenSSL uprobe programs:
- `#include "ssl_common.h"`
- `SEC("uprobe/ssl_write_entry")` — reads `SSL_write(SSL *ssl, const void *buf, int num)` args via PT_REGS. Extract pid/tid from `bpf_get_current_pid_tgid()`. Look up library enabled in `lib_enabled` map (key=LIB_OPENSSL). Reserve ring buffer entry. Read buf via `bpf_probe_read_user()` with min(num, MAX_PAYLOAD_LEN). Look up connection info from `conn_map` using pid_fd_key. Set direction=DIR_WRITE, library_type=LIB_OPENSSL. Submit.
- `SEC("uprobe/ssl_read_entry")` — stash buf pointer and fd in `ssl_read_args_map` keyed by pid_tgid. Buf pointer is arg2 of `SSL_read(SSL *ssl, void *buf, int num)`.
- `SEC("uretprobe/ssl_read_return")` — look up stashed args from `ssl_read_args_map` by pid_tgid. Delete entry. Get return value (bytes read) from `PT_REGS_RC(ctx)`. If return > 0, reserve ring buffer, `bpf_probe_read_user()` the buffer, set direction=DIR_READ, submit.
- Also create `SEC("uprobe/ssl_write_ex_entry")` for `SSL_write_ex` (same logic, 4th arg is `size_t *written`).
- Also create `SEC("uprobe/ssl_read_ex_entry")` and `SEC("uretprobe/ssl_read_ex_return")` for `SSL_read_ex`.

**connect.bpf.c** — connection correlation kprobes:
- `#include "ssl_common.h"`
- `SEC("kprobe/__sys_connect")` — extract `struct sockaddr_in` from args, read remote_ip and remote_port via `bpf_probe_read_user()`. Get pid from `bpf_get_current_pid_tgid()`, fd from arg1 (or use `bpf_get_current_task()` -> files). Store in `conn_map` with key `{pid, fd}`.
- `SEC("kprobe/__sys_accept4")` — similar, populate `conn_map` on accept.
- Note: use `SEC("kprobe/sys_connect")` if `__sys_connect` is not available on AL2023 kernel 6.1. The BPF verifier will tell us at load time.

Use the SAME header include pattern as pkg/ebpf/bpf.c: `#include <linux/bpf.h>`, `#include <bpf/bpf_helpers.h>`, plus `#include <linux/ptrace.h>` for PT_REGS macros. Do NOT use hand-rolled vmlinux.h.

Create a symlink or copy the headers directory. Since pkg/ebpf/tls/bpf/ is separate from pkg/ebpf/, the BPF programs need access to the same system headers. The bpf2go invocation will use `-I../headers` to reference pkg/ebpf/headers/ OR the Docker container will provide them via system paths. Use system includes (angle brackets) that Docker provides.

**pkg/ebpf/tls/gen.go** — bpf2go generate directives:
```go

package tls

// Two separate bpf2go invocations for the two BPF C programs:

```

Note: The `-I../../ebpf/headers` path references the existing `pkg/ebpf/headers/` directory where system headers live. The Docker container also has system includes, but the `-I` flag ensures local headers (common.h from parent package) are findable if needed. However, ssl_common.h is in `bpf/` so it's found via the default search path relative to the source file.

Actually, since system headers (`<linux/bpf.h>`, `<bpf/bpf_helpers.h>`) are provided by the Docker container's system packages, and ssl_common.h uses `#include "ssl_common.h"` (quoted, relative), no extra `-I` is needed for the BPF files themselves. But keep `-I../../ebpf/headers` for the vmlinux.h / bpf_helpers.h that pkg/ebpf/headers/ provides as a fallback. Test both paths — if Docker provides them, great; if not, the `-I` catches it.
