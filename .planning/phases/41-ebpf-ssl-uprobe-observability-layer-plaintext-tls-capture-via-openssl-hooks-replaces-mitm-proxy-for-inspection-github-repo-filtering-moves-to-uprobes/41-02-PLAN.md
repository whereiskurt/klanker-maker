**pkg/ebpf/tls/discovery.go** (`//go:build linux`):
- `type LibraryInfo struct { Path string; Type uint8; PIDs []uint32 }`
- `func DiscoverLibraries() ([]LibraryInfo, error)` — reads `/proc/*/maps` (glob), scans each file for lines containing `libssl.so`, `libgnutls.so`, `libnspr4.so`. Deduplicates by path. Returns list with library type classification.
- Helper `func classifyLibrary(path string) uint8` — returns `LibOpenSSL` for libssl, `LibGnuTLS` for libgnutls, `LibNSS` for libnspr4, 0 for unknown.
- `func FindSystemLibssl() (string, error)` — checks common paths: `/usr/lib64/libssl.so.3` (AL2023), `/usr/lib/x86_64-linux-gnu/libssl.so.3` (Debian/Ubuntu). Returns first that exists. This is the fallback when /proc scanning finds nothing (library might not be loaded yet).

**pkg/ebpf/tls/openssl.go** (`//go:build linux && amd64`):
- `type OpenSSLProbe struct { objs *opensslBpfObjects; connectObjs *connectBpfObjects; links []link.Link; closed bool }`
- `func AttachOpenSSL(libsslPath string) (*OpenSSLProbe, error)`:
  1. `rlimit.RemoveMemlock()`
  2. Load opensslBpf objects: `var objs opensslBpfObjects; if err := loadOpensslBpfObjects(&objs, nil); err != nil { ... }`
  3. Load connectBpf objects: `var connObjs connectBpfObjects; if err := loadConnectBpfObjects(&connObjs, nil); err != nil { ... }`
  4. `ex, err := link.OpenExecutable(libsslPath)`
  5. Attach uprobe on `SSL_write` -> `objs.SslWriteEntry`
  6. Attach uprobe on `SSL_read` -> `objs.SslReadEntry`
  7. Attach uretprobe on `SSL_read` -> `objs.SslReadReturn` (safe for C libraries, NOT Go)
  8. Attach uprobe on `SSL_write_ex` -> `objs.SslWriteExEntry` (if exists, some OpenSSL 3.x only)
  9. Attach uprobe on `SSL_read_ex` -> `objs.SslReadExEntry` (if exists)
  10. Attach uretprobe on `SSL_read_ex` -> `objs.SslReadExReturn` (if exists)
  11. Attach kprobes from connectBpf for connection correlation
  12. Store all links. Set default lib_enabled map entry for LIB_OPENSSL = 1.
  13. Return `&OpenSSLProbe{...}`
- Handle partial attach gracefully — if `SSL_write_ex` symbol doesn't exist (OpenSSL 1.1.x), log and skip, don't error.
- `func (p *OpenSSLProbe) EventsMap() *ebpf.Map` — returns the `tls_events` ring buffer map for consumer to read.
- `func (p *OpenSSLProbe) SetLibraryEnabled(libType uint8, enabled bool) error` — update `lib_enabled` BPF map.
- `func (p *OpenSSLProbe) Close() error` — close all links, close BPF objects. Idempotent.

**pkg/ebpf/tls/consumer.go** (`//go:build linux`):
- `type EventHandler func(event *TLSEvent) error` — handler function signature
- `type Consumer struct { reader *ringbuf.Reader; handlers []EventHandler; logger zerolog.Logger }`
- `func NewConsumer(eventsMap *ebpf.Map, logger zerolog.Logger) (*Consumer, error)`:
  1. `rd, err := ringbuf.NewReader(eventsMap)`
  2. Return `&Consumer{reader: rd, logger: logger}`
- `func (c *Consumer) AddHandler(h EventHandler)` — append to handlers slice
- `func (c *Consumer) Run(ctx context.Context) error`:
  1. Start goroutine watching ctx.Done() to call `c.reader.Close()`
  2. Loop: `record, err := c.reader.Read()`
  3. If `errors.Is(err, ringbuf.ErrClosed)` return nil
  4. Parse record.RawSample into TLSEvent via `binary.Read()` with LittleEndian
  5. Call each handler with the event. Log handler errors at Warn level, continue.
  6. Track metrics: events_total, events_per_library counters
- `func (c *Consumer) Close() error` — close reader

Follow the exact same ring buffer drain pattern as `pkg/ebpf/audit/consumer.go` from Phase 40.
