**pkg/ebpf/tls/http.go** (`//go:build linux`):
- `type HTTPRequest struct { Method string; Path string; Host string; Proto string }`
- `func ParseHTTPRequest(payload []byte) (*HTTPRequest, error)`:
  1. Find first `\r\n` — this is the request line
  2. Split request line by spaces: `METHOD PATH PROTO`
  3. Scan subsequent lines for `Host:` header
  4. Return parsed struct
  5. If payload doesn't start with a valid HTTP method (GET, POST, PUT, DELETE, PATCH, HEAD, OPTIONS, CONNECT), return nil, ErrNotHTTP
- Use `bufio.NewReader(bytes.NewReader(payload))` and `http.ReadRequest()` from stdlib as the parser — it handles edge cases correctly. Wrap in a limited reader to avoid unbounded reads.
- Add `var ErrNotHTTP = errors.New("not an HTTP request")`

**pkg/ebpf/tls/http_test.go** — comprehensive tests:
- Valid GET request with Host header
- Valid POST request with Content-Length
- Non-HTTP binary data returns ErrNotHTTP
- Truncated request (no \r\n\r\n) still parses request line
- HTTP/2 preface detection (return ErrNotHTTP — HTTP/2 binary framing not parseable as text)

**pkg/ebpf/tls/github.go** (`//go:build linux`):
- `func ExtractGitHubRepo(host, path string) (owner, repo string)`:
  1. Check host is `github.com`, `api.github.com`, or `*.github.com`
  2. For `api.github.com`: paths like `/repos/{owner}/{repo}/...` — extract owner/repo from positions 2,3 after split
  3. For `github.com`: paths like `/{owner}/{repo}.git/info/refs` or `/{owner}/{repo}/...` — extract from positions 1,2. Strip `.git` suffix from repo.
  4. Return empty strings if path doesn't contain owner/repo
- `type GitHubAuditHandler struct { AllowedRepos []string; Logger zerolog.Logger }`
- `func NewGitHubAuditHandler(allowedRepos []string, logger zerolog.Logger) *GitHubAuditHandler`
- `func (h *GitHubAuditHandler) Handle(event *TLSEvent) error` — implements `EventHandler`:
  1. Skip if `event.Direction != DirWrite` (only inspect outbound requests)
  2. Parse HTTP request from payload
  3. If not HTTP or parse error, skip
  4. Extract GitHub repo from host + path
  5. If no repo found (non-GitHub or non-repo path), skip
  6. Check if `owner/repo` is in `h.AllowedRepos` (case-insensitive)
  7. If NOT allowed: log at Warn level with full context: `{owner, repo, method, path, pid, remote_addr, sandbox_event: "github_repo_violation"}`
  8. If allowed: log at Debug level for audit trail

Note on EBPF-TLS-10 (budget metering): Per research findings, token counts are in HTTP/2 DATA frames (response bodies), NOT in HTTP/1.1 or HTTP/2 headers. Uprobes cannot reconstruct HTTP/2 data frames. Therefore, EBPF-TLS-10 is implemented as "audit logging of Bedrock/Anthropic request URLs" rather than actual token extraction. The MITM proxy continues to handle real budget metering. Add a stub `BedrockAuditHandler` that logs when it sees requests to `bedrock-runtime.*.amazonaws.com` or `api.anthropic.com` — URL and method only, no token extraction.
