  <action>
    Create `infra/live/use1/s3-replication/terragrunt.hcl` following the dynamodb-budget pattern but with critical additions for the dual-provider requirement.

    Structure:

    1. **locals block** — identical to dynamodb-budget pattern:
       - `repo_root` from `find_in_parent_folders("CLAUDE.md")`
       - `site_vars` from site.hcl
       - `region_config` from parent region.hcl
       - `region_label` and `region_full` from region_config

    2. **include "root"** — standard `find_in_parent_folders("terragrunt.hcl")`

    3. **generate "provider" block** — THIS IS THE CRITICAL DIFFERENCE from other live configs. The s3-replication module requires `provider "aws" { alias = "replica" }` targeting the destination region. The root terragrunt.hcl generates only a single default provider. This config MUST override the root's `generate "provider"` block by using the same block name ("provider") with `if_exists = "overwrite_terragrunt"`:

       ```hcl
       generate "provider" {
         path      = "provider.tf"
         if_exists = "overwrite_terragrunt"

         contents = <<-EOF
           terraform {
             required_version = ">= 1.6.0"

             required_providers {
               aws = {
                 source  = "hashicorp/aws"
                 version = ">= 5.0"
               }
             }
           }

           provider "aws" {
             region = "${local.region_full}"

             default_tags {
               tags = {
                 ManagedBy = "Terragrunt"
                 km_label  = "${local.site_vars.locals.site.label}"
               }
             }
           }

           provider "aws" {
             alias  = "replica"
             region = "${get_env("KM_REPLICA_REGION", "us-west-2")}"

             default_tags {
               tags = {
                 ManagedBy = "Terragrunt"
                 km_label  = "${local.site_vars.locals.site.label}"
               }
             }
           }
         EOF
       }
       ```

       IMPORTANT: The generate block name MUST be "provider" (same as the root terragrunt.hcl) so that `overwrite_terragrunt` replaces the root-generated file. If you name it "providers" (plural), both files will be generated and Terraform will see duplicate provider blocks.

    4. **remote_state block** — same pattern as dynamodb-budget but with key path `s3-replication`:
       ```
       key = "${local.site_vars.locals.site.tf_state_prefix}/${local.region_label}/s3-replication/terraform.tfstate"
       ```

    5. **terraform source** — point to the module:
       ```
       source = "${local.repo_root}/infra/modules/s3-replication/v1.0.0"
       ```

    6. **inputs** — the four required variables from the module:
       - `source_bucket_name` = `get_env("KM_ARTIFACTS_BUCKET", "")`
       - `source_bucket_arn` = `"arn:aws:s3:::${get_env("KM_ARTIFACTS_BUCKET", "")}"`
       - `destination_region` = `get_env("KM_REPLICA_REGION", "us-west-2")`
       - `destination_bucket_name` = `"${get_env("KM_ARTIFACTS_BUCKET", "")}-replica"`

    Do NOT use a Terragrunt `dependency` block for the source bucket. The bucket already exists; use `get_env` to read its name.
  </action>
