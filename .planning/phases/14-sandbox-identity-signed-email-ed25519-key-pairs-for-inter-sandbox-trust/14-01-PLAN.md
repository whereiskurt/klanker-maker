       - `aws_dynamodb_table "identities"` with PAY_PER_REQUEST billing

       - NO DynamoDB Streams (no Lambda trigger needed for identities)
       - TTL attribute: `expiresAt` (for automatic row cleanup after sandbox destroy grace period)
       - Tags from variable
       - No global table replicas in default config — single-region is sufficient for v1 where all sandboxes run in one region. The replica_regions variable is present to enable the same global table replication pattern used by km-budgets when multi-region is needed.
