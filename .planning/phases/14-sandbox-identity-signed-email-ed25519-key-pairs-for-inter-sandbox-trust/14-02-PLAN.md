<feature>
  <name>Sandbox Identity Library</name>
  <files>pkg/aws/identity.go, pkg/aws/identity_test.go</files>
  <behavior>
    **Key Generation + SSM Storage:**
    - GenerateSandboxIdentity(ctx, ssmClient, sandboxID, kmsKeyID) -> (ed25519.PublicKey, error)
    - Calls ed25519.GenerateKey(rand.Reader)
    - Stores base64-encoded 64-byte private key in SSM at /sandbox/{sandboxID}/signing-key
    - SSM PutParameter with Type=SecureString, KeyId=kmsKeyID, Overwrite=true (retry-safe per research Pitfall 6)
    - Returns public key bytes for DynamoDB publishing

    **Optional X25519 Encryption Key:**
    - GenerateEncryptionKey(ctx, ssmClient, sandboxID, kmsKeyID) -> (*[32]byte, error)
    - Generates X25519 key pair via nacl/box.GenerateKey() — separate from Ed25519 (per research recommendation)
    - Stores private key in SSM at /sandbox/{sandboxID}/encryption-key
    - Returns public key for DynamoDB publishing

    **DynamoDB Publishing:**
    - PublishIdentity(ctx, client, tableName, sandboxID, emailAddress, pubKey, encPubKey) -> error
    - PutItem with sandbox_id, public_key (base64), created_at (RFC3339), email_address
    - If encPubKey non-nil, includes encryption_public_key attribute
    - ConditionExpression: attribute_not_exists(sandbox_id) for idempotency

    **Public Key Fetch:**
    - FetchPublicKey(ctx, client, tableName, sandboxID) -> (*IdentityRecord, error)
    - GetItem by sandbox_id, returns IdentityRecord{SandboxID, PublicKey, EmailAddress, EncryptionPublicKey}
    - Returns nil, nil if item not found (not an error — sandbox may not have identity)

    **Email Signing:**
    - SignEmailBody(privKeyB64, body string) -> (signatureB64 string, error)
    - Decodes base64 private key, calls ed25519.Sign on body bytes, returns base64 signature
    - Input: body only (not headers — per research decision)

    **Email Verification:**
    - VerifyEmailSignature(pubKeyB64, body, sigB64 string) -> error
    - Returns nil if valid, error if verification fails or inputs are malformed

    **Signed Email Sending (with encryption policy):**
    - SendSignedEmail(ctx, sesClient, ssmClient, identityClient, from, to, subject, body, sandboxID, recipientSandboxID, tableName, encryptionPolicy string) -> error
    - Reads private key from SSM at /sandbox/{sandboxID}/signing-key
    - Signs body with Ed25519
    - **Encryption policy gate (ROADMAP requirement):**
      - If encryptionPolicy is "required" or "optional":
        a. Fetch recipient's identity record from DynamoDB via FetchPublicKey(ctx, identityClient, tableName, recipientSandboxID)
        b. If recipient has EncryptionPublicKey: encrypt body with EncryptForRecipient before MIME construction, add X-KM-Encrypted: true header
        c. If recipient has NO EncryptionPublicKey AND encryptionPolicy is "required": return error (reject send — per ROADMAP: "reject send if recipient has no published public key")
        d. If recipient has NO EncryptionPublicKey AND encryptionPolicy is "optional": send plaintext (no encryption)
        e. If encryptionPolicy is "off" or empty: skip encryption entirely
    - Constructs raw MIME bytes with X-KM-Sender-ID, X-KM-Signature, and optionally X-KM-Encrypted headers
    - CRITICAL: Uses Content.Raw (NOT Content.Simple) because SES Simple does not support custom headers
    - From/To/Subject in MIME headers + Destination in SDK input

    **NaCl Box Encryption (optional):**
    - EncryptForRecipient(recipientPubKey *[32]byte, plaintext []byte) -> ([]byte, error)
    - Uses box.SealAnonymous with crypto/rand.Reader
    - DecryptFromSender(privKey *[32]byte, pubKey *[32]byte, ciphertext []byte) -> ([]byte, error)
    - Uses box.OpenAnonymous

    **Cleanup:**
    - CleanupSandboxIdentity(ctx, ssmClient, dynClient, tableName, sandboxID string) -> error
    - Deletes /sandbox/{sandboxID}/signing-key from SSM (swallows ParameterNotFound)
    - Deletes /sandbox/{sandboxID}/encryption-key from SSM (swallows ParameterNotFound)
    - Deletes DynamoDB row by sandbox_id (DeleteItem is no-op for missing key)

    **Interfaces (narrow, per project pattern):**
    - IdentitySSMAPI: PutParameter, GetParameter, DeleteParameter
    - IdentityTableAPI: PutItem, GetItem, DeleteItem

    **Test Cases:**
    - GenerateSandboxIdentity: mock SSM receives PutParameter with correct path and SecureString type
    - GenerateSandboxIdentity: returned public key is 32 bytes
    - PublishIdentity: mock DynamoDB receives PutItem with sandbox_id, public_key, created_at, email_address
    - PublishIdentity: encryption_public_key attribute included when non-nil
    - FetchPublicKey: returns IdentityRecord from mock DynamoDB GetItem
    - FetchPublicKey: returns nil when item not found
    - SignEmailBody + VerifyEmailSignature: round-trip — sign then verify succeeds
    - VerifyEmailSignature: tampered body returns error
    - VerifyEmailSignature: wrong public key returns error
    - SendSignedEmail: mock SES receives raw MIME containing X-KM-Signature and X-KM-Sender-ID
    - SendSignedEmail: raw MIME body is correct (matches signing input)
    - SendSignedEmail with encryption=required and recipient has key: body is encrypted, X-KM-Encrypted header present
    - SendSignedEmail with encryption=required and recipient has NO key: returns error (send rejected)
    - SendSignedEmail with encryption=optional and recipient has key: body is encrypted
    - SendSignedEmail with encryption=optional and recipient has NO key: sends plaintext (no error)
    - SendSignedEmail with encryption=off: no FetchPublicKey call, plaintext sent
    - CleanupSandboxIdentity: SSM DeleteParameter called for signing-key path
    - CleanupSandboxIdentity: SSM ParameterNotFound is swallowed (idempotent)
    - CleanupSandboxIdentity: DynamoDB DeleteItem called with correct key
    - EncryptForRecipient + DecryptFromSender: round-trip succeeds
