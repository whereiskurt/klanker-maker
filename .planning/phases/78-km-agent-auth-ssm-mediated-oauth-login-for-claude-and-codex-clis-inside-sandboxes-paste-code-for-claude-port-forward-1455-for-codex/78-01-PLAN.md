<objective>
Ship Wave 1 of `km agent auth`: the `--claude` path that mediates the claude CLI's manual paste-the-code OAuth flow over SSM, plus the missing-credentials hint in `km shell --no-bedrock` and `km agent run --no-bedrock`.

This plan ships the thin-wrapper command that automates the SSM session, captures the success signal, and emits a clean hint at the two consumer entry points so missing credentials never silently cascade into confusing runtime failures.

Output: A new Cobra subcommand `km agent auth <sandbox-id> [--claude | --codex]` registered under the existing `km agent` parent (default `--claude`); `--codex` is wired but returns a "not implemented in Wave 1, ship in Plan 02" stub.
