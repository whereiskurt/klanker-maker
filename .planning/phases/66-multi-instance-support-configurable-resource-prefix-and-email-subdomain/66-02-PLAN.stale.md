<objective>
Migrate all ~30 Go call sites that hardcode `"sandboxes." + cfg.Domain` (or the literal `"sandboxes.klankermaker.ai"`) to call `cfg.GetEmailDomain()`. For Lambda handlers (`cmd/*/main.go`) which run without operator-side config access, switch to a `KM_EMAIL_DOMAIN` env var with the existing literal as a safe fallback. Also migrate the minor `pkg/aws/ec2_ami.go` AMI filter from literal `"km-"` to `cfg.GetResourcePrefix() + "-"` (per RESEARCH.md open question #4 — bundled here because it touches the same files-domain).

Purpose: Phase 65 established the Config-helper-method pattern. Plan 02 collapses 30+ redundant string concatenations to a single authoritative source. After this plan, changing `email_subdomain` in km-config.yaml flows automatically into every operator-side command. Lambda handlers get the same value injected by their TF modules in plan 04.

Output: 14 source files updated (8 internal/app/cmd, 3 cmd/*/main.go, 3 pkg/), one grep-audit gate green, full test suite green.
</objective>
