## Identified gaps (for `/gsd:plan-phase 84.3 --gaps`)

1. **`gap-drift-warn-viper-binding-masks-env-bound-keys`** (closure c)
   - viper auto-binds `KM_<KEY>` → `cfg.<Field>` at `config.Load()` time. By the time `ExportTerragruntEnvVars` checks `os.Getenv(key) != cfgVal`, cfgVal is already env. No drift detected.
   - Affected keys: `KM_REGION`, `KM_DOMAIN`, `KM_ARTIFACTS_BUCKET`, `KM_RESOURCE_PREFIX`, `KM_OPERATOR_EMAIL`, `KM_ROUTE53_ZONE_ID`, `KM_SCHEDULER_ROLE_ARN`, `KM_REGION_LABEL`.
   - Likely fix: snapshot pre-env-binding yaml values into a `cfg.YAMLDefaults` map at load time; drift check reads from that.

2. **`gap-drift-warn-runBootstrap-missing`** (closures c + h)
   - `runBootstrap` (the path for default `km bootstrap` without `--shared-ses`) does not call `ExportTerragruntEnvVars`, so drift WARN never fires from that command path even for yaml-authoritative keys.
   - Fix: add `ExportTerragruntEnvVars(loadedCfg)` call to `runBootstrap` right after `loadBootstrapConfig`.

3. **`gap-validate-artifacts-bucket-not-wired-into-load`** (closure e)
   - `validateArtifactsBucket` exists in `internal/app/cmd/configure.go` and passes its unit test, but no command code path calls it. Placeholder buckets (`km-artifacts-12345`, `<prefix>-artifacts-12345678`) silently pass through `km info`, `km doctor`, `km init`, etc.
   - Fix: call from `config.Load()` (or add to a common pre-flight hook) so every command rejects placeholder values at load.

4. **`gap-init-hardfail-not-triggered-on-placeholder`** (closure f.6)
   - `km init --dry-run=true` succeeds against a placeholder `artifacts_bucket`; operator sees full plan output for a misconfigured install.
   - Fix overlaps with gap 3: wiring `validateArtifactsBucket` into `config.Load()` short-circuits `km init` before any plan output.
