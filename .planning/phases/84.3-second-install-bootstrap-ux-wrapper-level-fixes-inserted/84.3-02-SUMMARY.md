## Decisions Made

- **validateArtifactsBucket angle-bracket detection:** Uses `strings.Index("<")` scan rather than `HasPrefix + HasSuffix` because the test case `"<prefix>-artifacts-12345678"` has a leading `<prefix>` token but does not end with `>`. The Index-based scan correctly identifies any embedded `<...>` token regardless of position.
- **warnShellEnvConflict placement:** Called before the `missing` validation check in the non-interactive path so WARNs reach the operator even when required flags are absent. This was required to make `TestConfigure_ShellEnvDriftWarn` pass (test calls runConfigure with empty ssoStartURL/ssoRegion but expects drift WARN).
- **validateArtifactsBucket wiring:** The helper is tested directly by Plan 01's tests; load-path wiring (PreRunE or config.Load) is deferred to Plans 03/04 as specified in the plan.
- **SetEnvKeyReplacer:** Added to fix TestConfig_AccountsTerraformEnvWins — viper's AutomaticEnv without a replacer doesn't correctly map dot-notation keys to KM_ env vars.
