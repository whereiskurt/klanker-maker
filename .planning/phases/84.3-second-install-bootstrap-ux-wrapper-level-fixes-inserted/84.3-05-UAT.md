## Closure (c) — ENV-CONFIG-DRIFT-WARN — PASS (gap closure 84.3-07/08)

**Gap closure update (84.3-10):** All drift WARN scenarios now pass. See evidence below.

**Where it works:** yaml-authoritative keys (`accounts.organization`, `accounts.dns_parent`, `accounts.application`) via the `--shared-ses` path which calls `ExportTerragruntEnvVars`:

```
$ KM_ACCOUNTS_ORGANIZATION=999999999999 ./km bootstrap --shared-ses --dry-run=true
(stderr) WARN: KM_ACCOUNTS_ORGANIZATION=999999999999 (env) overrides km-config.yaml accounts.organization=481723467561
```

**Where it does NOT work:**

1. **Env-bound keys** — viper binds `KM_REGION` → `cfg.PrimaryRegion` at `config.Load()` time, so by the time `ExportTerragruntEnvVars(cfg)` checks `envVal != cfgVal`, both already equal the env value. No drift detected.
   ```
   $ KM_REGION=us-west-2 ./km bootstrap --shared-ses --dry-run=true
   (stderr) (empty — no WARN despite km-config.yaml region=us-east-1)
   ```
   Same issue affects: `KM_DOMAIN`, `KM_ARTIFACTS_BUCKET`, `KM_RESOURCE_PREFIX`, `KM_OPERATOR_EMAIL`, `KM_ROUTE53_ZONE_ID`, `KM_SCHEDULER_ROLE_ARN`.

2. **`runBootstrap` (default `km bootstrap` without `--shared-ses`) never calls `ExportTerragruntEnvVars`** — so even the yaml-auth keys don't trigger the drift WARN from that path. The bootstrap banner block prints the cfg values (yaml-auth wins for the 3 locked keys), but no WARN fires.
   ```
   $ KM_ACCOUNTS_ORGANIZATION=999999999999 ./km bootstrap --dry-run=true
   (stderr) (empty)
   (stdout) Organization account: 481723467561   ← yaml wins, but no WARN telling operator about the env override
   ```

**Gap (original):** `gap-drift-warn-viper-binding-masks-env-bound-keys` + `gap-drift-warn-runBootstrap-missing`.

**Gap closure (84.3-07 + 84.3-08):**
- Plan 84.3-07 added `cfg.YAMLDefaults` snapshot (pre-env-binding) at `config.Load()` time; `ExportTerragruntEnvVars` now reads YAMLDefaults for the 8 env-bound keys to detect drift.
- Plan 84.3-08 added `ExportTerragruntEnvVars(loadedCfg)` call to `runBootstrap` right after `loadBootstrapConfig`.

**Re-verification evidence (84.3-10, km v0.2.672):**

