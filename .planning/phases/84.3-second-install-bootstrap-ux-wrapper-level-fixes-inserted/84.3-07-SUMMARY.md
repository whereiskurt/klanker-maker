# Phase 84.3 Plan 07: YAMLDefaults Drift Detection Fix Summary

**One-liner:** Snapshot km-config.yaml raw values into Config.YAMLDefaults before viper env-baking so ExportTerragruntEnvVars correctly detects KM_REGION/KM_ARTIFACTS_BUCKET drift even after AutomaticEnv() merges env into cfg fields.

## What Was Built

Gap 1 closure for Phase 84.3 UAT: `drift WARN doesn't fire for KM_REGION/KM_ARTIFACTS_BUCKET etc. because viper binds env ‚Üí cfg before the check runs`.

Root cause: `config.Load()` calls `v.AutomaticEnv()` which bakes `KM_REGION` into `cfg.PrimaryRegion`. When `ExportTerragruntEnvVars` runs, `cfg.PrimaryRegion == os.Getenv("KM_REGION")`, so `envVal != cfgVal` is always false ‚Äî no WARN fires.

Fix: Snapshot raw yaml values from `v2` (km-config.yaml viper instance) into `cfg.YAMLDefaults map[string]string` BEFORE building the cfg struct. `ExportTerragruntEnvVars.warnAndSetEnv` then compares `envVal` against `cfg.YAMLDefaults[yamlKey]` instead of `cfgVal`.

## Task 2: ExportTerragruntEnvVars Updated and t.Skip Removed

**Commit:** `80009d3`

**warnAndSetEnv change (before/after logic):**

Before:
```go
warnAndSetEnv := func(envKey, yamlKey, cfgVal string) {
    if envVal := os.Getenv(envKey); envVal != "" && cfgVal != "" && envVal != cfgVal {
        fmt.Fprintf(os.Stderr, "WARN: ...")
    }
    if cfgVal != "" && os.Getenv(envKey) == "" {
        os.Setenv(envKey, cfgVal)
    }
}
```
Problem: `cfgVal == envVal` after viper AutomaticEnv() baking ‚Äî comparison always false.

After:
```go
warnAndSetEnv := func(envKey, yamlKey, cfgVal string) {
    envVal := os.Getenv(envKey)
    if envVal == "" {
        if cfgVal != "" { os.Setenv(envKey, cfgVal) }
        return
    }
    yamlVal := cfgVal  // fallback when YAMLDefaults nil (Plan 01 tests, accounts.* keys)
    if cfg.YAMLDefaults != nil {
        if snap, ok := cfg.YAMLDefaults[yamlKey]; ok {
            yamlVal = snap  // raw yaml value, unaffected by env baking
        }
    }
    if yamlVal != "" && envVal != yamlVal {
        fmt.Fprintf(os.Stderr, "WARN: ...")
    }
    // env wins ‚Äî no override
}
```

**Special-cased blocks updated:**
- `KM_RESOURCE_PREFIX`: looks up `cfg.YAMLDefaults["resource_prefix"]` for `yamlPrefix`; falls back to `prefix` (from `cfg.GetResourcePrefix()`)
- `KM_EMAIL_SUBDOMAIN`: looks up `cfg.YAMLDefaults["email_subdomain"]` for `yamlSubdomain`; falls back to `cfg.EmailSubdomain`

**NOTE on KM_ACCOUNTS_* keys:** NOT changed. `accounts.*` are yaml-authoritative (Plan 02) ‚Äî cfg fields already hold yaml values (env does not bake in for these). `warnAndSetEnv` fallback path (cfgVal) works correctly as-is.

**TestRunBootstrap_DriftWarn_KM_REGION:** `t.Skip("needs Plan 07 ‚Äî YAMLDefaults field not yet added")` removed from `internal/app/cmd/bootstrap_drift_warn_test.go`. Test now runs and passes: cfg is built directly (no config.Load()), so `YAMLDefaults` is nil, `yamlVal` falls back to `cfgVal = "us-east-1"`, `envVal = "us-west-2"` ‚Üí WARN fires.
