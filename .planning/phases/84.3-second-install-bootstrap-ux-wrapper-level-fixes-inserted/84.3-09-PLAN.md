Gap 3 (closure e): placeholder bucket values silently pass through km info, km doctor, km init.
Gap 4 (closure f.6): km init --dry-run=true and km init --plan succeed against placeholder buckets.

Fix strategy (two callsites):

1. **config.Load()** — validate non-empty artifacts_bucket before returning. This catches
   placeholder values at the earliest possible point: every command that uses cfg will fail
   immediately. Handles gap 3 for all commands except fresh installs (empty bucket → skip validate).

2. **runInitPlan** in init.go — call validateArtifactsBucket early (before AWS creds, before
   ExportTerragruntEnvVars) to hard-fail `km init --plan` and `km init --dry-run=true` on
   placeholder values. This is defense-in-depth for gap 4: even if config.Load() somehow passes,
   the plan path also validates explicitly.

The `validateArtifactsBucket` function lives in configure.go. It is in the same package (cmd)
as init.go, so no import needed. But config.go is in a different package (config) — it cannot
import cmd. Solution: extract the placeholder-check logic inline in config.Load() OR move
validateArtifactsBucket to config package. The cleanest approach: inline a simple isPlaceholderBucket
check directly in config.Load() (config package, no cross-package import).

Design decision for config.Load():
- DO NOT import cmd package (circular). Instead, inline the same placeholder detection logic
  as a package-private isPlaceholderBucket helper in config.go.
- The logic is small (2 conditions): contains angle-bracket token OR equals "km-artifacts-12345".
- If km-config.yaml has an empty artifacts_bucket field, skip the check (fresh install).

The runInitPlan callsite in init.go CAN call configure.go's validateArtifactsBucket directly
(same package). The config.Load() check provides the primary gate; runInitPlan is secondary defense.

CRITICAL DESIGN NOTE — km configure wizard interaction:
The configure wizard (internal/app/cmd/configure.go::runConfigure) reads km-config.yaml using
`os.ReadFile + yaml.Unmarshal` DIRECTLY — it does NOT call `config.Load()`. This means the new
config.Load() validation does NOT block the wizard from reading an existing km-config.yaml with
a placeholder bucket. The configure wizard can always run regardless of the bucket value in the
existing yaml. This is the correct behavior: the wizard is the CURE for a placeholder bucket,
so it must not be blocked by the placeholder it exists to fix.

Additionally, the cobra command for `km configure` passes `_ = cfg` (reserved for future use),
so the configure subcommand does NOT require config.Load() to succeed at command startup.

The test `TestConfigurePath_PlaceholderBucketDoesNotBlockWizard` must confirm this invariant
is preserved after this plan's changes are applied.

Purpose: Every operator command on a misconfigured install immediately tells the operator:
"artifacts_bucket is a placeholder; re-run km configure." No silent pass-through.
km configure itself is immune to this block (by design).

```go
// isPlaceholderBucket reports whether the given artifacts_bucket value is a
// placeholder from km-config.example.yaml that an operator has not replaced.
// Returns true for:
//   - angle-bracket tokens (e.g. "<prefix>-artifacts-12345678")
//   - the literal example sentinel "km-artifacts-12345"
// Returns false for empty string (empty means unconfigured, not placeholder).
// Inline in config.go to avoid cross-package imports from config → cmd.
func isPlaceholderBucket(name string) bool {
    if name == "" {
        return false
    }
    if lt := strings.Index(name, "<"); lt >= 0 {
        if strings.Index(name[lt:], ">") >= 0 {
            return true
        }
    }
    return name == "km-artifacts-12345"
}
```

```go
}
```
