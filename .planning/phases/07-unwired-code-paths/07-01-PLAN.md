**Restructure `buildDest()` in `sidecars/audit-log/cmd/main.go`:**

1. Refactor `buildDest()` to wrap ALL returned destinations with `auditlog.NewRedactingDestination(inner, nil)` as the final return-path wrapper. Apply the wrapper AFTER the switch statement, not per-case. Pass `nil` for literals (regex patterns cover AWS keys, Bearer tokens, hex strings). This closes OBSV-07.

2. Extract the CloudWatch Logs client creation out of `buildDest()` so it is accessible to both the destination AND the idle detector. Specifically:
   - Create `newCWClient(ctx, region)` helper that returns `(kmaws.CWLogsAPI, error)` — constructs `cloudwatchlogs.NewFromConfig(cfg)`
   - In `main()`, when `destName == "cloudwatch"`, create the CW client BEFORE calling `buildDest()`
   - Pass the CW client into `buildDest()` as a parameter (signature becomes `buildDest(ctx, destName, cwLogGroup string, cwClient kmaws.CWLogsAPI)`)
   - The `newRealCWBackend` function takes the pre-created client instead of creating its own

3. **Wire IdleDetector** in `main()` after `buildDest()` returns successfully:
   - Read `IDLE_TIMEOUT_MINUTES` env var via `envOr("IDLE_TIMEOUT_MINUTES", "")`
   - If non-empty AND `destName == "cloudwatch"` (idle detection requires CW polling):
     - Parse as integer minutes using `strconv.Atoi`
     - Construct `lifecycle.IdleDetector` with: SandboxID from env, IdleTimeout from parsed duration, CWClient from the pre-created client, LogGroup/LogStream from existing vars
     - Set `OnIdle` to log a warning message and call `cancel()` to exit the binary (TTL Lambda handles actual sandbox destroy)
     - Launch with `go detector.Run(ctx)` in a goroutine
   - If empty or non-cloudwatch: skip silently (no warning needed)
   - Add import for `"strconv"`, `"time"`, and `lifecycle "github.com/whereiskurt/klankrmkr/pkg/lifecycle"`

4. **Create `sidecars/audit-log/cmd/main_test.go`:**
   - Test `buildDest` returns `*auditlog.RedactingDestination` for stdout and s3 cases (no AWS client needed)
   - Test that passing nil CW client with "stdout" dest works fine
   - Test that idle detector code path is reachable (verify the function exists and the type is correct)
   - Use `package main` (same package for internal function access) or export a thin wrapper

**Important constraints:**
- Do NOT make `buildDest()` changes fatal — if CW client construction fails, the error propagates as before
- `newRealCWBackend` still handles `EnsureLogGroup` — the refactor only moves CLIENT creation up, not log group setup
- The `realCWBackend` struct and its methods remain unchanged
