# Phase 115 UAT: Generic GitHub event→prompt router

Live E2E checklist for the operator to run after Tasks 1-2 of Plan 06 are merged
and Plans 02-05 are all deployed. This checklist covers:

- GH-EVENT-POLLER: the sandbox-side poller emits an event-context preamble (not PR:#0)
- GH-EVENT-E2E: the full pipeline fires end-to-end (config → manifest → webhook → cold-create → prompt)

## Prerequisites

- Plans 02-05 are merged and deployed (`make build-lambdas` + `km init --github` done)
- Plan 06 Tasks 1-2 are merged (userdata.go poller surgery + docs)
- A GitHub App is installed and the bridge Lambda is reachable

---

## Step 1: Add a github.events: rule to km-config.yaml

```yaml
github:
  events:
    - on: repository
      actions: [created]
      match: "<your-org>/*"
      exclude: ["<your-org>/km-e2e-skip-*"]
      profile: profiles/github-review.yaml
      cooldownSeconds: 0
      prompt: |
        A new repo {{repo}} was created by {{sender}} (default branch {{default_branch}}).
        Clone it and open a PR adding a minimal CI workflow. URL: {{html_url}}
```

Replace `<your-org>` with your GitHub org name.

**Expected:** km-config.yaml updated, no syntax errors.

---

## Step 2: Deploy

```bash
make build-lambdas
km init --github
# OR: km init --dry-run=false
```

Do NOT use `km init --sidecars` — it rebuilds binary zips but does NOT update
the Lambda env block (`KM_GITHUB_EVENTS` stays stale).

**Expected:** Apply succeeds, no error output.

---

## Step 3: Subscribe the App to the repository event

```bash
km github manifest
```

Copy the output JSON and paste it into your GitHub App settings:
1. Go to GitHub → Settings → Developer settings → GitHub Apps → your App
2. Click "Edit" → "App Manifest" (or update event subscriptions directly)
3. Add "Repository" to the list of subscribed events
4. Save and re-install the App

Confirm `metadata:read` permission is present (required for `repository` events).

**Expected:** App subscribed to `repository` event; re-install completed.

---

## Step 4: Confirm env reached the Lambda

After the next bridge Lambda cold-start (or invoke it once), inspect CloudWatch logs:

```
km logs --follow <lambda-name>
# OR: use AWS Console → CloudWatch → Log groups → /aws/lambda/{prefix}-github-bridge
```

**Expected log line:** `km-github-bridge: loaded event routing config rule_count=1`

If you see `KM_GITHUB_EVENTS not set; event routing dormant`, re-run `km init --github`.

## Step 5: POSITIVE — create a throwaway repo

Create a repo that matches your `match:` glob:

```bash
gh repo create <your-org>/km-e2e-onboard-$(date +%s) --private --confirm
```

Within ~1 minute, verify:

**a) Bridge logs show the event matched:**
```
github-bridge: event rule matched event=repository repo=<your-org>/km-e2e-onboard-... action=created
github-bridge: cold-creating sandbox for event rule alias= profile=profiles/github-review.yaml
```

**b) Sandbox was cold-created:**
```bash
km list
# Should show a new sandbox with profile=github-review.yaml
```

**c) On-box preamble (GH-EVENT-POLLER critical check):**
```bash
km shell <sandbox-id>
# Inside the box:
ls /workspace/.km-agent/runs/
cat /workspace/.km-agent/runs/<timestamp>/github-prompt.txt
```

The preamble MUST read:
```
[GitHub Event Trigger]
Repository: <your-org>/km-e2e-onboard-...
Event: repository / created
Sender: <your-login>
URL: https://github.com/<your-org>/km-e2e-onboard-...

--- Task ---
A new repo <your-org>/km-e2e-onboard-... was created by <sender> ...
```

The preamble MUST NOT contain:
- `PR: #0`
- `git fetch origin pull/0/head`
- `[GitHub Comment Trigger]`

**d) Prompt ran (agent produced output):**
```bash
cat /workspace/.km-agent/runs/<timestamp>/output.json | jq '.result // .result_text // .text'
```

**Expected:** All four checks pass.

## Step 6: NEGATIVE — exclude repo does not fire

Create a repo matching the `exclude:` glob:

```bash
gh repo create <your-org>/km-e2e-skip-foo --private --confirm
```

Wait ~1 minute and verify:

**a) Bridge logs show no dispatch:**
```
# Expected: "no matching event rule event=repository repo=<your-org>/km-e2e-skip-foo"
# OR: "event rule excluded" (depending on log level)
# NOT expected: "event rule matched" or "cold-creating sandbox"
```

**b) No new sandbox was created:**
```bash
km list
# Count should be unchanged from Step 5
```

**Expected:** No sandbox created for the excluded repo.

## Step 7: DEDUP/COOLDOWN (optional but recommended)

Redeliver the same `repository`/`created` webhook from Step 5:

1. GitHub App → Advanced → Recent Deliveries
2. Find the `km-e2e-onboard-*` repository/created event
3. Click "Redeliver"

**Expected:** Bridge logs either:
- `duplicate delivery suppressed delivery_id=...` (GUID dedup — same delivery GUID reused)
- `event cooldown suppressed event=repository repo=... action=created` (if cooldownSeconds > 0)

No second sandbox should be created.

**Bug found + fixed during E2E (commit `8df2e7fd`):** the on-box preamble line
`Event: <kind> / <action>` rendered with a **blank action** (`repository / `).
`GitHubEnvelope` had no `Action` field and `handleEventRoute` never set it, so the
poller's `jq -r '.action'` always read empty — even though the bridge parsed
`action=created` correctly (visible in the 6a drop log). Fixed by adding the `Action`
field to `GitHubEnvelope` + populating it from the parsed payload, plus a regression
assertion in `webhook_handler_phase115_test.go`. Rebuilt + redeployed (Lambda code
hash changed) + re-fired → preamble now reads `Event: repository / created`.
