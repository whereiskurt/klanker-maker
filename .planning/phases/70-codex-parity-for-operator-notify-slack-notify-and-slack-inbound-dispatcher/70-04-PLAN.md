<objective>
Add three new surface elements to the `km-slack` sandbox-side sidecar AND the bridge Lambda so Plan 70-06 (cross-agent thread switch) can:
1. Post a NEW top-level message to a sandbox's Slack channel (no thread_ts) and capture its `ts`
2. Fetch the permalink URL for any message ts in the sandbox's channel
3. Edit a previously-posted bot message (optional cleaner-UX path; safe ordering doesn't require it but the spec permits it)

Per CONTEXT.md locked decisions:
- `--new-message` is a flag on the existing `post` subcommand (NOT a new subcommand)
- `permalink` and `update` are new subcommands
- All three go through the existing bridge Lambda (signed envelopes — sandbox-side has no raw Slack bot token, per 70-RESEARCH.md security model)

Per 70-RESEARCH.md finding #2 (NOTE: this plan's task list MUST enumerate ALL touched files):

The "thin REST wrapper" description in CONTEXT.md is correct for the *sandbox-side* (the new flag/subcommands just build envelopes and POST). But because everything goes through the bridge, the bridge ALSO needs new action constants, new handler branches, new interface methods, and new client methods. Total file surface for Plan 70-04:
1. `cmd/km-slack/main.go` — `--new-message` flag + `permalink` + `update` subcommand handlers + dispatch table entries
2. `cmd/km-slack/main_test.go` — sidecar unit tests

6. `pkg/slack/bridge/interfaces.go` — extend `SlackPoster` interface
7. `pkg/slack/bridge/handler.go` — two new dispatch case branches with channel-scope authorization

This plan is independent of all other Plan 70-XX work — no dependency on the schema change (70-01), the compiler edits (70-02/03), or the poller fork (70-05). Can run in parallel with 70-01.

The `--new-message` flag on `post` is the simplest of the three — it just passes `thread=""` to the existing dispatch path (per 70-RESEARCH.md verified note: `pkg/slack/client.go` already does `if threadTS != "" { payload["thread_ts"] = threadTS }` at line ~133). func runUpdate(args []string, stderr io.Writer) int {
    fs := flag.NewFlagSet("update", flag.ContinueOnError)
    fs.SetOutput(stderr)
    var (
        channel  = fs.String("channel", "", "Slack channel ID (C...) — required")
        ts       = fs.String("ts", "", "Slack message ts — required")
        text     = fs.String("text", "", "New message text (or use --body file)")
        bodyPath = fs.String("body", "", "Path to body file (alternative to --text)")
    )
    if err := fs.Parse(args); err != nil {
        return 2
    }
    if *channel == "" || *ts == "" {
        fmt.Fprintln(stderr, "km-slack update: --channel and --ts are required")
        return 2
    }
    body := *text
    if body == "" && *bodyPath != "" {
        b, err := os.ReadFile(*bodyPath)
        if err != nil {
            fmt.Fprintf(stderr, "km-slack update: read body file: %v\n", err)
            return 1
        }
        body = string(b)
    }
    if body == "" {
        fmt.Fprintln(stderr, "km-slack update: one of --text or --body required")
        return 2
    }
    envelope := slack.SlackEnvelope{
        Action:    slack.ActionUpdate,
        Channel:   *channel,
        MessageTS: *ts,
        Text:      body,
    }
    _, err := signAndPostEnvelope(envelope)
    if err != nil {
        fmt.Fprintf(stderr, "km-slack update: bridge call failed: %v\n", err)
        return 1
    }
    return 0
}
