### Runner.Import method

Added to `pkg/terragrunt/runner.go` immediately after the `Apply` method:

```go
func (r *Runner) Import(ctx context.Context, dir, address, id string) error {
    cmd := r.BuildImportCommand(ctx, dir, address, id)
    return r.runCommand(ctx, cmd)
}

func (r *Runner) BuildImportCommand(ctx context.Context, dir, address, id string) *exec.Cmd {
    return r.buildCommand(ctx, dir, "import", address, id)
}
```

Mirrors the existing `Apply` / `Destroy` / `Plan` pattern exactly: `buildCommand` constructs the `exec.Cmd`, `runCommand` handles verbose/quiet modes and ctx-bounded execution (heartbeat, SIGTERM, SIGKILL). `BuildImportCommand` is exposed as a public method for command-construction tests â€” same pattern as `BuildApplyCommand`, `BuildDestroyCommand`.

Key constraints honored:
- NO `-auto-approve` (import doesn't support it)
- Args are exec.Command argv, not shell-quoted (no injection risk)
- Inherits Phase 84.1-02 heartbeat + ctx deadline via `runCommand` â†’ `runBounded`

Three tests added via TDD (RED then GREEN):
- `TestRunnerImportBuildsCorrectCommand` â€” argv shape assertion
- `TestRunnerImportCallsTerragrunt` â€” end-to-end via fake shim, exit 0
- `TestRunnerImportPropagatesError` â€” non-zero exit propagates as error

### scripts/inventory-diff.sh

AWS-CLI inventory snapshot+diff helper. Two subcommands:

**`snapshot <output.json> [--prefix <prefix>]`**
- Snapshots 9 AWS services: EFS filesystems, IAM roles, Lambda functions, DynamoDB tables, S3 buckets, KMS aliases, SES receipt rule sets + rules, Route53 hosted zones
- All filtered by `${prefix}-` name pattern or `km:resource-prefix` tag
- Output is `jq -S` (sorted keys) for stable line diffs
- Writes top-level JSON keyed by service, with `meta.prefix` and `meta.captured_at`

**`diff <before.json> <after.json> [--prefix <prefix>]`**
- Strips volatile `meta.captured_at` before comparing
- Exits 0 if no changes, exits 1 with unified diff output if changes detected

Requirements: `aws` CLI with `$AWS_PROFILE` set (fails with clear error if unset), `jq`. Script passes `bash -n` syntax check and is chmod +x.
