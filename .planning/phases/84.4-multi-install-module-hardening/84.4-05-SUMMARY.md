## Operational Notes

**Direct terragrunt invocations require KM_RESOURCE_PREFIX export.** The `km` wrapper calls `ExportTerragruntEnvVars(cfg)` before invoking terragrunt, so `km init --plan` and `km init` always export `KM_RESOURCE_PREFIX` correctly. If running bare `terragrunt plan` or `terragrunt apply` directly (e.g. for debugging or recovery), operators must first:
```
export KM_RESOURCE_PREFIX=<prefix>   # e.g. whereiskurt or km
```
Without this, terragrunt defaults to the `"km"` fallback in `get_env("KM_RESOURCE_PREFIX", "km")` — which will read the km install's state bucket (`tf-km-state-use1`) regardless of which install directory you're working in.
