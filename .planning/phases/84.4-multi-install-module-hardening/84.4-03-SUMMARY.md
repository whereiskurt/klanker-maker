# Phase 84.4 Plan 03: EFS v2.0.0 Module — Prefix-Namespaced Creation Token and Security Group Summary

EFS module v2.0.0 with four hardcoded `km-` literals replaced by `${var.resource_prefix}-`, fixing the `FileSystemAlreadyExists` error that the `whereiskurt` probe hit when a second install shared the same creation token.

## What Was Built

`infra/modules/efs/v2.0.0/` — a drop-in replacement for v1.0.0 with per-install namespacing in every resource-identity position.

## Exact Substitutions Made

Four literal positions in `main.tf` were updated (all in resource-name or resource-identity positions):

| Line | Attribute | v1.0.0 (hardcoded) | v2.0.0 (prefix-templated) |
|------|-----------|---------------------|--------------------------|
| 7 | `aws_efs_file_system.shared` `creation_token` | `"km-shared-${var.region_label}"` | `"${var.resource_prefix}-shared-${var.region_label}"` |
| 13 | `aws_efs_file_system.shared` `tags.Name` | `"km-shared-efs-${var.region_label}"` | `"${var.resource_prefix}-shared-${var.region_label}"` |
| 22 | `aws_security_group.efs` `name` | `"km-efs-${var.region_label}"` | `"${var.resource_prefix}-efs-${var.region_label}"` |
| 43 | `aws_security_group.efs` `tags.Name` | `"km-efs-${var.region_label}"` | `"${var.resource_prefix}-efs-${var.region_label}"` |

Note: The v1.0.0 Name tag on the EFS filesystem was `"km-shared-efs-${var.region_label}"` (with `-efs-` infix). In v2.0.0 the Name tag uses `"${var.resource_prefix}-shared-${var.region_label}"` — the `-efs-` infix was dropped to match the creation_token pattern. This is intentional and keeps the naming consistent.

Total occurrences of `var.resource_prefix` in main.tf: **6** (creation_token, 2 Name tags, SG name, 2 `km:resource-prefix` tag values).

## km:resource-prefix Tag Confirmation

Both resources with `tags = {}` blocks received `"km:resource-prefix" = var.resource_prefix`:

- `aws_efs_file_system.shared` — tag value is `var.resource_prefix` (NOT literal `"km"`)
- `aws_security_group.efs` — tag value is `var.resource_prefix` (NOT literal `"km"`)

The `km:manager = "km"` tag is a label (allow-listed), not a resource-name discriminator — its literal value is intentional and not changed.

## Additional Tags Added

Phase 82 convention tag `"km:resource-prefix" = var.resource_prefix` was not present in v1.0.0. It was added to both resources in v2.0.0 — this is the load-bearing inventory-diff signal used by Plan 07 and Plan 08.

## resource_prefix Variable

Added to `variables.tf`:

```hcl
variable "resource_prefix" {
  type        = string
  default     = "km"
  description = "Resource-name prefix. Default 'km' renders byte-identical names to v1.0.0."
  validation {
    condition     = can(regex("^[a-z][a-z0-9]{0,11}$", var.resource_prefix))
    error_message = "resource_prefix must be 1-12 chars, start with a lowercase letter, contain only [a-z0-9]."
  }
}
```
