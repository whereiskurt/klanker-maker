## The Four Audited Attribute Names

The CONTEXT.md LOCKED list covers every resource-identity position where a hardcoded `km-` prefix would cause a collision between two parallel installs:

| Attribute | Used by |
|-----------|---------|
| `name` | Nearly all `aws_*` resources (IAM roles, SGs, SES rules, etc.) |
| `creation_token` | `aws_efs_file_system` — identity-immutable once created |
| `alias` | `aws_kms_alias` — the human-visible key name |
| `policy_name` | `aws_iam_role_policy` inline policy name |

**Not audited:** `description`, `tags` key names (like `km:manager`), `variable.default` values — these are labels or backward-compat defaults, not resource identity positions. HCL block types other than `resource` (e.g., `variable`, `locals`, `data`) are skipped entirely.

## The Allow-List Mechanism

`pkg/terragrunt/testdata/allowlist.txt` contains explicit permitted patterns (one per non-comment line). The audit extracts the quoted RHS from each pattern line and compares by equality:

```
km:manager = "km-cluster"      → permits literal value "km-cluster"
km:component = "km-slack-inbound" → permits "km-slack-inbound"
```

This is conservative: only an exact match permits the value. A pattern `km:foo = "km-cluster"` does NOT permit the value `"km-cluster-suffix"`. The allow-list is consulted only when a literal `LiteralValueExpr` or a template literal part contains `"km-"`.

## Module Walk Strategy

`walkModulesV2Plus` uses two-level `os.ReadDir` (avoids `filepath.Glob("**")` which doesn't support `**`):

1. `ReadDir("infra/modules")` → all module names
2. For each module, `ReadDir(moduleDir)` → all version directories
3. Skip entries not starting with `"v"` (non-version subdirs)
4. Skip `"v1.0.0"` explicitly (historical reference, unaudited)
5. All other version directories (v1.1.0, v2.0.0, v2.1.0, ...) are audited

Current modules audited: `ses/v2.0.0`, `dynamodb-identities/v1.1.0`, `dynamodb-sandboxes/v1.1.0`. Plans 02/03/04 will add `scp/v2.0.0`, `efs/v2.0.0`, `s3-replication/v2.0.0`.

## Negative Test Approach

`TestModuleAuditCatchesBadFixture` creates a temp directory with:

```hcl
resource "aws_iam_role" "bad" {
  name = "km-bad"
}
```

It passes a `*mockT` (not `*testing.T`) to `checkTFFile`. `mockT.Errorf` sets `m.errored = true` and records messages without calling `os.Exit`. After `checkTFFile` returns, the test asserts `mt.errored == true`. If `checkTFFile` silently passed the bad fixture, the outer test fails with "checkTFFile should have reported an error for km-bad literal but did not".

This gives explicit confidence the audit is not a no-op — essential for a regression gate.
