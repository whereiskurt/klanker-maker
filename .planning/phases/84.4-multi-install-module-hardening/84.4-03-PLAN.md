<objective>
Create `infra/modules/efs/v2.0.0/` with every hardcoded `km-` literal replaced by
`${var.resource_prefix}-`. The load-bearing fix is `aws_efs_file_system.shared.creation_token`
— AWS rejects two filesystems sharing the same creation_token in the same account/region,
which is the exact `FileSystemAlreadyExists` failure the `whereiskurt` probe hit.

Wave-ordering note: Plan 03 depends on both Plan 00 (toolchain) AND Plan 01 (audit test
scaffold). The `<verify>` step invokes `TestModuleNamesUseResourcePrefix`, which is
created by Plan 01 — without that dependency, parallel Wave 1 execution would race.
After this revision, Plan 03 runs in Wave 2 (after both 00 and 01 land).

Purpose: EFS-PREFIX-NAMESPACING. Each install owns its own filesystem; no cross-install
mount surface (CONTEXT.md decision). Sandboxes only mount their own install's filesystem.

Output: efs/v2.0.0/ module + `test/test_harness.tf.skip` (LOCKED versioning-strategy
requirement). Backward compat: with `resource_prefix=km` (default), creation_token
renders to `km-shared-${region_label}` — byte-identical to v1.0.0 — verified by
zero-diff `km init --plan` in Plan 05.

Smaller than Plan 02 (no Go-code mirror, no size guard, just 4-line literal substitution
per RESEARCH.md). The audit test (Plan 01) provides the static-analysis half of the
LOCKED test requirement; the `test_harness.tf.skip` provides the HCL-fixture half
(mirrors `infra/modules/cluster-irsa/v1.0.0/test_harness.tf.skip` precedent).
</objective>
