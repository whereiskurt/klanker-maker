<objective>
Operator-driven UAT. Cleanly tear down the `klanker-maker-whereiskurt` probe install
(resource_prefix=whereiskurt, AWS account 052251888500) using the standard sequence:

1. BEFORE snapshot — `scripts/inventory-diff.sh snapshot` captures every km-/whereiskurt-
   tagged/named resource in the account.
2. `km uninit --dry-run=false` from the `klanker-maker-whereiskurt/` working directory —
   regional teardown.
3. Manual SCP detach + delete (whereiskurt SCP is imported into whereiskurt state but
   policy content actively DENIES whereiskurt's lambdas — must be detached from the
   application account via `aws organizations detach-policy`).
4. Manual EFS cleanup — whereiskurt's failed apply left an orphan EFS filesystem (or
   none, if the failure was pre-creation; capture state via the BEFORE snapshot).
5. `km unbootstrap --dry-run=false` — foundation teardown (deletes whereiskurt state
   bucket, KMS key alias, S3 artifacts bucket).
6. AFTER snapshot — re-run the same inventory-diff.
7. Diff — all `whereiskurt-*` resources gone, all `km-*` resources byte-identical.

This is operator-driven (not Claude-autonomous) because:
- Live AWS account ops with cross-account permissions
- SCP detach requires management-profile credentials
- EFS delete may need mount-target deletion sequence + retry (AWS is eventually consistent)
- The diff verification is a human judgment call (what counts as "unrelated" volatility)

This UAT is the load-bearing gate for closure criterion (f) in ROADMAP.md Phase 84.4.
A successful teardown proves Phase 84.4's reach: the second install was created with
partial v1.0.0 hardcoded literals, hit the exact failure modes 84.4 codifies, and can
be cleanly removed without affecting the first install.

before opening the fresh-prefix `rg` UAT (Plan 08).

    ```
    EFS_ID=$(jq -r '.efs_filesystems[] | select(.CreationToken | startswith("whereiskurt-")) | .FileSystemId' .planning/phases/84.4-multi-install-module-hardening/before.json)
    if [ -n "$EFS_ID" ] && [ "$EFS_ID" != "null" ]; then
      echo "Whereiskurt EFS id: $EFS_ID"
      # List mount targets
      aws efs describe-mount-targets --file-system-id $EFS_ID --query 'MountTargets[].MountTargetId' --output text | xargs -n1 aws efs delete-mount-target --mount-target-id
      # Wait for mount-target deletion (poll until empty)
      until [ "$(aws efs describe-mount-targets --file-system-id $EFS_ID --query 'length(MountTargets)' --output text)" = "0" ]; do
        echo "waiting for mount targets to delete..."
        sleep 5
      done
      aws efs delete-file-system --file-system-id $EFS_ID
    else
      echo "No whereiskurt EFS to clean up"
    fi
    ```
