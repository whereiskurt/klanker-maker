## Phase Requirements

| Req ID | Description |
|--------|-------------|
| **REQ-ACK-RETRY-CLASSIFY** | `SlackReactorAdapter.Add` classifies each response into Success / Terminal-auth / Terminal-bad-input / Transient / RateLimited buckets per the locked taxonomy; unknown error strings default to Transient. |
| **REQ-ACK-RETRY-BUDGET** | Retry loop attempts at most 3 calls (1 initial + 2 retries) with backoff 200ms → 600ms ± 25% jitter; total worst-case sleep budget ≤ 1s; deterministic under test via injected `Rand`. |
| **REQ-ACK-RETRY-RETRYAFTER** | On `ErrSlackRateLimited` with `RetryAfterSeconds: N`: if `N ≤ remaining ctx budget`, sleep N and retry; else return the typed error immediately without sleeping. |
| **REQ-ACK-RETRY-CTXCANCEL** | Every backoff sleep is cancellable via `ctx.Done()`; if ctx cancels during sleep or between attempts, return `ctx.Err()` or the last observed error promptly. |
| **REQ-ACK-RETRY-JITTER-DETERMINISM** | Jitter uses `math/rand` (NOT `crypto/rand`) seeded via an injected source so tests assert exact sleep durations; production defaults to time-seeded source. |
| **REQ-ACK-RETRY-LOGS** | One Warn line on final give-up with `attempt=N` field preserving the existing `events: reaction failed` grep target; one Debug line per intermediate retry; Error level for auth-class terminal errors; existing structured fields (`channel`, `ts`, `emoji`, `err`) preserved. |
| **REQ-ACK-RETRY-HANDLER-TIMEOUT** | `pkg/slack/bridge/events_handler.go:235` `context.WithTimeout(..., 5*time.Second)` becomes `10*time.Second`; the goroutine still does not block the 200 response. |
| **REQ-ACK-RETRY-DEPLOY** | Bridge-only deploy via `make build && km init --lambdas`. No sandbox redeploy. No `km init --sidecars`. No schema migration. Rollback is PR revert + `km init --lambdas`. |
