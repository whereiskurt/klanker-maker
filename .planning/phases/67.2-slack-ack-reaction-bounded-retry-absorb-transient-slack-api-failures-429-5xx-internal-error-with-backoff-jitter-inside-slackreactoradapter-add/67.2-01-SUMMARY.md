## Decisions Made

- **Internal test file vs test-only exporter shim.** Plan 67.2-01-PLAN.md offered both options; chose the cleaner alternative (new file `aws_adapters_internal_test.go` in `package bridge`) over adding `ClassifyReactionErrorForTest` + `ClassXxxForTest` constants to production `aws_adapters.go`. Matches existing convention — `events_handler_test.go` is already `package bridge`. Keeps production file free of test-only artifacts.
- **Enumerate RESEARCH.md's extra codes vs rely on default-unknown.** Chose enumeration despite the larger switch statement (~10 extra lines). The default-transient policy is safe but loses the operator-actionable distinction — `token_expired` deserves Error-level log so operators see it during token rotation incidents, not Warn buried with bad-input failures.
- **Helpers named `captureBridgeLogger` not `captureLogger`.** `handler_logging_test.go` already declares `captureLogger` in `package bridge_test`; reusing the name would collide. The new helper is intentionally distinct so plan 02's retry-loop tests + the existing handler-logging tests can coexist.
