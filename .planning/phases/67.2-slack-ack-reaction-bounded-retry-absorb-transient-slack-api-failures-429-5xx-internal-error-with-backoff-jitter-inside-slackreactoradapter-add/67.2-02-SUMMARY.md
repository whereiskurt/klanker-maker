## Decisions Made

- **`math/rand` v1 over `math/rand/v2`.** CONTEXT.md locks v1; v2 would have required slightly different seed/source plumbing for the test-injection pattern. v1's `*rand.Rand` is the most natural fit for "set this in tests, leave nil in production". RESEARCH.md noted v2 has cleaner Float64 ergonomics but the difference is cosmetic; the locked decision stands.
- **No log line for `classTerminalBadInput`.** The handler-side `events_handler.go:238` already emits `h.log().Warn("events: reaction failed", ...)` on any error returned from `Reactor.Add`. Adding a second Warn line inside the adapter would double-log every bad-input failure. The wrapped error returned by Add carries the Slack code (e.g., `bridge: reactions.add: message_not_found`), preserving the existing operator grep target without duplication.
- **Removed inline `already_reacted` shortcut at old line 573-577.** Plan-01's classifier already returns `classSuccess` for `httpStatus==200 && apiErr=="already_reacted"`. Keeping the inline check would have been a single-source-of-truth violation: a future change to the idempotency-success rule could be made in one place but not the other. The new `Add` body has exactly one path that returns nil (`case classSuccess`).
- **Updated `TestSlackReactorAdapter_RateLimited` in-place rather than deleting/replacing.** The plan explicitly called out this update as Task 1 Step 6. Deleting it would have lost the "*ErrSlackRateLimited{Method:reactions.add, RetryAfterSeconds:N}` is the typed contract" assertion. The in-place update preserves that contract while neutralizing the wall-clock cost (no-op Sleep stub) under the new retry loop.
- **Added a small `reactorTokenFetcher()` factory in test code.** All 10 new tests need an `SSMBotTokenFetcher` returning `xoxb-test`, and the boilerplate would have made each test ~6 lines noisier. The factory keeps every test scannable while maintaining the same end-to-end stub behavior the existing 4 `TestSlackReactorAdapter_*` tests use.
- **`captureBridgeLogger` test fixture's restore closure resets to `slog.NewTextHandler(&bytes.Buffer{}, nil)` (a discard sink), not to `slog.Default()`.** This was already established by plan-01 — it ensures parallel tests don't pollute each other's captured-log buffers via the package-level `bridge.SetLogger` global.
