<objective>
Replace the typed per-section reflection merge with a single generic recursive
`deepMerge` over the decoded `map[string]any`, and rewrite `resolve()` as a DAG
(multi-parent, memoized, diamond-idempotent) instead of a single-parent chain.

THE LYNCHPIN: this plan DELETES the entire hand-written typed-merger zoo
(`mergeSpecSection`, `mergeNotificationSpec` + all its children, `mergeAgentSpec`
+ children, `mergePermissionsPassthrough`, `pickBoolPtr/pickIntPtr/pickString`).
The generic map-level merge subsumes them and fixes BOTH known defect classes in
one stroke: (1) the all-or-nothing whole-section replace, and (2) the
Budget/Artifacts/Email/OTP/CLI/Secrets sections that were never merged at all.

The proof that the generic merge preserves field-level behavior is that the
EXISTING `inherit_notification_test.go` and `inherit_agent_test.go` MUST stay
green after the zoo is deleted.

Purpose: composable multi-parent inheritance with uniform list-union semantics.
Output: deepMerge engine, DAG resolve, exhaustive table-driven tests, fixtures.
</objective>

Public API unchanged:
```go
func Resolve(name string, searchPaths []string) (*SandboxProfile, error)
```
Resolve still returns a fully-merged *SandboxProfile and does NOT call Validate
(per the existing doc comment — keep that contract).

New private engine (RESEARCH Pattern 2 & 3):
```go
func deepMerge(dst, src map[string]any) map[string]any   // maps recurse; scalars src-wins; slices concat+dedup
func concatDedup(a, b []any) []any                        // order-preserving, first-occurrence kept; objects de-dup by deep-equality
func toSlice(v any) ([]any, bool)                         // []interface{} coercion
// resolve recurses producing a map[string]any per profile, folds bases L→R then child, then:
//   yaml.Marshal(acc) -> yaml.Unmarshal -> *SandboxProfile
```

DAG resolve signature (RESEARCH Pattern 3 — add memo + path-scoped visited):
```go
func resolveMap(name, baseDir string, searchPaths []string, ancestry map[string]bool, depth int, memo map[string]map[string]any) (map[string]any, string, error)
```
- ancestry is COPIED per branch (Pitfall 1: a single shared visited map false-flags
  diamond bases as cycles). memo (keyed by resolved abs path / builtin name) caches
  the resolved map so a shared base is resolved once.
- depth guard: raise maxInheritanceDepth 3 → 10 (RESEARCH recommendation).

initCommandsAppend handling (Pitfall 4 / open question D): after building the
merged map, append the leaf-chain's collected `execution.initCommandsAppend`
entries onto `execution.initCommands` (concat+dedup), then clear/leave
initCommandsAppend (it is a merge-time concept; the compiler reads initCommands).
Decide impl: simplest is to let deepMerge concat initCommandsAppend across the
chain like any list, then in a post-merge pass move execution.initCommandsAppend
onto the tail of execution.initCommands and delete the append key. Document the
chosen approach in the SUMMARY.

Pitfall 2 (non-pointer bool zero-value trap): document in code comment that base
FRAGMENTS must only declare fields they intend to set (a fragment that writes a
full spec.runtime block forces spot:false etc. onto children). The engine itself
needs no special-casing — scalar child-wins handles the override; Plan 04
authors fragments narrowly.
