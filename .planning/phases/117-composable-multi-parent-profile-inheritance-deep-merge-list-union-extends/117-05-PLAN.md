  <name>Task 1: OPERATOR-GUIDE.md § Composable inheritance</name>
  <files>OPERATOR-GUIDE.md</files>
  <action>
    Add a new section "§ Composable inheritance (multi-parent profiles)" to
    OPERATOR-GUIDE.md. Cover (use the SHIPPED dc34.yaml / learn.v2.yaml as
    concrete examples — read them first for accurate field names):
    - `extends:` accepts a single string (back-compat) OR an ordered list of base
      references. Bases are resolved left→right; the child applies LAST. Last/child
      wins on scalar collisions — ORDER MATTERS (especially initCommands).
    - Deep-merge rules: maps recurse (key-union at every depth — env, configFiles,
      metadata.labels, nested spec blocks); scalars child-wins; LISTS concat then
      de-dup (order-preserving, first occurrence kept) UNIFORMLY across
      initCommands, allowedDNSSuffixes/allowedHosts, allowedRepos/allowedRefs,
      email.allowedSenders, agent.claude.tools.autoApprove, rsyncPaths,
      artifacts.paths. Object lists (additionalSnapshots) de-dup by deep-equality.
    - Authoring base fragments: put them in `profiles/base/`, mark each with
      `metadata.abstract: true`, declare ONLY the fields the fragment owns.
      Abstract fragments are NOT validated standalone (km validate skips them with
      a message) and cannot be `km create`d directly.
    - `execution.initCommandsAppend`: append leaf-specific commands after the
      merged initCommands (use it for version-pinned installs that can't live in a
      shared base).
    - Diamond inheritance (D→[A,B], A→Core, B→Core) is idempotent and Core is
      resolved once (memoized). Max inheritance depth is 10; cycles are rejected.
    - LIMITATIONS (v1):
      * NO NARROWING: because lists always union, a child CANNOT remove/shrink a
        base's allowedHosts/allowedDNSSuffixes/etc. To use a smaller allowlist,
        don't extend the broad base — compose from a narrower one. A `!replace` /
        `__replace:` directive is a deferred follow-up.
      * Bool zero-value trap: a fragment that writes a FULL block containing
        non-pointer bools (e.g. a full spec.runtime with spot/hibernation) forces
        those bools (default false) onto children. Fragments must declare only the
        fields they intend to set; keep mixed-bool blocks like runtime in the leaf.
    Show a complete worked example based on the real refactored dc34.yaml.
  </action>
