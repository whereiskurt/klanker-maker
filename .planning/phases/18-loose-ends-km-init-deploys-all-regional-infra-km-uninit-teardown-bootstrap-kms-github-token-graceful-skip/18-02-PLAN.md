    - Test 3: uninit proceeds when active sandboxes exist and --force is set
    - Test 4: uninit proceeds when no active sandboxes exist (--force not needed)
    - Test 5: uninit skips modules whose directories don't exist (warning, continues)
    - Test 6: uninit continues past modules with no Terraform state (Destroy on empty state is no-op)
    - Test 7: uninit warns and requires --force when StateBucket is empty (can't verify sandboxes)

  <action>
1. Create `internal/app/cmd/uninit_test.go` first. Define mock interfaces:
   - `MockUninitRunner` with a `Destroy(ctx, dir) error` method that records calls in order
   - Use the existing `SandboxLister` interface with a mock that returns configurable sandbox records

2. Create `internal/app/cmd/uninit.go`:

3. Define `NewUninitCmd(cfg *config.Config) *cobra.Command` following the exact same pattern as NewInitCmd:
   - Flags: `--aws-profile` (default "klanker-application"), `--region` (default "us-east-1"), `--force` (default false)
   - Use: "uninit", Short: "Tear down all shared regional infrastructure for a region"
   - Long: helpText("uninit")

4. Define `UninitRunner` interface with `Destroy(ctx context.Context, dir string) error` for testability.

5. Implement `runUninit(cfg *config.Config, awsProfile, region string, force bool) error`:
   a. Validate AWS credentials (same pattern as runInit)
   b. Check active sandboxes:
      - If cfg.StateBucket is empty, warn "cannot verify active sandboxes — state_bucket not configured" and require --force
      - Otherwise, use `awspkg.NewS3SandboxLister(s3Client, cfg.StateBucket)` to list sandboxes
      - Filter by region and status == "running"
      - If count > 0 and !force, return error listing the count and suggesting --force
   c. Build reverse module list (reuse `regionalModules` from init.go but reversed — if init.go doesn't export it yet, define the same list locally in reverse order)
   d. For each module in reverse order:
      - Check if directory exists (skip with warning if not)
      - Call runner.Destroy(ctx, mod.dir)
      - On error: print warning but continue (partial teardown is better than stopping midway — unlike init where we stop on failure)
   e. Print summary of what was destroyed

6. For testability, add `runUninitWithDeps(cfg *config.Config, runner UninitRunner, lister SandboxLister, ...)` that the tests call directly.
