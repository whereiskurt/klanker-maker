<objective>
Expand `km init` from network-only to all 6 regional infrastructure modules, and add `state_bucket` prompt to `km configure`.

Purpose: Operators currently must manually `terragrunt apply` five additional modules after `km init`. This plan makes `km init` a single command that deploys all regional infra. Additionally, `km list`/`km status` fail without `state_bucket` in config — adding the prompt to `km configure` closes that gap.

Output: Updated init.go with sequential multi-module apply, updated configure.go with state_bucket field, and unit tests for both.
</objective>
