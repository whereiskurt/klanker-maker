<objective>
Add safe-phrase validation for external (non-sandbox) inbound email at the km-mail-poller bash script. Per the research recommendation (Pattern 6), the validation point is km-mail-poller — NOT a new SES receipt Lambda. The roadmap goal mentioned a Lambda but research found the SES receipt rule is a pure S3 action with no Lambda; adding one would require Terraform changes outside Phase 57's scope. km-mail-poller already filters on To-header and sender allowlist; safe phrase is the natural third gate.

Behavior:
- **Sandbox-to-sandbox** (X-KM-Sender-ID present): no safe phrase check (signature is the trust mechanism).
- **External** (X-KM-Sender-ID absent): require `KM-AUTH: <phrase>` in body. Without it, drop to `$MAIL_DIR/skipped/`.
- **Fail-open** when SSM unreachable: log warning, skip the check (consistent with existing fail-open allowlist behavior).

This is the third and final plan modifying `pkg/compiler/userdata.go`. Plans 01 (km-send `--no-sign`) and 02 (km-recv parsing fixes) are already complete.

Purpose: External email is a vector for unsolicited or hostile traffic into the sandbox mailbox. Requiring an operator-defined safe phrase ensures only deliberate external messages (forwarded by the operator with the phrase, or sent by trusted parties who know it) reach the sandbox. The phrase is already established and used by `email-create-handler` Lambda at the operator inbox — extending it to sandbox inbound is consistent with existing security architecture.

```bash
# At script start (one-shot, before main loop):
KM_SAFE_PHRASE_CACHED=""
if KM_SAFE_PHRASE_CACHED=$(aws ssm get-parameter \
    --name "/km/config/remote-create/safe-phrase" \
    --with-decryption \
    --query 'Parameter.Value' \
    --output text 2>/dev/null); then
  : # success — cached
else
  echo "[km-mail-poller] WARNING: failed to fetch safe phrase from SSM, external email validation will fail-open" >&2
  KM_SAFE_PHRASE_CACHED=""
fi

# Per-message check (after sender allowlist passes, before move to new/):
sender_id=$(head -c 8192 "$local_file" | grep -i "^X-KM-Sender-ID:" | head -1 | sed 's/^[^:]*:[[:space:]]*//' | tr -d '\r')
if [[ -z "$sender_id" ]]; then
  # External sender — require KM-AUTH phrase
  if [[ -n "${KM_SAFE_PHRASE_CACHED:-}" ]]; then
    body_check=$(head -c 4096 "$local_file" || true)
    if ! echo "$body_check" | grep -qP "KM-AUTH:\s*${KM_SAFE_PHRASE_CACHED}"; then
      sender_email=$(head -c 8192 "$local_file" | grep -i "^From:" | head -1 | tr -d '\r')
      echo "[km-mail-poller] External email from $sender_email missing/invalid safe phrase, skipping $key" >&2
      rm -f "$local_file"
      mkdir -p "$MAIL_DIR/skipped"
      touch "$MAIL_DIR/skipped/$key"
      continue
    fi
  fi
  # If KM_SAFE_PHRASE_CACHED is empty (SSM fetch failed at startup), fall through (fail-open).
fi
```
