<objective>
Add a third inbound-mail gate to km-mail-poller: external (non-sandbox) email must contain the operator-configured safe phrase to be delivered. Without it, external mail is silently dropped to `MAIL_DIR/skipped/` (same fate as a non-allowlisted sender).

Purpose: Plan 01 enables outbound external email; Plan 02 enables external mail to be readable once delivered; this plan closes the loop by ensuring inbound external mail is filtered. The phase-description language ("the SES receipt rule Lambda validates this before delivery") does NOT match the codebase: per RESEARCH.md the `sandbox_inbound` SES receipt rule (verified in `infra/modules/ses/v1.0.0/main.tf` line 126) is a pure S3 action with NO Lambda trigger. There is no Lambda to extend. The correct enforcement layer is `km-mail-poller`, which already runs as a per-sandbox systemd service and already has sender-allowlist filtering â€” adding a safe-phrase gate alongside the allowlist is a localized, additive bash change.

Two follow-on issues from RESEARCH.md must also be fixed in this plan:

1. **sender_id scoping bug (research Pitfall 5):** Today `sender_id` is only extracted INSIDE the `if [ -n "${KM_ALLOWED_SENDERS:-}" ]` block (line 644). When no allowlist is configured, `sender_id` is never set, so the safe-phrase gate cannot tell sandbox vs external senders. Fix: hoist `sender_id`/`sender_email` extraction OUT of the allowlist conditional so it always runs after the To-header match.
2. **Regex injection from SSM phrase value (research Pitfall 4):** The phrase comes from SSM and may contain `.`, `+`, `?`, `*`, etc. Using `grep -P "KM-AUTH:\s*${PHRASE}\b"` exposes the gate to false positives/negatives and arguably injection. Fix: use `grep -F` (fixed-string match) on the literal `KM-AUTH: <phrase>` substring.

Output: Two surgical edits inside the `pkg/compiler/userdata.go` km-mail-poller heredoc (lines ~606-684) that turn 7 Wave 0 RED tests GREEN: `TestUserData_MailPoller_ExtractsSenderIdUnconditionally`, `TestUserData_MailPoller_FetchesSafePhrase`, `TestUserData_MailPoller_DropsExternalNoPhrase`, `TestUserData_MailPoller_DeliversExternalWithPhrase`, `TestUserData_MailPoller_SkipsCheckForSandbox`, `TestUserData_MailPoller_SsmFailOpen`, `TestUserData_MailPoller_PhraseMatchUsesFixedString`. Pre-existing km-mail-poller behavior (allowlist enforcement when KM_ALLOWED_SENDERS is set, To-header matching, S3 list/copy, processed/skipped quarantine) MUST remain GREEN.

Honors locked decision: "Inbound external emails must contain the configured safe phrase (from km configure) to be accepted" (roadmap Phase 57 goal). The implementation deviates from the goal text in one specific way explicitly cleared in the planning context: enforcement happens in km-mail-poller (bash systemd service inside the sandbox) rather than a "SES receipt rule Lambda" (no such Lambda exists for `sandbox_inbound`). This deviation is documented in this objective section.
