<objective>
Add a `--no-sign` flag to the in-sandbox `km-send` bash script so sandboxes can send plain RFC 5322 email to external recipients (Gmail, etc.) without Ed25519 signing or `X-KM-*` headers. This is the first of three sequential plans modifying `pkg/compiler/userdata.go` (Plans 02 and 03 modify km-recv and km-mail-poller respectively, in different sections of the same file).

Purpose: External recipients (Gmail, Outlook) reject or quarantine messages with unknown headers. Removing the X-KM-Sender-ID and X-KM-Signature headers (and the SSM/openssl work that produces them) gives sandboxes a deliverable plain-email path while preserving the default signed-email path for sandbox-to-sandbox traffic.

Output: One file modified (`pkg/compiler/userdata.go`). Three RED tests from Plan 00 turn GREEN: `TestUserData_KmSend_NoSignFlag`, `TestUserData_KmSend_NoSignOmitsHeaders`, `TestUserData_KmSend_NoSignKeepsAuthPhrase`.
</objective>
