<objective>
Wire the GitHub repo allowlist from profile YAML through the compiler to the proxy sidecar environment.

Purpose: The proxy core (Plan 01) implements filtering logic where configured GitHub repos implicitly allow GitHub hosts through the proxy (no need for `network.egress.allowedHosts` entries). The allowedRepos list must flow from the SandboxProfile through the compiler (EC2 userdata and ECS service.hcl templates) as the `KM_GITHUB_ALLOWED_REPOS` env var, and the proxy `main.go` must read it and pass it to `NewProxy` via `WithGitHubRepoFilter`. The DNS proxy also needs GitHub DNS suffixes added implicitly when GitHub repos are configured.

Output: End-to-end wiring from profile YAML to proxy runtime configuration on both EC2 and ECS substrates.
</objective>
