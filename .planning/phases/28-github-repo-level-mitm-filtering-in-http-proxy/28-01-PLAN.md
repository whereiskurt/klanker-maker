<feature>
  <name>GitHub Repo-Level MITM Filtering</name>
  <files>sidecars/http-proxy/httpproxy/github.go, sidecars/http-proxy/httpproxy/github_test.go, sidecars/http-proxy/httpproxy/proxy.go, sidecars/http-proxy/httpproxy/http_proxy_test.go</files>
  <behavior>
    ExtractRepoFromPath:
    - ("github.com", "/owner/repo") -> "owner/repo"
    - ("github.com", "/owner/repo.git/info/refs") -> "owner/repo"
    - ("github.com", "/owner/repo/tree/main") -> "owner/repo"
    - ("api.github.com", "/repos/owner/repo/commits") -> "owner/repo"
    - ("api.github.com", "/rate_limit") -> "" (non-repo, pass through)
    - ("api.github.com", "/user") -> "" (non-repo, pass through)
    - ("raw.githubusercontent.com", "/owner/repo/main/README.md") -> "owner/repo"
    - ("codeload.githubusercontent.com", "/owner/repo/tar.gz/main") -> "owner/repo"
    - ("github.com", "/login") -> "" (single segment, non-repo)
    - ("github.com", "/") -> "" (root, pass through)
    - ("github.com:443", "/owner/repo") -> "owner/repo" (port stripped)
    - Case normalization: ("github.com", "/Owner/Repo") -> "owner/repo"

    IsRepoAllowed:
    - ("owner/repo", ["owner/repo"]) -> true (exact match)
    - ("owner/repo", ["other/repo"]) -> false (not in list)
    - ("Owner/Repo", ["owner/repo"]) -> true (case-insensitive)
    - ("org/anyrepo", ["org/*"]) -> true (org wildcard)
    - ("otherorg/repo", ["org/*"]) -> false (wrong org)
    - ("owner/repo", []) -> false (empty list)
    - ("owner/repo", ["github.com/owner/repo"]) -> true (github.com/ prefix stripped)

    GitHubBlockedResponse:
    - Returns 403 status code
    - Body is JSON with "error": "repo_not_allowed", "repo": "<blocked-repo>", "reason": "<message>"

    WithGitHubRepoFilter:
    - When repos is non-empty, sets cfg.githubRepos
    - When cfg.githubRepos is nil, NO GitHub MITM handlers registered in NewProxy

    Integration (via NewProxy):
    - Request to allowed repo passes through (200)
    - Request to blocked repo returns 403 JSON
    - Non-repo GitHub URL passes through
    - No githubRepos configured -> no MITM, normal host-level filtering only
  </behavior>
  <implementation>
    1. Create `github.go` in `sidecars/http-proxy/httpproxy/` with:
       - `var githubHostsRegex = regexp.MustCompile("^(github\\.com|api\\.github\\.com|raw\\.githubusercontent\\.com|codeload\\.githubusercontent\\.com)(:\\d+)?$")`
       - `func ExtractRepoFromPath(host, urlPath string) string` â€” strips port from host, uses `strings.ToLower`, switches on host: `api.github.com` extracts after `/repos/`, all others extract first two path segments. Strips `.git` suffix. Returns `""` for non-repo URLs.
       - `func IsRepoAllowed(repo string, allowed []string) bool` â€” case-insensitive comparison, org wildcard `owner/*` support, normalizes `github.com/` prefix in allowlist entries.
       - `func GitHubBlockedResponse(req *http.Request, sandboxID, repo string) *http.Response` â€” returns `goproxy.NewResponse` with 403 and JSON body `{"error":"repo_not_allowed","repo":"...","reason":"..."}`.
       - `func WithGitHubRepoFilter(allowedRepos []string) ProxyOption` â€” sets `cfg.githubRepos = allowedRepos`.

    2. Update `proxy.go`:
       - Add `githubRepos []string` field to `proxyConfig` struct.
       - In `NewProxy`, after existing Bedrock/Anthropic MITM blocks and BEFORE the general CONNECT handler (line ~452), add: if `len(cfg.githubRepos) > 0`, register `HandleConnectFunc` returning `goproxy.MitmConnect` for `githubHostsRegex`, then register `OnRequest.DoFunc` that calls `ExtractRepoFromPath` + `IsRepoAllowed` and returns `GitHubBlockedResponse` for blocked repos. Use zerolog structured logging with `event_type` fields: `github_mitm_connect`, `github_repo_blocked`, `github_repo_allowed`.

    3. Create `github_test.go` with table-driven tests for `ExtractRepoFromPath`, `IsRepoAllowed`, and `GitHubBlockedResponse`.

    4. Add integration tests to `http_proxy_test.go`:
       - `TestHTTPProxy_GitHubAllowedRepo` â€” proxy with `WithGitHubRepoFilter(["owner/repo"])`, request to allowed repo passes
       - `TestHTTPProxy_GitHubBlockedRepo` â€” request to disallowed repo returns 403 JSON
       - `TestHTTPProxy_GitHubNonRepoPassthrough` â€” request to `api.github.com/rate_limit` passes through
       - `TestHTTPProxy_GitHubNoFilter` â€” no `WithGitHubRepoFilter` option, GitHub hosts handled by normal host-level filtering only

    Anti-patterns to avoid:
    - Do NOT register GitHub HandleConnectFunc AFTER the general CONNECT handler (goproxy first-match semantics)
    - Do NOT block non-repo URLs (login, rate_limit, session must pass through)
    - Do NOT use case-sensitive comparison for repo names
    - Do NOT block when allowedRepos is empty/nil â€” just skip handler registration
