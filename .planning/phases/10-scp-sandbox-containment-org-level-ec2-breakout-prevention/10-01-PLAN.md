  <action>
Create `infra/modules/scp/v1.0.0/` directory with three files:

**variables.tf** — Three required variables:
- `application_account_id` (string) — 12-digit AWS account ID where sandboxes run
- `allowed_regions` (list(string)) — Regions allowed for non-global services (e.g. ["us-east-1"])
- `trusted_role_arns` (list(string)) — Role ARN patterns exempt from containment Deny statements. Default should include `AWSReservedSSO_*_*` pattern for operator SSO roles. The variable description should note that callers should add `km-provisioner-*` and `km-lifecycle-*` patterns when dedicated provisioner roles are introduced.

**main.tf** — Use `data "aws_iam_policy_document"` (NOT hand-coded JSON strings) to build the SCP document. The document must contain exactly 8 statements:

1. `DenySGMutation` — Deny ec2:CreateSecurityGroup, DeleteSecurityGroup, AuthorizeSecurityGroup{Ingress,Egress}, RevokeSecurityGroup{Ingress,Egress}, ModifySecurityGroupRules, UpdateSecurityGroupRuleDescriptions{Ingress,Egress}. Condition: ArnNotLike aws:PrincipalARN matching `var.trusted_role_arns`.

2. `DenyNetworkEscape` — Deny ec2:CreateVpc, CreateSubnet, CreateRouteTable, CreateRoute, CreateInternetGateway, AttachInternetGateway, CreateEgressOnlyInternetGateway, CreateNatGateway, CreateVpcPeeringConnection, AcceptVpcPeeringConnection, CreateTransitGateway, CreateTransitGatewayVpcAttachment. Condition: ArnNotLike var.trusted_role_arns.

3. `DenyInstanceMutation` — Deny ec2:RunInstances, ModifyInstanceAttribute, ModifyInstanceMetadataOptions. Condition: ArnNotLike concat of var.trusted_role_arns AND km-ecs-spot-handler pattern (arn:aws:iam::${var.application_account_id}:role/km-ecs-spot-handler).

4. `DenyIAMEscalation` — Deny iam:CreateRole, AttachRolePolicy, DetachRolePolicy, PassRole, AssumeRole. Condition: ArnNotLike concat of var.trusted_role_arns AND km-budget-enforcer-* pattern (arn:aws:iam::${var.application_account_id}:role/km-budget-enforcer-*). Budget-enforcer needs AttachRolePolicy/DetachRolePolicy for Bedrock IAM revocation.

5. `DenyStorageExfiltration` — Deny ec2:CreateSnapshot, CopySnapshot, CreateImage, CopyImage, ExportImage. Condition: ArnNotLike var.trusted_role_arns.

6. `DenySSMPivot` — Deny ssm:SendCommand, ssm:StartSession. Condition: ArnNotLike list of km-ec2spot-ssm-* pattern AND AWSReservedSSO_*_* (operator SSO). This is a SEPARATE carve-out list — only operator and SSM instance roles, NOT the full trusted_role_arns.

7. `DenyOrganizationsDiscovery` — Deny organizations:ListAccounts, DescribeOrganization, ListRoots, ListOrganizationalUnitsForParent, ListChildren. NO condition — applies to ALL roles in the application account. (Management account is exempt by AWS design — SCPs don't apply to the management account itself.)

8. `DenyOutsideAllowedRegions` — Region lock using `not_actions` (NotAction equivalent in aws_iam_policy_document). The not_actions list MUST include global services: iam:*, sts:*, organizations:*, support:*, cloudfront:*, route53:*, route53domains:*, budgets:*, ce:*, cur:*, health:*, trustedadvisor:*, waf:*, shield:*, globalaccelerator:*, networkmanager:*, pricing:*, s3:GetAccountPublic*, s3:ListAllMyBuckets, s3:PutAccountPublic*. Condition: StringNotEquals aws:RequestedRegion matching var.allowed_regions. NO trusted role carve-out — region lock applies to ALL roles.

Use `data "aws_iam_policy_document" "sandbox_containment"` with multiple `statement` blocks. Each statement has its own `sid`, `effect = "Deny"`, `actions` list, `resources = ["*"]`, and `condition` block where applicable.

For the SCP resource:
```hcl
resource "aws_organizations_policy" "sandbox_containment" {
  name        = "km-sandbox-containment"
  description = "Org-level backstop preventing sandbox role EC2/network/IAM breakout"
  type        = "SERVICE_CONTROL_POLICY"
  content     = data.aws_iam_policy_document.sandbox_containment.json
  tags = {
    "km:component" = "scp"
    "km:managed"   = "true"
  }
}
```

For the attachment:
```hcl
resource "aws_organizations_policy_attachment" "sandbox_containment" {
  policy_id = aws_organizations_policy.sandbox_containment.id
  target_id = var.application_account_id
}
```

Use `locals` for role pattern construction:
- `trusted_arns_base` = var.trusted_role_arns (operator passes their SSO + provisioner patterns)
- `trusted_arns_instance` = concat trusted_arns_base with km-ecs-spot-handler
- `trusted_arns_iam` = concat trusted_arns_base with km-budget-enforcer-*
- `trusted_arns_ssm` = [km-ec2spot-ssm-* pattern, AWSReservedSSO_*_* pattern] (hardcoded, not from var)

CRITICAL: Do NOT use `NotPrincipal` — it is not supported in SCPs. Use `Condition.ArnNotLike.aws:PrincipalARN` via the `condition` block in `aws_iam_policy_document`.

CRITICAL: Do NOT carve out `km-ecs-task-*` from any Deny statement — that IS the sandbox workload role and should be fully contained.

**outputs.tf** — Two outputs:
- `policy_id` = aws_organizations_policy.sandbox_containment.id
- `policy_arn` = aws_organizations_policy.sandbox_containment.arn
