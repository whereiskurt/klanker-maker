  <name>Task 1: Audit log sidecar binary (OBSV-01, OBSV-02, OBSV-03)</name>
  <files>sidecars/audit-log/main.go, sidecars/audit-log/audit_log_test.go</files>
  <behavior>
    - TestAuditLogFormat_ShellCommand: parse a shell_command JSON line from stdin → output matches exact schema (timestamp, sandbox_id, event_type="shell_command", source="audit-log", detail.command)
    - TestAuditLogFormat_NetworkEvent: parse a dns_query JSON line from stdin → output matches schema with event_type="dns_query", detail.domain, detail.allowed
    - TestAuditLogDest_Stdout: AUDIT_LOG_DEST=stdout → events written to stdout as JSON lines
    - TestAuditLogDest_CloudWatch: AUDIT_LOG_DEST=cloudwatch → events sent to mock CloudWatch PutLogEvents
    - TestAuditLogFormat_InvalidJSON: non-JSON line on stdin → emit warning log, do not crash
  </behavior>
  <action>
    Split `sidecars/audit-log/` into:
    - `auditlog.go` (package auditlog): exported types and logic
    - `main.go` (package main): wires env vars, starts reading

    Define `AuditEvent` struct matching the locked JSON schema:
    ```go
    type AuditEvent struct {
        Timestamp  time.Time              `json:"timestamp"`
        SandboxID  string                 `json:"sandbox_id"`
        EventType  string                 `json:"event_type"`  // shell_command, dns_query, http_request
        Source     string                 `json:"source"`
        Detail     map[string]interface{} `json:"detail"`
    }
    ```

    Define `Dest` interface:
    ```go
    type Dest interface {
        Write(ctx context.Context, event AuditEvent) error
        Flush(ctx context.Context) error
    }
    ```

    Implement three `Dest` implementations:
    1. `StdoutDest`: writes JSON-marshaled event + newline to `os.Stdout`
    2. `CloudWatchDest`: buffers events in memory, flushes to CW via `PutLogEvents` every 5 seconds or 25 events (whichever comes first). Calls `pkg/aws.EnsureLogGroup()` on initialization. Takes a `CWLogsAPI` interface (for testing).
    3. `S3Dest` (stub only — writes to `io.Discard` for now; full S3 archive is Phase 4 territory; log the intent): write warning "S3 dest: flushing at exit not yet implemented" and fall back to stdout.

    `main()`:
    - Read env vars: AUDIT_LOG_DEST (default "stdout"), SANDBOX_ID, CW_LOG_GROUP (default "/km/sandboxes/<SANDBOX_ID>/"), AWS_REGION
    - Read newline-delimited JSON from stdin in a loop
    - Parse each line as `AuditEvent`; if parse fails, emit zerolog warning with raw line
    - Route to configured `Dest`
    - On SIGTERM/SIGINT, call `Dest.Flush()` and exit cleanly

    On EC2: the audit-log sidecar is piped from shell audit hook (`PROMPT_COMMAND` in /etc/profile.d/km-audit.sh) and receives events from dns-proxy and http-proxy via systemd journal pipe. On ECS: awslogs driver on the audit-log container captures stdout.

    Tests use `bytes.Buffer` as stdin, capture stdout output.
  </action>

    <automated>go test ./sidecars/audit-log/... -run TestAuditLog -v -timeout 30s</automated>
  </verify>
  <done>All TestAuditLog_* tests pass. `go build ./sidecars/audit-log/` compiles without error.</done>

  <files>pkg/aws/cloudwatch.go, pkg/aws/cloudwatch_test.go</files>
  <behavior>

    - TestEnsureLogGroup_AlreadyExists: CreateLogGroup returns ResourceAlreadyExistsException → function returns nil (idempotent)
    - TestPutLogEvents_Success: PutLogEvents with 3 events → SDK called once with correct log group/stream
    - TestGetLogEvents_Success: GetLogEvents returns 2 events → both returned as []LogEvent
    - TestGetLogEvents_Empty: GetLogEvents returns no events → empty slice, no error
  </behavior>
  <action>
    Create `pkg/aws/cloudwatch.go`:

    Define `CWLogsAPI` interface (narrow — only methods used):
    ```go
    type CWLogsAPI interface {
        CreateLogGroup(ctx context.Context, input *cloudwatchlogs.CreateLogGroupInput, optFns ...func(*cloudwatchlogs.Options)) (*cloudwatchlogs.CreateLogGroupOutput, error)
        CreateLogStream(ctx context.Context, input *cloudwatchlogs.CreateLogStreamInput, optFns ...func(*cloudwatchlogs.Options)) (*cloudwatchlogs.CreateLogStreamOutput, error)
        PutLogEvents(ctx context.Context, input *cloudwatchlogs.PutLogEventsInput, optFns ...func(*cloudwatchlogs.Options)) (*cloudwatchlogs.PutLogEventsOutput, error)
        GetLogEvents(ctx context.Context, input *cloudwatchlogs.GetLogEventsInput, optFns ...func(*cloudwatchlogs.Options)) (*cloudwatchlogs.GetLogEventsOutput, error)
    }
    ```

    Implement these functions:
    1. `EnsureLogGroup(ctx, client CWLogsAPI, logGroup, logStream string) error` — calls CreateLogGroup then CreateLogStream, ignores ResourceAlreadyExistsException for both.
    2. `PutLogEvents(ctx, client CWLogsAPI, logGroup, logStream string, events []LogEvent) error` — batches up to 10,000 events per PutLogEvents call; LogEvent is `{Timestamp int64; Message string}`.
    3. `GetLogEvents(ctx, client CWLogsAPI, logGroup, logStream string, limit int32) ([]LogEvent, error)` — calls GetLogEvents, returns slice.
    4. `TailLogs(ctx, client CWLogsAPI, logGroup, logStream string, follow bool, w io.Writer) error` — polls GetLogEvents every 2s if follow=true; prints each message to w; stops on ctx cancellation.

    Add dependency: `go get github.com/aws/aws-sdk-go-v2/service/cloudwatchlogs`

    Tests in `pkg/aws/cloudwatch_test.go` use `mockCWLogsAPI` struct implementing `CWLogsAPI`. Follow exact same pattern as `mockTagAPI` in `discover_test.go`.
