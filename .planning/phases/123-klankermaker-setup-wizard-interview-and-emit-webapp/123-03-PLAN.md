<objective>
Wave 3: make REAL km the arbiter of the wizard's output. Add a thin non-interactive
km seam that validates a km-config.yaml, generate one config fixture per preset via
the Wave-1 emitter, and wire a CI job that fails when the wizard's output drifts
from km's actual config schema.

Purpose: this is the anti-drift guarantee of approach #1. The wizard never
re-derives km's truth; CI proves the emitted config still loads cleanly against the
current km on every change to either side.
</objective>
