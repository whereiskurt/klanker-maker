setup wizard — the declarative question-graph (data) and the deterministic

contract test) build against a stable, deterministic core.

Output: `web/wizard/` TS package; `npm test` green with golden bundles for all
three presets; account-vend + quota emitted as forward-compatible placeholders.

<context>
Sub-project A of the self-setup-from-zero milestone. Approach #1 (declarative
question-graph + CI contract test) was chosen over reimplementing the interview
in JS. Two layers: gated foundation spine (accounts/auth/Org/email-DNS/quota) +
flat capability catalog (Slack, GitHub, H1, checks, desktop, GPU-serving). Only
two capabilities inject a slow gate — email (DNS) and GPU (quota); everything
else is domain-free. The emitted artifact is a gate-segmented script BUNDLE (one
part per human gate), readable and idempotent, NOT a single auto-run megascript —
because the from-zero journey has multi-day human gates (DNS propagation, quota
grants). See the design spec §§4–12 for full detail.

This wave is intentionally km-binary-free: correctness-against-real-km is Wave 3
(REQ-123-CONTRACT). Here, golden snapshots pin the emitter's shape.
</context>
