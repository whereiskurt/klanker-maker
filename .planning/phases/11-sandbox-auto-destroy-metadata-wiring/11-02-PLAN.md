<objective>
Wire TTL expiry and idle timeout to actually destroy sandbox resources instead of just uploading artifacts (TTL) or exiting the sidecar (idle).

Purpose: PROV-05 (TTL auto-destroy) and PROV-06 (idle auto-destroy) are unsatisfied because the TTL Lambda stops at artifact upload/notification and the idle detector only calls cancel(). This plan adds the missing destroy paths.
Output: TTL Lambda calls TeardownFunc to destroy resources via AWS SDK; idle sidecar publishes EventBridge event routed to the same Lambda; sandbox IAM roles include events:PutEvents.
</objective>

    2. In infra/modules/ttl-handler/v1.0.0/main.tf — ADD EventBridge rule routing SandboxIdle events to the TTL Lambda:
       - This is CRITICAL for PROV-06. Without this rule, SandboxIdle events published by the sidecar are silently dropped.
       ```hcl
       # ============================================================
       # EventBridge rule: route SandboxIdle events to TTL Lambda
       # ============================================================

       resource "aws_cloudwatch_event_rule" "sandbox_idle" {
         name        = "km-sandbox-idle"
         description = "Routes SandboxIdle events from audit-log sidecar to TTL Lambda for sandbox teardown"

         event_pattern = jsonencode({
           source      = ["km.sandbox"]
           detail-type = ["SandboxIdle"]
         })

         tags = {
           "km:component" = "ttl-handler"
           "km:managed"   = "true"
         }
       }

       resource "aws_cloudwatch_event_target" "idle_to_ttl" {
         rule      = aws_cloudwatch_event_rule.sandbox_idle.name
         target_id = "km-ttl-handler-idle"
         arn       = aws_lambda_function.ttl_handler.arn

         # Transform EventBridge envelope to match TTLEvent struct shape
         input_transformer {
           input_paths = {
             sandbox_id = "$.detail.sandbox_id"
           }
           input_template = <<-TEMPLATE
           {"sandbox_id": <sandbox_id>, "event_type": "idle"}
           TEMPLATE
         }
       }

       # Lambda permission: allow EventBridge Events to invoke the TTL Lambda (for idle events)
       resource "aws_lambda_permission" "eventbridge_events" {
         statement_id  = "AllowEventBridgeEventsInvoke"
         action        = "lambda:InvokeFunction"
         function_name = aws_lambda_function.ttl_handler.function_name
         principal     = "events.amazonaws.com"
         source_arn    = aws_cloudwatch_event_rule.sandbox_idle.arn
       }
       ```
       - Note: This is separate from the existing `aws_lambda_permission.eventbridge_scheduler` which allows the EventBridge Scheduler (different service) to invoke the Lambda for TTL schedule events.
