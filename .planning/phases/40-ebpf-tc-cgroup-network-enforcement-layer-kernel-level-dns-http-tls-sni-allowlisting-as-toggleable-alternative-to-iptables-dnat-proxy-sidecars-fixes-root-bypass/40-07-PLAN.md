Purpose: Without the generated files, `GOOS=linux go build ./...` fails with `undefined: bpfObjects`. The km binary cannot be built for the Linux target platform.
Output: Dockerfile for BPF compilation, updated Makefile target, committed generated .go and .o files for both pkg/ebpf/ and pkg/ebpf/sni/.

<task type="auto">
  <name>Task 1: Create Dockerfile and update Makefile generate-ebpf target</name>
  <files>Dockerfile.ebpf-generate, Makefile</files>
  <action>
Create `Dockerfile.ebpf-generate` at repo root. This is a build-tool image for running bpf2go inside Docker from macOS.

The Dockerfile should:
- Use `golang:1.25-bookworm` as base (matches project Go version 1.25.5)
- Install clang, llvm, libbpf-dev, linux-headers (for BPF target support)
- Install the cilium/ebpf/cmd/bpf2go tool at the version matching go.mod (v0.21.0): `go install github.com/cilium/ebpf/cmd/bpf2go@v0.21.0`
- Set WORKDIR to /src
- The image does NOT copy source — source is bind-mounted at runtime

Update the `generate-ebpf` target in Makefile to:
1. Build the Docker image (with --quiet, tagged `km-ebpf-generate:latest`)
2. Run `docker run --rm -v $(PWD):/src km-ebpf-generate:latest sh -c "cd /src/pkg/ebpf && go generate && cd /src/pkg/ebpf/sni && go generate"`
3. The generated files land in the host filesystem via the bind mount
4. Print a message listing the generated files

Keep the existing Makefile comment about generated files being committed. Do NOT make `build` depend on `generate-ebpf` — generated files are committed and only need regeneration when BPF C source changes.

Important: The go generate directives in gen.go reference `github.com/cilium/ebpf/cmd/bpf2go` which needs to be available inside the container. The Dockerfile must `go install` it. The container also needs access to go.mod/go.sum for module resolution, which comes via the bind mount.
  </action>
  <verify>
    <automated>docker build -q -f Dockerfile.ebpf-generate -t km-ebpf-generate:latest . 2>&1 | tail -1</automated>
  </verify>
  <done>Dockerfile.ebpf-generate exists with clang + bpf2go toolchain. Makefile generate-ebpf target uses Docker instead of bare `go generate`.</done>
