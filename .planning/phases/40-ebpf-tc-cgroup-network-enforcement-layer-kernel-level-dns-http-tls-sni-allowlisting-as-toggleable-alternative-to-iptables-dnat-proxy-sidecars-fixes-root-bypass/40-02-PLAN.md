<objective>
Create the Go-side enforcer that loads BPF programs, manages sandbox cgroups, attaches programs, and handles pin/unpin lifecycle.

Purpose: The enforcer is the bridge between the km CLI and kernel-side BPF programs. It creates a sandbox-specific cgroup (so BPF programs only affect sandbox processes, not the host), attaches all four BPF programs, populates volatile constants (proxy PID, port numbers, firewall mode), and pins everything to bpffs so enforcement survives the km process exiting. On destroy, it cleans up all pinned artifacts.

Output: `pkg/ebpf/enforcer.go`, `pkg/ebpf/cgroup.go`, `pkg/ebpf/pin.go` with full lifecycle management.
</objective>

**pkg/ebpf/cgroup.go** - Sandbox cgroup management:

`CgroupPath(sandboxID string) string` — returns `/sys/fs/cgroup/km.slice/km-{sandboxID}.scope`. Uses systemd-style cgroup naming under a dedicated `km.slice` parent.

`CreateSandboxCgroup(sandboxID string) (string, error)`:
1. Detect cgroup2 mount point by parsing `/proc/mounts` for `cgroup2` filesystem type (typically `/sys/fs/cgroup`)
2. Create directory `{cgroupMount}/km.slice/km-{sandboxID}.scope` via `os.MkdirAll`
3. Return the full path
4. Note: The sandbox user's shell process will be moved into this cgroup by writing its PID to `{cgroupPath}/cgroup.procs` during user-data bootstrap

`RemoveSandboxCgroup(sandboxID string) error`:
1. Remove the cgroup directory (must be empty of processes first)
2. Best-effort removal of parent km.slice if empty

**pkg/ebpf/enforcer.go** - Core enforcer:

```go
type Config struct {
    SandboxID       string
    DNSProxyPort    uint32  // port for km-dns-resolver (e.g. 5353)
    HTTPProxyPort   uint32  // port for HTTP proxy (e.g. 3128)
    HTTPSProxyPort  uint32  // port for HTTPS proxy (e.g. 3128)
    ProxyPID        uint32  // PID of proxy process (exempt from redirection)
    FirewallMode    uint16  // 0=log, 1=allow, 2=block
    MITMProxyAddr   uint32  // 127.0.0.1 in network byte order
}

type Enforcer struct {
    config      Config
    cgroupPath  string
    objs        *bpfObjects      // generated by bpf2go
    connectLink link.Link        // MUST be stored to prevent GC detachment
    sendmsgLink link.Link
    sockopsLink link.Link
    egressLink  link.Link
    pinPath     string           // /sys/fs/bpf/km/{sandbox-id}/
    closed      bool
}
```

`NewEnforcer(cfg Config) (*Enforcer, error)`:
1. Call `rlimit.RemoveMemlock()` for kernels < 5.11 compatibility (safe no-op on newer kernels)
2. Create sandbox cgroup via `CreateSandboxCgroup(cfg.SandboxID)`
3. Determine pin path: `/sys/fs/bpf/km/{sandboxID}/`
4. `os.MkdirAll(pinPath, 0700)`
5. Load BPF objects with pin path:
   ```go
   var objs bpfObjects
   err := loadBpfObjects(&objs, &ebpf.CollectionOptions{
       Maps: ebpf.MapOptions{PinPath: pinPath},
   })
   ```
6. Set volatile constants via `spec.Variables`:
   - Load spec first: `spec, err := loadBpf()` (the generated CollectionSpec loader)
   - `spec.Variables["const_dns_proxy_port"].Set(cfg.DNSProxyPort)`
   - `spec.Variables["const_proxy_pid"].Set(cfg.ProxyPID)`
   - `spec.Variables["const_http_proxy_port"].Set(cfg.HTTPProxyPort)`
   - `spec.Variables["const_https_proxy_port"].Set(cfg.HTTPSProxyPort)`
   - `spec.Variables["const_firewall_mode"].Set(cfg.FirewallMode)`
   - `spec.Variables["const_mitm_proxy_address"].Set(cfg.MITMProxyAddr)`
   - Then load from spec: `spec.LoadAndAssign(&objs, &ebpf.CollectionOptions{Maps: ebpf.MapOptions{PinPath: pinPath}})`
7. Attach all four programs to cgroup:
   ```go
   connectLink, _ := link.AttachCgroup(link.CgroupOptions{
       Path:    cgroupPath,
       Attach:  ebpf.AttachCGroupInet4Connect,
       Program: objs.Connect4,
   })
   ```
   - sendmsg4: `ebpf.AttachCGroupUDP4Sendmsg`
   - sockops: `ebpf.AttachCGroupSockOps`
   - egress: `ebpf.AttachCGroupInetEgress`
8. Pin all links: `connectLink.Pin(pinPath + "connect4_link")`, etc.
9. Store everything in Enforcer struct, return

`(e *Enforcer) AllowCIDR(cidr string) error` — parse CIDR, put into `allowed_cidrs` LPM_TRIE map.

`(e *Enforcer) AllowIP(ip net.IP) error` — put single IP (/32) into `allowed_cidrs`.

`(e *Enforcer) MarkForProxy(ip net.IP) error` — put IP into `http_proxy_ips` map.

`(e *Enforcer) Close() error` — unpin links, close links, close objs, remove cgroup. Set closed=true. Idempotent.

`(e *Enforcer) Events() *ebpf.Map` — returns the ring buffer map for userspace consumers.

**pkg/ebpf/pin.go** — Pin lifecycle management (all `//go:build linux`):

`PinPath(sandboxID string) string` — returns `/sys/fs/bpf/km/{sandboxID}/`.

`Cleanup(sandboxID string) error`:
1. Remove all files in PinPath directory (links + maps)
2. `os.RemoveAll(PinPath(sandboxID))`
3. Best-effort remove parent `/sys/fs/bpf/km/` if empty
4. Also call `RemoveSandboxCgroup(sandboxID)`

`RecoverPinned(sandboxID string) (*Enforcer, error)`:
1. Check if pin path exists — if not, return error "no pinned state"
2. Load pinned links:
   ```go
   connectLink, err := link.LoadPinnedLink(pinPath + "connect4_link", nil)
   sendmsgLink, err := link.LoadPinnedLink(pinPath + "sendmsg4_link", nil)
   sockopsLink, err := link.LoadPinnedLink(pinPath + "sockops_link", nil)
   egressLink, err := link.LoadPinnedLink(pinPath + "egress_link", nil)
   ```
3. Load pinned maps:
   ```go
   allowedCidrs, err := ebpf.LoadPinnedMap(pinPath + "allowed_cidrs", nil)
   ```
4. Reconstruct Enforcer with recovered handles
5. Return Enforcer (no Config needed — programs already running with their constants)

`IsPinned(sandboxID string) bool` — check if pin path exists and contains expected link files.

`ListPinned() ([]string, error)` — list sandbox IDs under `/sys/fs/bpf/km/` by listing subdirectories.
