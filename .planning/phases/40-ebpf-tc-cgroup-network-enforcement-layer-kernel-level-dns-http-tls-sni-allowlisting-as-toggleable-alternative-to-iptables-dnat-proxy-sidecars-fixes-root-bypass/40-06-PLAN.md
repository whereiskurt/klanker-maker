**pkg/ebpf/audit/audit.go** (`//go:build linux`):

```go
type Consumer struct {
    reader    *ringbuf.Reader
    sandboxID string
    logger    zerolog.Logger
}
```

`NewConsumer(eventsMap *ebpf.Map, sandboxID string, logger zerolog.Logger) (*Consumer, error)`:
1. Create `ringbuf.NewReader(eventsMap)`
2. Return Consumer

`(c *Consumer) Run(ctx context.Context) error`:
1. Loop reading from ring buffer (blocking `c.reader.Read()`)
2. On `ringbuf.ErrClosed` or ctx.Done: return nil
3. Deserialize `record.RawSample` into `bpfEvent` struct using `binary.Read` with `binary.LittleEndian`
4. Convert event to structured JSON log:
   ```json
   {
     "timestamp": "2026-03-31T12:00:00Z",
     "sandbox_id": "sb-abcd1234",
     "pid": 12345,
     "src_ip": "10.0.1.5",
     "dst_ip": "93.184.216.34",
     "dst_port": 443,
     "action": "deny",
     "layer": "connect4",
     "comm": "curl",
     "event_type": "ebpf_network_deny"
   }
   ```
5. Log via zerolog at Warn level for denies, Debug level for allows/redirects
6. IP addresses converted from uint32 network byte order to dotted notation via `net.IP`

`(c *Consumer) Close() error` — close ring buffer reader (unblocks Read())

Helper functions:
- `actionString(action uint8) string` — "deny", "allow", "redirect"
- `layerString(layer uint8) string` — "connect4", "sendmsg4", "egress_skb", "sockops"
- `uint32ToIP(ip uint32) net.IP` — convert network byte order uint32 to net.IP

**internal/app/cmd/ebpf_attach.go** — Cobra subcommand `km ebpf-attach`:

```go
var ebpfAttachCmd = &cobra.Command{
    Use:   "ebpf-attach",
    Short: "Attach eBPF network enforcement to sandbox cgroup",
    Long:  "Loads BPF programs, attaches to sandbox cgroup, starts DNS resolver, and runs audit consumer. Invoked by EC2 user-data bootstrap.",
    RunE:  runEbpfAttach,
    Hidden: true,  // internal command, not user-facing
}
```

Flags:
- `--sandbox-id` (required)
- `--dns-port` (default 5353)
- `--http-proxy-port` (default 3128)
- `--firewall-mode` (default "block"; values: log, allow, block)
- `--allowed-dns` (comma-separated DNS suffixes)
- `--allowed-hosts` (comma-separated hosts for L7 proxy)
- `--proxy-hosts` (comma-separated hosts needing L7 proxy)
- `--cgroup` (cgroup path, default auto-detected)

`runEbpfAttach(cmd *cobra.Command, args []string)`:
1. Parse flags
2. Create Enforcer via `ebpf.NewEnforcer(cfg)` with:
   - FirewallMode from --firewall-mode flag
   - ProxyPID = uint32(os.Getpid())
   - MITMProxyAddr = 127.0.0.1 in network byte order
3. Pre-populate initial CIDR allowlist:
   - 169.254.169.254/32 (IMDS)
   - VPC CIDR (10.0.0.0/8 or from IMDS)
4. Start DNS resolver via `resolver.NewResolver()` in goroutine
5. Start audit consumer via `audit.NewConsumer()` in goroutine
6. If --proxy-hosts specified, mark resolved IPs for L7 proxy
7. Wait for SIGTERM/SIGINT
8. On signal: close audit consumer, stop resolver, close enforcer (unpin handled by km destroy)

Register `ebpfAttachCmd` as subcommand of root command in the existing cmd registration pattern. Follow the same pattern as other internal/app/cmd/*.go files.

**pkg/ebpf/enforcer_integration_test.go** (`//go:build linux && integration`):

These are integration tests that require root on an EC2 instance. They use the `integration` build tag so they don't run in normal CI. They document and verify the security guarantees of EBPF-NET-12.

```go
// TestRootCannotBypassEBPF verifies that a root process inside the sandbox
// cgroup cannot bypass eBPF network enforcement.
//
// Prerequisites:
// - Must run as root on EC2 (AL2023, kernel 6.1+)
// - BPF programs must be loaded and attached to test cgroup
//
// Test scenarios:
// 1. iptables -F has no effect on eBPF enforcement
// 2. Direct connect() to blocked IP returns EPERM
// 3. Raw socket to blocked IP is dropped by cgroup_skb/egress
// 4. Cannot call bpf() syscall to detach programs (EPERM, no CAP_BPF)
// 5. DNS query for blocked domain returns NXDOMAIN
// 6. Hardcoded IP (bypassing DNS) is blocked by connect4 LPM_TRIE check
```

Test structure:
1. `TestRootIPTablesFlushIrrelevant`:
   - Create enforcer, attach to test cgroup
   - Allow 1.1.1.1, block everything else
   - Run `iptables -F -t nat` (this would break iptables DNAT enforcement)
   - Verify connect to blocked IP (93.184.216.34) from cgroup still returns EPERM
   - This proves eBPF enforcement is independent of iptables

2. `TestRootDirectConnectBlocked`:
   - Create enforcer with block mode
   - Attempt `net.Dial("tcp", "93.184.216.34:443")` from cgroup process
   - Verify error contains EPERM / connection refused

3. `TestRootCannotDetachBPF`:
   - Document that `bpf()` syscall requires CAP_BPF in the **init** (host) user namespace
   - Sandbox processes run in the host user namespace but without CAP_BPF
   - Even `CAP_NET_ADMIN` is insufficient to detach cgroup BPF programs
   - The test verifies by attempting `link.AttachCgroup()` from a non-privileged process in the cgroup and expecting EPERM

4. `TestDNSBlockedDomain`:
   - Configure allowlist with only ".github.com"
   - From cgroup process: resolve "evil.com" via DNS query
   - Verify NXDOMAIN response

5. `TestHardcodedIPBlocked`:
   - Do NOT resolve any domain
   - Attempt connect to a specific IP not in allowlist
   - Verify EPERM from connect4

Each test creates its own cgroup, attaches BPF programs, runs the test, and cleans up. Tests are independent and can run in any order.

Include detailed comments explaining WHY each test proves the security guarantee. These tests are documentation as much as verification.
