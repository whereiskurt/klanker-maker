<objective>
Create the TC egress classifier that inspects TLS ClientHello SNI as a best-effort defense-in-depth layer.

Purpose: The cgroup/connect4 program enforces at the connect() syscall level, but raw sockets and hardcoded IPs can bypass it. The cgroup_skb/egress program catches those by IP, but cannot inspect TLS hostnames. The TC SNI classifier adds hostname-level inspection for TLS connections, catching cases where a DNS-allowed IP serves multiple hostnames (SNI-based virtual hosting). This is explicitly best-effort: Chrome ClientHellos > 1500 bytes get TCP-segmented, and SNI may land in the 2nd segment where TC cannot reassemble.

Output: `pkg/ebpf/sni/` package with TC classifier BPF program and Go attachment code.
</objective>

**pkg/ebpf/sni/sni.c** — TC classifier for TLS SNI:

Maps:
- `allowed_sni`: BPF_MAP_TYPE_HASH, key=char[256] (null-padded hostname), value=u8(1), max_entries=4096. Pre-populated from profile allowlist.
- `sni_events`: BPF_MAP_TYPE_RINGBUF, max_entries=1<<20 (1MB, smaller than main ring buffer since SNI events are rare)

SEC("classifier/sni_filter") program:
1. Parse Ethernet header (if present, may not be on some TC hooks) -> IP header -> TCP header
2. Check if dest port == 443 (network byte order). If not, return TC_ACT_OK (pass).
3. Calculate TCP payload offset: eth_hdr_len + ip_hdr_len + tcp_hdr_len (use data_offset field * 4)
4. Check TCP payload starts with TLS ContentType 22 (Handshake), then TLS version (0x0301, 0x0303), then HandshakeType 1 (ClientHello)
5. Skip: TLS record header (5 bytes) + Handshake header (4 bytes) + ClientHello fixed fields (2 bytes version + 32 bytes random + session_id_len + session_id + cipher_suites_len + cipher_suites + compression_methods_len + compression_methods)
6. Parse extensions: loop through extensions looking for type 0x0000 (SNI)
7. Extract SNI hostname bytes (after SNI extension header: ext_type(2) + ext_len(2) + sni_list_len(2) + sni_type(1) + sni_len(2) + hostname)
8. Copy hostname into char[256] buffer, null-pad, lowercase it
9. Lookup in `allowed_sni` map:
   - Found: return TC_ACT_OK (pass)
   - Not found: emit event to sni_events ring buffer, return TC_ACT_SHOT (drop)
10. If ANY parsing fails (packet too short, not TLS, SNI not in first segment, extension loop exceeds bounds): return TC_ACT_OK (pass through — best-effort only)

Use bounded loops with `#pragma unroll` or explicit iteration limits for BPF verifier compliance. Maximum SNI parse depth: 512 bytes into TCP payload (covers most ClientHellos). Use `bpf_skb_load_bytes()` for safe packet reads.

**IMPORTANT:** AL2023 kernel 6.1 does NOT support TCX (needs 6.6+). Use classic TC cls_bpf via netlink. The section name `classifier/sni_filter` is correct for cls_bpf.

**pkg/ebpf/sni/sni.go** — Go-side attachment:

```go
type SNIFilter struct {
    objs    *sniObjects
    qdisc   netlink.Qdisc   // clsact qdisc handle
    filter  netlink.Filter   // TC filter handle
    ifIndex int
    pinPath string
}

type SNIConfig struct {
    SandboxID string
    Interface string   // e.g. "eth0"
    PinPath   string
}
```

`NewSNIFilter(cfg SNIConfig) (*SNIFilter, error)`:
1. Load BPF objects
2. Get interface index via `net.InterfaceByName(cfg.Interface)`
3. Add clsact qdisc to interface via netlink:
   ```go
   qdisc := &netlink.GenericQdisc{
       QdiscAttrs: netlink.QdiscAttrs{
           LinkIndex: ifIndex,
           Handle:    netlink.MakeHandle(0xffff, 0),
           Parent:    netlink.HANDLE_CLSACT,
       },
       QdiscType: "clsact",
   }
   netlink.QdiscAdd(qdisc)
   ```
4. Add TC filter on egress:
   ```go
   filter := &netlink.BpfFilter{
       FilterAttrs: netlink.FilterAttrs{
           LinkIndex: ifIndex,
           Parent:    netlink.HANDLE_MIN_EGRESS,
           Protocol:  unix.ETH_P_ALL,
       },
       Fd:           objs.SniFilter.FD(),
       DirectAction: true,
   }
   netlink.FilterAdd(filter)
   ```
5. Pin objects

`(f *SNIFilter) AllowSNI(hostname string) error` — lowercase, pad to 256 bytes, put in `allowed_sni` map.

`(f *SNIFilter) Close() error` — remove TC filter, remove qdisc, unpin, close objects.

Uses `github.com/vishvananda/netlink` for TC/qdisc management (add to go.mod).
