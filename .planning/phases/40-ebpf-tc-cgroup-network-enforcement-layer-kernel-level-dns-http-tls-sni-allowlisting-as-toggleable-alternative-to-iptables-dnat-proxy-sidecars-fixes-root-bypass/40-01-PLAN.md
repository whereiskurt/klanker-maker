<objective>
Create the `pkg/ebpf/` package scaffold with BPF C programs, map definitions, and bpf2go code generation pipeline.

Purpose: This is the foundation for all eBPF network enforcement. The BPF C programs define the kernel-side behavior: connect4 intercepts TCP connections, sendmsg4 redirects DNS, sockops bridges source ports to socket cookies for the transparent proxy, and cgroup_skb/egress provides packet-level defense-in-depth. All seven BPF maps are defined here. The bpf2go pipeline compiles these into Go-embeddable bytecode.

Output: `pkg/ebpf/` package with compilable BPF C programs, generated Go loader code, and Go type definitions.
</objective>

Create `pkg/ebpf/` directory structure.

**pkg/ebpf/headers/common.h** - Shared definitions:
- `struct event` with fields: `u64 timestamp`, `u32 pid`, `u32 src_ip`, `u32 dst_ip`, `u16 dst_port`, `u8 action` (0=deny, 1=allow, 2=redirect), `u8 layer` (1=connect4, 2=sendmsg4, 3=egress_skb, 4=sockops), `u8 comm[16]`
- `struct ip4_trie_key` with `u32 prefixlen` and `u8 addr[4]` (for LPM_TRIE)
- Volatile const declarations: `const_dns_proxy_port` (u32), `const_proxy_pid` (u32), `const_http_proxy_port` (u32), `const_https_proxy_port` (u32), `const_firewall_mode` (u16: 0=log, 1=allow, 2=block), `const_mitm_proxy_address` (u32, default 127.0.0.1)
- Helper macros for `bpf_htonl`/`bpf_ntohl` (network byte order)

**pkg/ebpf/headers/bpf_helpers.h** - Minimal BPF helper declarations (from libbpf headers). Include `bpf_get_current_pid_tgid`, `bpf_get_socket_cookie`, `bpf_ringbuf_reserve`, `bpf_ringbuf_submit`, `bpf_probe_read_kernel`, `bpf_ktime_get_ns`, `bpf_get_current_comm`, `bpf_skb_load_bytes`. Use `__attribute__((section("...")))` macro `SEC()`.

**pkg/ebpf/headers/vmlinux.h** - Minimal vmlinux type subset needed for CO-RE. Include: `struct bpf_sock_addr` (for connect4/sendmsg4), `struct __sk_buff` (for cgroup_skb), `struct bpf_sock_ops` (for sockops). Use BTF-defined types (`__u8`, `__u16`, `__u32`, `__u64`, `__be16`, `__be32`). This is NOT the full vmlinux.h — only the types our programs reference, following the pattern from cilium/ebpf examples.

**pkg/ebpf/bpf.c** - All BPF programs in a single C file (bpf2go compiles one file):

1. **Seven BPF maps** (SEC(".maps")):
   - `events`: RINGBUF, max_entries 1<<24 (16MB)
   - `allowed_cidrs`: LPM_TRIE, key=ip4_trie_key, value=u32, max_entries=65535, BPF_F_NO_PREALLOC (MANDATORY for LPM_TRIE)
   - `http_proxy_ips`: HASH, key=u32(IP), value=u32(IP), max_entries=256K — IPs needing L7 proxy inspection
   - `sock_to_original_ip`: HASH, key=u64(socket_cookie), value=u32(IP), max_entries=256K
   - `sock_to_original_port`: HASH, key=u64(socket_cookie), value=u16(port), max_entries=256K
   - `src_port_to_sock`: HASH, key=u16(src_port), value=u64(socket_cookie), max_entries=256K
   - `socket_pid_map`: HASH, key=u64(socket_cookie), value=u32(PID), max_entries=10K

2. **emit_event()** helper: reserves ring buffer slot, populates event struct with `bpf_ktime_get_ns()`, `bpf_get_current_pid_tgid() >> 32`, `bpf_get_current_comm()`, submits.

3. **SEC("cgroup/connect4") connect4** (EBPF-NET-02, EBPF-NET-03):
   - Get PID, check if PID == const_proxy_pid -> return 1 (exempt proxy from redirection)
   - Check if dest IP == 169.254.169.254 (IMDS) -> return 1 (exempt)
   - Check if dest IP == 127.0.0.0/8 (localhost) -> return 1 (exempt)
   - Store PID in socket_pid_map keyed by bpf_get_socket_cookie()
   - Lookup dest IP in `allowed_cidrs` LPM_TRIE:
     - NOT found AND const_firewall_mode == 2 (block): emit deny event, return 0 (EPERM)
     - NOT found AND const_firewall_mode == 1 (allow): emit deny event, return 1
     - NOT found AND const_firewall_mode == 0 (log): emit deny event, return 1
   - If found, check if dest IP is in `http_proxy_ips` (needs L7 inspection):
     - Yes: store original IP/port in sock_to_original_ip/port keyed by socket cookie, rewrite ctx->user_ip4 to const_mitm_proxy_address, rewrite ctx->user_port to const_http_proxy_port or const_https_proxy_port based on original port
   - Return 1 (allow)

4. **SEC("cgroup/sendmsg4") sendmsg4** (EBPF-NET-04):
   - Check if PID == const_proxy_pid -> return 1 (exempt)
   - Check if dest port != 53 -> return 1 (only intercept DNS)
   - Store original IP/port via socket cookie
   - Rewrite ctx->user_ip4 to 127.0.0.1, ctx->user_port to const_dns_proxy_port
   - Return 1

5. **SEC("sockops") sockops** (bridge for transparent proxy):
   - Check for BPF_SOCK_OPS_ACTIVE_ESTABLISHED_CB
   - Map local_port -> socket_cookie in `src_port_to_sock`

6. **SEC("cgroup_skb/egress") egress_filter** (EBPF-NET-06):
   - Parse IP header from skb (bpf_skb_load_bytes)
   - Check if dest IP == 169.254.169.254 -> return 1 (IMDS exempt)
   - Check if dest IP == 127.0.0.0/8 -> return 1 (localhost exempt)
   - Lookup dest IP in `allowed_cidrs` LPM_TRIE
   - NOT found AND const_firewall_mode == 2: emit deny event, return 0 (drop)
   - Return 1 (allow)

Use `__attribute__((always_inline))` for helper functions. Use network byte order consistently with `bpf_htonl`/`bpf_ntohl`. All IPs in maps are stored in network byte order.

**pkg/ebpf/gen.go** - The bpf2go code generation directive:
```go
package ebpf

//go:generate go run github.com/cilium/ebpf/cmd/bpf2go -tags linux -type event bpf bpf.c -- -I./headers -O2 -g -Wall -Werror
```
The `-type event` flag generates a Go struct `bpfEvent` matching the C `struct event` for type-safe ring buffer deserialization. The `-tags linux` ensures generated code only compiles on Linux. The `-O2 -g` flags optimize and include debug info. `-Wall -Werror` catch BPF verifier issues at compile time.

**pkg/ebpf/types.go** - Go-side type definitions:
```go
package ebpf

// LpmKey is the Go representation of struct ip4_trie_key for LPM_TRIE lookups.
// PrefixLen is 0-32 for IPv4. Addr is in network byte order (big-endian).
type LpmKey struct {
    PrefixLen uint32
    Addr      [4]byte
}

// Event action constants matching BPF-side values.
const (
    ActionDeny     = 0
    ActionAllow    = 1
    ActionRedirect = 2
)

// Event layer constants matching BPF-side values.
const (
    LayerConnect4  = 1
    LayerSendmsg4  = 2
    LayerEgressSKB = 3
    LayerSockops   = 4
)

// FirewallMode constants for volatile const_firewall_mode.
const (
    ModeLog   = 0
    ModeAllow = 1
    ModeBlock = 2
)
```

Add build constraint `//go:build linux` to both gen.go and types.go since eBPF is Linux-only.
