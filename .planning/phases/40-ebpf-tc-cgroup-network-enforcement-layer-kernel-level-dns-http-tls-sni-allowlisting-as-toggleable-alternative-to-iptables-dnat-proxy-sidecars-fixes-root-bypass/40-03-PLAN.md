<objective>
Create the userspace DNS resolver daemon that receives DNS queries redirected by BPF sendmsg4, enforces the domain allowlist, resolves allowed domains, and pushes resolved IPs into the BPF LPM_TRIE map.

Purpose: BPF programs handle L3/L4 enforcement but cannot evaluate domain names at the kernel level. The DNS resolver daemon is the bridge: when a sandbox process tries to resolve a domain, BPF redirects the DNS query to this daemon. The daemon checks the domain against the profile's allowlist, resolves allowed domains via upstream DNS, and adds the resulting IPs to the BPF allowlist map. This is the Cilium FQDN model adapted for sandbox enforcement.

Output: `pkg/ebpf/resolver/` package with DNS server, domain allowlist, and IP-to-BPF-map population.
</objective>

func IsAllowed(name string, suffixes []string) bool {
    name = strings.TrimSuffix(name, ".")
    name = strings.ToLower(name)

}

**pkg/ebpf/resolver/resolver.go** — DNS resolver daemon:

```go
type ResolverConfig struct {
    ListenAddr      string        // e.g. "127.0.0.1:5353"
    UpstreamAddr    string        // e.g. "169.254.169.253:53" (VPC DNS)
    AllowedSuffixes []string      // from profile
    SandboxID       string        // for logging
    MapUpdater      MapUpdater    // interface to push IPs into BPF map
    ProxyHosts      []string      // hosts that need L7 proxy (GitHub, Bedrock)
}

// MapUpdater is the interface for updating BPF maps from resolved IPs.
// Enforcer implements this. Allows testing without BPF.
type MapUpdater interface {
    AllowIP(ip net.IP) error
    MarkForProxy(ip net.IP) error
}
```

`NewResolver(cfg ResolverConfig) *Resolver`

`(r *Resolver) Start(ctx context.Context) error`:
1. Create miekg/dns server on cfg.ListenAddr (UDP + TCP)
2. Handler function:
   a. Extract query domain from question
   b. Check allowlist: `r.allowlist.IsAllowed(domain)`
   c. If denied: return NXDOMAIN, log denial
   d. If allowed:
      - Forward query to upstream resolver
      - Extract A record IPs from response
      - Refuse AAAA queries (return empty answer, NOERROR) — IPv6 blocked
      - For each resolved IP: call `r.cfg.MapUpdater.AllowIP(ip)`
      - If domain matches ProxyHosts: also call `r.cfg.MapUpdater.MarkForProxy(ip)`
      - Track resolved IPs with TTL from DNS response: `r.allowlist.AddResolved(domain, ips, ttl)`
      - Return DNS response to client
3. Start background goroutine for TTL sweep (every 30 seconds, call `r.allowlist.Sweep()`)
4. Block on ctx.Done()

`(r *Resolver) Stop() error` — shutdown DNS server gracefully.

Use `github.com/miekg/dns` which is already a dependency (used by sidecars/dns-proxy). Use `github.com/rs/zerolog` for structured logging (also already a dependency).

Key differences from the existing `sidecars/dns-proxy`:
1. Runs in-process with km (not a separate sidecar binary)
2. Pushes resolved IPs into BPF maps (the sidecar doesn't do this)
3. Uses MapUpdater interface for BPF integration
4. Has TTL-based IP expiry (the sidecar doesn't track IPs)
5. Marks proxy-needed hosts (the sidecar doesn't distinguish)
