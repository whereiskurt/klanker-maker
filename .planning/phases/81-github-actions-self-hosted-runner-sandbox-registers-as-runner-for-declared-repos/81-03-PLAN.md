<objective>
Build the sandbox-side binary that systemd template units (`klanker-actions-runner-register@.service`,
`klanker-actions-runner-shutdown.service`) invoke at boot and at shutdown. This is the
operational counterpart to Plan 81-02's Lambda: where the Lambda mints tokens, this
sidecar receives them and feeds them to the official `actions/runner` `config.sh` script.

Purpose: Keep the heavy lifting (App JWT, installation-token exchange, DDB writes) in
the Lambda. The sidecar's job is to bridge from systemd unit invocation to (1) a sigv4
Lambda call (2) running config.sh with the returned token (3) reading back the runner_id.
This split keeps the GitHub App private key off the sandbox and respects the Phase 67
slack-bridge precedent for sandbox→Lambda sigv4.

Note: The systemd template units themselves (klanker-actions-runner-register@.service,
klanker-actions-runner@.service, klanker-actions-runner-shutdown.service) are emitted by
the compiler userdata template in Plan 81-04 — that plan owns RUNNER-SYSTEMD-UNITS. This
plan produces the binary that the units' ExecStart lines invoke; it does not itself
write any unit-file content.

Output: A static-linked Linux/amd64 binary deployable via the existing sidecar Makefile
target. Plan 81-04 wires this binary into systemd template units written by the
userdata template, and Plan 81-05 (compiler wiring) makes `km init --sidecars` upload it.
</objective>

type RunnerTokenResponse struct {

Lambda Function URL endpoint (from Plan 81-02 outputs):
- URL pattern: `https://<random>.lambda-url.<region>.on.aws/`
- AuthType: AWS_IAM → request must be sigv4-signed as `lambda` service
- Method: POST
- Body: `{"sandbox_id":"...","action":"register","repos":["my-org/repo-a"]}` (JSON)

config.sh signature (from actions/runner installer — preinstalled in /opt/km/actions-runner-template/):
```bash
./config.sh --url https://github.com/<repo> \
            --token <reg-token> \
            --name "klanker-<sandbox-id>-<repo-slug>" \
            --labels "self-hosted,klanker,klanker-<sandbox-id>,klanker-<alias>,<user-labels>" \
            --unattended --replace
./config.sh remove --token <removal-token>
```

After successful config.sh, the runner directory contains `.runner` — a JSON file with `{"agentId":N, "agentName":"...", ...}`. Sidecar reads it to extract runner_id.
