<objective>
Wire the profile-schema additions (Plan 81-01), the Lambda (Plan 81-02), and the sidecar
binary (Plan 81-03) into the userdata that the compiler emits when a sandbox is created.
This is the integration plan — it stitches the three Wave-1 outputs into a coherent
boot sequence. Every previous sidecar phase (63 / 67 / 68 / 73 / 79) followed exactly
this same pattern: heredoc systemd unit + binary download + EnvironmentFile bridge.

Purpose: When an operator runs `km create profile.yaml` with `runner.enabled:true`, the
resulting EC2 instance must, on first boot, automatically (1) download the
km-actions-runner binary from S3, (2) write two systemd template-unit files into
/etc/systemd/system, (3) write /etc/km/runner-repos.json listing the repos, (4) for
each repo, `systemctl enable klanker-actions-runner-register@<slug>.service`, and (5)
have the network allowlist already extended so config.sh can reach
*.actions.githubusercontent.com.

Output: A compiler change set that:
- Adds an Actions-runner section to `pkg/compiler/userdata.go`, gated on
  `p.Spec.SourceAccess.GitHub.Runner != nil && p.Spec.SourceAccess.GitHub.Runner.Enabled`
- Adds an allowlist-injection helper in `pkg/compiler/security.go`
- Adds a per-sandbox IAM session-policy statement granting `lambda:InvokeFunctionUrl`
  on `arn:aws:lambda:<region>:<acct>:function:${resource_prefix}-actions-runner-token`
- Adds compiler unit tests verifying every conditional fragment
</objective>

    2. Add ActionsRunnerExtraDNSSuffixes constant and a helper:
       ```go
       // ActionsRunnerExtraDNSSuffixes is the DNS allowlist the actions/runner binary needs
       // beyond api.github.com + github.com. Reference: design spec § Network policy.
       var ActionsRunnerExtraDNSSuffixes = []string{
           ".actions.githubusercontent.com",
           ".pkg.actions.githubusercontent.com",
           "objects.githubusercontent.com",
           "codeload.github.com",
       }

       // mergeDNSAllowSuffixes returns base concatenated with extras, de-duplicated, preserving order.
       func mergeDNSAllowSuffixes(base, extras []string) []string {
           seen := make(map[string]bool, len(base)+len(extras))
           out := make([]string, 0, len(base)+len(extras))
           for _, s := range base {
               if !seen[s] {
                   seen[s] = true
                   out = append(out, s)
               }
           }
           for _, s := range extras {
               if !seen[s] {
                   seen[s] = true
                   out = append(out, s)
               }
           }
           return out
       }

       // EffectiveDNSAllowSuffixes returns the effective DNS allowlist after Phase 81 auto-injection.
       // Callers should use this instead of p.Spec.Network.Egress.AllowedDNSSuffixes directly.
       func EffectiveDNSAllowSuffixes(p *profile.SandboxProfile) []string {
           base := p.Spec.Network.Egress.AllowedDNSSuffixes
           if p.Spec.SourceAccess.GitHub != nil &&
              p.Spec.SourceAccess.GitHub.Runner != nil &&
              p.Spec.SourceAccess.GitHub.Runner.Enabled {
               return mergeDNSAllowSuffixes(base, ActionsRunnerExtraDNSSuffixes)
           }
           return base
       }
       ```
