<objective>
Implement the operator-facing CLI surface (`km runner status|reattach|detach`) and wire
runner-aware logic into the existing `km create` and `km destroy` flows. This is the
human-touch layer — everything below this plan is autonomous (Lambda + sidecar +
systemd); this plan is what the operator types at their workstation.

Purpose: Without this plan, runners would register at boot and never be visible to
operators, and `km destroy` would leak ghost runners on GitHub's side. The CLI plus
destroy-time DELETE path closes both gaps. The km create injection (KM_ACTIONS_RUNNER_TOKEN_URL
into runner.env on the running instance) is the operational equivalent of Phase 67's
slack-bridge URL injection — same SSM SendCommand pattern, same visible-stderr lesson
from SLCK-11.

Output: Three new Cobra subcommands under `km runner`, a teardown hook in `km destroy`
that always invokes the Lambda's DELETE action per repo before terragrunt destroy
begins, and a create-time SSM SendCommand that pushes the Lambda Function URL into the
sandbox's runtime env file so systemd units can read it. All paths emit visible
success + failure messages on stderr (no zerolog → io.Discard silencing, per CLAUDE.md
SLCK-11 lesson).
</objective>

       ```go
       package cmd

       import (
           "context"
           "fmt"
           "os"

           "github.com/aws/aws-sdk-go-v2/aws"
           kmaws "github.com/whereiskurt/klanker-maker/pkg/aws"
       )

       // RunnerLambdaInvoker is the same interface declared in runner.go — reuse via single source.
       // (If go has no forward declarations, put the interface in a shared file e.g. runner_types.go.)

       // destroyActionsRunners is the belt-and-suspenders DELETE path. For each repo where the
       // sandbox registered as a runner, calls the actions-runner-token Lambda with action=delete
       // (operator-side, sigv4 with operator's AWS creds). Failures are WARN-logged but do not
       // propagate — the SSM RunCommand to the sandbox already attempted local cleanup; this
       // call catches the case where SSM was unreachable (sandbox dead, network partition).
       func destroyActionsRunners(
           ctx context.Context,
           awsCfg aws.Config,
           meta *kmaws.SandboxMetadata,
           invoker RunnerLambdaInvoker,
       ) {
           if meta == nil || len(meta.ActionsRunners) == 0 {
               return
           }
           for repo, info := range meta.ActionsRunners {
               if info.RunnerID == 0 {
                   continue
               }
               _, err := invoker.Invoke(ctx, meta.SandboxID, "delete", repo, info.RunnerID)
               if err != nil {
                   fmt.Fprintf(os.Stderr, "⚠ Actions runner: failed to delete runner %d for %s: %v\n", info.RunnerID, repo, err)
                   continue
               }
               fmt.Fprintf(os.Stderr, "✓ Actions runner: deleted %s runner %d from GitHub\n", repo, info.RunnerID)
           }
       }
       ```
