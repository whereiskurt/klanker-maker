<objective>
Close out Phase 81 with operational hardening: doctor checks for the four failure modes
documented in the spec, a presence-daemon extension so active runner jobs do not trigger
idle-pause, and end-user documentation in docs/github.md plus CLAUDE.md. This plan
contains a checkpoint at the end where the operator performs the GitHub App permission
re-install (cannot be automated — GitHub's UI is the only path) and runs the migration
sequence.

Purpose: A feature that ships without doctor checks is invisible to operations — broken
runners would silently fail until a workflow run failed and someone investigated.
Without presence signal #6, a long-running runner job (e.g. integration tests, large
builds) could trigger the idle-timeout pause path and kill its own host mid-job.
Without docs and CLAUDE.md updates, the feature is undiscoverable for any operator who
did not write it.

Output: A green km doctor run for a sandbox with runner.enabled set, once the operator
has performed the GitHub App permission re-install (one-time manual step, documented
in docs/github.md and signalled by the actions_runner_app_perms check). Phase 81 is
then ready to declare complete via SUMMARY.md.
</objective>
