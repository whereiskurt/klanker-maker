<objective>
Build the centralised token-minting Lambda + its Terraform module + its live terragrunt
unit. The Lambda is the only component holding the GitHub App private key — sandboxes
call it via sigv4-signed Function URL invocations to mint per-repo registration tokens,
removal tokens, and to issue DELETE/LIST on runners. This is the operator-side teardown
load-bearing piece (belt-and-suspenders DELETE in `km destroy` Plan 81-04 calls this
Lambda).

Purpose: Sandboxes must never hold long-lived GitHub credentials. The App private key
stays in SSM, the Lambda mints short-lived per-repo tokens on demand, and AWS IAM
(`lambda:InvokeFunctionUrl` on the specific ARN) gates who can call. Matches Phase 67
slack-bridge's auth model — sigv4 from sandbox role → Lambda Function URL with AuthType:
AWS_IAM. Lambda's IAM role is a strict reuse pattern from the existing
`infra/modules/github-token/v1.0.0/main.tf` — same SSM access + KMS decrypt + CW Logs.

Output: A deployable Lambda module that exposes a Function URL. Plan 81-03 will give the
sandbox instance role `lambda:InvokeFunctionUrl` on this Lambda's ARN. Plan 81-04 will
hit this URL from the operator workstation for the belt-and-suspenders DELETE.

The module also writes the function_url output to SSM at
`/${resource_prefix}/actions-runner-token/function-url` so that Plans 81-05/81-06
(which run on the operator workstation and inside doctor checks) have a single
source-of-truth discovery path — no terragrunt-output fallback needed.
</objective>
