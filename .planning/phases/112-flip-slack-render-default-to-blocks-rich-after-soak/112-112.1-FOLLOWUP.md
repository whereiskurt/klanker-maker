# Phase 112.1 — FOLLOWUP: reconcile the Slack inbound steer prompt with `blocks-rich`

**Status:** complete (2026-06-22)
**Type:** in-place follow-up to Phase 112 (small, in-scope extension of a shipped phase — not a new GSD phase)

## Problem

When a sandbox replied to a Slack @-mention (after the bridge posted the 👀 ACK),
the agent kept emitting flat, monospace replies — no native tables, no headings —
even though Phase 112 made `--render blocks-rich` (Tier-3: native `markdown` +
`table` blocks) the default for the inbound-poller reply path.

The render path was correct; the **agent's instructions** were not. The Slack
inbound poller hands Claude an `--append-system-prompt` (`pkg/compiler/userdata.go`,
the `.SlackInboundEnabled` block) that — written in the pre-Phase-111 Tier-2 era,
when Slack had no native table support — told the agent to:

- *"Wrap … any tabular or columnar data in triple-backtick fenced blocks so they
  render in monospace."*
- *"Prefer short bullet lists over wide Markdown tables…"*

`pkg/slack/rich.go` only upgrades **raw** GFM pipe-tables (≥2 consecutive pipe
lines, *outside* a code fence) into native `table` blocks, and promotes leading
Markdown headings into `header`/`markdown` blocks. Fencing a table keeps it
verbatim monospace (`inCodeFence` guard), and "prefer bullets over tables" means
the agent rarely emits a GFM table at all — so the steer actively defeated the
Phase 112 renderer. Phase 111/112 added the rich renderer but nobody reconciled
this steer prompt.

(Note: the steer is Claude-only — the Codex inbound paths have no
`--append-system-prompt` equivalent and were already unaffected.)

## Change

Rewrote the single `--append-system-prompt` steer in `pkg/compiler/userdata.go`
to embrace `blocks-rich`:

- Tell the agent Slack renders GFM natively (headings, bold, italic, lists, links,
  tables).
- Use `##`/`###` headings + bold + bullet lists for structure.
- Present tabular/columnar data as a **real GFM pipe table**, and explicitly do
  **NOT** fence tables.
- Reserve triple-backtick fences for content that must stay verbatim (code, file
  paths, commands, shell output, logs) — unchanged intent for actual code.
