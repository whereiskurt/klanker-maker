<feature>
  <name>Local Number State Management</name>
  <files>pkg/localnumber/localnumber.go, pkg/localnumber/localnumber_test.go</files>
  <behavior>
    State type: `{"next": int, "map": {"sandbox-id": number}}`

    StateFilePath:
    - Returns `os.UserConfigDir()/km/local-numbers.json`
    - Falls back to `os.UserHomeDir()/.config/km/local-numbers.json` if UserConfigDir fails

    Load:
    - Missing file -> returns `&State{Next: 1, Map: {}}`, nil
    - Corrupt JSON -> returns `&State{Next: 1, Map: {}}`, nil (fresh start)
    - Valid JSON -> returns parsed state
    - Nil map in JSON -> initialized to empty map
    - Next < 1 in JSON -> corrected to 1

    Save:
    - Creates parent directory `~/.config/km/` if missing (MkdirAll with 0o700)
    - Writes JSON to `path.tmp` then `os.Rename` to `path` (atomic)
    - File permissions 0o600

    Assign(state, sandboxID) -> int:
    - If sandboxID already in map, returns existing number (idempotent)
    - Otherwise assigns state.Next, increments state.Next, returns assigned number
    - Initializes nil map if needed

    Remove(state, sandboxID):
    - Deletes sandboxID from map
    - If map is now empty, resets Next to 1
    - No-op if sandboxID not in map

    Resolve(state, num) -> (string, bool):
    - Returns sandboxID for given number, true
    - Returns "", false if no sandbox has that number

    Reconcile(state, liveSandboxIDs []string):
    - Prunes map entries whose IDs are not in liveSandboxIDs
    - Assigns numbers (via Assign) to liveSandboxIDs not in map
    - If map is empty after pruning (and no live IDs), resets Next to 1

    Test cases (use t.TempDir() for isolation -- override StateFilePath via a testable helper or by testing Load/Save with explicit paths):
    - TestAssign: fresh state assigns 1, second assign gets 2, re-assign same ID returns original
    - TestRemove: remove entry, verify gone; remove last entry, verify Next resets to 1
    - TestResolve: resolve existing number succeeds; resolve nonexistent returns false
    - TestReconcile: prunes stale, assigns new, resets when all pruned
    - TestLoad: missing file returns empty state; corrupt file returns empty state; valid file parsed
    - TestSave: creates parent dir, atomic write (file exists after save), content matches expected JSON
  </behavior>
  <implementation>
    Implement per the code examples in 53-RESEARCH.md. Key decisions:

    1. Use `os.UserConfigDir()` for XDG compliance (per locked decision: `~/.config/km/local-numbers.json`)
    2. Atomic writes via tmp+rename pattern (per locked decision and codebase precedent in ebpf_attach.go)

    4. All functions are pure (operate on `*State` pointer) except Load/Save which do I/O
    5. `json.MarshalIndent` with 2-space indent for human readability
  </implementation>
</feature>
