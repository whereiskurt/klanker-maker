## Decisions Made

- `register_oidc_provider` defaults to `true` — zero-impact upgrade for all existing Phase 80 stacks
- `moved {}` block placed before `locals {}` — Terraform state migration resolves before plan diff evaluation
- `data.tls_certificate.oidc` gated with same count as the resource — no unnecessary TLS handshake when not creating the provider
- `data.aws_iam_openid_connect_provider.existing` references `local.oidc_provider_url` (not `var.oidc_provider_arn`) to satisfy Terraform 1.6.x count=0 data source evaluation safety
