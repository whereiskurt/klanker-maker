```markdown
## Same-account installation

When your EKS cluster is in the **same AWS account** as your klanker install, the OIDC
provider for that cluster is typically already registered in IAM (by `eksctl`, the AWS
Console, or a Terraform EKS module). Running `km cluster add` against a cluster in
the same account will fail with `EntityAlreadyExists` unless you tell klanker to reuse
the existing provider.

**Phase 80.1 handles this automatically.** `km cluster add` calls
`aws iam list-open-id-connect-providers` before provisioning and detects whether the
cluster's OIDC issuer URL is already registered. When it is, the module sets
`register_oidc_provider = false` and references the existing provider via a Terraform
data source instead of creating a new one. The role's trust policy still uses the
correct OIDC provider ARN as its Federated principal.

### Manual override

If auto-detect produces the wrong result (e.g. you want to force-create a new provider),
use the explicit flag:

```bash
# Force create a new OIDC provider (fails if one already exists for the same URL)
km cluster add --name my-cluster --oidc-provider-arn <arn> --register-oidc-provider=true

# Force reuse an existing OIDC provider (skips auto-detect, fails if none exists)
km cluster add --name my-cluster --oidc-provider-arn <arn> --register-oidc-provider=false
```

### km cluster rm behavior

When `register_oidc_provider = false` (reuse path), `km cluster rm` destroys only the IAM
role — it does NOT destroy the pre-existing OIDC provider, which is owned by the EKS
cluster provisioning toolchain (eksctl / Terraform EKS module / Console).
```

**Scenario A: Same-account (register=false path) — the primary Phase 80.1 use case**

```bash
make build

# Step 1: Pre-register an OIDC provider in klanker-application (simulate same-account EKS)
ACCT=$(aws sts get-caller-identity --profile klanker-application --query Account --output text)
# Pick any well-formed issuer URL (does NOT need to be a real EKS cluster)
ISSUER_URL="https://oidc.eks.us-east-1.amazonaws.com/id/PHASE801SMOKETEST"
PROV_ARN=$(aws iam create-open-id-connect-provider \
  --url "$ISSUER_URL" \
  --client-id-list sts.amazonaws.com \
  --thumbprint-list "0000000000000000000000000000000000000000" \
  --profile klanker-application \
  --query OpenIDConnectProviderArn \
  --output text)
echo "Pre-registered OIDC provider: $PROV_ARN"

# Step 2: Run km cluster add (auto-detect should detect and reuse)
km cluster add \
  --name phase801-smoke \
  --oidc-provider-arn "arn:aws:iam::${ACCT}:oidc-provider/oidc.eks.us-east-1.amazonaws.com/id/PHASE801SMOKETEST" \
  --namespace default \
  --service-account km \
  --dry-run=false
```

- `OIDC provider auto-detected: reusing existing arn:aws:iam::...:oidc-provider/oidc.eks.us-east-1.amazonaws.com/id/PHASE801SMOKETEST`
- `terragrunt apply` completes without `EntityAlreadyExists`
- `Cluster "phase801-smoke" provisioned: arn:aws:iam::...:role/...`
