## Decisions Made

- Used the simpler single-statement `Resource: "*"` form (as the plan recommended) rather than the two-statement split (`oidc-provider/*` + `*`). This matches the existing `kms` and `ses_send` policies in the same file. `iam:ListOpenIDConnectProviders` does not support resource-level restrictions per IAM documentation, making a split statement functionally identical but more verbose.
- Added `iam:UntagOpenIDConnectProvider` alongside the 5 plan-specified actions. Terraform may call it during destroy of a tagged provider — omitting it would cause a non-obvious AccessDenied on teardown.

## Deviations from Plan

None - plan executed exactly as written. The plan explicitly offered the single-statement form as preferred; that was used.
