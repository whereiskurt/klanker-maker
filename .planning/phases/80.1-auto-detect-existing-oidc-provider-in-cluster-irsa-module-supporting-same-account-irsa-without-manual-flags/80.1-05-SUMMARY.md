## Integration test results

| Scenario | Behavior | Outcome |
|---|---|---|
| **A — Same-account reuse** | Pre-registered an OIDC provider for issuer URL `oidc.eks.us-east-1.amazonaws.com/id/PHASE801SMOKETEST`, then ran `km cluster add --name phase801-smoke --oidc-provider-arn arn:aws:iam::052251888500:oidc-provider/oidc.eks.us-east-1.amazonaws.com/id/PHASE801SMOKETEST` | Auto-detect logged `reusing existing arn:aws:iam::052251888500:oidc-provider/oidc.eks.us-east-1.amazonaws.com/id/PHASE801SMOKETEST`. | **B — Reuse path: rm preserves pre-existing provider** | Ran `km cluster rm phase801-smoke --dry-run=false` against the Scenario A stack | Cluster destroyed (IAM role removed; `aws iam get-role` returns `NoSuchEntity`). Pre-existing OIDC provider STILL EXISTS post-rm. ✓ |
| **C — Cross-account create** | Ran `km cluster add --name phase801-cross --oidc-provider-arn arn:aws:iam::123456789012:oidc-provider/oidc.eks.us-east-1.amazonaws.com/id/PHASE801NEWISSUER` (issuer URL never seen before in 052251888500) | Auto-detect logged `creating`. Apply succeeded. Locally-registered provider was created. `km cluster rm` destroyed both the role AND the owned provider. ✓ |
| **D — Idempotency** | Re-ran the Scenario C `km cluster add` command after the first apply succeeded | Returned `Cluster "phase801-cross" already registered: arn:...:role/km-cluster-phase801-cross` and exited 0 (no terragrunt call). ✓ |

Final IAM state: zero `PHASE801*` artifacts remain. Both test stacks cleanly destroyed.
