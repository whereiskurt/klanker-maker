```

km init --dry-run=false  # apply Terraform module changes (defaults to --dry-run=true; explicit false required)
```

<safety_rationale>
This plan is `autonomous: false` because:
- `km init --dry-run=false` performs Terraform applies that mutate shared platform infra (SES, email-handler, ECS-substrate modules).
- An accidental apply with an unexpected recreate could trigger a brief inbound-mail outage (SES rule set) or break ECS task SSM access.
- Per CLAUDE.md "Executing actions with care," operator review of the dry-run plan output is mandatory before commit.

The operator checkpoint task pauses execution and requires the operator to:
1. Read the dry-run plan output.
2. Confirm zero `must be replaced` lines.
3. Confirm SES rule set shows zero diff.
4. Confirm tag-only `~` updates on sandbox-creating resources.
5. Type "approved" to proceed to the apply task.
</safety_rationale>
