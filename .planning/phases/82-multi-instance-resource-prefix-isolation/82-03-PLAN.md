<objective>
Stamp every operator-baked AMI with the install-discriminator tag `km:resource-prefix=${prefix}` at bake time, so Plan 04's doctor tag filter can cleanly separate one install's AMIs from a neighboring install's.

Purpose: D1 in CONTEXT.md `<decisions>` — without this tag, `ListBakedAMIs` returns BOTH installs' AMIs, and downstream `km doctor` cleanup logic risks flagging a foreign install's AMIs as orphans eligible for deletion.

Output: `KMBakeTags` accepts `resourcePrefix` as a new parameter and includes the new tag; the existing call site in `internal/app/cmd` passes `cfg.GetResourcePrefix()`; the existing unit test asserts the new tag is present.

This plan is intentionally scoped to *writing* the tag at bake time. Reading the tag (filtering `ListBakedAMIs` and `checkOrphanedEC2`) is Plan 04's job — it depends on this plan because the tag-write must land first.
</objective>
