<objective>
Make `doctor` and `ListBakedAMIs` respect the new `km:resource-prefix` tag from Plan 03, so a second install's `km doctor` cannot accidentally classify the `km` install's resources as orphans (and vice versa).

Purpose: Close the doctor cross-install destruction risk (D1 in CONTEXT.md). Without this filter, `ListBakedAMIs` and `checkOrphanedEC2` both filter only by `km:sandbox-id=*` — which matches EVERY install's resources.

Output: Two surfaces updated:
1. `pkg/aws/ec2_ami.go` `ListBakedAMIs` gains a prefix filter parameter (functional option or string).
2. `internal/app/cmd/doctor.go` `checkOrphanedEC2` reads the tag, skips foreign-prefix instances, WARNs on untagged.

Depends on Plan 03 (which writes the tag at bake time) — without that, this plan's reads return empty / cause false WARNs on every existing AMI.
</objective>
