<objective>
Extend `km shell --learn` so it observes AWS API calls and emits the inferred allowlist into the generated profile. Per `69-CONTEXT.md` (locked decision): on `aws_api_allowed` or `aws_api_blocked` events from non-platform UIDs, deduplicate the service slug into the generated profile's `spec.sourceAccess.aws.allowlist`. The generated profile sets `inspection: observe` (not `enforce`) so the operator reviews the inferred list before flipping it to enforce.

`aws_api_platform` and `aws_api_unsigned` events are intentionally NOT collected:
- `aws_api_platform`: those calls come from system users; the gate already exempts them. Adding them to the allowlist would be redundant noise.
- `aws_api_unsigned`: no service slug is available (the call had no SigV4 signature). Out of scope for v1 enforcement per 69-CONTEXT.md.

```go
// RecordAWSService deduplicates an AWS SigV4 service slug into the recorder's
// AWS service set. Phase 69. Mirrors RecordHost/RecordRepo (mutex-guarded
// map-of-empty-struct pattern).
func (r *Recorder) RecordAWSService(slug string) {
    if slug == "" {
        return
    }
    r.mu.Lock()
    defer r.mu.Unlock()
    r.awsServices[slug] = struct{}{}
}

// AWSServices returns the deduplicated, alphabetized list of recorded AWS
// service slugs.
func (r *Recorder) AWSServices() []string {
    r.mu.Lock()
    defer r.mu.Unlock()
    out := make([]string, 0, len(r.awsServices))
    for s := range r.awsServices {
        out = append(out, s)
    }
    sort.Strings(out)
    return out
}
```
