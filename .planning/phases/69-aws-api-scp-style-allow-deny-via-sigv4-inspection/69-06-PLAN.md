**Design rationale (do not skip).** Loading `sock_to_uid` inside the existing `loadMaps()` sync.Once would break ALL transparent proxy traffic on sandboxes where the bpf.o has not yet been refreshed to pin `sock_to_uid` (or where inspection is off, since Plan 69-02 only pins the new map when the eBPF program is loaded with the Phase 69 additions). 3. Add a NEW lazy loader, `loadUIDMap()`, with its own sync.Once. Mirror the existing `loadMaps` shape but check pin presence before loading:

```go
// loadUIDMap loads the Phase 69 sock_to_uid pinned map on first call. Safe to
// call from any goroutine. Subsequent calls reuse the cached result.
//
// Returns errUIDMapAbsent (a non-fatal sentinel) when the pin file does not
// exist — this is expected on sandboxes that have not been redeployed with the
// Phase 69 eBPF program. Callers MUST treat this sentinel as "uid unknown" and
// proceed with proxy traffic as normal; the AWS allowlist gate handles the
// no-platform-bypass case via platformUIDUnknown.
//
// Any other error (load failure on a present pin file) is returned verbatim
// and also treated as "uid unknown" by GetCallerUID's caller in proxy.go.
func (tl *TransparentListener) loadUIDMap() error {
    tl.uidMu.Do(func() {
        pin := tl.mapDir + "sock_to_uid"
        if _, err := os.Stat(pin); err != nil {
            if os.IsNotExist(err) {
                tl.uidMapErr = errUIDMapAbsent
                return
            }
            tl.uidMapErr = fmt.Errorf("stat sock_to_uid pin: %w", err)
            return
        }
        m, err := ebpf.LoadPinnedMap(pin, nil)
        if err != nil {
            tl.uidMapErr = fmt.Errorf("load sock_to_uid: %w", err)
            return
        }
        tl.sockToUID = m
    })
    return tl.uidMapErr
}
```

4. Add the `GetCallerUID` method. It calls `loadUIDMap()` first (NOT `loadMaps()` — sock_to_uid is independent of the transparent proxy maps), then `loadMaps()` only if needed for the portToSock lookup. Both errors are returned to the caller; proxy.go treats any non-nil error as platformUIDUnknown.

```go
// GetCallerUID returns the UID of the process that opened the connection
// identified by the supplied peer (source) port. Chains the existing
// src_port_to_sock map (loaded by loadMaps) with the Phase 69 sock_to_uid
// map (loaded by loadUIDMap).
//
// Returns errUIDMapAbsent when sock_to_uid is not pinned (sandbox without
// Phase 69 eBPF bits — graceful degradation: callers MUST fall back to
// platformUIDUnknown so the AWS gate applies but the proxy remains operational
// for non-AWS traffic).
//
// Returns other errors when the lookup fails post-load. proxy.go's OnRequest
// closure also treats these as platformUIDUnknown — never bubbles them up
// as proxy-blocking failures.
func (tl *TransparentListener) GetCallerUID(peerPort uint16) (uint32, error) {
    if err := tl.loadUIDMap(); err != nil {
        return 0, err // includes errUIDMapAbsent
    }
    if err := tl.loadMaps(); err != nil {
        return 0, err
    }
    var cookie uint64
    if err := tl.portToSock.Lookup(&peerPort, &cookie); err != nil {
        return 0, fmt.Errorf("port→cookie lookup: %w", err)
    }
    var uid uint32
    if err := tl.sockToUID.Lookup(&cookie, &uid); err != nil {
        return 0, fmt.Errorf("cookie→uid lookup: %w", err)
    }
    return uid, nil
}
```

// is bypassed and an aws_api_platform event is emitted instead.

// maxUID of 0 is normalized to 1000 (default boundary between system users and
// the sandbox user). Phase 69.
func WithAWSPlatformUIDLookup(uidFunc func(remotePort uint16) (uint32, error), maxUID uint32) ProxyOption {

        if maxUID == 0 {

        cfg.awsPlatformUIDMax = maxUID

}
