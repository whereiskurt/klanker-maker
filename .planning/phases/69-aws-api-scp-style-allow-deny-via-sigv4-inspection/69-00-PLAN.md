Plain-text file, one example per logical block. Format: `# label`\n`<value>`\n. Plan 03's parser tests will iterate this file and assert expected service+region per label. The format below is canonical AWS SigV4 per the SDK reference docs.

```

AWS4-HMAC-SHA256 Credential=AKIAIOSFODNN7EXAMPLE/20260504/us-east-1/s3/aws4_request, SignedHeaders=host;x-amz-date, Signature=fe5f80f77d5fa3beca038a248ff027d0445342fe2855ddc963176630326f1024
# sts-frankfurt
AWS4-HMAC-SHA256 Credential=AKIAIOSFODNN7EXAMPLE/20260504/eu-central-1/sts/aws4_request, SignedHeaders=host;x-amz-date, Signature=abc
# dynamodb-tokyo
AWS4-HMAC-SHA256 Credential=AKIAIOSFODNN7EXAMPLE/20260504/ap-northeast-1/dynamodb/aws4_request, SignedHeaders=host;x-amz-date;x-amz-target, Signature=def
# bedrock-runtime-virginia
AWS4-HMAC-SHA256 Credential=AKIAIOSFODNN7EXAMPLE/20260504/us-east-1/bedrock-runtime/aws4_request, SignedHeaders=host;x-amz-date, Signature=ghi
# sigv4a-multiregion
AWS4-ECDSA-P256-SHA256 Credential=AKIAIOSFODNN7EXAMPLE/20260504/*/s3/aws4_request, SignedHeaders=host;x-amz-date;x-amz-region-set, Signature=jkl
# query-presigned-s3
X-Amz-Credential=AKIAIOSFODNN7EXAMPLE%2F20260504%2Fus-east-1%2Fs3%2Faws4_request
# query-presigned-sts
X-Amz-Credential=AKIAIOSFODNN7EXAMPLE/20260504/us-west-2/sts/aws4_request
# malformed-too-few-parts
AWS4-HMAC-SHA256 Credential=AKIAIOSFODNN7EXAMPLE/20260504/us-east-1, SignedHeaders=host, Signature=xyz
```

Note line 14 (`malformed-too-few-parts`) intentionally has only 3 credential-scope parts and must produce `found=false` from the parser — Plan 03's tests assert this.
