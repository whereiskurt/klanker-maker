# Phase 60.1 ‚Äî Record pausedAt for all EC2 stop paths, not just hibernation

## Origin

Surfaced 2026-05-31 during a sanity-check of Phase 60 against the TTL-handler
stop path:

> "I want to make sure stopped/paused machines aren't running compute budget‚Ä¶
> sometimes a TTL will fire from lambda and stop the box etc."

Phase 60 (`calculateComputeCost` subtracts `pausedSeconds + (now - pausedAt)`)
is internally consistent ‚Äî but the **producer side** has a gap. `RecordPauseStart`
fires only when the EC2 was actually hibernated, not when it was stopped via the
default plain-stop path. That leaves the budget enforcer ticking wall-clock
spend against a stopped box that is incurring $0 AWS compute.

## The gap

Two call sites guard `RecordPauseStart` behind a `hibernate==true` (or
`status=="paused"`) check that doesn't match the real-cost semantics:

### 1. `cmd/ttl-handler/main.go` ‚Äî TTL-fired `handleStop`

```go
hibernate := h.lookupHibernation(ctx, event.SandboxID)  // L267 ‚Äî defaults false
...
if hibernate {
    StopInstances(..., Hibernate: true)   // L297-302
} else {
    StopInstances(...)                    // L314 ‚Äî plain stop
}
...
status := "stopped"
if hibernate { status = "paused" }        // L333
if status == "paused" && h.BudgetClient != nil && h.BudgetTable != "" {
    RecordPauseStart(...)                 // L338 ‚Äî GATED on hibernate
}
```

`Spec.Runtime.Hibernation` is `bool` with `omitempty` (`pkg/profile/types.go:181-182`).
Default is `false`. Spot instances are skipped entirely at `main.go:292-295`. So
any profile without an explicit `runtime.hibernation: true` takes the `else`
branch and **never writes `pausedAt`**. The budget enforcer keeps firing every
minute (it only self-deletes when the sandbox is *destroyed*, not stopped ‚Äî
`budget-enforcer/main.go:174-181`) and recomputes
`billableSecs = elapsedSecs - 0 = elapsedSecs` for the entire stopped window.

### 2. `internal/app/cmd/stop.go` ‚Äî `km stop` CLI

`runStop` calls `StopInstances` at line 134 and updates DynamoDB status to
`"stopped"` at line 149 but **never calls `RecordPauseStart` at all** ‚Äî not
gated, just absent. `km pause` (which writes `pausedAt` at `pause.go:187`) and
`km stop` had drifted out of sync.

## Why the resume side is already correct

All three resume paths call `RecordResumeClose` unconditionally:

- `internal/app/cmd/resume.go:123`
- `cmd/ttl-handler/main.go:408` (handleResume)
- `cmd/ttl-handler/main.go:596` (handleAgentRun auto-start)

`RecordResumeClose` is a documented safe no-op when `pausedAt` is absent
(Phase 60 verification truth #4, `pkg/aws/budget.go:194-199`). So legacy
sandboxes that were stopped pre-60.1 (no `pausedAt` written) won't
double-credit on resume ‚Äî they just resume normally with no interval to close.

## What ships

### Code change #1 ‚Äî `cmd/ttl-handler/main.go:338`

Drop the `status == "paused"` half of the gate. The remaining gate
(`h.BudgetClient != nil && h.BudgetTable != ""`) preserves the test-time
"BudgetClient absent" safety. The function is already EC2-only by
construction (spot is skipped, Docker/EKS don't reach `handleStop`).

```diff
-if status == "paused" && h.BudgetClient != nil && h.BudgetTable != "" {
+if h.BudgetClient != nil && h.BudgetTable != "" {
     if err := awspkg.RecordPauseStart(ctx, h.BudgetClient, h.BudgetTable, event.SandboxID, time.Now().UTC()); err != nil {
         log.Warn().Err(err).Str("sandbox_id", event.SandboxID).Msg("failed to record pause start in budget table (non-fatal)")
     }
 }
```

The comment at L336-337 (`"Record pause start in budget table so paused
intervals are excluded from cost. EC2 substrate only (handleStop is EC2-only
by construction). Non-fatal."`) stays accurate ‚Äî only the hibernate
discrimination is removed.

### Code change #2 ‚Äî `internal/app/cmd/stop.go`

Mirror `runPause`'s exact pattern (`pause.go:132-135, 187`): derive
`budgetTable` from `cfg`, call `RecordPauseForEC2` after the stop loop
succeeds, before the metadata update. Same non-fatal warn semantic.

```diff
+budgetTable := cfg.GetResourcePrefix() + "-budgets"
+if cfg.BudgetTableName != "" {
+    budgetTable = cfg.BudgetTableName
+}
+
 ec2Client := ec2.NewFromConfig(awsCfg)
 ...
 if stopped == 0 {
     return fmt.Errorf("no running instances found for sandbox %s", sandboxID)
 }

+// Record pause start so the budget enforcer doesn't tick wall-clock spend
+// against the stopped box. Non-fatal.
+RecordPauseForEC2(ctx, dynamoClient, budgetTable, sandboxID)
+
 // Update metadata status to "stopped" ...
```

Re-uses the existing `RecordPauseForEC2` helper from `pause.go:28-34`.

- `cmd/ttl-handler/main_test.go` ‚Äî no existing test for `handleStop` (the

## Deployment

```sh
make build                       # km binary + Lambda zips rebuild
km init --dry-run=false          # deploys ttl-handler Lambda zip via terragrunt
```

The fix touches:
- `cmd/ttl-handler/main.go` ‚Üí ttl-handler Lambda zip ‚Üí requires `km init`
  (the `--sidecars` flag rebuilds binaries but does NOT re-upload Lambda zips,
  per the existing `project_km_init_lambdas_doesnt_deploy` memory).
- `internal/app/cmd/stop.go` ‚Üí operator-side `km` binary only; no deploy
  needed beyond `make build`.

Existing stopped sandboxes don't get retro-pausedAt; the fix only affects
*new* stop events post-deploy. If a sandbox is currently stopped without
pausedAt and you want to seal it, run `km resume <id> && km stop <id>` (or
`km pause`) post-upgrade.

## Edge cases

- **Idempotency on double-stop.** `RecordPauseStart` uses
  `SET pausedAt = if_not_exists(pausedAt, :now)` (`pkg/aws/budget.go:165`).
  If the budget-enforcer auto-stop fires *after* a TTL stop, the pausedAt
  from the TTL stop is preserved. No double-pause skew.

- **Stop-while-already-paused.** Same `if_not_exists` guard. A TTL stop on
  an already-paused box (status race) preserves the original pausedAt.

- **BudgetClient/BudgetTable absent (test mode).** Existing
  `h.BudgetClient != nil && h.BudgetTable != ""` gate preserved. Unit tests
  with empty handler state continue to work.

- **Spot instances.** Unchanged ‚Äî TTL handleStop skips spot at L292-295,
  `km pause` rejects spot at `pause.go:155-159` with a clear error. Spot
  was never reached by the previous gate either.
