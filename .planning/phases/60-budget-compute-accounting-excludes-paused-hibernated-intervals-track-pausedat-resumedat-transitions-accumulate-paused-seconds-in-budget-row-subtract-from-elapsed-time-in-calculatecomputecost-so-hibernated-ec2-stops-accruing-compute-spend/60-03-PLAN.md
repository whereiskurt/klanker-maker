<objective>
Teach the budget-enforcer Lambda to honor paused-interval accounting: subtract accumulated pause time from elapsed before computing compute cost, and handle the current open interval (sandbox is paused right now) by extending `pausedSeconds` with `now - PausedAt` on the fly. Also wire the budget-enforcer's own pause path so exhausting the budget stops the clock (not just the EC2 instance).

Purpose: This plan is where the bug actually gets fixed â€” paused sandboxes stop accruing compute spend. Plan 01 gave us the primitives; Plan 02 wrote `pausedAt` on every external transition; this plan is the consumer side.

Output:
- `calculateComputeCost(event, pausedSeconds int64) (float64, error)` â€” signature change with clamp
- `HandleBudgetCheck` computes `effectivePaused = PausedSeconds + (PausedAt != nil ? now - PausedAt : 0)` before calling `calculateComputeCost`
- `enforceBudgetCompute` calls `awspkg.RecordPauseStart` after StopInstances succeeds so the Lambda does not re-accumulate elapsed time on its next tick
- Test suite covering paused-subtraction, zero-unchanged, open-interval-returns-zero, never-negative, multi-cycle
</objective>
