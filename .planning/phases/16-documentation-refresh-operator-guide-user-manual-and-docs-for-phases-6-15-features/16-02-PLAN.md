3. **AI budget dual-layer enforcement** — Enhance existing enforcement section. Layer 1: http-proxy returns 403 for Bedrock calls immediately (real-time). Explain why dual-layer: proxy catches SDK calls through the proxy, IAM revocation catches SDK/CLI calls that bypass the proxy.

4. **Top-up flow details** — Enhance existing top-up section. For AI: restores Bedrock IAM permissions + proxy unblocks. For compute EC2: calls StartInstances (resumes from stopped state). For compute ECS: re-provisions Fargate task from stored profile in S3. Show the `km budget add` command and what happens at each layer.

   - What SCPs are and why they exist (account-level backstop, prevents breakout even if IAM role is compromised)
   - Each deny statement: DenyEC2BreakoutActions (RunInstances, CreateVpc, etc.), DenyIAMEscalation (CreateRole, AttachRolePolicy, etc.), DenySSMBreakout, DenyOrganizationsDiscovery, DenyRegionOutsideAllowed

   - DenyOrganizationsDiscovery has no condition (applies to all roles)
   - Management account exempt by AWS design

2. **GitHub App Token Security** — Read pkg/github/. Add a section documenting:
   - Short-lived installation tokens (not PATs or SSH keys)
   - Repo-scoped permissions (only repos listed in profile sourceAccess.github.allowed_repos)
   - Token stored in SSM Parameter Store (not environment variables)
   - GIT_ASKPASS reads token at git time (no long-lived credential in memory)
   - Lambda refreshes token every 45 minutes (before 1-hour GitHub expiry)
   - KMS encryption of the private key in SSM

3. **Sandbox Identity and Email Signing** — Read pkg/identity/. Add a section documenting:
   - Ed25519 key pair generated per sandbox at creation time

   - Private key stored in SSM Parameter Store (KMS encrypted)
   - Email signing: body signed with Ed25519, X-KM-Signature header
   - Email verification: receiver fetches sender's public key from DynamoDB
   - Optional encryption: X25519 key exchange from Ed25519 keys, NaCl box.SealAnonymous
   - Sender identity: X-KM-Sender-ID header (not in ciphertext)
   - Profile controls: spec.email.signing, spec.email.verifyInbound, spec.email.encryption
