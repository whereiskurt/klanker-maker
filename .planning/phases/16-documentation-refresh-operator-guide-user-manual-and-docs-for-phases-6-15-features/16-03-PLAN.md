Read the full current docs/multi-agent-email.md. Add the following sections covering Phase 14 features. Read pkg/identity/ and pkg/ses/ source code for accuracy.

1. **Signed Email** — Add a section documenting the email signing protocol:
   - SignEmailBody signs body only (not headers) for simplicity and transit resilience
   - X-KM-Signature header: base64-encoded Ed25519 signature of the email body
   - X-KM-Sender-ID header: sender's sandbox ID for public key lookup
   - SendSignedEmail uses Content.Raw (not Content.Simple) because SES Simple strips custom X-KM-* headers
   - Verification flow: receiver reads X-KM-Sender-ID, fetches public key from DynamoDB km-identities table, verifies X-KM-Signature against body
   - Show example email headers with X-KM-* fields

2. **Optional Encryption** — Add a section documenting the encryption protocol:
   - X25519 key exchange derived from Ed25519 keys (Curve25519 conversion)
   - NaCl box.SealAnonymous for encryption, box.OpenAnonymous for decryption
   - Sender identity is in X-KM-Sender-ID header, not embedded in ciphertext
   - X-KM-Encrypted header indicates encrypted payload
   - DynamoDB public key discovery: sender fetches receiver's public key to encrypt
   - Show example flow: sandbox A encrypts message to sandbox B

3. **Profile spec.email Controls** — Add a section documenting profile-driven email policy:
   - `spec.email.signing` (boolean): enables Ed25519 signing of outbound email
   - `spec.email.verifyInbound` (boolean): verify signatures on inbound email, reject unsigned
   - `spec.email.encryption` (boolean): enables NaCl encrypted email
   - EmailSpec is a pointer on Spec (nil means not specified, defaults apply)
   - Show example profile YAML snippet with spec.email section
   - Note how built-in profiles handle email: hardened/sealed vs open-dev/restricted-dev

Update the Table of Contents to include new sections.
