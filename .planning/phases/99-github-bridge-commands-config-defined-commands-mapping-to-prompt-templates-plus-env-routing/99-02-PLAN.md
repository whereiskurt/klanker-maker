Parsing rules (design spec ┬з"Parsing rules", CONTEXT D3├ЧD5├ЧD6):
1. StripCode: remove fenced ``` blocks AND `inline code` before scanning.
2. Scan whitespace-bounded tokens matching ^/[A-Za-z][A-Za-z0-9_-]*$ (single segment;
   /usr/bin/patch fails тЖТ not a candidate). Use strings.Fields + prefix/segment check,
   NOT a heavy regex/lexer (RESEARCH "Don't Hand-Roll").
3. Match candidates тЖТ distinct KNOWN commands (defined in the map). /help recognized
   FIRST as a reserved built-in BEFORE the defined-command lookup (RESEARCH Pitfall 4).
   >1 distinct known тЖТ multi-command error (report the offending names). Repeats of the
   same command dedupe. 0 known тЖТ caller applies effective default else free-form.
4. ExtractArgs(body, botLogin, commandToken): strip the FIRST @mention occurrence and the
   FIRST /commandToken occurrence by position (NOT ReplaceAll тАФ RESEARCH Pitfall 5),
   whitespace-normalize. When commandToken == "" (default-command path), strip only the mention.
5. ExpandTemplate(template, args): strings.ReplaceAll(template, "{{args}}", args); if the
   template has no "{{args}}", append args on a new line.

Routing (D2): alias = command.alias || repo.alias; profile = command.profile || repo.profile || default_profile.
