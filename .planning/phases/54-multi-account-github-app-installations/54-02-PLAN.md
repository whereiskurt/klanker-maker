<objective>
Update `generateAndStoreGitHubToken` in create.go to resolve the installation ID from the repo owner rather than using the single global installation-id. Extract the owner from `sourceAccess.github.allowedRepos` entries (format: `owner/repo`), look up `/km/config/github/installations/{owner}`, and fall back to the legacy key when per-account is not found.

Purpose: At sandbox create time, automatically select the correct GitHub App installation based on which account owns the repos in the profile.
Output: Updated create.go with owner-aware installation resolution; updated github_token_hcl.go to pass resolved installation_id; tests.
</objective>

  <files>internal/app/cmd/create.go, internal/app/cmd/create_test.go</files>
  <behavior>
    - Test: extractRepoOwner("userA/my-repo") returns "userA"
    - Test: extractRepoOwner("orgB/other-repo") returns "orgB"
    - Test: extractRepoOwner("bare-repo") returns "" (no owner)
    - Test: extractRepoOwner("*") returns "" (wildcard)
    - Test: resolveInstallationID with repos ["userA/repo1"] looks up /km/config/github/installations/userA first
    - Test: resolveInstallationID falls back to legacy /km/config/github/installation-id when per-account key not found

    - Test: resolveInstallationID with mixed owners (should not happen in v1) uses first owner found
    - Test: resolveInstallationID with only bare repos or wildcard falls back to legacy key
  </behavior>
  <action>
    1. Add helper function `extractRepoOwner(repo string) string` that splits on "/" and returns the first segment, or empty string for bare repos and wildcards ("*").

    2. Add helper function `resolveInstallationID(ctx context.Context, ssmClient SSMGetPutAPI, allowedRepos []string) (string, error)` that:
       a. Extracts unique owners from allowedRepos using extractRepoOwner
       b. For the first non-empty owner found, attempts GetParameter at `/km/config/github/installations/{owner}`
       c. If found, returns that installation ID
       d. If not found (ParameterNotFound), falls back to legacy `/km/config/github/installation-id`
       e. If legacy also not found, returns ErrGitHubNotConfigured
       f. If no owners extracted (all bare/wildcard), goes directly to legacy key

    3. Refactor `generateAndStoreGitHubToken` (line ~1776) to replace the direct read of `/km/config/github/installation-id` with a call to `resolveInstallationID(ctx, ssmClient, allowedRepos)`. The resolved installation ID is then used in the existing `ExchangeForInstallationToken` call.

    4. Write unit tests in create_test.go using the existing mock SSM pattern. Test: per-account found, per-account missing (fallback to legacy), both missing (ErrGitHubNotConfigured), bare repos (legacy only), wildcard repos (legacy only).
  </action>
