    1. List the AMI via DescribeImages (single image filter) to confirm existence and capture snapshot IDs.
    2. Run profile refcount scan.
    3. If refcount > 0 AND --force not set: print "AMI <id> is referenced by <N> profile(s):" + list of paths; exit non-zero with helpful message ("rerun with --force to override").
    4. If --dry-run: print "would delete AMI <id> + N snapshots: <ids>"; exit 0 without calling DeleteAMI.
    5. If --yes not set: print confirmation prompt "Delete AMI <id> and <N> associated snapshots? [y/N]:"; abort on anything but y/Y/yes.
    6. Call `aws.DeleteAMI(ctx, client, amiID, false)` (dryRun=false in real path).
    7. Print "deleted AMI <id> (and N snapshots)".

    `km ami bake` flow:
    1. Resolve sandbox ID via `ResolveSandboxID(ctx, cfg, args[0])`.
    2. Fetch DynamoDB record via `fetcher.FetchSandbox(ctx, sandboxID)`.
    3. Extract instance ID, region, profile name, alias, instance type from the record.
    4. Call shared `bakeFromSandbox(ctx, cfg, rec, sandboxID, profileName)` — returns `(amiID string, err error)`.
    5. Print resulting AMI ID + tag set on success.
