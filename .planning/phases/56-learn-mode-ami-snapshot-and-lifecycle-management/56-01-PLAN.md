From the SDK (already pinned):

```go
// CreateImage — supports atomic tagging via TagSpecifications.
type CreateImageInput struct {
    InstanceId        *string
    Name              *string
    Description       *string
    NoReboot          *bool
    TagSpecifications []types.TagSpecification // ResourceTypeImage + ResourceTypeSnapshot
}
type CreateImageOutput struct {
    ImageId *string // returned immediately; AMI in "pending" state
}

// DescribeImages — NOT paginated when Owners=["self"].
type DescribeImagesInput struct {
    Owners  []string
    Filters []types.Filter
    ImageIds []string
}
type DescribeImagesOutput struct {
    Images []types.Image // includes BlockDeviceMappings, Tags, State, CreationDate
}

// DeregisterImage — DeleteAssociatedSnapshots cleans up EBS snapshots atomically.
type DeregisterImageInput struct {
    ImageId                   *string
    DeleteAssociatedSnapshots *bool // shared snapshots silently skipped (intentional)
    DryRun                    *bool
}

// CopyImage — does NOT carry tags. Re-tag in destination region required.
type CopyImageInput struct {
    SourceImageId *string
    SourceRegion  *string
    Name          *string
    Description   *string
}
type CopyImageOutput struct {
    ImageId *string // new AMI in destination region; "pending" until copy completes
}

// Waiter — preferred over manual sleep loops.
ec2.NewImageAvailableWaiter(client) // satisfies ec2.DescribeImagesAPIClient
```

  <behavior>
    - `EC2AMIAPI` interface exposes exactly: CreateImage, DescribeImages, DeregisterImage, CopyImage, CreateTags (the five operations needed across all Wave 2 consumers).
    - `KMBakeTags(sandboxID, profileName, alias, instanceType, sourceRegion string) []types.Tag` returns a Tag slice with: km:sandbox-id, km:profile, km:alias (skipped/empty value if alias is ""), km:baked-at (RFC3339 UTC), km:source-region, km:instance-type, km:baked-by=km, km:km-version (from runtime.Version OR a Version global), Name (=AMIName output).
    - `AMIName(profileName, sandboxID string, t time.Time) string` returns `km-{sanitized-profile}-{sandboxID}-{YYYYMMDDHHMMSS}` UTC. Sanitizer regex `[^a-zA-Z0-9._/-]` → `-`. Length capped at 128 chars (truncate sanitized profile if needed). Handles empty profileName (uses literal "profile" placeholder). MUST NOT use colons in name.
    - `BakeAMI(ctx, client EC2AMIAPI, instanceID, amiName, description string, tags []types.Tag, waitTimeout time.Duration) (string, error)`:
      - Calls CreateImage with NoReboot=true, TagSpecifications carrying tags on BOTH ResourceTypeImage and ResourceTypeSnapshot.
      - Returns the ImageId immediately on CreateImage error; otherwise emits a `[ami] snapshot started: <id>` line to stderr.
      - Calls `ec2.NewImageAvailableWaiter(client.(ec2.DescribeImagesAPIClient))` and Wait()s up to waitTimeout (default 15 min if zero). Returns ImageId + nil on success. On wait failure, returns ImageId + wrapped error so caller can decide whether to clean up.
      - All error returns wrap with `fmt.Errorf("create image %s: %w", instanceID, err)` style.
    - `ListBakedAMIs(ctx, client EC2AMIAPI) ([]types.Image, error)`:
      - Calls DescribeImages with `Owners=["self"]` AND filter `tag:km:sandbox-id=*` AND `state=available`.
      - Returns images sorted newest first by `CreationDate` (string compare works because format is `YYYY-MM-DDThh:mm:ss.SSSZ`).
      - No pagination loop (DescribeImages is not paginated for self-owned images per SDK).
    - `DeleteAMI(ctx, client EC2AMIAPI, amiID string, dryRun bool) error`:
      - Calls DeregisterImage with `DeleteAssociatedSnapshots=true` and `DryRun` matching the param.
      - Wraps errors as `deregister image %s: %w`.
    - `SnapshotIDsFromImage(img types.Image) []string`:
      - Returns the SnapshotId from each BlockDeviceMapping where Ebs != nil. Used by `--dry-run` previews.
    - `CopyAMI(ctx, srcClient, dstClient EC2AMIAPI, srcRegion, dstRegion, srcAMIID, name, description string, tags []types.Tag, waitTimeout time.Duration) (string, error)`:
      - Calls dstClient.CopyImage with SourceImageId, SourceRegion=srcRegion. Returns new ImageId.
      - Waits via `ec2.NewImageAvailableWaiter(dstClient.(ec2.DescribeImagesAPIClient))`.
      - After available, calls dstClient.CreateTags with the same tag set on BOTH the new AMI and its snapshot IDs (discovered via DescribeImages on dst). This is the re-tag step required because CopyImage does NOT inherit tags (Pitfall 3 from RESEARCH.md).
      - Returns dstAMIID + nil. On any post-create failure, returns dstAMIID + error so caller knows the AMI exists but is partial.

    - Test 2: `TestKMBakeTags_EmptyAlias_OmitsAliasOrLeavesBlank` — alias="" should NOT crash and should produce a deterministic tag set (planner's choice: omit the tag entirely; document it).
