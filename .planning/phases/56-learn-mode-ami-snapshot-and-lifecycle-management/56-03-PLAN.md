**Important: SCP vs IAM allow distinction.** AWS authorization is a two-step decision:
1. **SCP** (Service Control Policy) acts as a maximum permissions ceiling at the OU/account level. A Deny statement with ArnNotLike exemption means "this action is denied for everyone EXCEPT these principals". For exempted principals, the SCP no longer blocks the action — but the principal still needs an affirmative `Allow` in their IAM policy. **Un-blocking is not granting.**
2. **IAM allow policy** on the principal (SSO permission set, role inline policy, or attached managed policy) must include the action with `Effect: Allow`. Default IAM policy is `Deny`.

For Phase 56 to work end-to-end:
- Mutating ops (DeregisterImage, DeleteSnapshot, CreateTags) — added to SCP Deny + trustedBase exemption (this plan); operator role's existing IAM allow policy already grants `ec2:*` or includes these via the SSO permission set, so the SCP exemption is the only gap.
- Read-only ops (DescribeImages, DescribeSnapshots) — NOT in any SCP Deny statement (read-only ops shouldn't be SCP-gated); must be in the operator's IAM allow policy. Per Pitfall 6, these may not currently be explicit. Bootstrap.go must EMIT GUIDANCE so the operator knows to grant them.

**Where bootstrap.go emits operator IAM guidance:**
The `runShowSCP` function (line 296-540) prints the SCP JSON and the km-org-admin role policy. It does NOT currently emit a separate operator IAM allow policy (the operator role is managed via SSO permission sets, out of bootstrap.go's scope). However, bootstrap.go CAN print a guidance/documentation block instructing operators to ensure their SSO permission set / klanker-terraform role grants the necessary AMI-lifecycle permissions. This is text-block guidance, not programmatic policy emission.

**Decision (per checker feedback and RESEARCH Pitfall 6):**
- **DO add to SCP DenyInfraAndStorage**: `ec2:DeregisterImage`, `ec2:DeleteSnapshot`, `ec2:CreateTags` (mutating operations matching the existing pattern alongside CreateImage/CopySnapshot).
- **DO NOT add to SCP**: `ec2:DescribeImages`, `ec2:DescribeSnapshots` (read-only; SCPs don't gate these).
- **DO emit positive-allow guidance** in `runShowSCP` output: a new printed text block titled "Operator IAM Positive-Allow Requirements (Phase 56 AMI Lifecycle)" listing all 5 ops (3 mutating + 2 read-only) with rationale and example IAM JSON snippet for the operator's SSO permission set or klanker-terraform role.
