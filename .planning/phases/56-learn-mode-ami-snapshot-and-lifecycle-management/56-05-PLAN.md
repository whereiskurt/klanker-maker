<objective>
Wire the `--ami` flag onto `km shell --learn` and integrate the AMI-baking step into `runLearnPostExit` so the generated profile YAML includes `spec.runtime.ami` when the bake succeeds. Honors the CONTEXT.md locked decision: bake fires BEFORE the SIGUSR1 flush.

Purpose: This is the user-facing payoff of Phase 56 — the operator goes from "I just shaped this sandbox the way I want" to "now bake it" with a single flag, and the generated profile is immediately deployable in any region (raw-AMI path from Phase 33.1).

- `runLearnPostExit` calls `BakeFromSandbox` (from Plan 04) before `flushEC2Observations` for EC2 substrates.

- `pkg/allowlistgen.Recorder` gains `RecordAMI(string)` + `Generate()` emits `Spec.Runtime.AMI` when set.
- Test coverage for both the flag wiring and the generator extension.
