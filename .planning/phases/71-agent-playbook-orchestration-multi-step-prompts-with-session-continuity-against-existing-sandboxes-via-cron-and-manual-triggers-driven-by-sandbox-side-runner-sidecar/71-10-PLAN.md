```markdown
## Playbook Orchestration

Phase 71 ships multi-step Claude prompt sequences against existing sandboxes
via per-sandbox SQS FIFO + sandbox-side bash runner. Operators declare
playbooks as YAML, register them with `km playbook apply`, and fire
manually or via `km at '... playbook run ...'`.

### One-time setup

(commands)

### Profile fields (`spec.cli`)

| Field | Type | Default | Effect |
|---|---|---|---|
| `playbookEnabled` | bool | false | Provision per-sandbox SQS FIFO + DLQ; install km-playbook-runner.service |

### Sandbox env vars

| Variable | Source |
|---|---|
| `KM_PLAYBOOK_QUEUE_URL` | runtime (SSM /sandbox/{id}/playbook-queue-url fallback when env var absent — SCP workaround) |
| `KM_PLAYBOOK_RUNS_TABLE` | runtime |
| `KM_PLAYBOOK_SESSIONS_TABLE` | runtime |
| `KM_PLAYBOOK_ARTIFACTS_BUCKET` | runtime |

### SSM parameters

| Parameter | Purpose |
|---|---|
| `/sandbox/{id}/playbook-queue-url` | Per-sandbox FIFO queue URL (SCP workaround) |

### DynamoDB tables

| Table | Purpose |
|---|---|
| `{prefix}-playbooks` | Registered playbook metadata (name → s3_uri) |
| `{prefix}-playbook-sessions` | (playbook, sandbox_id) → claude_session_id map |
| `{prefix}-playbook-runs` | Per-run record (run_id → status, current_step, ...) |

See `docs/playbooks.md` for the full operator guide.
```
