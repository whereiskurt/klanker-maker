<objective>
Add SQS helpers for playbook FIFO + DLQ provisioning. The existing CreateSlackInboundQueue (`pkg/aws/sqs.go`) creates a single FIFO without a DLQ — playbook needs a paired DLQ with RedrivePolicy, which is new territory per RESEARCH §5 Pitfall 4. The two-step DLQ flow (create DLQ → fetch ARN → create main queue with RedrivePolicy) is encapsulated in a new `CreatePlaybookQueues` helper so callers (plan 71-08's create_playbook.go) get a single atomic call.

Purpose: Plan 71-08 needs a single function call to provision both queues atomically; plan 71-04's TTL Lambda needs the queue-name pattern to send messages; plan 71-09's doctor checks need the DLQ-name pattern to query DLQ depth. All three downstream consumers depend on this shared helper. Independent of profile schema and DDB tables — fits in Wave 1.

Output: 4 new functions in pkg/aws/sqs.go (2 naming, 2 lifecycle), unit tests with a stub SQSClient covering the RedrivePolicy JSON shape.
</objective>

```go
// PlaybookQueueName returns the FIFO main-queue name for a sandbox's playbook runner.
func PlaybookQueueName(resourcePrefix, sandboxID string) string {
    return fmt.Sprintf("%s-playbook-%s.fifo", resourcePrefix, sandboxID)
}

// PlaybookDLQName returns the FIFO DLQ name paired with PlaybookQueueName.
func PlaybookDLQName(resourcePrefix, sandboxID string) string {
    return fmt.Sprintf("%s-playbook-%s-dlq.fifo", resourcePrefix, sandboxID)
}

// CreatePlaybookQueues provisions a paired (main FIFO, DLQ FIFO) queue set
// with RedrivePolicy maxReceiveCount=5 from main → DLQ.
// Two-step flow per RESEARCH §5 Pitfall 4: create DLQ → GetQueueAttributes(QueueArn)
// → create main queue with RedrivePolicy referencing DLQ ARN.
// kmsKeyID empty ⇒ AWS-managed SSE (matches CreateSlackInboundQueue behavior).
func CreatePlaybookQueues(ctx context.Context, client SQSClient, mainName, dlqName, kmsKeyID string) (mainURL, dlqURL string, err error)

// DeletePlaybookQueues deletes both queues. Tolerates NonExistentQueue on either
// side for idempotency (km destroy may run twice). Returns the first non-tolerable error.
func DeletePlaybookQueues(ctx context.Context, client SQSClient, mainURL, dlqURL string) error
```

RedrivePolicy JSON (literal that goes into CreateQueue Attributes):
```json
{"deadLetterTargetArn":"<DLQ_ARN>","maxReceiveCount":"5"}
```
Note: SQS expects maxReceiveCount as a *string* in the RedrivePolicy JSON (AWS quirk).
