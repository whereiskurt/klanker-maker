<objective>
Add the `playbook-run` event handler to the existing TTL Lambda, plus the IAM and env-var plumbing to make it work. This is the central server-side dispatch surface — `km playbook run` and `km at … playbook run` both invoke this Lambda, which checks sandbox readiness and enqueues a single SQS FIFO message that the runner sidecar (plan 71-05) picks up.

Purpose: Per RESEARCH §3, the TTL Lambda's event switch is the obvious extension point (lines 199-226 of cmd/ttl-handler/main.go). Adding one case + one handler function + one IAM policy block reuses the entire existing Lambda packaging, deploy, and observability plumbing. The Lambda's IAM role currently has no SQS perms and no playbook-DDB perms — RESEARCH §3 flagged this gap; this plan closes it.

Output: handlePlaybookRun function with full {running, stopped, paused, terminated, missing} state coverage, IAM policies extended, env vars wired through Terraform variables.
</objective>
