<objective>
Extend the `km-github-threads` DDB store to read and write the new `agent_type` attribute so a
PR/issue thread can persist which agent (claude/codex) it is bound to.

Purpose: This is the persistence half of Phase 102. `agent_type` is a schema-on-write DDB
attribute — NO Terraform change, NO migration, the Phase 98 `km-github-threads` module is
untouched. The poller (Plan 03) reads `agent_type` directly via the AWS CLI, but the bridge's
`/help` reply (Plan 04) reads it through `LookupSandbox`, so the Go store must surface it.

Output: `GitHubThreadStore.LookupSandbox` returns a fourth `agentType` value; `UpdateSession`
takes an `agentType` parameter and writes it in the existing UpdateItem; the fake store + tests
are updated; the single `Handle()` caller of `LookupSandbox` is updated for the new signature.
</objective>
