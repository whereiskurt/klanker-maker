<objective>
Add dual-PID exemption to BPF hooks so both the enforcer and HTTP proxy processes are exempt from connect4/sendmsg4 interception. Add L7 proxy host derivation from profile fields. Add --proxy-pid CLI flag to km ebpf-attach.

Purpose: In gatekeeper mode (Phase 42), connect4 runs in block mode. The HTTP proxy must be exempt or its outbound connections to github.com get redirected back to itself (infinite loop). The existing single-PID exemption covers only the enforcer process. Additionally, --proxy-hosts currently receives GitHub repo names (wrong) instead of domain suffixes (correct).

Output: BPF C with dual-PID exemption, regenerated Go bindings, enforcer wiring, --proxy-pid flag, L7ProxyHosts helper with unit tests.
</objective>

func buildL7ProxyHosts(p *profile.SandboxProfile) string {

}
