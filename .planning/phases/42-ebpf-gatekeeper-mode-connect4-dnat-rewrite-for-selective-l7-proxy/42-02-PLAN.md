<objective>
Flip the `both` enforcement mode userdata template from passive eBPF + iptables to active eBPF gatekeeper. This is the core template change that makes connect4 the primary enforcer in `both` mode.

Purpose: Currently `both` mode runs eBPF in log mode while iptables handles all enforcement. After this plan, eBPF runs in block mode as kernel-level gatekeeper, iptables DNAT is removed, km-dns-proxy is replaced by the eBPF resolver, and only L7-required hosts are routed to the proxy via connect4 sockaddr rewrite.

Output: Updated userdata.go template with gatekeeper mode for `both` enforcement + comprehensive unit tests.
</objective>
