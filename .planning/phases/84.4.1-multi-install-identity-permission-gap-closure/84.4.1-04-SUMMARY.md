## Decisions Made

- Tests for unexported helpers (`ensureRegionHCL`, `terraformIsCurrent`, `isPlaceholderBucket`) placed in internal test files (`package cmd` / `package config`) alongside the existing external `cmd_test` / `config_test` files. External scaffold stubs updated to source-grep assertions (safe from `cmd_test` package).
- Terraform version sidecar file (`build/terraform.version`) chosen over exec-and-parse of the cross-arch linux/arm64 binary. Running an arm64 binary on macOS without Rosetta 2 or QEMU fails silently or crashes; the text sidecar is reliable and cheap.
- configure HeadBucket probe skipped when `in == nil` (caller passes nil for stdin) rather than checking `nonInteractive` flag. This is more defensive: some non-interactive callers might still pass a real `io.Reader`, and the existing test suite uses `nonInteractive=false` with `strings.NewReader("")`.
- `deleteDynamoDBLockTable` warns (not errors) when ResourceNotFoundException is returned, matching the general unbootstrap pattern where partial cleanup is preferred over stopping mid-teardown.
- newS3HeadBucketClient does not exist — inlined the SDK construction (`awspkg.LoadAWSConfigInRegion` + `s3.NewFromConfig`) directly in the configure interactive path. Silently skips probe when AWS credentials are unavailable at configure time.
