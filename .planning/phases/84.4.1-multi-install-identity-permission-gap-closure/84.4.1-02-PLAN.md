<objective>
TIER-1 gap closure: ssm-session-doc per-install rename.

The Phase 84.4-08 UAT proved that hardcoded resource names cause cross-install
destruction. `KM-Sandbox-Session` was hardcoded in `infra/modules/ssm-session-doc/
v1.0.0/main.tf`. Operator workaround was `terragrunt import` into rg's state —
which then DESTROYED the shared document from km's state when rg was torn down
(`km uninit` on rg deleted `KM-Sandbox-Session`, leaving km's state pointing at
a missing resource and `km shell` broken on km until manual recovery).

CONTEXT.md's recovery from this load-bearing failure is **per-install rename to
${var.resource_prefix}-Sandbox-Session** in a new ssm-session-doc/v2.0.0/ module,
with a `moved {}` block declaring the rename from v1.0.0's resource address.

RESEARCH.md Finding 1 contradicted CONTEXT.md's assumption that "km shell looks
up the document name from config" — VERIFIED FALSE. The document name is
hardcoded in 5 Go callsites: shell.go:500, agent.go:356/430/619, agent_auth.go:
157/411. Plus error-message wrappings at shell.go:506, agent.go:363/437/626. ALL
must update to cfg.GetSandboxSessionDocumentName() (the helper added in Wave 0).

This plan also extends the modulehygiene_test.go audit to case-insensitive
km-/KM- matching — the v1.0.0 audit missed `KM-Sandbox-Session` because the match
was case-sensitive. This is the same audit that Phase 84.4 Plan 01 shipped;
extending it here prevents the next ssm-session-doc-shape gap from sliding in.

`autonomous: true` because all changes are code/HCL-side. The live apply (which
will destroy `KM-Sandbox-Session` and create `km-Sandbox-Session` on the canonical
km install — a 1-2s window where new SSM sessions fail) happens in Wave 3 plan
84.4.1-05 (canonical km install transition).

Purpose: SSM-SESSION-DOC-PREFIX-NAMESPACING requirement. Closure criterion (a)
in ROADMAP §Phase 84.4.1.

Output: new v2.0.0 module (3 HCL files) + live wiring flip + 5 Go callsites
updated + 4 error message wraps updated + 2 Go test files updated + 1 audit
test extension.
