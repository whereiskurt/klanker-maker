## Notable findings (carry to Wave 4)

1. **SSM doc was already converged.** The case-sensitivity gap was Go-side only —
   the live SSM doc was `km-Sandbox-Session` lowercase the whole time, but
   shell.go / agent.go / agent_auth.go hardcoded `"KM-Sandbox-Session"` uppercase.
   Either AWS SSM document names are case-insensitive in practice, or the prior
   Go callsites never actually worked against this install for those code paths.
   Wave 4 UAT-1 (km shell from rebuilt binary) will validate this empirically.

2. **`./bin/km` was stale.** The committed `bin/km` predated Phase 84.3's
   `km env` command. `make build && cp km bin/km` was required before any
   terragrunt invocation could pick up the `KM_*` env exports. The
   feedback memory `rebuild_km` already covers this — operator runbook in
   Phase 84.4.1 UAT (Wave 4) should re-emphasize.

3. **klanker-management profile lacks Organizations perms.** The
   HostedZoneAdmin role on the klanker-management profile can't call
   `organizations:ListPolicies` / `DescribePolicy`. SCP body captures must
   chain-assume `km-org-admin` via the klanker-terraform profile. Terragrunt
   handles this transparently via the provider `assume_role` block; ad-hoc
   AWS-CLI use needs an explicit `sts:AssumeRole`.
