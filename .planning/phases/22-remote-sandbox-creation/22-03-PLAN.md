   - ARG TERRAGRUNT_VERSION=0.67.16
   - Download terraform arm64 zip from releases.hashicorp.com, unzip to /usr/local/bin/
   - Download terragrunt arm64 binary from github.com/gruntwork-io/terragrunt/releases, chmod +x

   Stage 2 (runtime): FROM public.ecr.aws/lambda/provided:al2023-arm64
   - COPY --from=downloader terraform and terragrunt to /var/task/
   - COPY build/km-create-handler /var/task/bootstrap (Lambda entry point)
   - COPY build/km /var/task/km (km binary for subprocess)

   a. IAM role (`km-create-handler`) with lambda.amazonaws.com trust policy
   b. IAM policy with BROAD permissions needed for `km create` subprocess:
      - EC2: RunInstances, DescribeInstances, CreateSecurityGroup, AuthorizeSecurityGroupEgress, DeleteSecurityGroup, TerminateInstances, CreateTags, DescribeSecurityGroups, RevokeSecurityGroupEgress
      - IAM: CreateRole, PutRolePolicy, DeleteRolePolicy, DeleteRole, CreateInstanceProfile, AddRoleToInstanceProfile, RemoveRoleFromInstanceProfile, DeleteInstanceProfile, PassRole (scoped to `arn:aws:iam::*:role/km-*`), GetRole, GetInstanceProfile
      - ECS: CreateCluster, DeleteCluster, RegisterTaskDefinition, DeregisterTaskDefinition, CreateService, DeleteService, UpdateService, DescribeServices, DescribeClusters
      - S3: GetObject, PutObject, ListBucket, DeleteObject on artifact bucket
      - DynamoDB: GetItem, PutItem, UpdateItem, DeleteItem on state lock table and budget table
      - EventBridge Scheduler: CreateSchedule, DeleteSchedule, GetSchedule (for TTL setup)
      - SSM: GetParameter, PutParameter, DeleteParameter (safe phrase, GitHub token)
      - SES: SendEmail (notifications)
      - Lambda: CreateFunction, DeleteFunction, GetFunction, UpdateFunctionCode, InvokeFunction (for budget-enforcer per-sandbox Lambda)
      - KMS: GenerateDataKey, Decrypt
      - CloudWatch Logs: CreateLogGroup, CreateLogStream, PutLogEvents (Lambda's own logs)
      - sts:GetCallerIdentity (for account ID detection)
      Scope with conditions where possible: `aws:RequestTag/km:managed = "true"`, resource ARN patterns `km-*`
   c. `aws_lambda_function` "create_handler":
      - function_name = "km-create-handler"
      - role = IAM role ARN
      - package_type = "Image"
      - image_uri = var.ecr_image_uri
      - architectures = ["arm64"]
      - timeout = 900
      - memory_size = 1536
      - ephemeral_storage { size = 2048 }
      - environment variables: KM_ARTIFACTS_BUCKET, KM_EMAIL_DOMAIN, KM_OPERATOR_EMAIL, KM_STATE_BUCKET, KM_STATE_PREFIX, KM_REGION_LABEL
   d. `aws_cloudwatch_event_rule` "sandbox_create":
      - name = "km-sandbox-create"
      - event_pattern matching source=["km.sandbox"], detail-type=["SandboxCreate"]
   e. `aws_cloudwatch_event_target` "create_to_lambda":
      - rule = event rule name
      - target_id = "km-create-handler"
      - arn = Lambda function ARN
      - retry_policy { maximum_retry_attempts = 0 } (create is NOT idempotent)
   f. `aws_lambda_permission` for events.amazonaws.com to invoke the Lambda
   g. `aws_iam_role_policy_attachment` for scheduler PassRole (if needed for TTL schedule creation)
