  <action>
1. Create `cmd/email-create-handler/main.go`:
   - Define `S3EventRecord` struct matching S3 event notification shape (Records[].S3.Bucket.Name, Records[].S3.Object.Key)
   - Define `EmailCreateHandler` struct with injectable dependencies:
     * `S3Client` — interface with `GetObject` and `PutObject`
     * `SSMClient` — interface with `GetParameter`
     * `EventBridgeClient` — `awspkg.EventBridgeAPI` (or local interface if Plan 01 not yet merged)
     * `SESClient` — `awspkg.SESV2API`
     * `ArtifactBucket string`
     * `Domain string`
     * `SafePhraseSSMKey string` (default `/km/config/remote-create/safe-phrase`)

   - Implement `Handle(ctx context.Context, event S3EventRecord) error`:
     a. Extract bucket and key from `event.Records[0].S3`
     b. `GetObject` to download raw MIME email from S3
     c. Parse with `net/mail.ReadMessage(reader)` to get headers + body
     d. Extract sender from `From` header
     e. Extract body text and YAML profile:
        - Check `Content-Type` header for multipart boundary
        - If multipart: iterate parts with `mime/multipart.NewReader`, find part with Content-Type `text/yaml`, `application/x-yaml`, or filename `*.yaml` — that is the profile YAML; concatenate remaining text parts as the body text
        - If single-part (text/plain or text/yaml): if Content-Type is text/yaml, entire body is profile YAML and body text is empty; if text/plain, entire body is both body text and profile YAML (operator sends YAML as plain text with KM-AUTH line above it)
     f. Extract KM-AUTH phrase: `awspkg.KMAuthPattern.FindStringSubmatch(bodyText)` — if no match, also check the full raw body (KM-AUTH may be in the YAML part headers or body)
     g. If no phrase found, send rejection email via SES ("Missing KM-AUTH phrase") and return nil (not an error — the email was processed, just rejected)
     h. `GetParameter` from SSM at `SafePhraseSSMKey` to get expected phrase
     i. Compare phrases (constant-time: `subtle.ConstantTimeCompare`) — if mismatch, send rejection email and return nil
     j. `profile.Parse(yamlBytes)` to validate — if parse error, send rejection email with validation errors and return nil
     k. Generate sandbox ID: `crypto/rand` hex (same pattern as create.go Step 5)
     l. Upload profile to S3 at `remote-create/{sandbox-id}/.km-profile.yaml`
     m. Call `PutSandboxCreateEvent` with detail (sandbox_id, artifact_bucket, prefix, sender email, on_demand=false)
     n. Send acknowledgment email to sender: "Sandbox creation request received: {sandbox-id}"
     o. Return nil

   - In `main()`:
     a. Load AWS config (empty profile)
     b. Create S3, SSM, EventBridge, SES clients
     c. Read env vars: `KM_ARTIFACTS_BUCKET`, `KM_EMAIL_DOMAIN`, `KM_SAFE_PHRASE_SSM_KEY` (default `/km/config/remote-create/safe-phrase`)
     d. Create handler, call `lambdaruntime.Start(handler.Handle)`

   NOTE: If Plan 01 has not been merged yet and `pkg/aws/eventbridge.go` does not exist, define a local `EventBridgeAPI` interface and `PutSandboxCreateEvent` function inside this package. They will be consolidated in Plan 03 or during merge. The interface shape is documented in the research.

   - Test invalid YAML profile -> rejection email with parse errors
