## Manual-Only Verifications

| Behavior | Criterion | Why Manual | Test Instructions |
|----------|-----------|------------|-------------------|
| audit-log opens FIFO on resumed sandbox (no `permission denied`) AND `km doctor` reports `✓ Presence daemon healthy` within 5 min of liveness signal | UAT-RESUME | Requires live AWS sandbox, `km pause`+`km resume` cycle, and CloudWatch ingestion of `source:"presence"` events | See UAT script below |

### UAT Script (operator-runnable)

```bash
# 1. Provision a fresh post-Phase-79.1 sandbox
km create profiles/learn.yaml --alias p79-1-test

# 2. Confirm presence daemon healthy on fresh boot
SB=$(km list | awk '/p79-1-test/ {print $1}')
km shell $SB -- journalctl -u km-audit-log | grep "reading from audit pipe"
# Expected: "reading from audit pipe pipe=/run/km/audit-pipe"

km doctor 2>&1 | grep -A1 "Presence daemon healthy"
# Expected: OK

# 3. Pause + resume
km pause $SB
km resume $SB

# 4. Wait for presence daemon to emit (60s tick + CloudWatch ingestion delay)
sleep 90

# 5. Assert audit-log opened the FIFO correctly (not permission denied)
km shell $SB -- journalctl -u km-audit-log | grep -E "audit pipe|permission denied"
# Expected: "reading from audit pipe pipe=/run/km/audit-pipe" (no "permission denied")

# 6. Assert km doctor goes green
km doctor 2>&1 | grep -A1 "Presence daemon healthy"
# Expected: ✓ Presence daemon healthy (OK, not WARN)

# 7. Verify /run/km/audit-pipe is a FIFO (not a regular file)
km shell $SB -- ls -la /run/km/audit-pipe
# Expected: prw-rw-rw- (p = FIFO)

# 8. Cleanup
km destroy $SB --remote --yes
```
