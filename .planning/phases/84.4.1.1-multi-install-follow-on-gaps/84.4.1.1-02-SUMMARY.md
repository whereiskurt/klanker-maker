## What Was Built

### Task 1: configure.go derive-default wiring (Gap #2a)

Added `deriveArtifactsBucket(resourcePrefix, applicationAcct)` call-sites in two places:

1. **Non-interactive path** (after `operatorEmail` derivation block, ~line 463): mirrors the existing operator_email derivation pattern. Only fires when `artifactsBucket == "" && resourcePrefix != "" && applicationAcct != ""`.

2. **Interactive path** (before the prompt at configure.go:603): same guard condition. The operator sees `tg-artifacts-052251888500` as the prompt default and can press Enter to accept.

Preserve-on-rerun: when `artifactsBucket` already has a value, neither derivation fires.

### Task 2: config.Load() ValidateArtifactsBucket wiring (Gap #2b + Gap #4)

Replaced the `isPlaceholderBucket` check in `config.Load()` with `ValidateArtifactsBucket`. Key design decision: validate the **yaml-authoritative value** (`yamlDefaults["artifacts_bucket"]`) rather than the viper-merged `cfg.ArtifactsBucket`. This preserves the existing `TestConfigLoad_DriftWarn_KM_ARTIFACTS_BUCKET` behavior where `KM_ARTIFACTS_BUCKET=env-override-bucket` (non-canonical) must not block `config.Load()`.

Logic:
```go
bucketToValidate := cfg.ArtifactsBucket
if yamlVal, ok := yamlDefaults["artifacts_bucket"]; ok {
    bucketToValidate = yamlVal
}
if err := ValidateArtifactsBucket(bucketToValidate); err != nil {
    return nil, err
}
```

**1. [Rule 1 - Bug] KM_ARTIFACTS_BUCKET env override broke TestConfigLoad_DriftWarn_KM_ARTIFACTS_BUCKET**
- **Found during:** Task 2 GREEN phase
- **Issue:** The initial `ValidateArtifactsBucket(cfg.ArtifactsBucket)` call validated the viper-merged value, which includes `KM_ARTIFACTS_BUCKET` env var overrides. The drift-warn test sets `KM_ARTIFACTS_BUCKET=env-override-bucket` (non-canonical) to test the drift warning, but the new validation blocked `config.Load()` before the drift-warn code ran.
- **Fix:** Validate `yamlDefaults["artifacts_bucket"]` (raw yaml value) when available, falling back to `cfg.ArtifactsBucket` only when no yaml config exists. This preserves env-override bypass semantics while still rejecting non-canonical yaml values.
