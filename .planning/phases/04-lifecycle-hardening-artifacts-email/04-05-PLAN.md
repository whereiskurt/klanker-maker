Close two verification gaps from Phase 04 VERIFICATION.md:

1. TTL expiry path does not upload artifacts -- EventBridge scheduler fires a Lambda but no TTL handler Lambda exists. Create a Go Lambda handler that receives `{"sandbox_id":"sb-xxx"}`, loads the sandbox profile from S3, uploads artifacts via `pkg/aws.UploadArtifacts`, sends a "destroyed" lifecycle notification, then destroys the sandbox via terragrunt. Create the matching Terraform module.

2. Lifecycle notifications for idle-timeout and error/crash events are not wired -- `IdleDetector.OnIdle` has no `SendLifecycleNotification` call, and `ExecuteTeardown` has no notification for error paths. Add notification callbacks to both.

Purpose: Without these fixes, artifact upload only works on manual `km destroy` (not TTL expiry), and operators miss idle-timeout and error notifications -- the two most common real-world termination scenarios.

Output: TTL handler Lambda (Go + Terraform), enhanced lifecycle notification wiring in teardown and idle packages.

Two changes to add lifecycle notification callbacks:

**1. Enhance TeardownCallbacks in `pkg/lifecycle/teardown.go`:**

Add a new optional field to `TeardownCallbacks`:
```go
// OnNotify sends a lifecycle notification after teardown completes.
// Called with (ctx, sandboxID, event) where event is "destroyed", "stopped",
// "retained", or "error".
// If nil, no notification is sent (backward compatible with existing callers).
// Failure is best-effort: logged as warning, does not affect teardown result.
OnNotify func(ctx context.Context, sandboxID string, event string) error
```

Modify `ExecuteTeardown`:
- After the `switch` block completes successfully, call `callbacks.OnNotify(ctx, sandboxID, policy)` if OnNotify is non-nil. Use the policy name as the event ("destroy" -> "destroyed", "stop" -> "stopped", "retain" -> "retained"). Log warning on error, do not propagate.
- If the Destroy or Stop callback returns an error, call `callbacks.OnNotify(ctx, sandboxID, "error")` before returning the error. Log warning on notification error, still return the original teardown error.

**2. Enhance IdleDetector in `pkg/lifecycle/idle.go`:**

Add a new optional field to `IdleDetector`:
```go
// OnIdleNotify is called when idle is detected, to send a lifecycle notification.
// Called with (sandboxID). If nil, no notification is sent.

OnIdleNotify func(sandboxID string)
```

Modify `Run()`: after calling `d.OnIdle(d.SandboxID)`, also call `d.OnIdleNotify(d.SandboxID)` if non-nil. Log warning if OnIdleNotify panics (recover). This keeps idle notification decoupled from idle teardown action.
