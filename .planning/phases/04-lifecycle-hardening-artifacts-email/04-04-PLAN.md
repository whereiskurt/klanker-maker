      ```go
      // Provision sandbox email identity
      sesClient := sesv2.NewFromConfig(awsCfg)
      emailDomain := "sandboxes.klankermaker.ai"
      emailAddr, err := awspkg.ProvisionSandboxEmail(ctx, sesClient, sandboxID, emailDomain)
      if err != nil {
          log.Warn().Err(err).Msg("failed to provision sandbox email (non-fatal)")
      } else {
          log.Info().Str("email", emailAddr).Msg("sandbox email provisioned")
          fmt.Fprintf(os.Stdout, "Email: %s\n", emailAddr)
      }
      ```
      Email provisioning is non-fatal — sandbox is still usable without email.

   b. Add import for `sesv2` package: `"github.com/aws/aws-sdk-go-v2/service/sesv2"`

   c. After create output, if `KM_OPERATOR_EMAIL` is set, send a lifecycle notification:
      ```go
      if operatorEmail := os.Getenv("KM_OPERATOR_EMAIL"); operatorEmail != "" {
          _ = awspkg.SendLifecycleNotification(ctx, sesClient, operatorEmail, sandboxID, "created", emailDomain)
      }
      ```

2. **Update `internal/app/cmd/destroy.go`** (`runDestroy` function):

   a. Before cleanup, add artifact upload:
      ```go
      // Upload artifacts before destroy (best-effort)
      if profile != nil && profile.Spec.Artifacts != nil && len(profile.Spec.Artifacts.Paths) > 0 {
          s3Client := s3.NewFromConfig(awsCfg)
          bucket := os.Getenv("KM_ARTIFACTS_BUCKET")
          if bucket == "" { bucket = "km-sandbox-artifacts-ea554771" }
          uploaded, skipped, err := awspkg.UploadArtifacts(ctx, s3Client, bucket, sandboxID, profile.Spec.Artifacts.Paths, profile.Spec.Artifacts.MaxSizeMB)
          if err != nil {
              log.Warn().Err(err).Msg("artifact upload failed (best-effort)")
          } else {
              log.Info().Int("uploaded", uploaded).Int("skipped", len(skipped)).Msg("artifacts uploaded")
          }
      }
      ```
      Note: The `destroy` command currently doesn't load the profile. Add profile loading from sandbox metadata (stored in S3 by `km create`). If profile is unavailable, skip artifact upload.

   b. After successful destroy, add SES email cleanup:
      ```go
      sesClient := sesv2.NewFromConfig(awsCfg)
      emailDomain := "sandboxes.klankermaker.ai"
      if err := awspkg.CleanupSandboxEmail(ctx, sesClient, sandboxID, emailDomain); err != nil {
          log.Warn().Err(err).Msg("failed to cleanup sandbox email (non-fatal)")
      }
      ```

   c. Send lifecycle notification on destroy:
      ```go
      if operatorEmail := os.Getenv("KM_OPERATOR_EMAIL"); operatorEmail != "" {
          _ = awspkg.SendLifecycleNotification(ctx, sesClient, operatorEmail, sandboxID, "destroyed", emailDomain)
      }
      ```

   d. Add imports for `sesv2` and update existing s3 import.

3. **Update ECS service.hcl template** in `pkg/compiler/service_hcl.go`:

   Add SES IAM permission block to the ECS task role policy. The ECS template already has IAM policy generation — add a conditional `ses:SendEmail` permission scoped to the sandbox's own email address:
   ```
   {{- if .HasEmail }}
   {
     effect   = "Allow"
     actions  = ["ses:SendEmail", "ses:SendRawEmail"]
     resources = ["*"]
     condition = {
       test     = "StringEquals"
       variable = "ses:FromAddress"
       values   = ["{{ .SandboxEmail }}"]
     }
   }
   {{- end }}
   ```
   Add `HasEmail bool` and `SandboxEmail string` to the ECS params struct. Set `HasEmail = true` always (Phase 4 creates email for every sandbox). Set `SandboxEmail = sandboxID + "@sandboxes.klankermaker.ai"`.

   Also add `KM_EMAIL_ADDRESS` to the main container's environment variables so the agent knows its own email:
   ```
   { name = "KM_EMAIL_ADDRESS", value = "{{ .SandboxEmail }}" }
   ```

4. **Update EC2 user-data template** in `pkg/compiler/userdata.go`:

   ```
   {{- if .SandboxEmail }}
   export KM_EMAIL_ADDRESS="{{ .SandboxEmail }}"
   {{- end }}
   ```
   Add `SandboxEmail string` to `userDataParams`.

IMPORTANT: Artifact upload in destroy uses `UploadArtifacts` from Plan 01 — same logic as the teardown callback from Plan 03.

   - `source_bucket_name` (string) — the primary artifact bucket name (e.g. "km-sandbox-artifacts-ea554771")
   - `source_bucket_arn` (string) — ARN of the source bucket
   - `destination_region` (string) — the replication target region (e.g. "us-west-2")
   - `destination_bucket_name` (string) — name for the replica bucket (e.g. "km-sandbox-artifacts-ea554771-replica")

2. Create `infra/modules/s3-replication/v1.0.0/main.tf`:
   a. `aws_s3_bucket` for the destination bucket in the destination region (using a provider alias):
      ```hcl
      provider "aws" {
        alias  = "replica"
        region = var.destination_region
      }
      resource "aws_s3_bucket" "replica" {
        provider = aws.replica
        bucket   = var.destination_bucket_name
      }
      ```
   b. `aws_s3_bucket_versioning` on BOTH source and destination buckets (required for replication):
      ```hcl
      resource "aws_s3_bucket_versioning" "source" {
        bucket = var.source_bucket_name
        versioning_configuration { status = "Enabled" }
      }
      resource "aws_s3_bucket_versioning" "replica" {
        provider = aws.replica
        bucket   = aws_s3_bucket.replica.id
        versioning_configuration { status = "Enabled" }
      }
      ```
   c. `aws_iam_role` for S3 replication with trust policy for `s3.amazonaws.com`
   d. `aws_iam_role_policy` allowing `s3:GetReplicationConfiguration`, `s3:ListBucket` on source, and `s3:ReplicateObject`, `s3:ReplicateDelete`, `s3:ReplicateTags` on destination
   e. `aws_s3_bucket_replication_configuration` on the source bucket:
      - Role: the IAM role ARN
      - Rule: status=Enabled, prefix="artifacts/" (only replicate artifact uploads, not mail/)
      - Destination: bucket ARN of replica, storage class STANDARD

3. Create `infra/modules/s3-replication/v1.0.0/outputs.tf`:
   - `replica_bucket_arn` — ARN of the replica bucket
   - `replica_bucket_name` — name of the replica bucket
   - `replication_role_arn` — ARN of the IAM replication role

IMPORTANT: S3 replication requires versioning on both buckets — the module must enable it.
IMPORTANT: Only replicate the `artifacts/` prefix — mail/ objects are ephemeral inbox data and don't need replication.
IMPORTANT: This module is applied when the operator sets `replicationRegion` in the profile. It's a one-time infrastructure setup, not per-sandbox. The compiler does NOT need to do anything for replication — Terraform handles it automatically once the rule is in place.
