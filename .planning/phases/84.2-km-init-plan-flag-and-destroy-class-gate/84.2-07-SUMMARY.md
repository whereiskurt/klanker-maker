## Phase 84.3 Gap Entries

### GAP-1 (blocking, regression-of-purpose) — aws_ses_receipt_rule missing from ProtectedTypes

**Requirement violated:** DESTROY-CLASS-GATE — the gate must trip on any destroy of resource types whose loss caused the Phase 82→84 incident.

**Observed:** `aws_ses_receipt_rule` destroys pass the gate (exit 0, no trip block). Only the parent `aws_ses_receipt_rule_set` is in `pkg/terragrunt/planreport/protected.go`.

**Why this matters:** The Phase 82→84 cutover destroyed child rules (operator inbound + sandbox catchall) — `aws_ses_receipt_rule` instances — via the `removed { destroy = true }` orphan path. Today's gate does not catch a re-occurrence of the exact incident it was built for.

**Fix sketch:**
- Add `aws_ses_receipt_rule` to `pkg/terragrunt/planreport/protected.go` with incident-link comment
- Add gate_test.go fixture exercising a single `aws_ses_receipt_rule` destroy
- Re-run Scenarios 2 + 3 in Phase 84.3 UAT

### GAP-2 (non-blocking, validation-doc drift) — Scenario 4b blocked by prevent_destroy

**Observed:** `aws_ses_receipt_rule_set.shared` has `lifecycle.prevent_destroy = true` (foundation module lines 92-94), so a synthetic destroy edit causes terraform to error before the gate runs — a different failure mode than the validation doc projects.

**Fix sketch:** Rewrite Scenario 4b to target a foundation-owned resource without `prevent_destroy` (e.g. `aws_ses_active_receipt_rule_set`), or document that `prevent_destroy` is the active defense at the foundation level and the gate is a second-tier net at the regional level.

### GAP-3 (non-blocking, validation-doc drift) — Scenario 6 unset path

**Observed:** `unset KM_ROUTE53_ZONE_ID; km init --plan` does not skip the ses module on this install because km loads `route53_zone_id` from `km-config.yaml` via `ExportTerragruntEnvVars`, not from the shell env.

**Fix sketch:** Update the validation doc to describe the skip-trigger as a `km-config.yaml` field absence, or convert Scenario 6 into a unit test against `runInitPlan` with a fake config that omits the field.
