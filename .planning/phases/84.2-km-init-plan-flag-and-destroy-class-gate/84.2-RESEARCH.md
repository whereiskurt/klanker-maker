## Summary

Phase 84.2 adds a real `terragrunt plan` + JSON-parse + curated destroy-class gate to `km init` and `km bootstrap --shared-ses`. The plumbing is well-trodden: the runner already exposes a `Plan` method (`pkg/terragrunt/runner.go:130`), `ExportTerragruntEnvVars` is the canonical 14-call helper (`internal/app/cmd/init.go:698`), and Phase 84.1-02 timeouts/heartbeat are shipped and exercised by 8 production callers. The only genuinely new code is the `pkg/terragrunt/planreport/` package (pure JSON-in / decision-out), two thin `Runner` siblings (`PlanWithOutput`, `ShowPlanJSON`), and wiring into the two cobra commands.

The dominant unknowns from the design spec all resolve in research:
1. Phase 84.1-01 (`ExportTerragruntEnvVars`) ‚Äî **shipped** (`92daa0e`, 8 callers, exported).
2. Phase 84.1-02 (runner timeouts + heartbeat) ‚Äî **shipped** (`e67e8b2` + `c6839c5`, fully tested).
3. Terraform schema version pinned by terragrunt ‚Äî **`format_version "1.0"`** (stable Terraform 1.1.0 ‚Üí 1.15.x; repo runs Terraform v1.8.2, terragrunt v0.99.1). The `change.actions` array contract is fully documented.
4. `terragrunt show -json <planfile>` works on plans from `terragrunt plan -out=<file>` provided we invoke both from the same `cmd.Dir` (the module's live dir) ‚Äî terragrunt resolves `.terragrunt-cache/<hash>/<module>/` automatically. Don't reach for `terraform show` directly.

**Primary recommendation:** Build `pkg/terragrunt/planreport/` as a pure-Go package (no `terragrunt-json` dep ‚Äî it's actively recommended against by the terraform-core team for parsers because it drops unknown attrs). Hand-roll a minimal `Report` struct that mirrors the documented `format_version "1.0"` schema. Two new `Runner` methods are thin sibling-shaped wrappers over the existing `buildCommand`. The cobra wiring follows the existing `--dry-run` flag-validation pattern; `--i-accept-destroys` should be a hidden flag (`MarkHidden`) to keep `--help` clean.

## User Constraints (from CONTEXT.md)

### Locked Decisions

All decisions below are LOCKED ‚Äî captured from the approved design spec.

**Decision 1 ‚Äî Flag shape:**
- New `--plan` flag, independent of `--dry-run`.
- Combinable with `--dry-run=true` (shows both info dump and plan).
- `--plan` itself implies no apply ever runs.
- **Why:** `--dry-run=true` info dump surfaces wrapper-level steps (DNS, build, upload) not visible in `terragrunt plan`. Replacing its semantics would silently change default behavior.

**Decision 2 ‚Äî Scope:**
- All regional modules planned, in `runInit` order (sequential).
- No selectable subset / per-module flag.
- **Why:** A real safety gate must cover everything `apply` would touch; a subset flag invites plan-missing the module about to drift.

**Decision 3 ‚Äî Safety gate:**
- Destroy-class gate with curated compiled-in `ProtectedTypes` allowlist.
- Any destroy OR replace of a protected resource type halts with exit 1.
- Override: `--i-accept-destroys` (per-invocation only, never persisted, does NOT auto-apply ‚Äî only clears the `--plan` exit code).
- **Why:** An all-destroys gate is too noisy; no-gate is just `terragrunt plan` with extra steps.

**Decision 4 ‚Äî Bootstrap parity:**
- `km bootstrap --shared-ses` gets a parallel `--plan` flag in this same phase.
- **Why:** Phase 84 Gaps 2, 3, 6 happened in the bootstrap path too. Symmetric coverage is cheap once the plumbing exists.

**Decision 5 ‚Äî Output model:**
- Sequential execution (matches `runInit` order ‚Äî never parallel).
- Default: per-module one-line summary (`ses: 2 to add, 0 to change, 1 to destroy ‚öÝ`).
- `--verbose` streams the full plan text.
- Destroy-gate trip block ALWAYS prints in full regardless of verbose.
- **Why:** Terragrunt's own `run --all plan` parallelism interleaves output, complicates JSON aggregation for the gate, and bursts AWS API calls.

**Decision 6 ‚Äî No config file for protected list:**
- `ProtectedTypes` lives compiled-in `pkg/terragrunt/planreport/protected.go`.
- Updates require PR review; each addition references the motivating incident in a comment.
- **Why:** An operator-side config file would let an organization silently disable the safety. The safety value comes from review pressure.

### Pattern 2: Pure-logic planreport package (no AWS, no terragrunt imports)

```go
// pkg/terragrunt/planreport/report.go

package planreport

import (
    "encoding/json"
    "fmt"
)

// Report is the trimmed-down per-module view of `terraform show -json`.
// Adds, Changes, Destroys, Replaces are populated from resource_changes[].
// ParseFailed is set when the underlying JSON couldn't be unmarshaled ‚Äî
// callers (and the gate) treat this as a conservative trip.
type Report struct {
    Module      string
    Adds        []ResourceChange
    Changes     []ResourceChange
    Destroys    []ResourceChange
    Replaces    []ResourceChange
    ParseFailed bool
    ParseError  error  // non-nil iff ParseFailed
}

type ResourceChange struct {
    Address string
    Type    string
    Action  string  // "create" | "update" | "delete" | "replace"
}

// Parse consumes terraform show -json output (format_version "1.0") and returns
// a Report. The module name is set by the caller.
//
// Schema reference: https://developer.hashicorp.com/terraform/internals/json-format
// (format_version "1.0", stable Terraform 1.1.0 ‚Üí 1.15.x)
func Parse(jsonBytes []byte) (Report, error) {
    var raw struct {
        FormatVersion   string `json:"format_version"`
        ResourceChanges []struct {
            Address string `json:"address"`
            Type    string `json:"type"`
            Change  struct {
                Actions []string `json:"actions"`
            } `json:"change"`
        } `json:"resource_changes"`
    }
    if err := json.Unmarshal(jsonBytes, &raw); err != nil {
        return Report{}, fmt.Errorf("parse plan JSON: %w", err)
    }
    // We accept "1.x" ‚Äî schema is forward-compatible by HashiCorp's stated rule
    // ("minor version increments for backward-compatible additions").
    if !isCompatibleFormatVersion(raw.FormatVersion) {
        return Report{}, fmt.Errorf("unsupported plan format_version %q (want 1.x)", raw.FormatVersion)
    }
    r := Report{}
    for _, rc := range raw.ResourceChanges {
        action := classifyAction(rc.Change.Actions)
        ch := ResourceChange{Address: rc.Address, Type: rc.Type, Action: action}
        switch action {
        case "create":  r.Adds = append(r.Adds, ch)
        case "update":  r.Changes = append(r.Changes, ch)
        case "delete":  r.Destroys = append(r.Destroys, ch)
        case "replace": r.Replaces = append(r.Replaces, ch)
        // "no-op", "read", "forget" ‚Äî ignore
        }
    }
    return r, nil
}

// classifyAction collapses the terraform actions slice into the four categories
// the gate cares about.
//
// Schema (per https://developer.hashicorp.com/terraform/internals/json-format):
//   ["no-op"]                              ‚Üí no-op
//   ["create"]                             ‚Üí create
//   ["read"]                               ‚Üí read (data source)
//   ["update"]                             ‚Üí update
//   ["delete"]                             ‚Üí delete
//   ["delete","create"] / ["create","delete"] ‚Üí replace
func classifyAction(actions []string) string {
    if len(actions) == 0 { return "no-op" }
    if len(actions) == 1 {
        return actions[0]  // "no-op", "create", "read", "update", "delete", "forget"
    }
    // Two-action sequences = replace (either order)
    if len(actions) == 2 {
        a, b := actions[0], actions[1]
        if (a == "delete" && b == "create") || (a == "create" && b == "delete") {
            return "replace"
        }
    }
    return "unknown"  // defensive ‚Äî gate doesn't trip on this; logged as warning by caller
}
```

```go
// pkg/terragrunt/planreport/gate.go

package planreport

import "slices"

type GateResult struct {
    Blocked bool
    Trips   []Trip
}

type Trip struct {
    Module  string
    Type    string  // empty when ParseFailed
    Address string  // empty when ParseFailed
    Action  string  // "DESTROY" or "REPLACE" or "PARSE-FAIL"
    Reason  string  // populated for parse-fail trips
}

// Evaluate runs the locked gate algorithm from the design spec.
// Trips are populated even when acceptDestroys clears the block ‚Äî
// the operator still sees what was overridden.
func Evaluate(reports []Report, acceptDestroys bool) GateResult {
    var trips []Trip
    for _, r := range reports {
        if r.ParseFailed {
            trips = append(trips, Trip{
                Module: r.Module,
                Action: "PARSE-FAIL",
                Reason: "plan JSON parse failed ‚Äî conservative trip",
            })
            continue
        }
        for _, ch := range r.Destroys {
            if slices.Contains(ProtectedTypes, ch.Type) {
                trips = append(trips, Trip{Module: r.Module, Type: ch.Type, Address: ch.Address, Action: "DESTROY"})
            }
        }
        for _, ch := range r.Replaces {
            if slices.Contains(ProtectedTypes, ch.Type) {
                trips = append(trips, Trip{Module: r.Module, Type: ch.Type, Address: ch.Address, Action: "REPLACE"})
            }
        }
    }
    blocked := len(trips) > 0 && !acceptDestroys
    return GateResult{Blocked: blocked, Trips: trips}
}
```

**Source:** Algorithm verbatim from CONTEXT.md lock (lines 117-135) and design spec lines 99-117.

### Pattern 4: Module loop with skip + per-module timeout

Mirror `RunInitWithRunner` (`init.go:749-845`) ‚Äî same skip semantics, same per-module timeout wrapper.

```go
// internal/app/cmd/init.go ‚Äî new function adjacent to RunInitWithRunner

func runInitPlan(cfg *config.Config, awsProfile, region string, verbose, acceptDestroys bool) error {
    ctx := context.Background()

    // Validate AWS credentials (same path as runInit ‚Äî init.go:389-395)
    awsCfg, err := awspkg.LoadAWSConfig(ctx, awsProfile)
    if err != nil {
        return fmt.Errorf("failed to load AWS config (profile=%s): %w", awsProfile, err)
    }
    if err := awspkg.ValidateCredentials(ctx, awsCfg); err != nil {
        return fmt.Errorf("AWS credential validation failed: %w", err)
    }

    // Export env vars (Phase 84.1-01, init.go:399)
    ExportTerragruntEnvVars(cfg)

    repoRoot := findRepoRoot()
    regionLabel := compiler.RegionLabel(region)
    regionDir := filepath.Join(repoRoot, "infra", "live", regionLabel)

    runner := terragrunt.NewRunner(awsProfile, repoRoot)
    runner.Verbose = false  // Always capture stdout for --verbose decision per-module

    printBanner("km init --plan", fmt.Sprintf("%s (%s)", region, regionLabel))

    modules := regionalModules(regionDir)
    reports := make([]planreport.Report, 0, len(modules))

    for _, m := range modules {
        if _, err := os.Stat(m.dir); os.IsNotExist(err) {
            fmt.Printf("  [skip] %s ‚Äî directory not found\n", m.name)
            continue
        }
        skipped := false
        for _, envVar := range m.envReqs {
            if os.Getenv(envVar) == "" {
                fmt.Printf("  [skip] %s ‚Äî %s not set\n", m.name, envVar)
                skipped = true
                break
            }
        }
        if skipped { continue }

        report, err := planModule(ctx, runner, m, verbose)
        if err != nil {
            // Hard plan failure ‚Äî stop loop, surface with module prefix
            return fmt.Errorf("planning %s: %w", m.name, err)
        }
        reports = append(reports, report)
    }

    result := planreport.Evaluate(reports, acceptDestroys)
    if result.Blocked {
        printTripBlock(result.Trips)
        return fmt.Errorf("destroy-class gate tripped (use --i-accept-destroys to override)")
    }
    if len(result.Trips) > 0 {
        // Override active ‚Äî still print the list for visibility
        printTripBlock(result.Trips)
        fmt.Println("  (override active via --i-accept-destroys ‚Äî exit 0)")
    }
    printAggregateSummary(reports)
    fmt.Println("Run 'km init --dry-run=false' to apply.")
    return nil
}

// planModule runs PlanWithOutput + ShowPlanJSON + Parse for a single module.
// Returns Report{ParseFailed: true} when JSON parse fails (gate conservative-trips).
func planModule(ctx context.Context, runner *terragrunt.Runner, m regionalModule, verbose bool) (planreport.Report, error) {
    timeout := ModuleTimeoutFunc(m.name)
    pctx, cancel := context.WithTimeout(ctx, timeout)
    defer cancel()

    // Temp planfile ‚Äî absolute path required so terragrunt's cache-dir doesn't swallow it
    tmpFile, err := os.CreateTemp("", "km-plan-"+m.name+"-*.tfplan")
    if err != nil {
        return planreport.Report{Module: m.name, ParseFailed: true, ParseError: err}, nil
    }
    tmpPath := tmpFile.Name()
    tmpFile.Close()  // PlanWithOutput owns the file from here
    defer os.Remove(tmpPath)

    var stdoutBuf bytes.Buffer
    fmt.Printf("  Planning %s...", m.name)
    if err := runner.PlanWithOutput(pctx, m.dir, tmpPath, &stdoutBuf); err != nil {
        fmt.Println(" FAILED")
        if errors.Is(err, context.DeadlineExceeded) {
            return planreport.Report{}, fmt.Errorf("module %s wedged after %s: %w", m.name, timeout, err)
        }
        return planreport.Report{}, fmt.Errorf("terragrunt plan failed: %w", err)
    }

    jsonBytes, err := runner.ShowPlanJSON(pctx, m.dir, tmpPath)
    if err != nil {
        fmt.Println(" parse-fail (warn)")
        return planreport.Report{Module: m.name, ParseFailed: true, ParseError: err}, nil
    }
    report, err := planreport.Parse(jsonBytes)
    if err != nil {
        fmt.Println(" parse-fail (warn)")
        return planreport.Report{Module: m.name, ParseFailed: true, ParseError: err}, nil
    }
    report.Module = m.name
    fmt.Printf(" %s\n", summarizeReport(report))
    if verbose {
        os.Stdout.Write(stdoutBuf.Bytes())
    }
    return report, nil
}
```

### Anti-Patterns to Avoid

- **Calling `terraform show` directly (not via terragrunt).** Terragrunt runs the underlying terraform in `.terragrunt-cache/<hash>/<module>/` ‚Äî `terraform show` requires the same cwd to find providers. Use `terragrunt show -json` with `cmd.Dir = m.dir`; terragrunt handles cache resolution.
- **Persisting `--i-accept-destroys` to disk or env.** Locked decision (Decision 3). The flag is read once, used once, never persisted, never auto-applies. Don't add a `km configure` knob for it.
- **Importing `github.com/hashicorp/terraform-json`.** Library authors explicitly recommend against it for forward-compat parsing ‚Äî it drops unknown attributes. We need exactly 3 fields (`address`, `type`, `change.actions`); hand-roll ~30 LoC.
- **Using `terragrunt run-all plan -out`.** Interleaved output, known JSON-mangling bug on large plans (gruntwork-io/terragrunt#3427), busts cache between plan and show (#1697). Sequential loop matches the existing apply path.
- **Touching `runInit` or `runInitDryRun`.** Locked (Decision 1 + design spec ¬ß Files touched). Phase 84.1 owns those. Surface separation avoids merge churn.

## Common Pitfalls

### Pitfall 1: Plan-file path resolution between terragrunt invocations

**What goes wrong:** `terragrunt plan -out=tfplan.binary` runs from `.terragrunt-cache/<hash>/<module>/`, so `tfplan.binary` ends up inside the cache dir. A naive follow-up `terragrunt show -json tfplan.binary` may or may not find it depending on whether terragrunt rebuilds the cache for `show`.
**Why it happens:** Documented in gruntwork-io/terragrunt#1970 ‚Äî caller assumed plan and show share working dir; they don't necessarily.
**How to avoid:** Always pass an **absolute** path to `-out=` (use `os.CreateTemp("", ...)` which returns abs path on macOS/Linux). Then `terragrunt show -json <abs-path>` invoked with `cmd.Dir = m.dir` works because terragrunt resolves its own cache.
**Warning signs:** `Failed to read plan file` or `Error reading the plan` when running `show` after a `plan`.

### Pitfall 2: Skipped modules counted toward gate

**What goes wrong:** A module with `[skip: KM_ARTIFACTS_BUCKET not set]` produces no `Report`. If the gate naively iterates all `regionalModules()` instead of the `reports` slice, it might insert a `ParseFailed` placeholder for skipped modules and conservative-trip.
**Why it happens:** Confusing "module exists but plan didn't run" with "plan ran and produced no JSON."
**How to avoid:** Only append to `reports` when `planModule` returns successfully. Skipped modules are loop-`continue`s before `planModule` is called.
**Warning signs:** UAT scenario (g) fails ‚Äî skipped modules show in the trip block.

### Pitfall 3: Plan failure conflated with parse failure

**What goes wrong:** A `terragrunt plan` that fails with `AWS auth error` is fundamentally different from a `terraform show -json` that returns malformed bytes. The first should HARD-STOP the loop (surface stderr); the second should mark `ParseFailed: true` and continue.
**Why it happens:** Treating both as "JSON couldn't be obtained."
**How to avoid:** Return `error` from `planModule` only when `PlanWithOutput` itself failed; return `Report{ParseFailed: true}` when `ShowPlanJSON` or `Parse` failed.
**Warning signs:** UAT scenario (h) fails ‚Äî auth errors silently become trips instead of hard-stops.

### Pitfall 4: Schema version drift from HashiCorp

**What goes wrong:** HashiCorp bumps `format_version` from "1.0" to "1.1" (backward-compatible add) or "2.0" (breaking). A hard-coded `== "1.0"` check rejects 1.1; a missing check accepts 2.0 garbage.
**Why it happens:** Treating the schema version as a constant rather than a contract.
**How to avoid:** Accept `1.x` (major version 1, any minor) and log a warning on unknown majors. Per HashiCorp's documented stability rule: "minor version increments for backward-compatible additions or changes; major version increments for changes not backward-compatible."
**Warning signs:** Plans fail to parse after a terraform upgrade; gate stops working.

### Pitfall 5: Test-fixture authoring without a real Phase-82.x plan

**What goes wrong:** The "Phase 84 trip scenario" fixture (`ses-82to84-destroy.json`) ideally captures a real plan from an in-place v1.0.0‚Üív2.0.0 cutover. The operator's account is already on Phase 84 ‚Äî that exact plan can't be reproduced.
**Why it happens:** Time-travel data-capture problem.
**How to avoid:** Hand-craft to the documented schema. Use a minimal plan with `format_version: "1.0"` and `resource_changes` containing exactly the 3 destroys from the design spec (`aws_ses_domain_identity.sandboxes`, `aws_ses_domain_dkim.sandboxes`, `aws_route53_record.dkim[0]`). Validate one capture against a real plan during operator UAT scenario 2 (synthetic destroy of catchall rule).
**Warning signs:** Fixture parses but real plans don't (or vice versa) ‚Äî schema deviation.

### Pitfall 6: `cmd.Output()` returns stderr as part of error

**What goes wrong:** `exec.Cmd.Output()` returns stderr in `*exec.ExitError.Stderr` only. If terragrunt prints diagnostics to stderr but exits 0, those won't show up ‚Äî `Output()` returns clean stdout. But if it exits non-zero, the error message is opaque.
**Why it happens:** `cmd.Output()` is convenience but doesn't compose with capture-everything semantics.
**How to avoid:** Use `cmd.Output()` for `ShowPlanJSON` (we want clean JSON; if terragrunt errors, surface its stderr via `*exec.ExitError.Stderr`). For `PlanWithOutput`, use the existing `runBounded` so heartbeat + stderr printing kick in.
**Warning signs:** Parse failures with no context; "exit status 1" with no clue why.

### Pitfall 7: Operator runs `km init --plan --dry-run=false` and expects an apply

**What goes wrong:** Decision 1 says `--plan` and `--dry-run` are independent. An operator could plausibly think `--plan` is a verbose mode of `--dry-run=false` and pass both. Per spec, `--plan` "implies no apply ever runs."
**Why it happens:** Flag overlap confusion.
**How to avoid:** Either reject `--plan --dry-run=false` explicitly (cleaner), or document loudly in `--help` that `--plan` overrides `--dry-run`. Design spec implies `--plan` always wins.
**Warning signs:** Operator surprise that `--plan` "ate" their apply intent.
