<objective>
Extend `pkg/terragrunt/runner.go` with two new Runner methods that the plan-and-gate workflow needs: `PlanWithOutput` (runs `terragrunt plan -out=<file>` with stdout captured into a caller buffer) and `ShowPlanJSON` (runs `terragrunt show -json <file>` and returns JSON bytes).

The methods are intentionally thin siblings of the existing `Plan` (runner.go:130), `Apply` (runner.go:62), and `Output` (runner.go:78) — same `buildCommand` factory, same `runBounded` ctx wrapper. The two new behaviours layered on top:
1. `PlanWithOutput` captures stdout into a caller-supplied `*bytes.Buffer` regardless of `r.Verbose` so the cmd-layer (Plan 04) can decide later whether to stream it to the terminal (when `--verbose`) or discard it (default summary mode).
2. `ShowPlanJSON` uses `cmd.Output()` (not `runBounded`) because we need a clean stdout-only byte slice for `planreport.Parse` — matches the existing `Output` method (runner.go:78-89).

Per RESEARCH § Pitfall 1, `terragrunt plan -out=<file>` puts the binary plan inside `.terragrunt-cache/<hash>/<module>/` if given a relative path; the caller (Plan 04) must always pass an absolute path. This plan documents that contract loudly in the doc comment so the implementer of Plan 04 / 05 can't miss it.

# Pitfall 1 reminder (84.2-RESEARCH.md § Common Pitfalls)
# `terragrunt plan -out=tfplan.binary` runs from `.terragrunt-cache/<hash>/<module>/`, so
# `tfplan.binary` (relative path) ends up inside the cache dir. A naive follow-up
# `terragrunt show -json tfplan.binary` may or may not find it.
# CONTRACT: callers MUST pass ABSOLUTE paths to planFile. `os.CreateTemp("", "...")` returns
# absolute paths on macOS/Linux (the only platforms km targets).

# Pitfall 6 reminder (84.2-RESEARCH.md § Common Pitfalls)
# cmd.Output() returns stderr in *exec.ExitError.Stderr only on non-zero exit.
# For ShowPlanJSON we WANT this — clean stdout bytes on success, stderr surfaced via
# ExitError.Stderr on failure. Don't switch to runBounded (which captures differently).

       ```go
       // PlanWithOutput runs `terragrunt plan -out=<planFile>` inside sandboxDir.
       //
       // Stdout is captured into stdoutBuf (for optional --verbose echo by the caller —
       // typically internal/app/cmd/runInitPlan and runBootstrapSharedSESPlan in Phase 84.2).
       // Stderr handling matches Apply: when r.Verbose, stderr streams to os.Stderr; when
       // r.Verbose is false, stderr is captured and only printed on failure (matches the
       // OPER-01 quiet-by-default contract).
       //
       // Phase 84.1-02: inherits the runner-layer ctx deadline + heartbeat via runBounded.
       // A bounded ctx returns context.DeadlineExceeded (wrapped) when the deadline trips —
       // do NOT add a local context.WithTimeout here (would double-wrap and confuse
       // ModuleTimeoutFunc bookkeeping in init.go).
       //
       // The planFile path MUST be absolute. Pass `nil` for stdoutBuf to discard stdout.
       func (r *Runner) PlanWithOutput(ctx context.Context, sandboxDir, planFile string, stdoutBuf *bytes.Buffer) error {
           cmd := r.BuildPlanWithOutputCommand(ctx, sandboxDir, planFile)
           if stdoutBuf == nil {
               stdoutBuf = &bytes.Buffer{}  // discard
           }
           // Always capture stdout to the caller-supplied buffer (independent of r.Verbose
           // — callers decide later whether to echo the buffer to the terminal).
           cmd.Stdout = stdoutBuf
           var stderrBuf bytes.Buffer
           if r.Verbose {
               cmd.Stderr = io.MultiWriter(os.Stderr, &stderrBuf)
           } else {
               cmd.Stderr = &stderrBuf
           }
           err := r.runBounded(ctx, cmd)
           if err != nil && !r.Verbose && stderrBuf.Len() > 0 {
               // Surface terragrunt's stderr on failure when quiet mode would have hidden it.
               fmt.Fprintln(os.Stderr, stderrBuf.String())
           }
           return err
       }
       ```
