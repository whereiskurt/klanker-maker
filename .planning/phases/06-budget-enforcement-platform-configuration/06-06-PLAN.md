Create internal/app/cmd/budget.go:

**Command structure:** `km budget add <sandbox-id> --compute <amount> --ai <amount>`

NewBudgetCmd(cfg) returns a cobra.Command with `add` subcommand.

**km budget add logic:**
1. Read current budget from DynamoDB via GetBudget
2. Calculate new limits: existing limit + top-up amount (additive, not replace)
3. Write new limits via SetBudgetLimits
4. **Auto-resume logic** — check if sandbox was suspended:
   - Read sandbox metadata from S3 (same pattern as km status)
   - If substrate=ec2 and instance state is stopped: call ec2.StartInstances
   - If substrate=ecs and task is stopped: re-provision task from stored S3 profile (call km create internally or use a dedicated resume function)
   - If Bedrock IAM was detached (check for missing policy): call iam.AttachRolePolicy to restore
5. Print summary: "Budget updated: compute $X.XX/$Y.YY, AI $X.XX/$Y.YY. Sandbox resumed."
