Create sidecars/http-proxy/httpproxy/bedrock.go:

1. **extractBedrockTokens(body io.Reader) (inputTokens, outputTokens int, err error):**
   - Use bufio.Scanner to read SSE events (delimited by blank lines)
   - On `data: ` prefix lines, JSON-parse for type field
   - `message_start`: extract `message.usage.input_tokens`
   - `message_delta`: extract `usage.output_tokens` (cumulative final — use this, NOT message_start output_tokens)
   - For non-streaming responses (no SSE events), try JSON-unmarshal the whole body for `usage.input_tokens`/`usage.output_tokens`
   - Cap buffer at 10MB — if response exceeds, return best-effort partial counts

2. **calculateCost(inputTokens, outputTokens int, rate BedrockModelRate) float64:**
   - `(inputTokens * rate.InputPricePer1KTokens / 1000) + (outputTokens * rate.OutputPricePer1KTokens / 1000)`

3. **AIBudgetExhaustedError struct** and **bedrockBlockedResponse(req, sandboxID, modelID, spent, limit) *http.Response:**
   - Returns goproxy.NewResponse with 403 status, application/json content type
   - Body: `{"error":"ai_budget_exhausted","spent":5.00,"limit":5.00,"model":"claude-sonnet-4","topUp":"km budget add sb-xxx --ai 5"}`

4. **extractModelID(urlPath string) string:**
   - Parse model ID from URL path `/model/{model-id}/invoke` or `/model/{model-id}/invoke-with-response-stream`

Create sidecars/http-proxy/httpproxy/budget_cache.go:
- budgetCache struct with sync.Mutex, map[string]*budgetEntry
- budgetEntry: computeLimit, aiLimit, aiSpent, fetchedAt time.Time
- get(sandboxID) returns entry if within 10s TTL, otherwise nil
- set(sandboxID, entry) stores entry with current timestamp
- updateLocalSpend(sandboxID, additionalCost) — adds cost to cached aiSpent (optimistic local tracking between DynamoDB refreshes)

Tests use table-driven patterns with real SSE payload strings captured from Anthropic docs.

**Extend NewProxy signature** to accept budget-related parameters. Add a new function or extend the existing one:


Use functional options pattern:
- WithBudgetEnforcement(budgetClient BudgetAPI, tableName string, modelRates map[string]BedrockModelRate) ProxyOption

When budget enforcement is enabled:

1. **Register Bedrock MITM handler BEFORE the general CONNECT handler** (goproxy uses first-match):
   ```go
   bedrockHostRegex := regexp.MustCompile(`^bedrock-runtime\..+\.amazonaws\.com`)
   proxy.OnRequest(goproxy.ReqHostMatches(bedrockHostRegex)).HandleConnect(goproxy.AlwaysMitm)
   ```

2. **Register OnResponse handler for Bedrock hosts:**
   - Check budget cache — if AI budget exhausted, return 403 immediately (bedrockBlockedResponse)
   - If under budget, read response body, call extractBedrockTokens
   - Calculate cost via calculateCost with model rate lookup
   - Call IncrementAISpend atomically to DynamoDB (fire-and-forget goroutine — don't block response)
   - Update budget cache local spend optimistically
   - Replace resp.Body with io.NopCloser(bytes.NewReader(capturedBody)) so client gets the full response
   - Log token usage and cost via zerolog

3. **Also register OnRequest handler** for pre-flight budget check:
   - Before proxying the Bedrock request, check budget cache
   - If AI budget >= limit, reject with 403 before even making the request

4. **Non-Bedrock traffic is unaffected** — the existing OkConnect handler fires for all other hosts.

**Update sidecars/http-proxy/main.go:**
- Read budget-related env vars: KM_BUDGET_TABLE, KM_SANDBOX_ID (already available), KM_BUDGET_ENABLED
- If KM_BUDGET_ENABLED=true, create DynamoDB client, load model rates (GetBedrockModelRates), pass WithBudgetEnforcement option
- If not enabled, proxy works exactly as before (backward compatible)
- Write budget remaining to /run/km/budget_remaining file on each cache refresh (per research recommendation)

**CA certificate injection** is handled by the compiler's user-data template (not this plan). The proxy uses goproxy's built-in CA (goproxy.GoproxyCa) by default. To use a custom CA, read KM_PROXY_CA_CERT env var (base64 PEM) and set proxy.CertStore. Add a TODO comment for custom CA support.
