    - Test 2: ECS sandbox (substrate="ecs") with cluster ARN and task ARN in Resources builds correct `aws ecs execute-command --cluster X --task Y --interactive --command /bin/bash` exec.Cmd
    - Test 3: Sandbox with status="stopped" returns error "sandbox is not running"
    - Test 4: Sandbox with substrate="unknown" returns error about unsupported substrate
  </behavior>
  <action>
    Create `internal/app/cmd/shell.go` with:

    1. `NewShellCmd(cfg *config.Config) *cobra.Command` — public constructor following the existing pattern. Use: "shell", Args: ExactArgs(1) for sandbox-id.

    2. `NewShellCmdWithFetcher(cfg *config.Config, fetcher SandboxFetcher) *cobra.Command` — DI variant for testing, following the same pattern as `NewStatusCmdWithFetcher`.

    3. `runShell(cmd *cobra.Command, cfg *config.Config, fetcher SandboxFetcher, sandboxID string) error`:
       a. Load AWS config (reuse pattern from status.go)
       b. Create real fetcher if nil (same newRealFetcher pattern)
       c. Call `fetcher.FetchSandbox(ctx, sandboxID)` to get SandboxRecord
       d. Check `rec.Status` — if "stopped", return error: `fmt.Errorf("sandbox %s is stopped — start it with 'km budget add %s --compute <amount>' first", sandboxID, sandboxID)`
       e. Based on `rec.Substrate`:
          - `"ec2"`: Extract instance ID from Resources (find ARN containing ":instance/"), call `execSSMSession(ctx, instanceID, rec.Region)`
          - `"ecs"`: Extract cluster ARN and task ARN from Resources, call `execECSCommand(ctx, clusterARN, taskARN, rec.Region)`
          - default: return error `fmt.Errorf("unsupported substrate %q for km shell", rec.Substrate)`

    4. `execSSMSession(ctx context.Context, instanceID, region string) error`:
       ```go
       // Build: aws ssm start-session --target <instanceID> --region <region>
       cmd := exec.CommandContext(ctx, "aws", "ssm", "start-session",
           "--target", instanceID, "--region", region)
       cmd.Stdin = os.Stdin
       cmd.Stdout = os.Stdout
       cmd.Stderr = os.Stderr
       return cmd.Run()
       ```

    5. `execECSCommand(ctx context.Context, clusterARN, taskARN, region string) error`:
       ```go
       // Build: aws ecs execute-command --cluster <cluster> --task <task>
       //        --interactive --command /bin/bash --region <region>
       cmd := exec.CommandContext(ctx, "aws", "ecs", "execute-command",
           "--cluster", clusterARN, "--task", taskARN,
           "--interactive", "--command", "/bin/bash", "--region", region)
       cmd.Stdin = os.Stdin
       cmd.Stdout = os.Stdout
       cmd.Stderr = os.Stderr
       return cmd.Run()
       ```

    6. Helper `extractResourceID(resources []string, pattern string) (string, error)` — finds ARN matching pattern, extracts the resource ID portion.

    For tests in `shell_test.go`:
    - Use `NewShellCmdWithFetcher` with a fake SandboxFetcher (same pattern as status_test.go)
    - For EC2/ECS tests: instead of running the actual aws CLI, test that the command construction logic produces the right arguments. Use a `shellExecFunc` type variable (package-level, overridable in tests) so tests can capture the exec.Cmd args without executing. Pattern: `var shellExec = defaultShellExec` with `defaultShellExec` calling `cmd.Run()` and tests replacing with a capture function.
    - Test error paths: stopped sandbox, unknown substrate, missing instance ID in resources.
