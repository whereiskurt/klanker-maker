**Create cmd/configui/handlers_budget.go:**

Define a BudgetDataFetcher that queries DynamoDB for budget summaries. Create a narrow interface following the established pattern:

```go
type BudgetFetcher interface {
    GetBudget(ctx context.Context, sandboxID string) (*BudgetDisplayData, error)
}
type BudgetDisplayData struct {
    ComputeSpent  string // formatted "$1.23"
    ComputeLimit  string // formatted "$5.00"
    ComputePct    int    // percentage 0-100+
    AISpent       string
    AILimit       string
    AIPct         int
    CSSClass      string // "budget-ok", "budget-warn", "budget-exceeded"
    HasBudget     bool   // false if no budget data exists
}
```

CSS class logic: less than 80% = "budget-ok" (green), 80-99% = "budget-warn" (yellow), 100%+ = "budget-exceeded" (red). These are Claude's discretion per CONTEXT.md.

**Update cmd/configui/handlers.go:**

In the dashboard handler (handleDashboard or equivalent), after fetching sandbox list:
- For each sandbox, call BudgetFetcher.GetBudget
- If budget data exists, populate BudgetDisplayData in the template data struct
- If no budget data (sandbox has no budget defined), set HasBudget=false
- Graceful degradation: if DynamoDB is unreachable, show "N/A" in budget columns

**Update cmd/configui/templates/dashboard.html:**

Add two columns to the sandbox table: "Compute Budget" and "AI Budget".
- If HasBudget: show "$spent / $limit" with color-coded background via CSSClass
