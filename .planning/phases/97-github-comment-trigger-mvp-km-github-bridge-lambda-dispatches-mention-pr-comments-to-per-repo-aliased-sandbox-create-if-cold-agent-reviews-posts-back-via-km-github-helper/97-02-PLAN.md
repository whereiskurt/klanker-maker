<objective>
Two foundation pieces the bridge depends on: (1) extend the GitHub installation-token permission
compilation so per-sandbox tokens can actually post comments/reviews, and (2) adopt the research's
**cold-create correction** — carry the github envelope through the `SandboxCreate` EventBridge event
and have `create-handler` drain it into the new sandbox's `github-inbound` queue after provisioning.

Purpose: The design spec's "Phase 86 prompt-queue carry" is NOT implementable as written (Phase 86
is operator-side post-create SSM push; `SandboxCreateDetail`/`CreateEvent` have no prompt field; the
bridge has no operator shell). This plan implements the recommended fix (Pitfall 1 option A) so the
cold path actually delivers the prompt. Without the write-scope fix, `km-github review` 403s.

Output: extended `CompilePermissions` + call-site; `SandboxCreateDetail`/`CreateEvent` envelope
field; create-handler enqueue hook.
</objective>
