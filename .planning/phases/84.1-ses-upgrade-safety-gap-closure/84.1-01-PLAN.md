<objective>
Close GAP-1 and GAP-7 from Phase 84 UAT: `km bootstrap` does not export `KM_ROUTE53_ZONE_ID` (causing foundation MX/DKIM/verification record applies to fail when `register_domain_identity=true`) and does not export `KM_ARTIFACTS_BUCKET` (causing direct-terragrunt operator workflows to produce malformed S3 ARNs).

Per Phase 84 UAT observation #7 (`84-10-UAT.md` lines 156-157): "Suggest a single `ExportTerragruntEnvVars(cfg)` helper that exports the full set: `KM_RESOURCE_PREFIX`, `KM_DOMAIN`, `KM_EMAIL_SUBDOMAIN`, `KM_REGION`, `KM_REGION_LABEL`, `KM_ROUTE53_ZONE_ID`, `KM_ARTIFACTS_BUCKET`, `KM_ACCOUNTS_*`."

The fix is the consolidation: rename `ExportConfigEnvVars` to `ExportTerragruntEnvVars`, make it a strict superset of the existing behaviour, and ensure every km command that subsequently calls terragrunt (bootstrap, init, configure when it persists state) routes through this helper.

Purpose: Make `km bootstrap --shared-ses` self-contained — operators do not need to manually `export KM_ROUTE53_ZONE_ID=...` before running it (Phase 84 UAT recovery procedure relied on this manual export).

Output: A single canonical env-var export path used by all terragrunt-invoking commands.
</objective>

```go
// ExportConfigEnvVars exports the config-derived env vars that Terragrunt's
// site.hcl uses via get_env(). Exported so cmd_test package can verify the
// correct vars are set without triggering a full runInit (which requires
// real AWS credentials).
func ExportConfigEnvVars(cfg *config.Config) {
    if cfg.ArtifactsBucket != "" && os.Getenv("KM_ARTIFACTS_BUCKET") == "" {
        os.Setenv("KM_ARTIFACTS_BUCKET", cfg.ArtifactsBucket)
    }
    if cfg.OrganizationAccountID != "" && os.Getenv("KM_ACCOUNTS_ORGANIZATION") == "" {
        os.Setenv("KM_ACCOUNTS_ORGANIZATION", cfg.OrganizationAccountID)
    }
    // ... similarly for DNSParentAccountID, ApplicationAccountID, Domain,
    //     PrimaryRegion, OperatorEmail, SchedulerRoleARN, KM_RESOURCE_PREFIX,
    //     KM_EMAIL_SUBDOMAIN.
    //
    // NOTE: KM_ROUTE53_ZONE_ID is NOT exported here (GAP-1).
    // NOTE: KM_REGION_LABEL is NOT exported here.
}
```
