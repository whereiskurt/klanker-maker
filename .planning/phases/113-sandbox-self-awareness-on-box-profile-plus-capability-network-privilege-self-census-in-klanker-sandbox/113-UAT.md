## Defect found & fixed (Section C) — commit `9e3918aa` (orchestrator)

Live UAT exposed four real bugs in the shipped Section C network census, all confined to
`skills/sandbox/SKILL.md` Section C:

1. **DNAT counter double-emit.** `grep -c DNAT || echo 0` double-emitted `0\n0` →
   `[: integer expression expected` error + garbled output. Fixed: bare `grep -c` + `${VAR:-0}`.
2. **Enforcement parse too shallow.** `grep -A2 '^  network:'` missed `enforcement:` below the
   `egress:` sub-block → wrongly reported "(not set, default=proxy)" for an `enforcement: both`
   profile. Fixed: match the unique `enforcement:` key directly.
3. **Egress-allowlist parse over-scooped.** Picked up unrelated YAML list items
   (`allowedRegions` `us-east-1`, `initCommands` `openssl`, `trustedDirectories` `/workspace`).
   Fixed: awk block-scoped to `allowedHosts`/`allowedDNSSuffixes`.
4. **Safe-active probe picked a garbage host.** Consequently chose `us-east-1` as the "allowed
   host" (curled `https://us-east-1/` → meaningless 502). Fixed: pick a concrete host from
   `allowedHosts` skipping wildcards; fall back to `api.anthropic.com` on a wildcard-only
   allowlist; dropped the `|| echo 000` http_code double-emit.

Also hardened proxy detection to use `HTTPS_PROXY` / injected-CA signals (no root needed)
instead of the root-only iptables DNAT count, so mode infers correctly on non-privileged boxes.

The SKILL.md fix is **already committed** as `9e3918aa`. Plugin version stays `0.4.10`
(113-02's unreleased bump; the fix rides it).
