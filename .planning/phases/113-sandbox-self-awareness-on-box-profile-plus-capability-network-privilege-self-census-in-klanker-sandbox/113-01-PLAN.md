<objective>
Thread the rendered SandboxProfile YAML into the EC2 userdata so every sandbox writes
`/opt/km/.km-profile.yaml` at boot. This is the platform half of Phase 113 — it makes the
file the `klanker:sandbox` skill (Plan 02) reads actually exist, sourced from the identical
`*profile.SandboxProfile` that drives all other userdata fields so the on-box copy can never
diverge from the S3 copy.

Purpose: An on-box agent must be able to read its own declarative configuration without S3
access or an IAM grant, even when egress is fully locked down.
Output: A new `ProfileYAML` field on `userDataParams`, a section "2.10" template write-block,
two new unit tests, and three regenerated golden files.
</objective>
