## Notes / findings

- **`conversations.info` needs `groups:read`** to read private-channel metadata; the live bot install **predates** `groups:read` (the manifest template at `slack_manifest_template.json:22` requests it, but the install was not re-authorized). Not a Phase 118 code issue — private-channel *creation* works via `groups:write`/`channels:manage`. To read private-channel metadata operationally, reinstall the App (`km slack manifest` → reinstall → `km slack rotate-token`).
- The silent-drop log line is `Debug`-level; at the bridge's INFO level a rejected trigger is invisible except by the absence of an `enqueued` line. Intentional ("silent ignore" per spec).
- Synthetic `event_callback` POSTs must target the **`/events`** path (path-based dispatch); the bare bridge URL routes to the sandbox→Slack envelope handler (`missing_fields`).
