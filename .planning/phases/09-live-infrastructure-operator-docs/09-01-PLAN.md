  <name>Task 1: Add build-lambdas target to Makefile</name>
  <files>Makefile</files>
  <action>
Add a `build-lambdas` target to the existing Makefile that cross-compiles both Lambda binaries for linux/arm64 (Graviton). The Lambda `provided.al2023` runtime requires the executable inside the zip to be named `bootstrap`.

Add the following target after the existing `ecr-push` target:

```makefile
## build-lambdas: cross-compile Go Lambda binaries for arm64 and package as deployment zips
build-lambdas:
	@mkdir -p build
	GOOS=linux GOARCH=arm64 CGO_ENABLED=0 go build -o build/bootstrap ./cmd/ttl-handler/
	cd build && zip -j ttl-handler.zip bootstrap && rm bootstrap
	GOOS=linux GOARCH=arm64 CGO_ENABLED=0 go build -o build/bootstrap ./cmd/budget-enforcer/
	cd build && zip -j budget-enforcer.zip bootstrap && rm bootstrap
	@echo ""
	@echo "Lambda deployment packages built:"
	@echo "  build/ttl-handler.zip"
	@echo "  build/budget-enforcer.zip"
```

CRITICAL: Use `GOARCH=arm64` (NOT amd64 like the sidecar targets). The Lambda modules declare `architectures = ["arm64"]`. Using amd64 causes `exec format error` at invocation time.

CRITICAL: The binary inside the zip MUST be named `bootstrap` (not the package name). The `-j` flag in zip strips paths.

Add `build-lambdas` to `.PHONY`.

Also add `build/` to .gitignore if not already present.
