<objective>
Extend the profile compiler to generate a per-sandbox budget-enforcer Terragrunt directory, and wire km create to deploy it after the main sandbox infrastructure.

Purpose: The budget-enforcer Lambda is per-sandbox (unique IAM, EventBridge schedule, sandbox_id scoping). The compiler already generates `budget_enforcer_inputs` in service.hcl but no Terragrunt wrapper instantiates those inputs into the module. This plan closes the gap so that `km create` with a budget-enabled profile deploys both the sandbox and its budget enforcer.

Output: Budget enforcer HCL generator in the compiler, extended CompiledArtifacts struct, km create wiring.
</objective>

1. Add `BudgetEnforcerHCL string` field to `CompiledArtifacts` in `compiler.go`. Add comment: "BudgetEnforcerHCL is the content for budget-enforcer/terragrunt.hcl (empty if no budget defined)."

2. Create `pkg/compiler/budget_enforcer_hcl.go` with:
   - A Go `text/template` that produces a Terragrunt HCL file for the per-sandbox budget-enforcer. The template reads `budget_enforcer_inputs` from the sibling `service.hcl` via `read_terragrunt_config`. Key structure:
     ```
     locals block: repo_root (find_in_parent_folders CLAUDE.md), site_vars (site.hcl), svc_config (../service.hcl), be_inputs (svc_config.locals.budget_enforcer_inputs)
     include "root": find_in_parent_folders("terragrunt.hcl")
     remote_state: S3 backend, key = "${tf_state_prefix}/sandboxes/${sandbox_id}/budget-enforcer/terraform.tfstate"
     terraform.source: "${repo_root}/infra/modules/budget-enforcer/v1.0.0"
     inputs: merge(be_inputs, { lambda_zip_path, budget_table_name, budget_table_arn (constructed from accounts.application + region), state_bucket (KM_ARTIFACTS_BUCKET), email_domain (sandboxes.${domain}), operator_email (KM_OPERATOR_EMAIL) })
     ```
   - The `budget_table_arn` must be constructed using `local.site_vars.locals.accounts.application` and `local.site_vars.locals.region.full` to build the full DynamoDB ARN.
   - Template data struct: `budgetEnforcerHCLData` with `SandboxID string` (for the state key).
   - Function `generateBudgetEnforcerHCL(sandboxID string) (string, error)` that executes the template.

3. In `Compile()` function (in `compiler.go` or wherever EC2/ECS compilation calls live), after generating ServiceHCL, check if the profile has `spec.budget` set. If so, call `generateBudgetEnforcerHCL(sandboxID)` and store the result in `artifacts.BudgetEnforcerHCL`.

Note: The budget-enforcer terragrunt.hcl itself is mostly static (it reads dynamic values from service.hcl's budget_enforcer_inputs block). The only templated value is the sandbox_id in the state key. This keeps the template simple.
