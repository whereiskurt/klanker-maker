  <action>
    For each file, read the current code first, then apply the specific change pattern:

    **DynamoDB client setup pattern (add to each command's Run function):**
    Each CLI command already creates an AWS config and S3 client. Add a DynamoDB client alongside:
    ```go
    dynamoClient := dynamodb.NewFromConfig(awsCfg)
    tableName := cfg.SandboxTableName  // "km-sandboxes" from config
    ```
    Where `cfg` is the loaded config (already available in all commands via `config.LoadConfig()`).

    **Backward-compat S3 fallback pattern:**
    For READ operations (ReadSandboxMetadataDynamo, ListAllSandboxesByDynamo), catch `ResourceNotFoundException`:
    ```go
    meta, err := kmaws.ReadSandboxMetadataDynamo(ctx, dynamoClient, tableName, sandboxID)
    if err != nil {
        var rnf *dynamodbtypes.ResourceNotFoundException
        if errors.As(err, &rnf) {
            // Table doesn't exist — fall back to S3
            meta, err = kmaws.ReadSandboxMetadata(ctx, s3Client, bucket, sandboxID)
        }
        if err != nil { return err }
    }
    ```
    For WRITE operations, no fallback — if table doesn't exist, the write to S3 is preserved as-is (backward compat means old environments keep working).

    **Per-file changes (from research call site inventory):**

    1. `create.go` — Find all 3 PutObject calls for metadata.json (EC2 path, Docker path, Remote path). Replace each with `kmaws.WriteSandboxMetadataDynamo(ctx, dynamoClient, tableName, meta)`. CRITICAL: Docker substrate must also use DynamoDB (user explicitly requested this). Ensure the SandboxMetadata struct is populated identically to current behavior before calling WriteSandboxMetadataDynamo.

    2. `extend.go` — Replace `kmaws.ReadSandboxMetadata` + `s3.PutObject` pattern with `kmaws.ReadSandboxMetadataDynamo` then `kmaws.WriteSandboxMetadataDynamo`. Add S3 fallback for read.

    3. `pause.go` — Replace `ReadSandboxMetadata` + status-update PutObject with `ReadSandboxMetadataDynamo` (for lock check) + `UpdateSandboxStatusDynamo`. Add S3 fallback for read.

    4. `resume.go` — Replace PutObject status update with `UpdateSandboxStatusDynamo`. Add S3 fallback for read if there is a ReadSandboxMetadata call.

    5. `lock.go` — Replace the entire read-modify-write pattern with single `kmaws.LockSandboxDynamo(ctx, dynamoClient, tableName, sandboxID)`. This is the key atomic improvement. Handle "already locked" error message from LockSandboxDynamo. Add S3 fallback: if ResourceNotFoundException, fall back to old S3 read-modify-write pattern.

    6. `unlock.go` — Replace read-modify-write with single `kmaws.UnlockSandboxDynamo(ctx, dynamoClient, tableName, sandboxID)`. Handle "not locked" error. Add S3 fallback.

    7. `destroy.go` — Replace `ReadSandboxMetadata` with DynamoDB read (with S3 fallback). Replace `DeleteSandboxMetadata` (S3) with `kmaws.DeleteSandboxMetadataDynamo`. Keep S3 artifact deletion unchanged (artifacts stay in S3).

    8. `stop.go` — Replace `ReadSandboxMetadata` (substrate routing) with DynamoDB read. Add S3 fallback.

    9. `list.go` — Replace `ListAllSandboxesByS3` with `kmaws.ListAllSandboxesByDynamo(ctx, dynamoClient, tableName)`. Add S3 fallback on ResourceNotFoundException to call existing `ListAllSandboxesByS3`.

    10. `status.go` — Replace the `awsSandboxFetcher` backend (or wherever ReadSandboxMetadata is called) with DynamoDB read. Add S3 fallback.

    11. `budget.go` — Replace `ReadSandboxMetadata` (used for substrate check) with DynamoDB read. Add S3 fallback.

    **Import changes:** Add `"github.com/aws/aws-sdk-go-v2/service/dynamodb"` and `dynamodbtypes "github.com/aws/aws-sdk-go-v2/service/dynamodb/types"` to each file. Add `kmaws` import alias if not already present.
