    1. Define `SandboxMetadataAPI` interface (narrow interface pattern matching identity.go):
       - GetItem, PutItem, UpdateItem, DeleteItem, Scan, Query

    2. Define private `sandboxItemDynamo` struct with `dynamodbav` tags mapping to DynamoDB attribute names:
       - `SandboxID string \`dynamodbav:"sandbox_id"\`` (hash key)
       - All fields from SandboxMetadata with proper dynamodbav tags
       - `TTLExpiryEpoch int64 \`dynamodbav:"ttl_expiry"\`` for Number storage
       - Add conversion functions `toSandboxMetadata()` and `fromSandboxMetadata()` to/from the public SandboxMetadata type
       - Per research Pitfall 1: do NOT rely on json tag fallback — use explicit dynamodbav tags

    3. Implement exported functions (per research patterns 1-5):
       - `ReadSandboxMetadataDynamo(ctx, client, tableName, sandboxID) (*SandboxMetadata, error)` — GetItem, unmarshal sandboxItemDynamo, convert to SandboxMetadata. Return ErrSandboxNotFound (from sandbox.go) on empty result.
       - `WriteSandboxMetadataDynamo(ctx, client, tableName, meta *SandboxMetadata) error` — convert to sandboxItemDynamo, marshal, override ttl_expiry with AttributeValueMemberN{Unix epoch seconds} if TTLExpiry non-nil. Omit alias attribute if empty string (per research Pitfall 5 — prevents GSI pollution).
       - `DeleteSandboxMetadataDynamo(ctx, client, tableName, sandboxID) error` — DeleteItem by sandbox_id.
       - `ListAllSandboxesByDynamo(ctx, client, tableName) ([]SandboxRecord, error)` — Scan with pagination loop using LastEvaluatedKey (per research Pitfall 7). Convert each sandboxItemDynamo to SandboxRecord via metadataToRecord helper (already exists in sandbox.go).
       - `ResolveSandboxAliasDynamo(ctx, client, tableName, alias) (string, error)` — Query alias-index GSI. Return error if 0 or >1 results.
       - `LockSandboxDynamo(ctx, client, tableName, sandboxID) error` — UpdateItem with ConditionExpression: `attribute_exists(sandbox_id) AND (attribute_not_exists(locked) OR locked = :f)`. Set locked=true, locked_at=now. Catch ConditionalCheckFailedException and return "already locked" message.
       - `UnlockSandboxDynamo(ctx, client, tableName, sandboxID) error` — UpdateItem with ConditionExpression: `attribute_exists(sandbox_id) AND locked = :t`. Set locked=false, remove locked_at. Catch ConditionalCheckFailedException.
       - `UpdateSandboxStatusDynamo(ctx, client, tableName, sandboxID, status string) error` — UpdateItem setting status field only. Used by pause/resume/stop/destroy for lightweight status transitions without full PutItem.

    5. Write tests for each function listed in the behavior section. For ConditionalCheckFailedException, construct `&dynamodbtypes.ConditionalCheckFailedException{}` as the mock return error.

    6. For the pagination test: mock Scan to return non-nil LastEvaluatedKey on first call and nil on second call. Verify two Scan calls were made and all items from both pages are returned.
